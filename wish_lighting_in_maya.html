Custom colorMattes

If you need custom or unique mattes as a shot solution...

Do not override GlobalMattes! (Create new custom mattes.)

  • Create colorMattes:
    • choose num and channel (RGB)
    • User var:
      • choose mapping (screenspaceTexture)
      • paste var name into 'colorMatte#' attribute field
  • screenspaceTexure parameters:
    • scale:
      • size of tile
    • latticeSize:
      • number of tiles
    • blend:
      • edge merging between tiles
    • ref_jitter:
      • angular offset of tiles
    • preserveScreenSize:
      • distance scale control
    • preserveContrast:
      • sharpen blend regions
    • usePref:
      • stick to char surface
    • seed:
      • not explicitly tested by developer/lighters
      • it might help with variations if needed
    • mode:
      • lattice
      • point (very expensive, more accurate)
  • Assign to items:
    • used to be called 'binding'

Working Interactively in Maya/DLight

ION

  • Set up dummy pass (named ION):
    • you can turn lights on/off harmlessly
  • Turn on ION in Maya viewport
  • BUG: Target assignments for chars need wildcards at the end of the string:
    • :wdas:shot:asha:*
  • ION doesn't preview XGen elements

RegionGrad

  • Add regionGrad to light
  • Click on RegionGrad visualizer button in Maya
  • Adjust RegionGrad in Maya or DLight

TMODs

  • Target objects (assignments):
    • diffuse
    • spec
    • color
    • etc.

Direct link to NUKE

Sending Helios Renders to Nuke

  • Render (Helios):
    • Pass or full Stage
  • In Render Log Viewer:
    • click on 'Send Interactive Render to Picdir' icon
    • Icon: Pic-->
    • Dialog:
      • 'OK'
    • In Nuke:
      • set Reads to 'interactive' in 'updateNuke' dialog
  • in ShadowBox:
    • WISH_comp_breakout
    • set for imported renders

Tips

  • Overlight in Maya
  • Processes in Nuke will darken images
  • Try to keep emission numbers consistent
