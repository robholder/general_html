Foundation

Foundation Process

  • Gig into show
  • navigate to sequence and stage:
    • n testBreakout seqData foundation
    • lighting_setup
    • foundation_help:
      • UI lists shots:
        • Candidate cameras
      • Pick shots (select)
      • Extract selected button:
        • Goes to the render queue to run tasks
        • At least 15 minutes, to hours

in Maya

  • Maya scenes:
    • version 001 is available
    • doMaya
  • Cameras are now installed to view

in DLight

  • In DLight:
    • Stage exists
  • Create new Render Pass:
    • name 'base'
    • default pass
    • OK
  • Lighting shelf:
    • Foundation Lighting Helper (colored triangle icon)
    • UI lists shots/cameras
    • RMB on shot to create pass:
      • You can select all and do en masse:
        • stage:
          • base:
            • shot_001_a
            • shot_002_00_A
            • shot_003_00_B

Hierarchy

  • In Hierarchy:
    • Two rigs exist:
      • lgtAssettestBreakoutseq
      • lrigtestBreakoutseq
    • Create lightrig:
      • sun with + icon
      • lrigElement:
        • rename:
          • lrigtestBreakoutseqFar
      • When publishing:
        • Publish Light Rigs (sun with tick icon)
        • Each rig will be listed to be published:
          • Light Rig Publish UI:
            • list of rigs
    • Select one of the auto rigs:
      • check on Render box
      • select 'base' render pass
      • Create new render pass:
        • far (default)
      • Now select the Far lrig we made in Hierarchy:
        • check on Render box for the new far rig
      • Drag camera pass into this new render pass variant:
        • perhaps shot_001_a
        • Now when selecting the far pass, the cam will show shot 1

Lighting setup

  • Selecting one of the camera passes:
    • in Palettes:
      • go to the relevant lrig:
        • Add the lr_sun, lr_practical, lr_state lights:
          • all_SKY
          • all_SUN
          • etc.
      • For Skydome IBL:
        • Texture Library tab:
          • PULP (show):
            • browse IBL
          • These IBLs get copied to the correct show dir
      • SUN:
        • scatter enable
        • angle 6.0
    • Get an interactive render out.

Next Lrig (next shot)

  • Select other camera passes:
  • Confirm the correct lrig is activated in the Hierarchy pane
    • Copy and paste all lights from the various individual lights to this lrig
    • Clear modifications (to prevent override situations)
    • Test render:
      • Then adjust lighting:
        • light adjusts
        • region_grads:
          • on skydome perhaps.
          • regionGradCoordSys don't work in foundation publishing
          • Only regular region_grads propagate.

Make sure to save!

Publish Utility

  • Open Publish Utility:
    • Sun with tick icon on lighting shelf
    • Message:
      • initial foundation lighting publish
    • Select lrig/passes:
      • click Publish
      • goes to render farm
      • This extracts lightrigs
    • AT TIME OF TUTORIAL:
      • EACH LRIG MUST BE PUBLISHED INDIVIDUALLY!

In Shell

sequence_manager
  • In Sequence Manager UI:
    • List of every shot, nested under seqData parent.
    • Assign rigs by choosing master rig in top right:
      • then choose variants per shot
      • in this case, 'far' for far shot.
    • Click save button/arrow:
      • publish info will be updated with publish details
