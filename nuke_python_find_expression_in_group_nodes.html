Find Driven Nodes Inside A Group (By Expressions)

import nuke

# 1) Grab the Group node (or any node) that has your MyExpressionKnob knob  
g = nuke.toNode("MyGroup")  # replace with your group’s name
# 2) Walk every node inside it  
for n in g.nodes(recurseGroups=True):  
    # 3) Check every knob on that node  
    for k in n.allKnobs():  
        # only knobs that support expressions have .hasExpression()
        try:  
            if k.hasExpression():  
                expr = k.expression()  
                # 4) Look for your slider’s name in the expression string  
                if "MyExpressionKnob" in expr:  
                    print("{node}.{knob}  →  {expr}".format(  
                        node=n.fullName(),  
                        knob=k.name(),  
                        expr=expr  
                    ))  
        except AttributeError:  
            # some knobs (like the label) won’t have hasExpression()
            continue

Explanation of the steps:

  • Use nuke.toNode() to grab the Group (or just nuke.thisGroup() if you’re running this from inside a Group’s Python callback).
  • group.nodes(recurseGroups=True) returns every node in that group—including nested groups.
  • node.allKnobs() gives you every knob object on a node. Knobs that can hold expressions respond to .hasExpression() and .expression().
  • You filter for any expression containing your user-knob’s name ("MyExpressionKnob"), and then print out exactly which node.knob is linked and what the expression is.

If you just want a one-liner to dump all expressions in the whole script and grep for your knob name, you could also do: 

for n in nuke.allNodes():  
    for k in n.allKnobs():  
        try:
            e = k.expression()
            if "MyExpressionKnob" in e:
                print(n.name(), k.name(), e)
        except:
            pass

But sticking to group.nodes() keeps the output scoped to that Group.
