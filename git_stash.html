Git - Stash

Sometimes called 'pseudo-commits'

Taken from:

https://opensource.com/article/21/4/git-stash

https://refine.dev/blog/git-stash/

"Git stash saves the uncommitted changes locally, allowing you to make
changes, switch branches, and perform other Git operations. You can then
reapply the stashed changes when you need them. A stash is locally scoped
and is not pushed to the remote by git push."

Stash stores both the indexed (staged) and unstaged working directory conditions.

Changing branches will completely overwrite your current files and indexed files. Stashing is a way of storing your current state for later recovery, without creating any commits.

Stashes are independent of branch, and can be restored to and from any branch.

Stashes are NOT pushed to a remote repository.

Stashes are stored in /.git/refs/stash.

Here's the sequence to follow when using git stash:
  • Save changes to branch A.
  • Run git stash.
  • Checkout branch B.
  • Fix the bug in branch B.
  • Commit and (optionally) push to remote.
  • Check out branch A
  • Run git stash pop to get your stashed changes back.

Commands

git stash - create a stash point

git stash --include-untracked - include untraced files (even those in .gitignore)
or:
git stash -u

git stash save "<description>" - create a named stash point, be more descriptive

git stash list - list stash points
git stash list -p - list stash points and show diffs per stash

git stash apply - apply most recent stash
git stash apply <stash_id> - apply a specific stash point
git statsh apply stash@{#} - apply specific stash point

git stash pop - apply most recent stash, and remove it from saved stashes
git stash pop <stash_id> - apply a specific stash, and remove it from saved stashes

git stash branch <branchName> - create new branch from Git stash
git stash branch <branch_name> stash@{revision} - generate a new branch from specific stash

git stash show <stash_id> - allows you to view the diff of a stash (stash_id can be number or stash@{#}
git stash show <stash_id> -p - (or --patch) allows you to view the diff of a stash with a more detail

Delete Stashes

git stash drop - deletes latest (most recent) stash from the stash list
git stash drop <stash_id> - deletes a particular stash from the stash list. (Stash indexes are adjusted following this command and the removal of the specified stash id.)
git stash clear - empties the stash list by removing all the stashes

Create Branshes

git stash branch <new_branch_name> <stash_id> - creates a new branch based on the commit the stash was created from and pops the stashed changes to it

Stash for a Single File

From:

  • https://betterstack.com/community/questions/git-stahs-specific-file/

To stash a specific file in Git, you can use the git stash push command with the path to the file you want to stash. Here's how to do it: 

git stash push -- <path-to-file>

Replace with the path to the file you want to stash.

For example, if you want to stash changes to a file named example.txt, you would run: 

git stash push -- example.txt

This command stashes the changes to the specified file, removing them from the working directory and staging area, but keeping them in the stash for later retrieval.

Note:

  • Stashing a specific file is useful when you only want to temporarily remove changes to that file from your working directory without stashing changes to other files.
  • You can stash multiple files by providing multiple arguments.
  • To retrieve the stashed changes later, you can use git stash apply or git stash pop.
  • Stashed changes can also be referred to using a stash index, which you can find using git stash list.
