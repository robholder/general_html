Challenge 1: Whitespace check

AI Insights

In Swift, there isn't a single property like .isNotWhitespace, but the standard way to handle this is to trim the whitespace and then check if the resulting string .isEmpty.

Here are the two best ways to handle it:

1. The Direct Approach

You can use trimmingCharacters(in:) to strip out all whitespace and newlines before checking the count. 

if myString.trimmingCharacters(in: .whitespacesAndNewlines).isEmpty {
    print("It's either empty or just whitespace.")
}

The trimmingCharacters(in:) method does not strip out all whitespace regardless of placement. It only removes whitespace from the start and the end of the string.

How it behaves:

  • " Hello " becomes "Hello" (Leading/trailing removed)
  • "Hello World" remains "Hello World" (Internal space is preserved)
  • " " (just spaces) becomes "" (An empty string)

Other Built-in Options

Think of a CharacterSet as a giant bucket of specific characters. When you pass that bucket to trimmingCharacters(in:), Swift looks at the start and end of your string and says, "If the character I'm looking at is inside this bucket, toss it away."

Because it uses CharacterSet, you have a lot of options beyond just spaces. Here are the other "buckets" you can use:

  • .whitespaces: Only removes standard spaces and tabs. Unlike .whitespacesAndNewlines, it will leave "Enter" key presses (newlines) alone.
  • .decimalDigits: Removes numbers (0-9).
  • .letters: Removes all letters, but leaves numbers and symbols.
  • .punctuationCharacters: Removes marks like !, ?, ., and ,.
  • .uppercaseLetters / .lowercaseLetters: Self-explanatory, but very handy for specific formatting.

Mixing and Matching

You aren't limited to the presets. You can combine them using "unions" or even create your own custom set.

To remove both punctuation AND whitespace: 

let combinedSet = CharacterSet.whitespaces.union(.punctuationCharacters)
let cleanString = myString.trimmingCharacters(in: combinedSet)

To remove specific characters you define: 

let customSet = CharacterSet(charactersIn: "0123456789#*")
let filtered = myString.trimmingCharacters(in: customSet)

Why this is perfect for your project:

For an address field in Cupcake Corner, you want to allow internal spaces (like "123 Main St"). You only want to prevent the user from:

  • Leaving the field entirely empty.
  • Typing a bunch of spaces to "trick" the .isEmpty check.

If you actually wanted to strip ALL whitespace

If you ever need to remove every single space in a string (e.g., for a username or a credit card number), you would use a different approach: 

let messyString = " 1 2 3 4 5 "
let cleanString = messyString.replacingOccurrences(of: " ", with: "")
// result: "12345"

2. The "Paul Hudson" Way (Recommended)

Since you are following 100 Days of SwiftUI, Paul often encourages writing extensions to make your code more readable. This allows you to keep using a simple property name throughout your project.

Add this extension to your project (usually at the bottom of the file or in a String-Extensions.swift file): 

extension String {
    var isReallyEmpty: Bool {
        self.trimmingCharacters(in: .whitespacesAndNewlines).isEmpty
    }
}

Now you can use it just like .isEmpty: 

if name.isReallyEmpty {
    print("Please enter a name that isn't just spaces.")
}

Why this matters for Cupcake Corner

In Project 10, users might try to "cheat" your validation by entering a single space for their name or address. Using .isReallyEmpty (or the trimming method) ensures that the Place Order button stays disabled until they provide actual text.

If you are using this inside your Order class for a computed property like hasValidAddress, it would look like this: 

var hasValidAddress: Bool {
    if name.isReallyEmpty || streetAddress.isReallyEmpty || city.isReallyEmpty || zip.isReallyEmpty {
        return false
    }
    return true
}

String-Extension.swift

import SwiftUI

extension String {

        self.trimmingCharacters(in: .whitespacesAndNewlines).isEmpty
    }

Order.swift

import SwiftUI


@Observable
class Order: Codable {
    static let types = ["Vanilla", "Strawberry", "Chocolate", "Rainbow"]
    
    var type = 0
    var quantity = 3
    var extraFrosting = false
    var addSprinkles = false
    var name = ""
    var streetAddress = ""
    var city = ""
    var zip = ""
    
    enum CodingKeys: String, CodingKey {
        case _type = "type"
        case _quantity = "quantity"
        case _specialRequestEnabled = "specialRequestEnabled"
        case _extraFrosting = "extraFrosting"
        case _addSprinkles = "addSprinkles"
        case _name = "name"
        case _city = "city"
        case _streetAddress = "streetAddress"
        case _zip = "zip"
    }
    
    var specialRequestEnabled = false {
        didSet {
            if specialRequestEnabled == false {
                extraFrosting = false
                addSprinkles = false
            }
        }
    }
    
    var hasValidAddress: Bool {
        // Using Swift-Extension (isReallyEmpty):
        if name.isReallyEmpty || streetAddress.isReallyEmpty || city.isReallyEmpty || zip.isReallyEmpty {
            return false
        }
        return true
    }
    
    var cost: Decimal {
        // $2 per cake
        var cost = Decimal(quantity) * 2
        
        // complicated cakes cost more
        cost += Decimal(type) / 2
        
        // $1/cake for extra frosting
        if extraFrosting {
            cost += Decimal(quantity)
        }
        
        // $0.50/cake for sprinkles
        if addSprinkles {
            cost += Decimal(quantity) / 2
        }
        
        return cost
    }
}

Challenge 2

AI Insights

You’ve hit on a common UI challenge: making a single alert "dynamic" so it reacts to the state of your data.

To solve this, you need a way to change the title string based on whether the network request succeeded or failed.

The Strategy

You already have a @State property for the message. You just need to add another @State property for the title.

Add a new State property:

@State private var alertTitle = ""

Update your placeOrder() function:

In the do block (Success): 

alertTitle = "Thank you!"
confirmationMessage = "Your order for..."
showingConfirmation = true

In the catch block (Failure): 

alertTitle = "Oops!"
confirmationMessage = "Check your internet connection and try again."
showingConfirmation = true

Update the Alert View:

Change the hardcoded "Thank you!" to your new variable: 

.alert(alertTitle, isPresented: $showingConfirmation) {
    Button("OK") { }
} message: {
    Text(confirmationMessage)
}

Why this is better than two separate alerts

You could create two different .alert modifiers (one for success, one for error), but that gets messy quickly. By using variables for both the title and the message, your code follows the Single Source of Truth principle—a core concept in SwiftUI.

A Quick "Pro" Tip for the Challenge

Paul’s challenge often implies catching actual network errors.

When you comment out request.httpMethod = "POST", URLSession defaults to a GET request.

httpbin.org/post will actually throw an error because it requires a POST. However, if you want to test a "No Internet" scenario without touching your code, you can simply:

  1. Turn off your Mac's Wi-Fi.
  2. Run the app.
  3. The catch block will trigger with a "The Internet connection appears to be offline" error automatically!
