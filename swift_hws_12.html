Hacking With Swift - 12 - How to use type annotations

Sometimes we don’t want to assign a value immediately, or sometimes we want to override Swift’s choice of type, and that’s where type annotations come in.

Inferred Typing

Swift normally 'infers' a data type: 

let surname = "Lasso" # inferred String
var score = 0 # inferred Integer

Explicit Typing

You can explicitly declare type like this: 

let surname: String = "Lasso" # Explicit String
let score: Int = 0 # Explicit Integer
// or to allow half scores later
var score: Double = 0 # Explicit Integer

Other data types:

Simple variables and constants:

let playerName: String = "Roy"
let luckyNumber: Int = 13
let pi: Double = 3.141
var isAuthenticated: Bool = true

Complex Types Such As Arrays

var albums: [String] = ["Red", "Fearless"] # Array of strings
var user: [String: String] = ["id" "@twostraws"]
var books: Set<String> = Set([
    "The Bluest Eye",
    "Foundation",
    "Girl, Woman, Other"
])

Setting up an empty String array: 

// If the array is populated at declaration, you can do this:
var soda: [String] = ["Coke", "Pepsi", "Irn-Bru"]

// If empty at declaration, you have choices.
// All these are the same result:

var teams: [String] = [String]()
var cities: [String] = []
var clues: [String]()

Enums Are their own Type/Class

We'll declare an enum: 

enum UIStyle {
    case light, dark, system
}

Now, 'style' is its own 'type', and this is inferred, so you don't even have to specify the Class or Type anymore (you can use the dot-attribute syntax - the script knows what 'style' is). 

var style = UIStyle.light
style = .dark

But you can add type annotations like this: 

var style: UIStyle = UIStyle.light
style = .dark

When to Use Type Annotations

Creating empty or blank variables or constants (especially constants) can really benefit from type annotations, since Swift can't infer from a blank entity.

Swift must always know what 'kind' a container is.

This kind of code requires type annotation: 

let username: String

// lots of complex logic
username = "@twostraws"

// lots more complex logic
print(username)
// >> @twostraws

Type-Safe Language

This subject is at the core of being a type-safe language, and stops us doing nonsense things like 5 + true or similar.

We cannot mix annotation type and assignment types: 

let score: Int = "zero"
