Hacking With Swift - 271 - ScrollView effects using GeometryReader

When we use the frame(in:) method of a GeometryProxy, SwiftUI will calculate the view’s current position in the coordinate space we ask for.

However, as the view moves those values will change, and SwiftUI will automatically make sure GeometryReader stays updated.

Previously we used DragGesture to store a width and height as an @State property, because it allowed us to adjust other properties based on the drag amount to create neat effects.

However, with GeometryReader we can grab values from a view’s environment dynamically, feeding in its absolute or relative position into various modifiers. Even better, you can nest geometry readers if needed, so that one can read the geometry for a higher-up view and the other can read the geometry for something further down the tree.

To try some effects with GeometryReader, we could create a spinning helix effect by creating 50 text views in a vertical scroll view, each of which has an infinite maximum width so they take up all the screen space, then apply a 3D rotation effect based on their own position.

Start by making a basic ScrollView of text views with varying background colors: 

struct ContentView: View {
    let colors: [Color] = [.red, .green, .blue, .orange, .pink, .purple, .yellow]

    var body: some View {
        ScrollView {
            ForEach(0..<50) { index in
                GeometryReader { proxy in
                    Text("Row #\(index)")
                        .font(.title)
                        .frame(maxWidth: .infinity)
                        .background(colors[index % 7])
                }
                .frame(height: 40)
            }
        }
    }
}

To apply a helix-style spinning effect, place this rotation3DEffect() directly below the background() modifier: 

.rotation3DEffect(.degrees(proxy.frame(in: .global).minY / 5), axis: (x: 0, y: 1, z: 0))

When you run that back you’ll see that text views at the bottom of the screen are flipped, those at the center are rotated about 90 degrees, and those at the very top are normal. More importantly, as you scroll around they all rotate as you move in the scroll view.

That’s a neat effect, but it’s also problematic because the views only reach their natural orientation when they are at the very top - it’s really hard to read. To fix this, we can apply a more complex rotation3DEffect() that subtracts half the height of the main view, but that means using a second GeometryReader to get the size of the main view: 

struct ContentView: View {
    let colors: [Color] = [.red, .green, .blue, .orange, .pink, .purple, .yellow]

    var body: some View {
        GeometryReader { fullView in
            ScrollView {
                ForEach(0..<50) { index in
                    GeometryReader { proxy in
                        Text("Row #\(index)")
                            .font(.title)
                            .frame(maxWidth: .infinity)
                            .background(colors[index % 7])
                            .rotation3DEffect(.degrees(proxy.frame(in: .global).minY - fullView.size.height / 2) / 5, axis: (x: 0, y: 1, z: 0))
                    }
                    .frame(height: 40)
                }
            }
        }
    }
}

With that in place, the views will reach a natural orientation nearer the center of the screen, which will look better.

We can use a similar technique to create CoverFlow-style scrolling rectangles: 

struct ContentView: View {   
    var body: some View {
        ScrollView(.horizontal, showsIndicators: false) {
            HStack(spacing: 0) {
                ForEach(1..<20) { num in
                    GeometryReader { proxy in
                        Text("Number \(num)")
                            .font(.largeTitle)
                            .padding()
                            .background(.red)
                            .rotation3DEffect(.degrees(-proxy.frame(in: .global).minX) / 8, axis: (x: 0, y: 1, z: 0))
                            .frame(width: 200, height: 200)
                    }
                    .frame(width: 200, height: 200)
                }
            }
        }
    }
}

All this code works well, and I think you'll agree these are fun effects. But they aren't exactly easy. Fortunately, SwiftUI provides better options - let's look at them next...

Ai Insight

Modulo Operator

QUESTION

In the loop, what is the % 7 doing?

ANSWER

The % symbol is the Modulo operator (or remainder operator). In plain math terms, it divides the first number by the second number and spits out whatever is left over.

In this specific code, index % 7 is a clever programming trick used to create a continuous, infinite loop through your list of 7 colors, preventing your app from crashing.

The Problem It Solves

Look at your data layout:

  • You have an array called colors containing exactly 7 items (indices 0 to 6).
  • Your ForEach loop counts from 0 all the way to 49 (index).

If you wrote colors[index] without the % 7, your app would crash the exact second the loop hit row #7. Why? Because colors[7] doesn't exist—the array ends at index 6. The computer throws an "Index out of range" error.

How % 7 Acts Like a Carousel

By using % 7, you are forcing the math to stay locked within a perfect 0 to 6 loop, no matter how high the row index goes.

Look at what happens to the math as the loop runs:

  • Row 0: 0÷7=0 with a remainder of 0 → colors[0] (.red)
  • Row 1: 1÷7=0 with a remainder of 1 → colors[1] (.green)
  • ...
  • Row 6: 6÷7=0 with a remainder of 6 → colors[6] (.yellow)

Here comes the magic loop-back:

  • Row 7: 7÷7=1 with a remainder of 0 → colors[0] (.red)
  • Row 8: 8÷7=1 with a remainder of 1 → colors[1] (.green)
  • Row 9: 9÷7=1 with a remainder of 2 → colors[2] (.blue)

Summary

The % 7 ensures that your 50 list rows will cycle beautifully through your 7 background colors over and over again (.red, .green, .blue... .yellow, .red, .green...). It turns a straight number line into a repeating circle!

QUESTION

Can you explain: 

.rotation3DEffect(.degrees(proxy.frame(in: .global).minY / 5), axis: (x: 0, y: 1, z: 0))

ANSWER

This line of code is where the power of combining a GeometryReader with 3D graphics really shines. It creates a dynamic animation where the rows appear to spin in 3D space as you scroll them up and down the screen.

Let's break this effect down into two parts: The Axis of Rotation (the hinge) and The Dynamic Angle (the movement).

1. The Axis: axis: (x: 0, y: 1, z: 0)

Before you spin a view in 3D space, you have to tell SwiftUI where to place the invisible hinge. You control this using 3D spatial coordinates: X, Y, and Z.

By setting y: 1 and leaving X and Z at 0, you are skewering the text view right down its vertical center with a metaphorical flagpole.

Because the hinge is vertical, the view can only spin left-to-right. Think of it exactly like a revolving door at a hotel entrance, or a sign spinning on a post.

2. The Dynamic Angle: proxy.frame(in: .global).minY / 5

Instead of passing a fixed number like .degrees(45), this code uses the GeometryReader to calculate a brand-new angle every single time the view moves even one pixel on the screen.

Let's look at the ingredients of this math:

  • proxy.frame(in: .global).minY: As we discussed earlier, this looks up the absolute pixel distance from the very top of the phone screen to the top edge (.minY) of the row.
  • / 5: This is a dampening factor. If a row is 400 pixels down the screen, rotating it by 400° would make it spin violently out of control. Dividing by 5 tames that number down to a gentle 80° rotation.

3. Putting it Together: The Perspective Illusion

Because this code lives inside a ScrollView, the value of .minY is constantly changing as the user drags their finger. Look at what happens to three different rows on your screen simultaneously:

  • A row near the top of the screen: Its .minY is close to 0.
    • 0° ÷ 5 = 0° rotation.
    • The row faces you completely flat.
  • A row in the middle of the screen: Its .minY is roughly 400.
    • 400° ÷ 5 = 80° rotation.
    • This row is swung heavily open like a door.
  • A row near the bottom of the screen: Its .minY is roughly 800.
    • 800° ÷ 5 = 160° rotation.
    • This row is twisted almost entirely backwards away from you.

Why it looks like "Depth"

As you scroll a row from the bottom of the screen to the top, its angle smoothly updates from 160° → 80° → 0°.

Because SwiftUI automatically applies perspective projection to 3D effects (meaning things that rotate away from you physically look smaller and narrower), your eyes perceive this smooth transition as a true depth effect. The rows look like they are unfolding from a 3D cylinder or spiral staircase as they enter the screen, flattening out perfectly just as they reach the top!

The Two Improvements

The two tweaks transition the 3D effect from a basic technical demo into polished, professional-looking UI animations. They fix two distinct visual issues by introducing a relative anchor point and changing the direction/axis of the scroll.

Let’s break down exactly how each tweak visually improves the experience.

Tweak One: The "Center Anchor" Math (- fullView.size.height / 2)

In the previous code, the rows only flattened out (0° rotation) when they hit the absolute top edge of the screen (.minY == 0). This looked a bit awkward because the rows spent the entire middle and bottom of your screen heavily twisted.

Look at how the math changes when we wrap the whole thing in an outer GeometryReader called fullView and change the formula: 

.rotation3DEffect(.degrees(proxy.frame(in: .global).minY - fullView.size.height / 2) / 5, axis: (x: 0, y: 1, z: 0))

...or across multiple lines: 

.rotation3DEffect(.degrees(proxy.frame(in: .global)
    .minY - fullView.size.height / 2) / 5,
          axis: (x: 0, y: 1, z: 0))

The Visual Improvement: Zeroing out at the center

By subtracting half of the screen's total height (fullView.size.height / 2), you are shifting the "sweet spot" of the animation to the exact vertical center of the screen.

  • A row at the dead center of the screen: Its .minY is exactly equal to half the screen height.
    • Midpoint−Midpoint=0.
    • Divided by 5 is 0°.
    • The row faces you perfectly flat right in the middle of the screen.
  • A row moving towards the top:
    • The math becomes negative (e.g., −200 ÷ 5 = −40°).
    • It twists to the left.
  • A row moving towards the bottom:
    • The math becomes positive (e.g., +200 ÷ 5 = +40°).
    • It twists to the right.

Why this is better:

Instead of a uniform twist that only resolves at the top, the rows now beautifully twist inward as they enter from the bottom, flatten out completely right where the user is looking (the center), and then twist outward as they exit the top. It creates a symmetrical, cylindrical "carousel" illusion.

Tweak Two: The Horizontal Cover Flow (-proxy.frame(in: .global).minX)

This tweak completely changes the layout orientation and the formula mechanics to create a horizontal card-flipper effect (similar to Apple's classic "Cover Flow"). 

ScrollView(.horizontal) {
    HStack {
        // ...
        .rotation3DEffect(.degrees(-proxy.frame(in: .global).minX) / 8, axis: (x: 0, y: 1, z: 0))

The Visual Improvement: Horizontal tracking with a negative flip

  • Switching Axes (.minX): Because the ScrollView is now horizontal, we no longer care about how far down the screen the view is (.minY). We care about how far across the screen it is. .minX tracks the left edge of each card as it slides horizontally.
  • The Negative Sign (-): Placing a minus sign in front of the proxy math inverts the rotation. As a card slides off the left side of the screen, instead of swinging open like a normal door, the negative sign forces it to twist backward into the screen background.
  • The / 8 Tame Factor: Dividing by 8 instead of 5 makes the rotation slightly more subtle and elegant, ensuring the text remains legible even while angled.

Why this is better:

As you swipe horizontally, the cards look like they are attached to a large spinning wheel deep inside your phone. Cards on the right face you more directly, and as you drag them to the left, they appear to seamlessly pivot away from you into the background, creating an excellent sense of 3D depth for a horizontal gallery.
