Swift - Location/Exif to Map

Ai - Untested

Extracting GPS coordinates embedded inside a photo and rendering them on a map is a classic iOS workflow.

To build this feature, your app needs to execute three discrete steps:

  • Let the user select an image using the SwiftUI PhotosPicker.
  • Extract the embedded EXIF metadata (specifically the GPS coordinate dictionary) out of the raw image data.
  • Feed that location into a SwiftUI Map view using the MapKit framework.

Here is a comprehensive, self-contained implementation blueprint demonstrating exactly how this pipeline is built.

1. The Core Infrastructure (Extracting GPS Data)

When a digital camera or iPhone snaps a photo, it injects a hidden dictionary of metadata into the image file called EXIF data. This metadata contains the camera settings, date, and—crucially—the GPS longitude and latitude.

To read this raw metadata dictionary without needing any invasive full-photo-library privacy permissions, use Apple's native ImageIO framework: 

import Foundation
import CoreLocation
import ImageIO

struct PhotoLocationExtractor {
    static func extractLocation(from data: Data) -> CLLocationCoordinate2D? {
        // 1. Convert raw Data into an unrendered image source pointer
        guard let source = CGImageSourceCreateWithData(data as CFData, nil) else { return nil }
        
        // 2. Extract the global metadata dictionary for index 0 (the primary image)
        guard let metadata = CGImageSourceCopyPropertiesAtIndex(source, 0, nil) as? [String: Any] else { return nil }
        
        // 3. Drill down specifically into the GPS sub-dictionary
        guard let gpsData = metadata[kCGImagePropertyGPSDictionary as String] as? [String: Any] else { return nil }
        
        // 4. Safely extract Latitude and Longitude values along with their direction vectors
        guard let latitude = gpsData[kCGImagePropertyGPSLatitude as String] as? Double,
              let latitudeRef = gpsData[kCGImagePropertyGPSLatitudeRef as String] as? String,
              let longitude = gpsData[kCGImagePropertyGPSLongitude as String] as? Double,
              let longitudeRef = gpsData[kCGImagePropertyGPSLongitudeRef as String] as? String else { return nil }
        
        // 5. Correct the coordinates to negative numbers if they are South or West
        let finalLatitude = (latitudeRef == "S") ? -latitude : latitude
        let finalLongitude = (longitudeRef == "W") ? -longitude : longitude
        
        return CLLocationCoordinate2D(latitude: finalLatitude, longitude: finalLongitude)
    }
}

2. Defining the Map Location Anchor

To pin the photo onto the SwiftUI Map, create a lightweight model conforming to Identifiable so the map loop layout can uniquely track it. 

import Foundation
import CoreLocation

struct PhotoAnnotation: Identifiable {
    let id = UUID()
    let coordinate: CLLocationCoordinate2D
    let name: String
}

3. Creating the Complete SwiftUI Interface

This unified view ties everything together. It presents the photo picker, uses the helper to scrape the GPS metadata, displays the selected image, and automatically updates a live interactive map view. 

import SwiftUI
import PhotosUI
import MapKit

struct PhotoMapConnectorView: View {
    // 1. PhotosPicker selection tracking state bindings
    @State private var selectedPickerItem: PhotosPickerItem? = nil
    @State private var loadedUIImage: UIImage? = nil
    
    // 2. Map view location and annotation states
    @State private var mapPosition: MapCameraPosition = .automatic
    @State private var mapAnnotations: [PhotoAnnotation] = []
    
    var body: some View {
        NavigationStack {
            VStack(spacing: 16) {
                
                // MAIN MAP ELEMENT
                Map(position: $mapPosition) {
                    ForEach(mapAnnotations) { marker in
                        Marker(marker.name, coordinate: marker.coordinate)
                            .tint(.blue)
                    }
                }
                .frame(height: 300)
                .clipShape(.rect(cornerRadius: 12))
                .padding(.horizontal)
                
                // INTERACTIVE PHOTO PREVIEW BOX
                if let loadedUIImage {
                    Image(uiImage: loadedUIImage)
                        .resizable()
                        .scaledToFill()
                        .frame(width: 140, height: 140)
                        .clipShape(.rect(cornerRadius: 12))
                        .shadow(radius: 4)
                } else {
                    ContentUnavailableView("No Photo Selected", systemImage: "photo.badge.plus", description: Text("Select an image with embedded GPS coordinates."))
                        .frame(height: 140)
                }
                
                // NATIVE PHOTOS PICKER TRIGGER BUTTON
                PhotosPicker(selection: $selectedPickerItem, matching: .images) {
                    Label("Choose Photo", systemImage: "photo")
                        .font(.headline)
                        .frame(maxWidth: .infinity)
                        .padding()
                }
                .buttonStyle(.borderedProminent)
                .padding(.horizontal)
            }
            .navigationTitle("Photo Mapper")
            // 3. Listen for changes when the user chooses an item
            .onChange(of: selectedPickerItem) { _, newItem in
                if let newItem {
                    processSelectedPhoto(newItem)
                }
            }
        }
    }
    
    // 4. Asynchronously stream data out of the system photo picker
    private func processSelectedPhoto(_ item: PhotosPickerItem) {
        Task {
            // Load the raw file bytes securely out of the Photo Library sandbox
            if let rawData = try? await item.loadTransferable(type: Data.self) {
                
                // Update image preview on the main thread
                if let uiImage = UIImage(data: rawData) {
                    self.loadedUIImage = uiImage
                }
                
                // Attempt to parse out location EXIF properties
                if let coordinate = PhotoLocationExtractor.extractLocation(from: rawData) {
                    
                    // Create a pin for the map
                    let newPin = PhotoAnnotation(coordinate: coordinate, name: "Photo Location")
                    self.mapAnnotations = [newPin]
                    
                    // Smoothly pan the map camera position to center on the photo coordinates
                    withAnimation {
                        let region = MKCoordinateRegion(
                            center: coordinate,
                            span: MKCoordinateSpan(latitudeDelta: 0.05, longitudeDelta: 0.05)
                        )
                        self.mapPosition = .region(region)
                    }
                } else {
                    print("⚠️ This image does not contain embedded GPS location metadata coordinates.")
                }
            }
        }
    }
}

Important Development Notes

  • Simulator Restrictions: If you test this on the Xcode iOS Simulator using default stock photos (like the pictures of flowers or waterfalls), they will fail to register a location. Those images do not contain GPS metadata. To test this feature properly, drag and drop a real photo taken by an iPhone (with Location Services active) straight into your simulator screen before running the app.
  • Privacy Considerations: The modern PhotosPicker runs out of process. This means your app does not need to ask the user for global "Photo Library Read Access" permissions in your Info.plist file. The user acts as the permission gatekeeper when they manually tap the photo.
