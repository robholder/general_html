Git - Comparing Branches

Let's take this repo and its branches as an example: 

git branch -vv
  macbook-intel 697ac86 fixed some truncation issues and added home icon
  macbook-m3    697ac86 [origin/macbook-m3] fixed some truncation issues and added home icon
  macmini       697ac86 [origin/macmini] fixed some truncation issues and added home icon
* main          697ac86 [origin/main] fixed some truncation issues and added home icon

1. The Quick Overview (git branch -vv)

The fastest way to see the status of all your local branches relative to their remote counterparts is: 

git branch -vv
  • What to look for: Look for labels like [ahead 1, behind 2] next to the branch name.
    • ahead: You have local commits (perhaps a new machine-specific palette) not yet pushed to GitHub.
    • behind: There are updates on GitHub (like a logic fix you "graduated" from another machine) that you haven't pulled yet.
    • diverged: You have both ahead and behind counts. This means you and the remote have "branched off" from a common point and both have unique work.

2. Comparing Machine Branches to main

To see if your macmini branch has diverged from your shared main branch, use the "triple-dot" diff or log:

  • See the commit difference: 
git log --oneline --left-right main...macmini
  • Commits marked with < are on main but missing from macmini.
  • Commits marked with > are unique to macmini (like its specific palette).
  • See the file difference: 
    git diff main...macmini --stat

    This shows a summary of which files are actually different between the two branches.

    3. Visualize the "Diamond" (git log --graph)

    If you prefer a visual "map" of how your branches have split and where they currently sit: 

    git log --oneline --graph --decorate --all

    This will show you exactly where macbook-m3 and macmini "split off" from main and if they are currently lagging behind it.
