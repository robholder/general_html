Git - Checkout Versions Using 'Switch' To New Branch

TL;DR

git switch -c <branch-name> HEAD~1

This is a fantastic, pro-level way to on a completed commit, by reverting to a previous version and safely practicing on a new branch. Using Git to create a safe "sandbox" branch from a past commit lets you code with a safety net, knowing your working solution is safely stored away.

The Crappy Way

While the command... 

git switch --detach HEAD~1

...followed by... 

git switch -c <branch-name>

...works perfectly fine to take you back in time, there is an even cleaner, more direct way to do this without getting caught in a "detached HEAD" state.

  • By The Way - What it does: This tells Git, "Take the exact commit I am currently standing on (even though it's detached) and turn it into the starting point for a brand new branch called ." It automatically hooks your HEAD up to this new branch so you are no longer detached.

Old Crappy Way

If you ever find yourself working in an environment with an older version of Git, or if you prefer the classic syntax, you can use checkout: 

git checkout -b <branch-name>
  • By The Way - What it does: This behaves exactly like Option 1. The -b flag stands for "branch" and forces Git to create the branch name right where you are standing and switch you onto it.

The Best Way: The One-Line Branch Shortcut

Instead of switching back to a detached state first, you can tell Git to create a new branch and switch to it immediately, pointing it directly at the previous commit (HEAD~1).

Run this command in your terminal: 

git switch -c <branch-name> HEAD~1

What this does:

  • -c <branch-name>: Creates a brand new branch named <branch-name> and switches you to it.
  • HEAD~1: Tells Git not to start the branch at your current working code, but instead to start it exactly one commit back in time (where you left off before the changes).

This keeps you safely on a real branch right from the start, so you don't have to worry about detached HEAD warnings.

Alternative: If you already ran the detach command

If you already typed 

git switch --detach HEAD~1

...and your terminal is currently warning you about being in a detached HEAD state, don't worry! You are just hovering in limbo.

To lock that position down into a safe branch, simply run: 

git switch -c <branch-name>

This instantly anchors your current "limbo" position into a real branch named .

How to Get Back to Your Working Code Later

Once you are done practicing line-by-line and want to return to the completed, working version of your app, you just need to switch back to your main development branch (usually named main or master): 

git switch main

Using this strategy means you can jump back and forth between your empty "" canvas and your completed "answer key" whenever you get stuck! It's one of the best ways to build muscle memory in iOS development.
