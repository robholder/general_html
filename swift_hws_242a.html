Hacking With Swift - 242a - Fixing 'Delete Selected' button

QUESTION

I have a 'delete selected' conditional item on this view, but it's appearing somewhat behind the tab row at the base. How do I make sure it's visible to the user

ANSWER

You've hit a classic layout friction point when building apps with a TabView.

When a view is nested inside a TabView, the placement .bottomBar creates a native bottom toolbar. However, iOS often renders this directly underneath or overlapping the background blur of the tab bar itself. Because the native TabView takes layout priority, any screen attempting to claim that real estate gets choked out.

To make sure your "Delete Selected" button is always clearly visible to the user and behaves cleanly, you have two great layout options.

Option 1: Move it to the Top Bar (Apple's Recommended Way)

From a pure iOS UX pattern perspective, global actions that affect list selections (like deleting or archiving) usually belong at the top of the screen next to the edit buttons.

Since you already have a topBarLeading and a topBarTrailing item, you can dynamically insert it into the top trailing section using an HStack. It will automatically hide when nothing is selected and appear smoothly when items are checked.

Replace your .toolbar block with this: 

.toolbar {
    ToolbarItem(placement: .topBarLeading) {
        EditButton()
    }
    
    ToolbarItem(placement: .topBarTrailing) {
        HStack(spacing: 16) {
            // Show the delete button first if selection exists
            if !selectedProspects.isEmpty {
                Button("Delete Selected", role: .destructive, action: delete)
                    .foregroundStyle(.red)
            }
            
            Button("Scan", systemImage: "qrcode.viewfinder") {
                isShowingScanner = true
            }
        }
    }
}

Option 2: Float a Custom Banner Above the Tab Bar (The Overlay Way)

If you specifically want it sitting at the base of the screen directly above the tab bar, do not use .bottomBar. Instead, attach an .overlay with a safeAreaInset to the List.

This injects a custom view container that sits perfectly on top of the list contents while forcing the list's bottom padding to adjust so your data never gets obscured behind your button.

Here is how to structure it: 

var body: some View {
    NavigationStack {
        List(prospects, selection: $selectedProspects) { prospect in
             // ... your existing row layout ...
        }
        .navigationTitle(title)
        // 1. Remove the .bottomBar item from the toolbar block entirely!
        .toolbar {
            ToolbarItem(placement: .topBarLeading) {
                EditButton()
            }
            ToolbarItem(placement: .topBarTrailing) {
                Button("Scan", systemImage: "qrcode.viewfinder") {
                    isShowingScanner = true
                }
            }
        }
        // 2. Add this safeAreaInset overlay onto the List
        .safeAreaInset(edge: .bottom) {
            if !selectedProspects.isEmpty {
                Button(role: .destructive, action: delete) {
                    Text("Delete Selected (\(selectedProspects.count))")
                        .bold()
                        .frame(maxWidth: .infinity)
                        .padding()
                        .background(.red)
                        .foregroundStyle(.white)
                        .clipShape(RoundedRectangle(cornerRadius: 12))
                }
                .padding(.horizontal)
                .padding(.bottom, 8) // Spaces it perfectly just clear of the TabBar height
                .transition(.move(edge: .bottom).combined(with: .opacity))
            }
        }
        .animation(.default, value: selectedProspects.isEmpty) // Smooth slide-up animation
        .sheet(isPresented: $isShowingScanner) {
            CodeScannerView(codeTypes: [.qr], simulatedData: "Paul Hudson\npaul@hackingwithswift.com", completion: handleScan)
        }
    }
}

Why Option 2 is a great fix

By using .safeAreaInset(edge: .bottom), SwiftUI calculates the precise height of the TabView beneath it and floats your red delete button directly on top of it. Adding the .animation modifier ensures that the moment a user checks an item, the delete panel smoothly slides up from the base of the screen like a native modal shelf.
