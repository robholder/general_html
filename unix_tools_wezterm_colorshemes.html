WezTerm - Switching Color Schemes

Adding a theme toggle is the perfect "finishing touch" for a custom terminal. Since you’ve already mastered the font toggle, we can use the same logic to cycle through different looks (e.g., a "Focus" dark mode, a "Vibrant" theme, or a "Light" mode for high-glare environments).

1. Define your Theme List

First, add a table of themes near the top of your wezterm.lua, right next to your my_fonts table: 

local my_themes = {
  "Catppuccin Macchiato",
  "Tokyo Night",
  "Kanagawa (Glow)",
  "Bamboo",
  "Nord",
  "Night Owl",
  "Github Light" -- Good to have one light mode just in case!
}

2. The Theme Toggle Keybind

Add this block into your config.keys. I've assigned it to <Ctrl><Shift>T (T for Theme): 

  {
    key = 't', mods = 'CTRL|SHIFT',
    action = wezterm.action_callback(function(window, pane)
      if wezterm.GLOBAL.theme_indices == nil then
        wezterm.GLOBAL.theme_indices = {}
      end
      
      local id = tostring(window:window_id())
      local current_index = wezterm.GLOBAL.theme_indices[id] or 1
      
      local next_index = current_index + 1
      if next_index > #my_themes then
        next_index = 1
      end
      
      local next_theme = my_themes[next_index]
      
      -- Get current overrides so we don't accidentally wipe out your font toggle!
      local overrides = window:get_config_overrides() or {}
      overrides.color_scheme = next_theme
      
      window:set_config_overrides(overrides)
      window:toast_notification("WezTerm", "Theme: " .. next_theme, nil, 2000)
      
      wezterm.GLOBAL.theme_indices[id] = next_index
    end),
  },

Important: Merging Overrides

You’ll notice a key difference here: I used window:get_config_overrides().

  • Why? If you toggle your Font, and then toggle your Theme, you don't want the theme toggle to "forget" your font choice.
  • The fix: This script grabs whatever is currently overridden (like your font), adds the new theme to that list, and then saves it all back.

Bonus: Visualizing the "Stack"

In WezTerm, your configuration acts like a stack of layers. Your wezterm.lua is the base, and set_config_overrides is the "live" layer on top.

How to browse themes without coding

If you want to find more theme names to add to your list, you can always use the built-in previewer:

  • Press Cmd + Shift + L
  • Type "Color Scheme"
  • Scroll through—when you find one you like, just add its name to your my_themes table.

QUESTION

I presume I have to have these themes installed. How do I do that?

Gemini said

Actually, you’re in luck: you don't have to install them!

One of the best things about WezTerm is that it comes "batteries included." It has a massive collection of over 700 built-in color schemes (including all the famous ones like Catppuccin, Tokyo Night, and Gruvbox) bundled directly into the binary.

How to see what's available

Since they are already on your system, you just need to know their exact names. You can see the full list and live-preview them by doing this:

  • Open WezTerm.
  • Press Cmd + Shift + L (Command Palette).
  • Type "color" and select "ActivateCommandPalette: Pick Color Scheme".
  • As you arrow up and down through the list, your terminal will change colors instantly.

Once you find one you love, note the exact name (like "AdventureTime", "NightFly", or "Subliminal") and add that string to your my_themes table in your config.

If you want a "Custom" theme from the web

If you ever find a theme online that isn't built-in (usually a .toml or .lua file), you would follow these steps:

  • Create a folder at ~/.config/wezterm/colors/.
  • Drop the theme file in there.
  • WezTerm will then treat that file's name as a valid color_scheme name.

Current Setup Check

Since you are using the "Toggle" script, just make sure the names in your my_themes table match the official names exactly.
