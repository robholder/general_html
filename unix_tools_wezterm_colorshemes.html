WezTerm - Switching Color Schemes

Adding a theme toggle is the perfect "finishing touch" for a custom terminal. Since you’ve already mastered the font toggle, we can use the same logic to cycle through different looks (e.g., a "Focus" dark mode, a "Vibrant" theme, or a "Light" mode for high-glare environments).

Where to Find Colors

The available color schemes in the current WezTerm release can be found here:

  • Wezterm Color Scheme Website

There are others you can install, like this Gogh site:

  • Gogh - Terminal Color Schemes

On this Gogh site, you run the following command in the terminal, and a list of color schemes are presented, and you choose by number which to install.

The installation failed at first for me. The shell was missing the $TERMINAL variable. I entered this first (to make a variable): 

export TERMINAL=wezterm

1. Define your Theme List

First, add a table of themes near the top of your wezterm.lua, right next to your my_fonts table: 

local my_themes = {
  "Catppuccin Macchiato",
  "Tokyo Night",
  "Earthsong",
  "Bamboo",
  "Nord (Gogh)",
}

2. The Theme Toggle Keybind

Add this block into your config.keys. I've assigned it to <Ctrl><Shift>T (T for Theme): 

  {
    key = 'c', mods = 'CTRL|SHIFT',
    action = wezterm.action_callback(function(window, pane)
      if wezterm.GLOBAL.theme_indices == nil then
        wezterm.GLOBAL.theme_indices = {}
      end
      
      local id = tostring(window:window_id())
      local current_index = wezterm.GLOBAL.theme_indices[id] or 1
      
      local next_index = current_index + 1
      if next_index > #my_themes then
        next_index = 1
      end
      
      local next_theme = my_themes[next_index]
      
      -- Get current overrides so we don't accidentally wipe out your font toggle!
      local overrides = window:get_config_overrides() or {}
      overrides.color_scheme = next_theme
      
      window:set_config_overrides(overrides)
      window:toast_notification("WezTerm", "Theme: " .. next_theme, nil, 2000)
      
      wezterm.GLOBAL.theme_indices[id] = next_index
    end),
  },

Important: Merging Overrides

You’ll notice a key difference here: I used window:get_config_overrides().

  • Why? If you toggle your Font, and then toggle your Theme, you don't want the theme toggle to "forget" your font choice.
  • The fix: This script grabs whatever is currently overridden (like your font), adds the new theme to that list, and then saves it all back.

Bonus: Visualizing the "Stack"

In WezTerm, your configuration acts like a stack of layers. Your wezterm.lua is the base, and set_config_overrides is the "live" layer on top.

QUESTION

I presume I have to have these themes installed. How do I do that?

Gemini said

Actually, you’re in luck: you don't have to install them!

One of the best things about WezTerm is that it comes "batteries included." It has a massive collection of over 700 built-in color schemes (including all the famous ones like Catppuccin, Tokyo Night, and Gruvbox) bundled directly into the binary.

How to see what's available

You can try any color scheme instantly using the Debug Overlay, which acts as a live Lua REPL for your current window. This is the fastest way to test a scheme you found on the WezTerm Color Schemes page without editing your config.

How to use the Debug Overlay:

  • Press CTRL + SHIFT + D to open the debug overlay.
  • Type the following command and hit Enter: 
window:set_config_overrides({ color_scheme = 'Batman' })
  • Replace 'Batman' with the name of the scheme you want to try.
  • The changes apply immediately to that specific window.
  • To close the overlay, press CTRL + SHIFT + L again or type exit.

Important Notes:

  • Temporary: These overrides only last until you close the window or reload your configuration (e.g., by saving your .wezterm.lua file).
  • Resetting: To revert to your configured theme, run this in the overlay: 
window:set_config_overrides({})

Finding Names: If you aren't sure of the exact internal name for a built-in scheme, you can list them all in the overlay by typing: 

wezterm.get_builtin_color_schemes()

QUESTION

I have a conflict to call schemes

ANSWER

Cross-platform compatibility is one of the best reasons to use WezTerm.

WezTerm provides a built-in variable wezterm.target_triple that tells us which operating system is currently running the config. We can use a simple Lua "if" statement to swap the modifier key dynamically.

The Cross-Platform Command Palette

Add this logic at the top of your config.keys list: 

-- Determine the "Super" key based on the OS
local platform_mod = "CMD"
if wezterm.target_triple:find("linux") then
  platform_mod = "ALT"
end

config.keys = {
  -- Your Command Palette shortcut using the dynamic modifier
  { 
    key = 'Enter', 
    mods = platform_mod, 
    action = wezterm.action.ShowLauncherArgs { flags = 'FUZZY|COMMANDS' } 
  },

  -- Your HJKL Resizing (keep these as they are)
  { key = 'l', mods = 'CTRL|SHIFT', action = wezterm.action_callback(function(window, pane) ... end) },
  -- ... other keys ...
}

Why use wezterm.target_triple?

The "target triple" is a technical string used by the Rust compiler (which WezTerm is built with). It looks like aarch64-apple-darwin for a Mac or x86_64-unknown-linux-gnu for Linux. By using :find("linux"), your config becomes "intelligent"—it adjusts itself the moment it loads on a different machine.
