Startship - Hostname & Custom Colors

While Starship doesn't support native Lua-style logic inside its starship.toml file, you can achieve the same result by using Custom Palettes or Environment Variables. Since Starship is a static configuration file (TOML), it cannot "calculate" a color based on your hostname on the fly like WezTerm's Lua engine. However, you can use one of these two methods to get the same effect:

Method 1: Using Custom Palettes (Recommended)

You can define multiple palettes in your starship.toml and then tell Starship which one to use by setting an environment variable in your .zshrc or .bashrc.

1. Update your starship.toml:

Put these at the end of the toml: 

# Define different palettes
[palettes.work_laptop]
accent = "#7aa2f7"

[palettes.home_desktop]
accent = "#9ece6a"

[palettes.default]
accent = "#bb9af7"

# Use the 'accent' variable in your modules
[directory]
style = "bold accent"

2. Update your shell config (.zshrc or .bashrc):

Add logic to your shell startup to pick the palette based on the hostname: 

if [[ $(hostname) == "work-laptop" ]]; then
    export STARSHIP_PALETTE="work_laptop"
elif [[ $(hostname) == "home-desktop" ]]; then
    export STARSHIP_PALETTE="home_desktop"
else
    export STARSHIP_PALETTE="default"
fi

Method 2: Direct Environment Variables

Starship allows you to use environment variables directly in your style strings. This is the most "WezTerm-like" way to do it.

1. Define the color in your shell config:

# Assign the color hex code to a variable based on host
if [[ $(hostname) == "work-laptop" ]]; then
    export MY_PROMPT_COLOR="#7aa2f7"
else
    export MY_PROMPT_COLOR="#bb9af7"
fi

2. Reference it in starship.toml:

[character]
success_symbol = "[$ ]($MY_PROMPT_COLOR bold)"

Summary of Differences

  • WezTerm: Handles the logic internally via Lua.
  • Starship: Requires the shell to handle the logic and pass the result in via an environment variable.

Both methods will default to a color if you provide a fallback in your shell's else statement.
