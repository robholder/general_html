Conditional "on/off" control for nodes based on frame number in Nuke

You can use expressions in Nuke to conditionally enable or disable a node based on the current frame number. This is useful for optimizing your script by disabling nodes that are not needed within a specific frame range.

Syntax for conditional disabling

To disable a node based on the frame number, you can use the following syntax in the node's disable knob expression:

  • Disable from a certain frame onwards: frame > X (where X is the frame number).
    • For example, frame > 79 will disable the node from frame 80 onwards.
  • Disable before a certain frame: frame < X (where X is the frame number).
    • For example, frame < 10 will disable the node before frame 10.

Example

Let's say you have a Blur node that you only want to be active between frames 50 and 100.

  • Select the Blur node.
  • Right-click on the disable knob and choose Add expression....
  • Enter the following expression: frame < 50 || frame > 100 ? 1 : 0.

This expression uses the logical OR operator (||) and a conditional statement (? :) to achieve the desired effect.

  • If the current frame is less than 50 or greater than 100, the condition is true, and the expression returns 1 (which means "disable").
  • Otherwise, the expression returns 0 (which means "enable").

Using comparison operators

You can use other comparison operators as well:

  • Equal to: frame == X
  • Not equal to: frame != X
  • Greater than or equal to: frame >= X
  • Less than or equal to: frame <= X

Other conditional expressions

You can also create more complex conditional expressions using logical operators like AND (&&) or NOT (!).

For example, to disable a node only on frame 50: frame == 50 ? 1 : 0.

Additional notes

  • When a node is disabled with an expression, it will have a single diagonal line across it in the Node Graph, indicating that the node has been disabled with an expression, according to a YouTube video.
  • When rendering outside the Nuke GUI (e.g., via command line), the disable expression using $gui will return 0 (meaning "enable"), according to Foundry Learn. This allows you to selectively bypass heavy nodes during interactive work without affecting the final render.
