Summary

  • Gig into the show gig
    • gig <show name> --perm
    • For Training ONLY:
    • gig OHIOTR
  • Navigate to the seqData lighting area of the sequence.
    • It is important that you type the entire navigation command, including lighting.
    • If you only type n <seq> seqData and then cd lighting, it will not set the correct environment variable and may cause issues for your session.
    • n <seq> seqData lighting
  • Run Scene Builder
    • scene_builder
  • Select the shot(s) or packet that is assigned to you.
    • Press "Show All" and check "Include OOP Shots" if you don't see your shots.
  • Open your shot
    • cd <user>/<shot or packet>/houdini
    • run_houdini lighting_v0001.hip

Creating and Placing Lights

Video Frame #: 22046

Shot/Packet Lighting

Light Box Approach

The Light Box node is a container for Light nodes.

  • In the 'Lights - Lightbox' backdrop area in the node graph:
    • Drop a Light Box node:
      • tab and type lightbox
      • The node name will be lightbox

Lightbox Node Panel

Video Frame #: 22993

The lightbox node has a parameter panel which is a template hierarchy.

Select a 'parent' container, and drop lights inside/below these.

Create lights in here, under the parent of appropriate headings (for example a new light called all_KEY_lgt:

The create icons are across the top of the panel:

  • quad
  • sphere
  • cone
  • etc.

A dialog will allow you to rename at creation.

Inside the Light Box node, the light nodes are being piped into the stream. 

lgtAsset:
  | matPalette:
      | asset_materials
      | volume_materials
lrigLOCAL:
    | matPalette:
        | lr_sunSky
        | lr_stage:
            | all_KEY_lgt
        | lr_practical
        | shot_modifactions

Lights - Broken Out Palettes Approach

Alternative to Light Box.

Viewport Lighting

Snapping Light To Geometry

Video Frame #: 31443

  • Select light in parameter panel:
    • Click on the transform icon (which appears next to the name when you hover over a light item in the panel).
  • In 3d view:
    • Select the magnet icon on the left vertical panel
    • Middle-click on any geo
    • Light should snap to that geo

Moving Lights

  • Move with manipulators:
    • Regular manipulators:
      • translate
      • rotate
    • Extra manipulators:
      • Select light in the parameter panel (using the transform handle on the light listing after name):
        • <Shift> L
      • You'll get extra controls:
        • target & rotation pivots
      • With the magnet on (right vert panel):
        • middle click on geo to snap the target.

Or:

  • Shelf tabs:
    • Lighting tools:
      • Look through selected
      • Lock viewport to camera
When going back to the shot cam, hit unlock (lock icon on r right vertical panel) to revert to the correct camera position. (Leaving the lock on doesn't change the shot camera's position data.)

Viewport Renders

Video Frame #: 39213

In viewport, look for the Persp dropdown, and choose:

  • Houdini VK:
    • No lighting.
  • Chimera:
    • Proprietary Houdini VK Replacement.
    • Turn on Scene lighting icon (right vert panel).
    • Turn off headlight icon (right vert panel).
    • Rough estimation of light influences.
  • ion:
    • It's a Render Preview (for lighting).
    • Estimate of full Renderer (it's a Hyperion proxy).
    • XGen not supported.
    • Materials are inaccurate.
  • Karma XPU
  • Karma

Interactive Rendering

Video Frame #: 52089

The render engine for interactive renders is Helios.

There is an interactiveRender node in Houdini that gets used for this (it floats in the node graph).

Parameters

Usual controls for Hyperion here in the parameters panel. For example:

  • Resolution
  • Features:
    • XGN
    • MBR
    • VOL
    • DOF
    • UNB
    • SSS
    • SUB
    • AUR
    • BON
    • IMG
These settings do not get passed to genuine renders and outputs.

Viewport Settings

Either use the side icons, or:

  • Click d in the viewport for the settings panel.
  • Useful to turn on/off volumes

There is a button to activate the node internals and show you the parameter panel:

  • Lighting Tool (shelf):
    • Interactive Render

If you make any changes (lighting) after the initiation of a render, as long as Helios is still active, you can click the Hyperedit button in the Helios parameters panel, or at the top of the Light Box panel, to push the changes to Helios.

To keep active

Video Frame #: 62026

Make sure the Edit > Preferences > Terminate on Completion checkbox is set to 'off'.

The progress area on the frame will be yellow if active.

Collections

Video Frame #: 72155

In order to make bindings to geometry/parts more manageable, you can create dpipe Collections.

dpipCollection

Video Frame #: 74780

There is a backdrop in the scene graph template, and collections can be organized here.

  • Create with tab and dpipCollection
  • Drop into the pipe flow

This node has a parameters panel (that is lifted from DLight). 

* shot
    * CulledElements
    * CulledMeshes
    * ElemsChar
    * ElemsComp
    * ElemsCrowd
    * ElemsFx
    * ElemsProp
    * ElemsSet
    * IndirectElems
    * IndirectMeshes

These default collections should be quite accurate to their descriptions.

Create your own by:

  • Clicking on the orange collection+ icon
  • Rename in the panel:
    • ch_all_CL for example
    • en_all_CL as another example

Adding items

  • Select the collection from the list.
  • Select items from the Stage Graph Tree or viewport.
  • In the Collections Bindings panel (at the base of the Collections panel):
    • Click the blue cube with the +

Selecting Collection Items

  • To select items in a collection, select the collection name in the panel:
    • Hit the orange collection icon with the cursor motif.

Render Pass Modifiers - dpipeModifier

Video Frame #: 79135

This is the procedure to set visibility and influence in the render passes.

Add Modifier

On the pass backdrop networks (ch_all, en_all, etc.):

  • Drop a dpipeModifier after the top nodes of the block:
    • Change Template drop down to a mode type:
      • Example: hiddenFromCamera (or whichever you need)
    • The name of the node will change to reflect your choice
    • Open the Assignments tab in the new modifiers parameters panel

Add Collections to the Modifier

Video Frame #: 83790

To add items (Collections) to the Targets of the modifier:

  • Select a collection in the Scene Graph Tree panel:
    • click the + in a circle button (tiny) in the hiddenFromCamera panel
    • For example:
      • Add chars collection to en_all hiddenFromCamera
      • Add env to collection ch_all hiddenFromCamera

ContextOptionsEditor Panel

This panel is used to set which passes to render in Helios (the interactive renderer).

Setup:

In the Scene Graph Tree panel:

  • Click on the + next to the panel title (Scene Graph Tree)
  • New Pane Tab >
    • Solaris > Context Options Editor

You can set the active passes here (for interactive rendering only) with the renderpass dropdown.

Rendering to the Farm

Video Frame #: 86610

Render Submitter panel

Launch:

  • Lighting Tools shelf:
    • Click on Render Submitter button

Dialog

This will show you shots in the 'packet' in a seq/scene/packet/shot hierarchy.

On the right of the panel are the overrides.

  • Select the shot pass on the tree (left)
  • Override settings on the parameter panel (right)
These overrides can be set on the modifier nodes; so choose wisely where to do them!

To render:

  • Select the passes in the submitter hierarchy/tree (left)
  • Hit Submit to Queue
  • Confirmation dialog should appear
  • Check in CODA

CODA

Video Frame #: 90467

Coda is available in a browser at:

  • coda.disneyanimation.com

You'll see a hierarchy of 'jobs' for your submit:

  • Job ID:
    • 1 - lighting_setup: <shot>
    • 2 - Submitting shot: <shot>

When the job is actually submitted by the 2 item, the actual render will appear in CODA.

  • Example:
    • OHIO-teaser-010.00-lighting: SubmitJobs

Inside that node you'll see the processes:

  • Render layers (ch_all, en_all, etc.)
  • Crossfilter passes
  • utility_ renders
  • Metrics
  • etc.

Renders

Video Frame #: 100189

All files go to the 'temp' directory.

  • Example:
    • /disney/shows/OHIO/work/Sequences/seq/<seq>/seqData/lighting/render/recover/<job_id> followed by checkpoint directories.

Houdini worked in seqData, but the compositing is done in the conventional directory.

To navigate:

  • n teaser <shot> lighting
  • Example:
    • /disney/shows/OHIO/work/Sequences/seq/<seq>/<shot>/lighting
  • cd pic:
    • current
    • old
    • temp

All renders go into temp, and we run a tool called manage_pics.

  • current -> old
  • temp -> current
  • This allows us to work with current while rendering to temp.

Inside these pic folders are the 'pass' directories:

  • ch_all
  • en_all
  • hy_all_utility
  • etc.

Example path: .../<seq>/<shot>/lighting/pic/temp/chall/1920x804/mono

  • color
  • geometry
  • lighting
  • mattes
  • variance

Manage Pics

Video Frame #: 107527

manage_pics is a CLI tool to manage the location of renders:

In the command line:

  • manage_pics

You'll get a printout of all the relevant passes, and the option:

  • The above will REPLACE current. Continue? [Y/n]

If you say Y, they will be transferred and replaced.

    • Lighting > updatenuke

Breakout

Video Frame #: 116756

The Read nodes are usually NOT connected to anything. We use 'instances'.

To make individual instances, select read node:

  • On the property panel:
    • Create Instance button
  • Keyboard shortcut:
    • <Shift> i

To make whole merge system:

  • On the property panel:
    • Break Out

Shadowbox

Video frame #: 119371

Best approach to break out passes:

  • Open Shadowbox tab in Nuke.
  • Switch to OHIO
  • Open the DEFAULT hierarchy:
    • EmissionCompBreakout or EmissionCompBreakout_FastLoat
    • then Import button
  • In the GUI:
    • Layer order is reversed, but this orders the comp:
      • Bottom item is top in the Nuke graph.
    • Select passes (and move up or down)
    • Set passes to instantiate
    • Set merge type
    • Set Backdrop styles
    • Select additional node options:
      • Emission CC Nodes
      • Final Comp Recipe:
        • CF_OhioFinalAdjustents
      • and optionals like:
        • Emission Sticky Notes
      • etc.
    • Hit Breakout button.
