Challenge 1

  • Allow the user to switch map modes, between the standard mode and hybrid.

Ai Insight

QUESTION

I want to add a 'map' options button on my Map() view inside my app.

My idea is to perhaps have an emum with .imagery, .hybrid etc, and be able to have a popup to choose setting. Is this possible in the SwiftUI app?

ANSWER

Yes, this is absolutely possible! In fact, SwiftUI's MapKit framework makes changing map styles incredibly easy using the .mapStyle modifier.

To build this beautifully, we can create an enum for your map styles, track the active style with a @State variable in your ViewModel, and use a SwiftUI Menu button to display a clean popup list of choices right on top of your map.

Here is how to implement this step-by-step.

Step 1: Update your ViewModel

Open your ViewModel class and add an enum to represent your style options, along with a variable to hold the currently selected style.

Using an enum that conforms to String and CaseIterable allows us to easily loop through all choices to build our picker menu automatically. 

@Observable
class ViewModel {
    // 1. Define your custom map style choices
    enum MapStyleOption: String, CaseIterable, Identifiable {
        case standard = "Standard"
        case imagery = "Satellite"
        case hybrid = "Hybrid"
        
        var id: String { self.rawValue }
    }
    
    // 2. Track the active style selection
    var selectedStyle: MapStyleOption = .standard
    
    // ... your existing properties (locations, selectedPlace, etc.)
}

Step 2: Create a View Helper (Optional but Clean)

Because your enum holds your custom app settings, we need a quick way to convert your enum cases into actual, native MapKit MapStyle structs.

Add this helper computed property to your enum inside the ViewModel: 

// Add this right inside your MapStyleOption enum:
var nativeStyle: MapStyle {
    switch self {
    case .standard: return .standard(elevation: .realistic)
    case .imagery:  return .imagery(elevation: .realistic)
    case .hybrid:   return .hybrid(elevation: .realistic)
    }
}

Step 3: Update ContentView with the Style Modifier and Menu Button

Now, we wrap your map inside a ZStack. This allows us to overlay a gorgeous floating map options button in the top-right corner of the screen, mimicking native Apple apps. 

var body: some View {
    if viewModel.isUnlocked {
        ZStack(alignment: .topTrailing) { // ZStack aligns the button to the top-right
            
            MapReader { proxy in
                Map(initialPosition: startPosition) {
                    ForEach(viewModel.locations) { location in
                        // ... your existing Annotation code
                    }
                }
                // 1. Apply the active map style directly to the Map view!
                .mapStyle(viewModel.selectedStyle.nativeStyle)
                .onTapGesture { position in
                    // ... your existing tap gesture code
                }
                .confirmationDialog("Manage Location", isPresented: $showingOptions, presenting: viewModel.selectedPlace) {
                    // ... your existing dialog buttons
                }
                .sheet(item: $placeToEdit) { place in
                    // ... your existing sheet code
                }
            }
            
            // 2. The Floating Map Options Menu Button
            Menu {
                Picker("Map Style", selection: $viewModel.selectedStyle) {
                    ForEach(ViewModel.MapStyleOption.allCases) { option in
                        Text(option.rawValue).tag(option)
                    }
                }
            } label: {
                Image(systemName: "map.circle.fill")
                    .font(.title)
                    .foregroundStyle(.primary)
                    .background(.white)
                    .clipShape(.circle)
                    .shadow(radius: 4)
                    .padding() // Keeps it safely away from the screen edges
            }
        }
    } else {
        // ... your existing unlock button layout
    }
}

How It Works

  • ZStack(alignment: .topTrailing): This layers your UI elements on top of each other. The map becomes the bottom layer, and the Menu button is pinned perfectly to the upper-right corner.
  • Menu + Picker: SwiftUI automatically renders a specialized context menu when you wrap a standard Picker inside a Menu element. Tapping the map icon brings up a smooth iOS popup menu right beneath the button.
  • .mapStyle(): The moment a user selects an option in the picker menu, viewModel.selectedStyle updates. SwiftUI notices the state change, recalculates .mapStyle(), and the map instantly morphs from flat vector lines into rich satellite photography!

We will drop the MapStyleOption enum and its helper into your ViewModel file, wrap your MapReader inside a ZStack in your ContentView, and swap out your fixed .mapStyle(.imagery) for the dynamic layout variable.

QUESTION

One issue is that now the map style drop-down menu sits directly on the existing compass icon. Can we separate them in screen space?

ANSWER

What you are seeing with the overlapping icons is very common. By default, MapKit drops its native utility overlays (like the Compass and the Zoom/Scale controls) directly into the top corners of the screen context. When we drop a floating Menu button into the exact same .topTrailing slot of our ZStack, they collide.

To fix this natively in SwiftUI, we can use the mapControls modifier. This lets us do two things at once:

  • Hide MapKit's automatic compass overlay.
  • Control exactly where it appears by turning it into a regular view element that we can layout side-by-side with our custom menu button!

The Layout Fix

Instead of relying on the ZStack alignment to position our menu, we will change the ZStack alignment to a neutral position, build a dedicated vertical spacing container (VStack), and place both the native MapCompass and our Menu button inside it.

Here is how your updated ContentView.swift layout block should look: 

var body: some View {
    //if viewModel.isUnlocked {
    ZStack { // 1. Changed to a standard neutral ZStack
        MapReader { proxy in
            Map(initialPosition: startPosition) {
                ForEach(viewModel.locations) { location in
                    // ... your existing Annotation code remains identical
                }
            }
            .mapStyle(viewModel.selectedStyle.nativeStyle)
            // 2. THIS IS THE MAGIC FILTER: It tells the map to hide the default compass overlay
            .mapControls {
                // Leaving this blank completely hides all native controls!
            }
            .onTapGesture { position in
                if let coordinate = proxy.convert(position, from: .local) {
                    viewModel.addLocation(at: coordinate)
                }
            }
            .confirmationDialog("Manage Location", isPresented: $showingOptions, titleVisibility: .visible, presenting: viewModel.selectedPlace) {
                // ... your existing dialog code remains identical
            }
            .sheet(item: $placeToEdit) { place in
                // ... your existing sheet code remains identical
            }
        }
        
        // 3. NEW FLOATING UTILITY COLUMN
        // We push this container up and out to the top-right using Spacers
        VStack {
            HStack {
                Spacer() // Pushes items to the right side
                
                VStack(spacing: 12) { // Vertically stacks the controls with a clean gap
                    
                    // A. NATIVE COMPASS CONTROL (Now behaves as an explicit view element!)
                    MapCompass()
                        .mapControlVisibility(.automatic) // Only displays if the map is actually rotated
                    
                    // B. YOUR CUSTOM STYLE MENU BUTTON
                    Menu {
                        Picker("Map Style", selection: $viewModel.selectedStyle) {
                            ForEach(ViewModel.MapStyleOption.allCases) { option in
                                Text(option.rawValue).tag(option)
                            }
                        }
                    } label: {
                        Image(systemName: "map.circle.fill")
                            .font(.system(size: 44))
                            .foregroundStyle(.primary)
                            .background(.white)
                            .clipShape(.circle)
                            .shadow(radius: 4)
                    }
                }
                .padding(.trailing, 12) // Safe padding distance away from the right screen edge
                .padding(.top, 60)      // Safe padding distance down to avoid hitting the iPhone notch / status bar
            }
            Spacer() // Pushes the entire row assembly to the top of the screen
        }
    }
}

Why this works so beautifully

  • .mapControls { }: By declaring an empty control configuration block on your Map, you effectively cut off the automatic generation layout logic. This prevents Apple from drawing its own rogue compass over your design.
  • MapCompass() View: MapKit exports its core utilities as modular standalone views. By putting MapCompass() directly into your layout hierarchy, it still maintains all its complex internal logic (spinning, pointing north, disappearing when the map is flat, and resetting orientation when tapped), but it obeys your custom VStack layout spacing guidelines!

Give this a try, and watch how cleanly the compass moves over to stack up neatly above your custom selector button.

EditView.swift

import SwiftUI

struct EditView: View {
    
    enum LoadingState {
        case loading, loaded, failed
    }
    
    @Environment(\.dismiss) var dismiss
    var location: Location
    
    @State private var name: String
    @State private var description: String
    var onSave: (Location) -> Void
   
    @State private var loadingState = LoadingState.loading
    @State private var pages = [Page]()
    
    var body: some View {
        NavigationStack {
            Form {
                Section {
                    TextField("Place name", text: $name)
                    TextField("Description", text: $description)
                }
                
                Section("Nearby…") {
                    switch loadingState {
                    case .loaded:
                        ForEach(pages, id: \.pageid) { page in
                            //Text("**\(page.title)**: *Page description here*")
                            Text("**\(page.title)**: *\(page.description)*")
                        }
                    case .loading:
                        Text("Loading…")
                    case .failed:
                        Text("Please try again later.")
                    }
                }
            }
            .navigationTitle("Place Details")
            .toolbar {
                Button("Save") {
                    var newLocation = location
                    newLocation.id = UUID()
                    newLocation.name = name
                    newLocation.description = description
                    
                    onSave(newLocation)
                    dismiss()
                }
            }
        }
        .task {
            await fetchNearbyPlaces()
        }
    }
    
    init(location: Location, onSave: @escaping (Location) -> Void) {
        self.location = location
        self.onSave = onSave
        
        _name = State(initialValue: location.name)
        _description = State(initialValue: location.description)
    }
    func fetchNearbyPlaces() async {
        let urlString = "https://en.wikipedia.org/w/api.php?ggscoord=\(location.latitude)%7C\(location.longitude)&action=query&prop=coordinates%7Cpageimages%7Cpageterms&colimit=50&piprop=thumbnail&pithumbsize=500&pilimit=50&wbptterms=description&generator=geosearch&ggsradius=10000&ggslimit=50&format=json"
        
        guard let url = URL(string: urlString) else {
            print("Bad URL: \(urlString)")
            return
        }
        
        do {
            let (data, _) = try await URLSession.shared.data(from: url)
            
            // we got some data back!
            let items = try JSONDecoder().decode(Result.self, from: data)
            
            // success – convert the array values to our pages array
            //pages = items.query.pages.values.sorted { $0.title < $1.title }
            pages = items.query.pages.values.sorted()
            loadingState = .loaded
        } catch {
            // if we're still here it means the request failed somehow
            loadingState = .failed
        }
    }
}

Location.swift

import Foundation
import MapKit

struct Location: Codable, Equatable, Identifiable {
    var id: UUID
    var name: String
    var description: String
    var latitude: Double
    var longitude: Double
    
    // Computed Property
    var coordinates: CLLocationCoordinate2D {
        CLLocationCoordinate2D(latitude: latitude, longitude: longitude)
    }
    
    // Return 'equal' if IDs are the same.
    static func ==(lhs: Location, rhs: Location) -> Bool {
        lhs.id == rhs.id
    }
    
    #if DEBUG
    // Example Location
    static let example = Location(
        id: UUID(),
        name: "Buckingham Palace",
        description: "Lit by over 40,000 lightbulbs.",
        latitude: 51.501,
        longitude: -0.141
    )
    #endif
}

Result.swift

import SwiftUI


struct Result: Codable {
    let query: Query
}

struct Query: Codable {
    // dictionary with Int as Key, Page as value
    let pages: [Int: Page]
}

struct Page: Codable, Comparable {
    let pageid: Int
    let title: String
    // dictionary with String as key and a String array as value (optional also)
    let terms: [String: [String]]?
    
    var description: String {
        terms?["description"]?.first ?? "No further information"
    }
    
    // The '<' function allows for parameterless '.sorted()' method
    static func <(lhs: Page, rhs: Page) -> Bool {
        lhs.title < rhs.title
    }
}
