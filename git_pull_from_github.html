Cloning and Pulling a Repo From GitHub

Here is a quick-reference guide for downloading an existing repository from GitHub to your Mac and managing your daily development workflow with pulls and pushes.

🧭 Quick Reference: The Workflow Commands

Whenever you need a quick reminder of the terminal commands, use this sequence: 

# 1. Clone a repository to your Mac (Only done once per project)
git clone git@github.com:USERNAME/REPO_NAME.git

# 2. Check the status of your files at any time
git status

# 3. Pull down the latest changes from other contributors
git pull origin main

# 4. Stage your changes, commit them, and push them back up
git add .
git commit -m "Your descriptive commit message ⚙️"
git push origin main

🛠️ Step 1: Cloning a Remote Repository

Cloning downloads a complete copy of the GitHub repository (including all files, folders, and its entire version history) and sets up the remote link automatically.

  • Go to the repository page on GitHub.
  • Click the green < > Code button.
  • Select the SSH tab and copy the provided URL string (it will look like git@github.com:username/repo-name.git).
  • Open Terminal on your Mac, navigate to where you want the project folder to live, and run the clone command: 
    # Navigate to your development directory
cd ~/Documents/Projects

# Clone the repository
git clone git@github.com:USERNAME/REPO_NAME.git
    Note: This creates a brand-new folder named after the repository inside your current directory. Make sure to step into that new folder before running any more Git commands: cd REPO_NAME.

🔄 Step 2: The Daily Routine (Pull, Commit, Push)

To keep your local project in sync with GitHub without running into conflicts, follow this standard three-step loop whenever you sit down to work:

1. Always Pull First (Stay Synced)

Before you write or modify any code, pull down any changes that might have been pushed to GitHub from another computer or team member. This prevents "divergent history" headaches later. 

git pull origin main

2. Make Your Changes and Commit Locally

Work on your files as normal. Once you reach a good stopping point, save your work to your local Git history: 

# Check what files you have modified
git status

# Stage all changes for the commit
git add .

# Create a local snapshot of your changes
git commit -m "Refined image sizing variables and updated padding logic ✨"

3. Push to GitHub

Once your local commits are ready, send them back up to your hosted repository on GitHub so they are safely backed up and visible to your team or your other machines: 

git push origin main

⚠️ Dealing with the "Branch Named Master" Legacy

If you are cloning an older repository created several years ago, the primary branch might be named master instead of main.

If your git pull or git push commands give you an error saying the branch cannot be found, run git branch to see what your local active branch is named. If it says master, simply swap out the word main in your routines: 

git pull origin master
git push origin master
