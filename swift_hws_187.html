Hacking With Swift - 187 - Challenge 1

Challenge 1

  • Try making the Slider and Change Filter buttons disabled if there is no image selected.

Disable Button

You use the .disabled() modifier to disable a button in SwiftUI.

Pass a boolean condition directly into this modifier. When the condition evaluates to true, the button becomes unclickable and visually greys out.

The Standard Pattern

swiftstruct ContentView: View {
    @State private var username = ""
    
    var body: some View {
        VStack {
            TextField("Enter username", text: $username)
            
            Button("Submit") {
                print("Form submitted")
            }
            // Disables the button if the text field is empty
            .disabled(username.isEmpty) 
        }
    }
}

3 Ways to Handle Conditions

  • Direct Boolean:
    • .disabled(isFormInvalid) (Disables when true)
  • Inverted Boolean:
    • .disabled(!isLoggedIn) (Disables when false by using the ! operator)
  • Comparison Expression:
    • .disabled(cartCount < 1) (Disables based on math or logic)

Quick Styling Tip

SwiftUI automatically dims disabled buttons. If you want to apply custom styling (like changing the background color) when a button is disabled, pair it with an environment variable or your state property: 

swift.background(username.isEmpty ? Color.gray : Color.blue)

You apply the .disabled() modifier directly to the end of the button declaration.

Here is the exact code showing how to disable your button based on a condition: 

Button("Change Filter", action: changeFilter)
    .disabled(isLoading) // Disables the button when 'isLoading' is true

Real-World Examples

1. Disable when a specific filter is already active

Button("Change Filter", action: changeFilter)
    .disabled(currentFilter == .sepia)

2. Disable while processing data

Button("Change Filter", action: changeFilter)
    .disabled(isProcessingImage)

3. Disable if no image is selected

Button("Change Filter", action: changeFilter)
    .disabled(selectedImage == nil)

Dimming Buttons

You can dim the button by using the .opacity() modifier combined with a ternary operator (condition ? trueValue : falseValue).

SwiftUI automatically lowers the opacity of the text inside a disabled button, but using .opacity() allows you to dim the entire button—including any custom background colors or borders.

The Code

Button("Change Filter", action: changeFilter)
    .disabled(isLoading)
    // Dims the button to 50% opacity when loading, otherwise keeps it at 100%
    .opacity(isLoading ? 0.5 : 1.0)

Add a Smooth Fade Animation

To make the transition look professional instead of snapping instantly, wrap the state change inside a withAnimation block in your code, or add the .animation() modifier directly to the button: 

Button("Change Filter", action: changeFilter)
    .disabled(isLoading)
    .opacity(isLoading ? 0.5 : 1.0)
    // Smoothly animates the opacity changes
    .animation(.default, value: isLoading)

In Context

ContentView.swift

import CoreImage.CIFilterBuiltins
import PhotosUI
import StoreKit
import SwiftUI

struct ContentView: View {
    @State private var selectedItem: PhotosPickerItem?
    @State private var processedImage: Image?
    @State private var filterIntensity = 0.5
    
    // We need a context to convert a CIImage -> CGImage
    // and a filter to apply
    // Core Image Context is an object for rendering a CIImage to a CGImage
    // Recipe -> image
    let context = CIContext()
    // Place holder to update:
    //@State private var currentFilter = CIFilter.sepiaTone()
    // Updated to set type to CIFilter, rather then CIFilter.sepiaTone only.
    // Now we lose acdess to the 'intensity' property.
    @State private var currentFilter: CIFilter = CIFilter.sepiaTone()
    
    // Keep beginImage in a var to prevent having to re-select everytime
    // the filter is selected.
    //@State private var beginImage: CIImage?
    
    @State private var showingFilters = false
    
    // Counter to count how many times the user is changing filter
    @AppStorage("filterCount") var filterCount = 0
    // Set up Request Review environment
    @Environment(\.requestReview) var requestReview

    var body: some View {
        NavigationStack {
            VStack {
                Spacer()
                
                // Wrap image secion with picker closure
                PhotosPicker(selection: $selectedItem) {
                    // Unwrap optional (processedImage)
                    if let processedImage {
                        processedImage
                            .resizable()
                            .scaledToFit()
                    } else {
                        // Show 'unavailable' screen if no image selected
                        ContentUnavailableView(
                            "No Picture",
                            systemImage: "photo.badge.plus",
                            description: Text("Tap to import a photo")
                        )
                    }
                }
                // Stop the 'No Picture' view from turning active blue
                .buttonStyle(.plain)
                .onChange(of: selectedItem, loadImage)

                Spacer()
                
                HStack {
                    // Add slider control (filter intensity)
                    Text("Intensity")
                    Slider(value: $filterIntensity)
                        // Add 'onChange' to watch for live changes of intensity slider
                        .onChange(of: filterIntensity, applyProcessing)
                }
                .padding(.vertical)
                
                HStack {
                    // Add filter button, but extract function outside of layout
                    Button("Change Filter", action: changeFilter).disabled(selectedItem == nil)
                    
                    Spacer()
                    }
                    
                    // Share the picture
                    if let processedImage {
                        ShareLink(item: processedImage,
                            preview: SharePreview("Instafilter image",
                                image: processedImage)
                        )
                    }
                }
            }
            .padding([.horizontal, .bottom])
            .navigationTitle("Instafilter")
            // Dialog to choose filter
            .confirmationDialog(
                "Select a filter",
                isPresented: $showingFilters) {
                    Button("Crystallize") { setFilter(CIFilter.crystallize()) }
                    Button("Edges") { setFilter(CIFilter.edges()) }
                    Button("Gaussian Blur") { setFilter(CIFilter.gaussianBlur()) }
                    Button("Pixellate") { setFilter(CIFilter.pixellate()) }
                    Button("Sepia Tone") { setFilter(CIFilter.sepiaTone()) }
                    Button("Unsharp Mask") { setFilter(CIFilter.unsharpMask()) }
                    Button("Vignette") { setFilter(CIFilter.vignette()) }
                    Button("Cancel", role: .cancel) { }
                }
    }
    // Move button action here (out of layout)
    func changeFilter() {
        showingFilters = true
     }
    
    func loadImage() {
        Task {
            // Load selected image as pure 'data' and not an image type.
            guard let imageData = try await selectedItem?.loadTransferable(
                type: Data.self) else { return }
            // Convert data to image
            guard let inputImage = UIImage(data: imageData) else { return }
            
            // Convert UIImage to CIImage
            let beginImage = CIImage(image: inputImage)
            currentFilter.setValue(beginImage, forKey: kCIInputImageKey)
            // Apply the chosen filter
            applyProcessing()
        }
    }
    
    func applyProcessing() {
        // Due to compatibiltiy with this old code, we have
        // to use 'Float' instead of 'Double'
        // Once we change the @State, we lose access to .intensity
        //currentFilter.intensity = Float(filterIntensity)
        // Instead, we feed our filterIntensity var into a Key
        //currentFilter.setValue(filterIntensity, forKey: kCIInputIntensityKey)
        // This works for sepiaTone, but not for others, and it crashes
        
        // Fix:
        let inputKeys = currentFilter.inputKeys
        
        // Map filterIntensity to kCIInputIntensityKey
        if inputKeys.contains(kCIInputIntensityKey) {
            currentFilter.setValue(
                filterIntensity,
                forKey: kCIInputIntensityKey
            )
        }
        // Map filterIntensity to kCIInputRadiusKey and multiply by 200
        if inputKeys.contains(kCIInputRadiusKey) {
            currentFilter.setValue(
                filterIntensity * 200,
                forKey: kCIInputRadiusKey
            )
        }
        // Map filterIntensity to kCIInputScaleKey and multipty by 100
        if inputKeys.contains(kCIInputScaleKey) {
            currentFilter.setValue(
                filterIntensity * 100,
                forKey: kCIInputScaleKey
            )
        }

        
        // Capture outputImage (after filtering)
        guard let outputImage = currentFilter.outputImage else { return }
        // Convert to actual pixels
        guard let cgImage = context.createCGImage(outputImage, from: outputImage.extent)
                else { return }
        
        let uiImage = UIImage(cgImage: cgImage)
        processedImage = Image(uiImage: uiImage)
    }
    
    // Set main actor so that this runs on the main thread
    @MainActor
    func setFilter(_ filter: CIFilter) {
        currentFilter = filter
        loadImage()
        
        // Track usage to guage when to trigger Review Request
        filterCount += 1
        
        // Ask for Review
        if filterCount >= 20 {
            requestReview()
        }
    }
}
