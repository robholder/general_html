Types

There are two main types of star possible in a shot:

  • Aurora (procedural):
    • Named: auroraPrototypes
  • Hero:
    • wishBubble(#)

( Use the Hierarchy filter to drill down to see part if necessary.)

Structure

  • outer shell
  • inner volumes:
    • motes
    • shards
  • externalLight
  • internalLight

Variants

  • Clouded (main variant: as this document)
  • Reveal (not established yet)

Pass Creation

TDs are working on a pass wizard.

Meanwhile:

  • Create Default renderpass
  • Use mods to control bubble lights:
    • From Shadowbox > usd file > wishBubble_MODS)
      • Brings in mods and bindings (hero and procedural)
    • Treat Mod like a light (exposure, emission etc.)
    • No color - controlled by FX drivers:
      • Lighters adjust exposure/intensity for the most part
    • Use crossfilter (quip)
  • colorMattes:
    • all in the import
  • Create Utility pass:
    • 2k
    • forwardCameraTag_Off_Utility (not clear how this is set, because it's duplicated from utility pass)
    • Confirm in demo scene if necessary:
      • /disney/shows/WISH/work/Sequences/seq/190.0/016.00/lighting/comp/fullcomp_v0001_wishBubbleDemo.nk
    • No crossfilter (quip)

Render outputs:

  • Add all outputs (lights) for lights you use:
    • esp. 40 thru 46
    • Nuke has channel limit of 1023, so only expand lights you NEED:
      • leave just beauty in other words

Comp/Nuke:

Breakout

  • Wish bubbles?
  • 40 to 46 emissions are bubble emissions
  • Later emission numbers have spec relfs in there
  • you can remove unused emission read nodes

Depthmatte

  • There's a depth matte:
    • use for depth variation in comp

BetterBloom

  • Adjust for screen size

Adjust Munging

  • Based on screen depth/space
  • Dial to taste

CC'ing

  • Color corrections done in CC'ing area

Edge Pooling

  • Get some pooling for the interior
  • Harder at the center
  • Softer at the edges

Ambient Occlusion (AO)

  • Useful if bubbles are close to eachother

Spec

  • Direct spec mostly
  • You might have to switch instance read:
    • directSpecular
    • specular
    • remainder

Chromatic aberration

  • Adjust based on screen space

Linework

  • Goes AFTER the bubble work:
  • Same as other elements
  • Lines only on outer edges

Notes

  • Core > transparent
  • Edges > opaque
  • Color mattes for facing-ratio/edge matting:
    • adjust for shot
  • Distortion:
    • Only if needed
    • plug in environment
    • uses iDistort
    • Very little distortion is the rule!
