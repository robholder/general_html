Undo Tree

How do I see a tree of undos in Vim?

The Undo Tree

Vim has an undo tree, which allows you to navigate through different branches of undo history rather than just a linear undo/redo stack. If your text has disappeared and u (undo) says there are no undos left, you might still be able to recover it using Vim’s undo tree commands.

Checking the Undo Tree

List all undo branches Run: 

:undolist

This will show a list of available undo states, each with a number. You can jump to a specific undo state using: 

:undo <number>

...where is one of the IDs from :undolist.

Navigate through the undo tree

  • g- moves backward through the undo tree (to previous branches).
  • g+ moves forward through the undo tree (to later branches). These commands allow you to recover text that might have been lost due to branching undo history.

Use time-based undo If you remember when the text was last present, you can go back in time: 

:earlier 10m

This moves the buffer state 10 minutes earlier. You can also use: 

:later 5m

...to move forward in time.

Persistent Undo (If Enabled)

If you have persistent undo enabled, Vim stores undo history even after closing the file. You can check if it’s enabled with: 

:set undofile?

If it’s off, you won’t be able to recover undo history after closing Vim.

Undo Tree Visualization

If you want a graphical representation of the undo tree, you can use the UndoTree plugin (Undotree), which provides a visual way to navigate undo branches.
