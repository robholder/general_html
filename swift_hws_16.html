Hacking With Swift - 16 - How to use the ternary conditional operator for quick tests

One last way to test conditions in Swift

Ternary Conditional Operator

This is similar to Nuke's expression conditionals: 

let age = 18

let canVote = age >= 18 ? "Yes" : "Now"

WTF

  • What? - what to test
  • True - What to do if true
  • False - What to do if false

Examples

let hour = 23
print(hour < 12 ? "It's before noon" : "It's after noon")

let names = ["Jayne", "Kaylee", "Mal"]   
let crewCount = names.isEmpty ? "No one" : "\(names.count) people"
print(crewCount)

It gets a little hard to read when your condition use == to check for equality, as you can see here: 

enum Theme {
    case light, dark
}

let theme = Theme.dark

let background = theme == .dark ? "black" : "white"
print(background)
  • What? theme == .dark
  • True: “black”
  • False: “white”

If you want (to make the conditional clearer) you can put the 'test' in parenthises: 

enum Theme {
    case light, dark
}

let theme = Theme.dark

let background = (theme == .dark) ? "black" : "white"
print(background)

If we wanted to write that out using if and else we’d either need to write this invalid code:

This is NOT allowed!! (Note the error) 

print(
    if hour < 12 {                  // Expected expression in list of expressions
        "It's before noon"
    } else {
        "It's after noon"
    }
)

To fix you could run print() twice, like this: 

if hour < 12 {
    print("It's before noon")
} else {
    print("It's after noon")
}

...but the 'ternary conditional operator' is more concise.
