Trace Commands

In most shells (such as Bash, Zsh, or even the Fish shell), you have several tools at your disposal to determine which commands are aliases. Here are a few common methods:

  • Using the alias Command: If you simply type 
alias

...the shell will list all current aliases and what command each one is tied to .

  • Using the type Command: To find out whether a specific command is an alias, function, builtin, or external command, use: 
type mycommand

For example, if you run... 

type ll

...and it shows something like: 

ll is aliased to `ls -l --color=auto'

...then you know that ll is indeed an alias.

  • Using the command -v Command: This prints the path or details about the command and can indicate if itâ€™s an alias. However, in many cases, command -v may not clearly indicate that something is an alias, so type is often more informative.
  • Using compgen -a (Bash-specific): This built-in command lists all aliases known to the shell: 
compgen -a

These tools allow you to quickly check what your shell is doing and confirm whether a command you've encountered is actually a defined alias. Each method can be useful depending on whether you want an overview of all aliases or details about a specific command.
