Starship - zshrc Settings

This is a snapshot of what I have in my .zshrc file. This takes into account NerdFonts also: 

#--------------------------------- Add Paths ----------------------------------#

# Option 1: Explicit add Path
#export PATH="$HOME/.local/bin:$PATH

# Option 2: Test before adding Path
[[ -d "$HOME/.local/bin" ]] && export PATH="$HOME/.local/bin:$PATH"

#---------------------------- Check for Nerd Fonts ----------------------------#
# --- Nerd Font Diagnostic ---
# Tests for the 'nf-fa-folder' icon (f115)
# If the terminal doesn't recognize it, it usually renders it as a 1-character '?' 
# Nerd Fonts render this as a 2-character wide glyph in most modern terminals.
check_nerd_font() {
  # We use a character from the Nerd Font 'Private Use Area' (e.g., \ue0b0 - Powerline Arrow)
  # Instead of measuring width, we check if the terminal handles it as a multi-byte UTF-8 char
  # while ensuring it doesn't fall back to a 1-byte placeholder.
  local test_char=$(printf "\ue0b0")
  
  # If the shell is Zsh, we can use the (%V) expansion to see how the terminal sees it.
  # If it shows as <e0b0> or \uE0B0, it's not being rendered.
  if [[ "${(%):-%V}" == *"$test_char"* ]]; then
     return 1
  fi

  # Secondary check: If the icon and a space together are still measured as '1' 
  # by certain terminal logic, it's likely a non-rendered glyph.
  if [[ ${#test_char} -lt 1 ]]; then
    return 1
  fi

  return 0
}

if ! check_nerd_font; then
  echo -e "\n\033[0;33m⚠️  Starship Warning: Nerd Font not detected!\033[0m"
  echo -e "Icons may look like [?] or blocks. To fix this, run:"
  echo -e "\033[0;34mbrew install --cask font-jetbrains-mono-nerd-font\033[0m"
  echo -e "Then, set your Terminal font to 'JetBrainsMono Nerd Font'.\n"
fi
#------------------------------------ END -------------------------------------#


#-------------------------------- Add Starship --------------------------------#

# # Set config from default of ~/.config to ~/.config/starship/
export STARSHIP_CONFIG=~/.config/starship/starship.toml

if [ -n "$TMUX" ]; then
  ## Inside Tmux, load this starship config:
  export STARSHIP_CONFIG=~/.config/starship/starship_tmux.toml
  ## Factory Settings prompt:
  #export STARSHIP_CONFIG=/dev/null
else
  # NOT in Tmux
  export STARSHIP_CONFIG=~/.config/starship/starship.toml
fi
#
# Prevent starship interfering with vi mode
if [[ "${widgets[zle-keymap-select]#user:}" == "starship_zle-keymap-select" || \
      "${widgets[zle-keymap-select]#user:}" == "starship_zle-keymap-select-wrapped" ]]; then
    zle -N zle-keymap-select "";
fi

# # Start Starship
eval "$(starship init zsh)"

# Restrict Starship to only print fatal errors by setting its logging level to error. 
# (Effectively silences [WARN] errors.)
export STARSHIP_LOG=error

Suppressing [WARN] Messages

Ai Insight

QUESTION I get this now and again in Starship environment: 

[WARN] - (starship::context): Scanning current directory timed out.
[WARN] - (starship::context): You can set scan_timeout in your config to a higher value to allow longer-running scans to keep executing.

...how can I suppress these (largely) useless errors?

ANSWER

To permanently suppress those warnings, you have two main options: silence warning logs completely via an environment variable, or increase/disable the scan timeout in your Starship configuration.

Option 1: Silence Warning Logs Globally (Recommended)

By default, Starship prints [WARN] messages directly to your standard error when operations run a bit long. You can restrict Starship to only print fatal errors by setting its logging level to error.

Add this line to your shell configuration file (~/.zshrc or ~/.bashrc): 

export STARSHIP_LOG=error

(Then reload your shell with source ~/.zshrc or open a new terminal window).

Option 2: Raise or Disable scan_timeout

If you'd rather fix the underlying cause (Starship taking more than its default 30ms limit to scan directories with many files or over network/slow drives), you can adjust the threshold in your starship.toml configuration file.

  • Open ~/.config/starship.toml (create it if it doesn't exist).
  • Add one of the following lines at the very top of the file (outside any [module] sections): 
    # Option A: Give it more time (e.g., 500ms instead of 30ms)
scan_timeout = 500

# Option B: Disable the scan timeout completely
scan_timeout = 0
