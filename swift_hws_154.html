Hacking With Swift - 154 - Deleting from a SwiftData query

We already used @Query to place SwiftData objects into a SwiftUI List, and with only a little more work we can enable both swipe to delete and a dedicated Edit/Done button.

Just as with regular arrays of data, most of the work is done by attaching an onDelete(perform:) modifier to ForEach, but rather than just removing items from an array we instead need to find the requested object in our query then use it to call delete() on our model context. Once all the objects are deleted, SwiftData's autosave system will kick in and apply the changes permanently.

So, start by adding this method to ContentView: 

func deleteBooks(at offsets: IndexSet) {
    for offset in offsets {
        // find this book in our query
        let book = books[offset]

        // delete it from the context
        modelContext.delete(book)
    }
}

We can trigger that by adding an onDelete(perform:) modifier to the ForEach of ContentView, but remember: it needs to go on the ForEach and not the List.

Add this modifier now: 

.onDelete(perform: deleteBooks)

That gets us swipe to delete, and we can go one better by adding an Edit/Done button too. Find the toolbar() modifier in ContentView, and add another ToolbarItem: 

ToolbarItem(placement: .topBarLeading) {
    EditButton()
}

That completes ContentView, so try running the app - you should be able to add and delete books freely now, and can delete by using swipe to delete or using the edit button.

AI Insights

The deleteBooks function is the "cleanup crew" for the List. To understand it, we have to look at how SwiftUI handles the user's physical gesture of swiping to delete.

1. What is at offsets?

In Swift, at is an argument label. It makes the code read like a sentence when you call the function elsewhere.

  • Inside the function, you use the name offsets.
  • Outside the function, you would call it like this: deleteBooks(at: someIndexSet).

The word offset is just a programmer's way of saying "position" or "distance from the start." If you have a list of 5 books, the first book is at offset 0, the second is at offset 1, and so on.

You could rename them almost anything you like, and the code would still work, as long as you use the new names consistently. However, there is a "Swift standard" happening here that makes these specific names very common.

A "Real World" Comparison

If you were writing a function to eat a box of chocolates: 

func eatChocolates(from box: [Chocolate]) {
    for piece in box {
        // eat piece
    }
}
  • box is your user name for the collection (like offsets).
  • piece is your user name for the individual item (like offset).
  • [Chocolate] is the Type (like IndexSet).

2. What is an IndexSet?

An IndexSet is like a collection of bookmarks.

Imagine you have a stack of 10 books. If I tell you, "Go pick up books 1, 3, and 5," those three numbers (1, 3, 5) together form an IndexSet.

Why not just a single number?

SwiftUI allows for "multi-select" deletion. Even if you only swipe one row, SwiftUI sends that single position inside an IndexSet container just in case there are more.

3. Walking through the Code

Let’s trace what happens when you swipe to delete the 3rd item in your list: 

func deleteBooks(at offsets: IndexSet) {
    // 1. The loop begins. 'offsets' contains the number [2] 
    // (remember, we count from 0, so the 3rd item is index 2).
    for offset in offsets {
        
        // 2. We use that '2' to look into our 'books' array.
        // It's like saying: "Hey array, give me the book sitting at chair #2."
        let book = books[offset]

        // 3. Now that we have the specific 'Book' object in our hand,
        // we tell the ModelContext to remove it from the database.
        modelContext.delete(book)
    }
}

4. The "Magic" Connection

You might be wondering: How does this function actually get triggered?

Usually, you connect it to your ForEach in the ContentView like this: 

ForEach(books) { book in
    // row content
}
.onDelete(perform: deleteBooks)

When you swipe a row, the .onDelete modifier automatically calculates exactly which row you swiped and passes that position into your deleteBooks function as an IndexSet.

The "Wait, what about saving?" Catch

In your current code, you tell the modelContext to delete the book. Depending on how your app is set up, SwiftData might "autosave" that change immediately, or you might need to call try? modelContext.save() if you want to be absolutely sure it's gone from the disk.

ContentView.swift

import SwiftData
import SwiftUI


struct ContentView: View {
    @Environment(\.modelContext) var modelContext
    @Query var books: [Book]
    
    @State private var showingAddScreen = false
    
    var body: some View {
        NavigationStack {
            List {
                ForEach(books) { book in
                    NavigationLink(value: book) {
                        HStack {
                            EmojiRatingView(rating: book.rating)
                                .font(.largeTitle)
                            
                            VStack(alignment: .leading) {
                                Text(book.title)
                                    .font(.headline)
                                Text(book.author)
                                    .foregroundStyle(.secondary)
                            }
                        }
                    }
                }
                // Add delete functinoality (call func)
                .onDelete(perform: deleteBooks)
            }
            .navigationTitle("Bookworm")
            .navigationDestination(for: Book.self) { book in
                DetailView(book: book)
            }
            .toolbar {
                ToolbarItem(placement: .topBarTrailing) {
                    Button("Add Book", systemImage: "plus") {
                        showingAddScreen.toggle()
                    }
                }
                // Add an edit button for the book list
                ToolbarItem(placement: .topBarLeading) {
                    EditButton()
                }
            }
            .sheet(isPresented: $showingAddScreen) {
                AddBookView()
            }
        }
    }
    
    func deleteBooks(at offsets: IndexSet) {
        for offset in offsets {
            // find this book in our query
            let book = books[offset]
            
            // delete it from the context
            modelContext.delete(book)
        }
    }
}
