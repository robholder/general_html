getcompletion()

The getcompletion() function in Vim is a powerful tool used to retrieve a list of 'completions' for a specific context. I became aware of it when trying to harvest a list of available colorschemes. Here's a breakdown of how it works:

Syntax: 

getcompletion({arg}, {type})

Parameters:

  • {arg}:
    • This is a string that specifies the argument or prefix for which completions are sought.
    • For example, an empty string ('') retrieves all possible completions for the specified type.
    • Think of this as a filter; so 'se' would filter all results and show only 'set', 'sequence' etc.
  • {type}:
    • This is a string that specifies the type of completion you want to generate.
    • Examples of types include:
  • color: Retrieves a list of available colorschemes.
  • command: Retrieves available commands.
  • option: Retrieves Vim options.
  • tag: Retrieves tags for the current file.
  • Other supported types can be found in the documentation (:help complete()).

Return Value:

The function returns a list of possible completions based on the given arg and type.

Use Case: Get all Available Colorschemes

To get a list of all available colorschemes: 

:echo getcompletion('', 'color')

This will return a list containing the names of all colorschemes installed in your Vim environment: 

:let @a = join(getcompletion('', 'color'), ', ')
  • getcompletion('', 'color'): Retrieves all available colorschemes.
  • join(..., ', '): Combines the list into a single string, separating items with ', '.:
    • You could use '\n' here for a new line after each list item.
