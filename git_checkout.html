Checking out a Commit

# To checkout a commit; and populate the working directory with a snapshot of the commit:
git checkout <commit_ID>

# WARNING: This will trigger a 'detached HEAD' warning dialog in the shell.

# This warning can be disabled with:
git config advice.detachedHead false
# or for ALL repositories on your system:
git config --global advice.detachedHead false

# To suppress this warning ONLY with the current command:
git -c advice.detachedHead=false checkout <commit_ID>
# or
git checkout --quiet <commit_ID>

Detached HEAD - sounds bad..

It kind of is...

  • A detached HEAD means you are no longer on a branch. Gulp.
  • Remember that a HEAD location is the indicator as to where a commit is made. (The HEAD becomes the 'parent' of the commit.)
  • Making a commit in this state will create an un-referenced branch.
  • To recover, you'll need to have recorded the commit IDs

MORAL: Always create a new branch before checking out a commit!

Reverting a Checkout

No problem: Switching back to the previous or other branch will cancel the checkout without leaving a trace: 

# To leave a 'checkout' state:
git switch -
# or
git switch <branch-name>

Creating a Branch after Checkout

# To create a new branch after a checkout:
git switch -c <new-branch-name>
