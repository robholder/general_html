Commands

git init - initialize a new repo in a directory
git add - move file(s) to the staging area
git commit -a -m "commit message" - commit all modified and tracked files in on command (bypass separate 'git add' command)

(Only the message of the last commit of a branch can be edited.)

git commit --amend -m "message" - edit the commit message of the last commit on the current branch. (Clean working dir only to do this!!)
git merge <branch-name> - merge branches (fast-forward and 3-way merges)
git merge --abort - abort a merge during a merge conflict situation

git status

git status - see the state of files in working tree, staging area vs latest commit in git history
git status -s - see the state of files in summary form (??-untracked, A-added, M-modified). Two columns, working and staged.

git log

git log - git history of the current branch
git log --all --decorate --oneline --graph - commit history graph

git log --pretty=oneline --abbrev-commit - shorter prettier output for log (from Head First Git - O'Reilly)
git log --onelne - This is the equivalent of the line above!! (from Head First Git - O'Reilly)
git log -p or --patch - patch shows you a diff message
git log --graph - show commit graph
git log -S <string term> - s-Pickaxe - search for additions and deletions of search string
git log -G <string term> - g-Pickaxe - search for lines if the search string is on the line of a diff
git log --grep <string term> - search through commit messages for the search string
git log - view the git history / git commit graph

git diff

  • git diff compares the working and index (staged) versions by default

git diff - diff of working tree and staging area
git diff --cached - diff of staging area and latest commit
git diff --staged - diff of staging area and latest commit (same result as above)
git diff <master> <SDN> - diff between 2 branches
git diff --word-diff - diff individual words (rather than lines). (Head First Git - O'Reilly)
git diff <hash_id target> <hash_id source> - diff between two commits (using hash IDs). First is old, second is recent (to get - & + correct).

git diff HEAD~1 HEAD - diff compare the current commit (HEAD) and 'Head' minus '1' on the current branch.

  • HEAD: Each increment is another further back in the commit history on the branch. This case casts the 'previous' commit (parent) as 'a' and current as 'b'. Note: HEAD~ is the same as HEAD~1. (Head First Git - O'Reilly)

git diff HEAD^1 HEAD^2 - diff compare parent one (on same branch as current) with parent two (from the merged branch)
git diff HEAD~1 HEAD^2~2 - The carat ('^') is parent number, so this compares the current branch immediate parent with the immediate parent on the merged branch plus two more back along the merged path.

  • Note: HEAD~1 on a merged commit refers to the parent commit on the merged-to branch.
  • Note: There is no way to specify 'forward' in commit history. Only parents are recorded, not children.

Compare two commits

git diff HEAD~1 HEAD            # This shows differences between previous and current commit.

git diff master origin/master   # To compare a branch with a 'remote branch'

git config

git config --global user.name "name" (--local option as well)
git config --global user.email "email" (--local option as well)
git config --global difftool.prompt false - prevents the prompt from asking if you want to open vimdiff
git config --global diff.tool vimdiff - simple setting of vimdiff for diffs
git config --global difftool.vimdiff.cmd 'vimdiff "\(LOCAL" "\)REMOTE"' - sets vimdiff as the difftool of choice (in write mode)

  • $LOCAL is starting state
  • $REMOTE is ending state
  • I had difficulty in getting the settings to get set in the .gitconfig file, but this is the desired effect: 
    [difftool]
	prompt = false
[difftool "vimdiff"]
	cmd = vimdiff \"$LOCAL\" \"$REMOTE\"
[diff]
	tool = vimdiff

git checkout

git checkout -- <filename> - retrieve a file from the staging area into the working tree
git checkout <commit hash> <filename> - retrieve a file from a previous commit

See Restore below - it replaces Checkout

git checkout <commit-hash> - checkout a commit directly, not through a branch, results in a detached HEAD state
git checkout <branch-name> - checkout a branch/move head pointer

git rm

git rm - remove a file from the working tree and the staging area
git rm -f - force the removal of a file if it differs from the committed version (modified)
git rm -r - remove a directory from the working tree and the staging area (from Head First Git - O'Reilly)
git rm --cached - remove from git but not the filesystem (on disk)
git rm --dry-run - git rm -n - don't run for real, but show what WOULD be removed

# Remove example:
git rm log/\*.log

# The backslash is to prevent the double expansion of the asterisk (git and the shell)
# The command deletes all '.log' files in a directory called 'log'.

git mv

git mv <orig file> <moved file> - move files (same as a regular 'mv', 'git rm' and a 'git add')

git branch & switch

git branch <branch-name> - create a branch
git switch <branch-name> - updated version of the 'checkout' command above (from Head First Git - O'Reilly)
git switch -c <new branch name> - create and switch to a new branch name (from Head First Git - O'Reilly)

git branch - show branches and which is active (asterisk)
git branch -a - show branches including remote branches (if remote repo exists)
git branch -r - show branches on remote repo (if remote repo exists)
git branch --merged - see branches merged into the current branch
git branch -d <branch-name> - delete a branch, only if already merged
git branch -D <branch-name> - delete a branch, including if not already merged (exercise caution here)
git branch -m <new_name> - rename a branch from current branch using the -m flag. (From Head First Git - O'Reilly)
git branch -m <old_name> <new_name> - rename a branch from any branch using the -m flag. (From Head First Git - O'Reilly)

git checkout <commit-hash> - checkout a commit directly, not through a branch, results in a detached HEAD state
git checkout <branch-name> - checkout a branch/move head pointer

Restore

  • Git - Restore - Restoring File from Previous Commit
Restore is the new version of checkout:
  • When there is a staged/index version (un-commited)

git restore <my_file(s)> - It deletes the staged version (nothing in staged). The working tree file is unchanged. (From Head First Git - O'Reilly)

git restore --staged <my_file(s)> - It 'restores' the staged or indexed version to the commit. This is like unstaging a file. (From Head First Git - O'Reilly)

Next two from https://www.git-tower.com/learn/git/commands/git-restore/

git restore --source <commit ID> <my_file> - restore to the version with the specified commit id.
git restore --source <main~2 (or #)> <my_file> - restore to the version two (or #) commits from the given branch's current version.

Reset - reverts back (commits)

  • Git - Reset - Resetting to a Previous Commit

git reset HEAD <filename> - retrieve a file from the latest commit into the staging area
git reset HEAD~1 - reverts to the previous commit.

Combining Restore and reset

  • To revert to a previous commit:
    • git restore <file_name>
    • git reset <file_name>

git stash

git stash - create a stash point
git stash list - list stash points
git stash list -p - list stash points and show diffs per stash
git stash apply - apply most recent stash
git stash pop - apply most recent stash, and remove it from saved stashes
git stash apply <stash reference> - apply a specific stash point
git stash save "<description>" - create a stash point, be more descriptive

Blame Game!

  • Git - Reflog, Bisect, Explore & Search

git blame <file_name> - to show what's changed, by whom and at what commit:

Examine the what, who, and when a file was edited 

# to show what's changed, by whom and at what commit:
git blame <file_name>

# The default is to look at HEAD:
git blame HEAD <file_name>
# ...is the same as:
git blame <file_name>
 
# to show at a given commit hash:
git blame <commit ID> <file_name>

Flags for Blame

# Suppress time stamps:
git blame -s <file_name>
