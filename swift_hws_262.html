Hacking With Swift - 262 - Challenge 3

Challenge 3

  • For a harder challenge: when the users gets an answer wrong, add that card back into the array so the user can try it again. Doing this successfully means rethinking the ForEach loop, because relying on simple integers isn’t enough - your cards need to be uniquely identifiable.

Ai Insight

QUESTION

I'm attempting to address the third challenge by including an 'id' parameter (UUID) to the Card struct.

This is what I've done for the Card struct: 

import Foundation

struct Card: Codable, Identifiable {
    var id: UUID
    var prompt: String
    var answer: String
    
    static let example = Card(
        id: UUID(),
        prompt: "Who played the 13th Doctor in Doctor Who?",
        answer: "Jodie Whittaker"
        )
}

However, I now need to alter my ContentView to deal with the new way of identifying each card. Right now we rely on the cards' index in the list. I guess we need to track the 'id' instead.

ANSWER

You are exactly on the right path. Because Card now conforms to Identifiable, you have the power to stop thinking in fragile array indices (0, 1, 2) and start working directly with the unique cards themselves.

The core issue with using indices (0..<cards.count) for a dynamic array like this is that if a user swipes a card away or inserts one back into the middle, the positions change instantly. SwiftUI gets confused trying to animate card index 3 if it suddenly becomes card index 4.

Let’s re-work the ForEach loop and update your removal tracking logic.

Step 1: Fix the ForEach Loop

Instead of counting through a range of indices, we can now loop through your cards array directly because every card has a unique id.

Find your existing ForEach inside the central ZStack: 

// Old way using an index range
ForEach(0..<cards.count, id: \.self) { index in
    CardView(card: cards[index]) {
        withAnimation {
            removeCard(at: index)
        }
    }
    // ...
}

Replace that entire loop block with this structure: 

ForEach(cards) { card in
    // Find the current position of this specific card in the stack
    let index = cards.firstIndex(of: card) ?? 0
    
    CardView(card: card) {
        withAnimation {
            // Pass the specific card to our removal method
            removeCard(card: card) 
        }
    }
    .stacked(at: index, in: cards.count)
    .allowsHitTesting(index == cards.count - 1)
    .accessibilityHidden(index < cards.count - 1)
}

Note: To use cards.firstIndex(of: card), make sure your Card struct conforms to Equatable. If you haven't added that yet, change your card definition to: struct Card: Codable, Identifiable, Equatable { ... }

Step 2: Update the Accessibility Buttons

Since your accessibility layout checks the very top card using cards.count - 1, we need to change those button actions too. Instead of passing an index, we find the exact card currently at the top of the deck.

Look at your two fallback accessibility buttons (Wrong and Correct). Update their withAnimation blocks to target the specific card at the top: 

// Inside your accessibility button block:
Button {
    withAnimation {
        // Find the top card and remove it
        if let topCard = cards.last {
            removeCard(card: topCard)
        }
    }
} label: {
    Image(systemName: "xmark.circle")
}

(Do the same for your checkmark.circle button!)

Step 3: Rewrite removeCard to handle "Wrong" answers

Right now, your removal function completely deletes the card. To complete Paul's challenge, we want to know why it's being removed. If the user got it wrong, we need to push it back to the bottom of the pile!

Let's modify removeCard to take the unique Card instance and a boolean flag determining if it should be sent back to the deck: 

// Replace your old removeCard(at:) function with this:
func removeCard(card: Card, reinsertIfWrong: Bool = false) {
    // 1. Find where this card currently sits in our array
    guard let index = cards.firstIndex(of: card) else { return }
    
    // 2. Temporarily remove it from its current position
    cards.remove(at: index)
    
    if reinsertIfWrong {
        // 3. For a harder challenge: stick a fresh copy at the very beginning (bottom of stack)
        // We create a copy with a new UUID so SwiftUI views it as a completely brand new entry!
        let recycledCard = Card(id: UUID(), prompt: card.prompt, answer: card.answer)
        cards.insert(recycledCard, at: 0)
    }
    
    if cards.isEmpty {
        isActive = false
    }
}

Step 4: Connecting the dots back in CardView

To make reinsertIfWrong work, your CardView needs to pass a signal back letting ContentView know whether the user swiped right (Correct) or left (Wrong).

Right now, your removal closure inside CardView looks like this: 

var removal: (() -> Void)? = nil

Change it to expect a boolean value representing success/failure: 

var removal: ((Bool) -> Void)? = nil

Then, down in your CardView drag gesture's .onEnded: 

.onEnded { _ in
    if abs(offset.width) > 100 {
        // If swiped right (offset > 0), they got it right! 
        // If swiped left (offset < 0), they got it wrong, so pass 'false'.
        let isCorrect = offset.width > 0
        removal?(isCorrect)
    } else {
        offset = .zero
    }
}

Finally, back in ContentView, update your main view loop to handle that incoming boolean value: 

CardView(card: card) { isCorrect in
    withAnimation {
        // If they got it wrong (isCorrect is false), we want to reinsert it!
        removeCard(card: card, reinsertIfWrong: !isCorrect)
    }
}

By generating a unique UUID() when putting the wrong card back, SwiftUI treats it as a totally fresh element, preventing any animation glitches where it awkwardly flies backward.
