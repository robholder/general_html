Restructure a Repo

Here’s a concise, step-by-step process to completely re-order your repository’s file and directory structure locally so that the changes are reflected when you push to GitHub:

Plan Your New Structure:

  • Outline the changes you’d like to make (e.g., moving files into new directories, renaming folders) so you know what moves to perform.

Use git mv for Trackable Moves:

  • Use the git mv command to move and rename files and directories. For example: 
    git mv old_dir new_dir
git mv new_dir/old_file.txt new_dir/new_file.txt
  • Using git mv ensures that Git recognizes these changes as moves, preserving history where possible.

Manually Move Files (Optional):

  • If you need to move things with your file manager, you can do that too, but you must then stage the changes: 
    git add -A
  • Git will detect moved files as deletes and adds, which is fine though not as neat as using git mv.

Review the Changes:

  • Run git status to check the list of changes. Confirm that files are showing up as “renamed” or “moved” as expected.

Commit Your Reorganization:

Commit the changes with a descriptive commit message: 

git commit -m "Completely re-order repository file/directory structure"

Push Your Changes:

Once committed, push the changes to your remote repository: 

git push

This will update the GitHub repository with the new file and directory structure.

Post-Push Verification:

  • Optionally, verify on GitHub (or by cloning the repo into a new directory) that the changes have been applied successfully.
  • Following these steps will allow you to safely re-order your repository’s structure locally, with Git tracking all the moves, so that when you push, GitHub reflects your updated organization.
