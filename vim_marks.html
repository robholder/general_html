Marks

Creating Marks

In current buffer

  • m<name> --> Mark current position
    • e.g. ma

In specific file/buffer

  • m<capitalized_name> --> Mark current position in a specific file/buffer
    • Uppercase marks (e.g. mA) preserve both position AND filename, so you can use them to move across different buffers

'Go To' Marks

  • '<name> --> Go to marked line e.g. 'a)
  • <backtick><name> --> Go to marked position (e.g. <backtick>a)

Shortcuts to Jump

  • '' --> Go to most recent mark (line)
  • <backtick><backtick> --> Go to most recent mark (exact position)
Vim automatically sets marks prior to each “jump.” So these commands are a common way to “go back to where I just was before this"
  • [' --> Go backwards through marks in buffer (line)
  • ]' --> Go forwards through marks in buffer (line)
  • [<backtick> ] --> Go backwards through marks in buffer (exact position)
  • <backtick> --> Go forwards through marks in buffer (exact position)

'Go To' Last Changes - Step by Step

Vim also keeps track of the locations of the “changes” that you make in the file, allowing you to navigate places you’ve been in your recent editing activity:

  • g; --> Go to previous edit location (e.g. “What was I doing before this?”)
  • g, --> Go to next edit location (e.g. “What was I doing after this?”)
Observe how g; and g, mirror a similar idea in ; and , which are used in f t searches, so you can think of them in the same way.

'Go To' Last Changes - Numbered Marks (0-9)

  • 0-9: These marks represent positions from the last 10 edits made to the file. For example, the mark 2 represents the position of the second-to-last edit:
    • e.g. '2

Previously Selected (Visual Mode)

  • Angle Bracket Marks (< and >):
    • '< --> This mark represents the beginning of the previously selected Visual mode text.
    • '> --> This mark represents the end of the previously selected Visual mode text.
    • For example, if you visually select a block of text and then exit Visual mode, you can quickly jump back to the beginning or end of that selection using the < or > marks.

Original Notes

m[lowercase letter] <-- mark current location of cursor to a letter.
[backtick][lowercase letter] <-- (backtick letter)return to mark stored to a letter.

mn <-- marks current position into 'n'
[backtick]n <-- (backtick - letter) returns to position in 'n'
'n <-- (single-quote letter) returns to first non-blank character in line of 'n' location
Lower case marks can be used with other operators (like delete till)

m[upper case letter] <-- marks the location across multiple open files
NOTE: Numbers are reserved for config files to record intersession locations.

]' <-- next mark (in file, not temporal)
[' <-- previous mark (no loop arounds)

:marks <-- will show current marks
:marks ac <-- will show marks between a & c
:delm <-- will delete marks (:delmarks)
:delm aAb <-- will delete marks a, A and b

Copilot's Guide To Marks

Marks in Vim are a powerful feature for navigating and editing text efficiently. They allow you to set specific locations within a file and quickly jump back to them later.

Basics of Marks

  • Setting a Mark: You can set a mark by typing m followed by a lowercase letter (e.g., ma sets a mark named 'a').
  • Jumping to a Mark: To jump to a mark, type followed by the same letter (e.g., a` jumps to mark 'a').

Local vs Global Marks:

  • Local marks are specific to the current buffer and are set using lowercase letters (e.g., 'a', 'b', 'c', etc.).
  • Global marks are accessible across all buffers and are set using uppercase letters (e.g., 'A', 'B', 'C', etc.).

Default Mark Locations

Vim has a set of predefined marks that have special meanings:

  • '' : (double-single-quote) Jump back to the position before the last jump.
  • \\` : (single backtick) Jump back to the position where the last edit was made.
  • '[ : Jump to the beginning of the previously changed or yanked text.
  • '] : Jump to the end of the previously changed or yanked text.
  • '< : Jump to the beginning of the previously visually selected text.
  • '> : Jump to the end of the previously visually selected text.

Practical Examples of the Default Marks

Here are practical examples of when you might use each of these default marks in Vim:

  • '' (double-single-quote): Jump back to the position before the last jump.
    • Example: You're navigating through a long document and make several jumps to different sections. If you want to return to the position before your last jump, you can use '' to quickly get back to where you were.
  • \\` (single backtick): Jump back to the position where the last edit was made.
    • Example: After making an edit in a specific part of your document, you navigate to a different section to review something else. If you need to return to the spot where you made your last edit, use ` to quickly jump back.
  • '[ : Jump to the beginning of the previously changed or yanked (original) text.
    • Example: You copy (yank) a paragraph from one part of your document and paste it somewhere else. If you want to jump back to the beginning of the text you just copied, you can use '[.
  • '] : Jump to the end of the previously changed or yanked text.
    • Example: Similar to the previous example, but if you want to jump to the end of the copied (original) text instead of the beginning, you can use '].
  • '< : Jump to the beginning of the previously visually selected text.
    • Example: You visually select a block of text to highlight or manipulate it. If you want to jump back to the start of that selection later on, use '<.
  • '> : Jump to the end of the previously visually selected text.
    • Example: After visually selecting a block of text, if you want to jump to the end of that selection, you can use '>. This can be useful for applying edits or commands to the end of the selected text.

Practical Uses of Marks

Marks are incredibly useful for various tasks in Vim:

  • Navigating Large Files: Set marks at important sections (e.g., chapters in a document) to quickly jump between them.
  • Temporary Bookmarks: When working on a specific section, you can set a mark to quickly return to that spot after exploring other parts of the file.
  • Recording Changes: Marks help keep track of positions before and after edits, making it easy to review changes.
  • Repeating Actions: If you need to perform repetitive tasks at specific locations, marks can save you time by allowing you to jump directly to those spots.

Here's a quick example:

# Set a mark 'a' at the current cursor position
ma
# Move around the file and later jump back to mark 'a'
`a

Marks can be a game-changer once you get the hang of them!
