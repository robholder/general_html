Marks

Link: Count Vajhula

Creating Marks

In current buffer

  • m<name> --> Mark current position
    • e.g. ma

In specific file/buffer

  • m<capitalized_name> --> Mark current position in a specific file/buffer
    • Uppercase marks (e.g. mA) preserve both position AND filename, so you can use them to move across different buffers

'Go To' Marks

  • '<name> --> Go to marked line e.g. 'a)
  • <backtick><name> --> Go to marked position (e.g. <backtick>a)

Shortcuts to Jump

  • '' --> Go to most recent mark (line)
  • <backtick><backtick> --> Go to most recent mark (exact position)
Vim automatically sets marks prior to each “jump.” So these commands are a common way to “go back to where I just was before this"
  • [' --> Go backwards through marks in buffer (line)
  • ]' --> Go forwards through marks in buffer (line)
  • [<backtick> ] --> Go backwards through marks in buffer (exact position)
  • <backtick> --> Go forwards through marks in buffer (exact position)

'Go To' Last Changes - Step by Step

Vim also keeps track of the locations of the “changes” that you make in the file, allowing you to navigate places you’ve been in your recent editing activity:

  • g; --> Go to previous edit location (e.g. “What was I doing before this?”)
  • g, --> Go to next edit location (e.g. “What was I doing after this?”)
Observe how g; and g, mirror a similar idea in ; and , which are used in f t searches, so you can think of them in the same way.

'Go To' Last Changes - Numbered Marks (0-9)

  • 0-9: These marks represent positions from the last 10 edits made to the file. For example, the mark 2 represents the position of the second-to-last edit:
    • e.g. '2

Previously Selected (Visual Mode)

  • Angle Bracket Marks (< and >):
    • '< --> This mark represents the beginning of the previously selected Visual mode text.
    • '> --> This mark represents the end of the previously selected Visual mode text.
    • For example, if you visually select a block of text and then exit Visual mode, you can quickly jump back to the beginning or end of that selection using the < or > marks.

Original Notes

m[lowercase letter] <-- mark current location of cursor to a letter.
[backtick][lowercase letter] <-- (backtick letter)return to mark stored to a letter.

mn <-- marks current position into 'n'
[backtick]n <-- (backtick - letter) returns to position in 'n'
'n <-- (single-quote letter) returns to first non-blank character in line of 'n' location
Lower case marks can be used with other operators (like delete till)

m[upper case letter] <-- marks the location across multiple open files
NOTE: Numbers are reserved for config files to record intersession locations.

]' <-- next mark (in file, not temporal)
[' <-- previous mark (no loop arounds)

:marks <-- will show current marks
:marks ac <-- will show marks between a & c
:delm <-- will delete marks (:delmarks)
:delm aAb <-- will delete marks a, A and b
