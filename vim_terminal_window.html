Terminal Window

The terminal mode in Vim is a powerful feature for integrating your workflow, allowing you to manage and interact with a terminal directly from within Vim. Here’s a breakdown of practical tips and key features to help you get the most out of it:

Basics of Terminal Mode

Switching Modes:

  • Insert Mode: When in a terminal buffer, you're automatically in Insert mode. Whatever you type is sent to the terminal.
  • Normal Mode: Press <C-\><C-n> (Control + Backslash, then Control + N) to enter Normal mode. This lets you navigate and manipulate the terminal buffer as if it were a normal Vim buffer.
  • Return to Terminal (Insert Mode): Use i or a in Normal mode to return to typing in the terminal.

Opening a Terminal:

  • Use the :terminal command to open a new terminal window: 
:terminal

# or
:term
  • You can also specify a shell to use: 
:term bash

# or
: term zsh

Navigation and Pane Management

Split Windows with a Terminal:

  • Open a terminal in a horizontal split:
  • NOTE: MacVim opens a split window by default. 
:split | terminal

" running :terminal in MacVim already opens a horizontal split window.
  • Open a terminal in a vertical split: 
:vsplit | terminal

" or
:vert term

Switch Between Windows:

Use standard Vim window navigation shortcuts, such as:

  • <C-w> h: Move to the left window.
  • <C-w> l: Move to the right window.
  • <C-w> j: Move to the window below.
  • <C-w> k: Move to the window above.

NOTE: This is a Vimrc set of shortcuts for those window navigation moves:

  • nnoremap <C-h> <C-w>h --> left
  • nnoremap <C-l> <C-w>l --> right
  • nnoremap <C-j> <C-w>j --> down
  • nnoremap <C-k> <C-w>k --> up

Practical Features

Copying Text from the Terminal:

  • In Normal mode, you can select text using Vim commands (e.g., v for visual mode).
  • Yank text using y, and it will be copied to your Vim register for use elsewhere.

Scrolling Through Terminal Output:

  • In Normal mode, you can scroll back through the terminal buffer using Vim navigation:
    • k or j: Move up or down one line.
    • <C-b>: Scroll up a full page.
    • <C-f>: Scroll down a full page.
  • Use g to jump to the start of the buffer and G to jump to the end.

Sending Signals to the Terminal:

  • Use key sequences in Insert mode to send signals like <C-c> to interrupt a process.

Customization and Tips

Resize Terminal Windows:

  • Adjust the size of terminal splits with :resize or :vertical resize commands: 
:resize 10     " Set the height to 10 lines
:vertical resize 40 " Set the width to 40 columns

Map Keys for Quick Mode Switching:

  • If you frequently switch modes, add key mappings to your Vimrc: 
tnoremap <esc> <C-\><C-n>          " Escape goes to Normal mode
tnoremap jk <C-\><C-n>             " Escape goes to Normal mode
nnoremap <leader>t :terminal<CR>   " Quick open terminal

Persistent Shell:

  • Keep a terminal open in the background while editing files. This is helpful for running build commands, testing scripts, or monitoring logs without leaving Vim.

Integrate with Plugins:

  • Use plugins like vim-floaterm or toggleterm to manage terminal workflows more dynamically. These plugins allow floating terminals or easily toggled terminal views.

Practical Use Cases

Compile and Test Code:

  • Run a build tool (like make) in one pane while editing source code in another.
  • Quickly switch back and forth with <C-w> commands.

Monitor Logs or Output:

  • Tail a log file or watch script output in real-time in a terminal split.

Version Control:

  • Run git commands directly in a terminal while keeping the file you're editing visible.

Run Interactive Programs:

  • Use the terminal for REPLs (Read-Eval-Print Loops) like Python or Node.js, keeping your editor context intact.

By leveraging Vim’s terminal mode, you can create a seamless, integrated development or productivity environment.

Processing In Background (and Closing)

Closing the terminal split doesn't automatically terminate the processes running in that terminal unless explicitly configured.

Here’s how you can ensure terminal jobs are stopped when you close the terminal window:

  • Manually Kill the Job
    • If you’re already in the terminal:
      • Use <C-c> to send an interrupt signal (like stopping a running script or process in a typical shell).
      • Or, if you're working with long-running processes, explicitly quit the shell process (e.g., by typing exit).
  • Why It Happens
    • Vim allows terminal jobs to continue running in the background to preserve their state across splits or buffer changes. This can be useful for long-running processes but might conflict with your intent to fully exit or clean up.
