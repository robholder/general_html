Git - Deleting a Branch

Deleting a branch in Git is very straightforward, but there is one golden rule:

You cannot delete a branch while you are currently standing on it. Before running a delete command, you must first switch to a different branch (like main). 

git switch main

Once you have stepped away from the branch you want to destroy, you can delete it using one of two commands, depending on whether you have saved your work.

1. The Safe Delete (Recommended)

If you are done with the branch and want to delete it, use the lowercase -d flag: 

git branch -d <branch-name>
  • What it does: This is the "safe" option. Git will check to see if you have merged your work into your current branch. If you have uncommitted or unmerged changes that would be permanently lost, Git will stop you and say: "error: The branch '' is not fully merged."

2. The Force Delete (The "Trash It" Option)

If you deliberately want to throw away your code and you don't care about losing any changes you made on that branch, use the uppercase -D flag: 

git branch -D <branch-name>
  • What it does: This forces the deletion. It tells Git, "I know exactly what I am doing, delete this branch immediately and permanently discard everything on it." Git will not warn you or check for merged changes.

Summary Checklist

  • git switch main (Get off the branch)
  • git branch -d <branch-name> (Delete it safely) OR git branch -D <branch-name> (Force delete it)
