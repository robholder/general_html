Swift - QR Codes

Creating a QR code in iOS feels a bit like using an old-school darkroom photo lab. Your code is actually a perfect snapshot of how Apple’s Core Image framework handles this.

Core Image is an incredibly powerful, hardware-accelerated image processing engine. It doesn't just "draw" shapes; it uses a recipe-based system. You hand it raw data, it processes it through a mathematical filter, and then you have to manually "render" the final image.

Here is a breakdown of how the whole technique works, mapping your code directly to the real-world process.

The Big Picture: The Photo Lab Analogy

To understand why we need things like a context and a filter, think of the process like this: 

[ Your Text String ] ──> Convert to Raw Bytes (Data)
                                 │
                                 ▼
                     ┌───────────────────────┐
                     │   CIFilter (Recipe)   │ <── Dictates how the squares are arranged
                     └───────────────────────┘
                                 │
                                 ▼
                     ┌───────────────────────┐
                     │ CIImage (Blueprint)   │ <── Pure math; cannot be displayed yet
                     └───────────────────────┘
                                 │
                                 ▼
                     ┌───────────────────────┐
                     │  CIContext (Lab/GPU)  │ <── Renders the math into actual pixels
                     └───────────────────────┘
                                 │
                                 ▼
                     ┌───────────────────────┐
                     │   UIImage (The Photo) │ <── Ready to display in SwiftUI
                     └───────────────────────┘

Step-by-Step Breakdown of the Technique

1. Setting up the Lab Equipment

let filter = CIFilter.qrCodeGenerator()
let context  = CIContext()
  • CIFilter.qrCodeGenerator() (The Recipe): A filter is an object that knows the mathematical rules of how to transform input data into an output image. By calling .qrCodeGenerator(), you are loading up Apple’s built-in mathematical engine specifically designed to turn text into those iconic black-and-white square matrix patterns.
  • CIContext() (The Digital Darkroom): This is the heavy lifter. Creating a context prepares the phone's GPU/processor to do the actual image rendering. It is resource-heavy to create, which is why you declare it outside the function once, rather than recreating it every time a QR code is generated.

2. Preparing the Input Data

filter.message = Data(string.utf8)

A QR code filter cannot read a standard Swift String like "Hello World". It requires raw, binary computer bytes. Data(string.utf8) translates your text string into a stream of 1s and 0s using UTF-8 encoding. You feed those bytes into the filter's .message property.

3. Fetching the Mathematical Blueprint

if let outputImage = filter.outputImage {

When you ask the filter for its outputImage, it returns a CIImage.

  • Crucial Concept: A CIImage is not an actual image file made of pixels. It is merely a "recipe" or a mathematical blueprint. It says, "If you ever decide to render me, I will look like a QR code with these specific dimensions." Because no pixels have been drawn yet, this step happens instantly.

4. Developing the Photo (The Render)

if let cgImage = context.createCGImage(outputImage, from: outputImage.extent) {

This is where the magic happens. You hand your mathematical blueprint (outputImage) and its boundaries (outputImage.extent) over to your darkroom (context).

  • The CIContext fires up the graphics processor, executes the math, and renders raw pixels out into a CGImage (Core Graphics image). A CGImage is a concrete, bitmap array of pixels living in memory.

5. Framing it for SwiftUI / UIKit

return UIImage(cgImage: cgImage)

Finally, while a CGImage is great for computers, iOS views prefer a UIImage wrapper to display things nicely on screen. You wrap the pixel data into a UIImage and send it on its way. If anything in this multi-step process fails, your code safely falls back to a default "X" icon (xmark.circle).

A Common Gotcha with this Technique

If you display this generated UIImage directly in SwiftUI, you might notice it looks incredibly blurry or pixelated.

This is because Core Image generates a QR code where 1 module (square) = exactly 1 pixel. The blueprint it outputs is incredibly tiny (often just 27×27 pixels!). When SwiftUI stretches that tiny image to fill a large Image view, iOS tries to smooth it out, making it blurry.

To fix it in SwiftUI, you usually want to tell the image view to skip the smoothing and keep the edges sharp: 

Image(uiImage: qrCodeImage)
    .interpolation(.none) // <--- Keeps the QR code crisp and blocky!
    .resizable()
    .frame(width: 200, height: 200)
