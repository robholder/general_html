Examining a Repo

Blame Game!

Examine the what, who, and when a file was edited

Blame

The 'git blame' command works at line level (it looks at one line at a time).

It can tell you when a line changed, but not what changed. 

# to show what's changed, by whom and at what commit:
git blame <file_name>

# The default is to look at HEAD:
git blame HEAD <file_name>
# ...is the same as:
git blame <file_name>
 
# to show at a given commit hash:
git blame <commit ID> <file_name>

Flags for Blame

# Suppress time stamps:
git blame -s <file_name>

Grep

Git grep

Git grep only searches files that have been staged or committed, and not untracked files in the working area.

Git grep can use expressions and expanded expressions. 

# To search through repo for a search-term
git grep <search-term>
# To search for multi-word terms (surround with quotes):
git grep "multi word term"

Grep Options

# Case insensitive -i or --ignore-case
git grep -i <search-term>
# Display line numbers -n or --line-number
git grep -n <search-term>
# Display only filenames -l or --name-only
git grep -l <search-term>

# Mixing flags:
git grep -ni <search-term>
# or:
git grep -n -i <search-term>

Log's Diff Options

Git log w/Pickaxe

The 'git log' command can be given the '-S' flag to add the pickaxe option.

The 'pickaxe' option will show a commit if a given search-term was changed, added or removed. 

# To show commits for a file where the word "test" was changed, added or removed:
git log -S test

Git log w/Patch

Adding a 'patch' option to 'git log' will show you a diff display of a log output. 

git log -p

Log options Combined!

Combine pickaxe and patch for an even better result. 

# To show the commit with the search-term and how it changed:
git log -p -S <search-term>
# Or move the patch option (however, -S has to be followed by the <search-term>:
git log -S <search-term> -p
