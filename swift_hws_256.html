Hacking With Swift - 256 - Ending the app with allowsHitTesting()

Making Timer Stop The Session

SwiftUI lets us disable interactivity for a view by setting allowsHitTesting() to false, so in our project we can use it to disable swiping on any card when the time runs out by checking the value of timeRemaining.

Start by adding this modifier to the innermost ZStack - the one that shows our card stack: 

.allowsHitTesting(timeRemaining > 0)

That enables hit testing when timeRemaining is 1 or greater, but sets it to false otherwise because the user is out of time.

Stopping The Timer And Reset

The other outcome is that the user flies through all the cards correctly, and ends with none left. When the final card goes away, right now our timer slides down to the center of the screen, and carries on ticking. What we want to happen is for the timer to stop so users can see how fast they were, and also to show a button allowing them to reset their cards and try again.

This takes a little thinking, because just setting isActive to false isn’t enough - if the app moves to the background and returns to the foreground, isActive will be re-enabled even though there are no cards left.

Let’s tackle it piece by piece. First, we need a method to run to reset the app so the user can try again, so add this to ContentView: 

func resetCards() {
    cards = Array<Card>(repeating: .example, count: 10)
    timeRemaining = 100
    isActive = true
}

Second, we need a button to trigger that, shown only when all cards have been removed. Put this after the innermost ZStack, just below the allowsHitTesting() modifier: 

if cards.isEmpty {
    Button("Start Again", action: resetCards)
        .padding()
        .background(.white)
        .foregroundStyle(.black)
        .clipShape(.capsule)
}

Now we have code to restart the timer when resetting the cards, but now we need to stop the timer when the final card is removed - and make sure it stays stopped when coming back to the foreground.

We can solve the first problem by adding this to the end of the removeCard(at:) method: 

if cards.isEmpty {
    isActive = false
}

As for the second problem - making sure isActive stays false when returning from the background - we should just update our scene phase code so it explicitly checks for cards: 

.onChange(of: scenePhase) {
    if scenePhase == .active {
        if cards.isEmpty == false {
            isActive = true
        }
    } else {
        isActive = false
    }
}

Done!

CardView.swift

import SwiftUI

struct CardView: View {
    @Environment(\.accessibilityDifferentiateWithoutColor) var accessibilityDifferentiateWithoutColor

    let card: Card
    var removal: (() -> Void)? = nil
    
    @State private var isShowingAnswer = false
    @State private var offset = CGSize.zero
    
    var body: some View {
        ZStack {
            RoundedRectangle(cornerRadius: 25)
                .fill(
                    accessibilityDifferentiateWithoutColor
                    // Ternary condition
                    ? .white : .white
                        .opacity(1 - Double(abs(offset.width / 50)))
                    
                )
                .background(
                    accessibilityDifferentiateWithoutColor
                    // Ternary condition
                    ? nil : RoundedRectangle(cornerRadius: 25)
                        .fill(offset.width > 0 ? .green : .red)
                )
                .shadow(radius: 10)
            VStack {
                Text(card.prompt)
                    .font(.largeTitle)
                    .foregroundStyle(.black)
                if isShowingAnswer {
                    Text(card.answer)
                        .font(.title)
                        .foregroundStyle(.secondary)
                }
            }
            .padding(20)
            .multilineTextAlignment(.center)
        }
        .frame(width:450, height: 250)
        .rotationEffect(.degrees(offset.width / 5.0))
        .offset(x: offset.width * 5)
        .opacity(2 - Double(abs(offset.width / 50)))
        .gesture(
            DragGesture()
                .onChanged { gesture in
                    offset = gesture.translation
                }
                .onEnded { _ in
                    if abs(offset.width) > 100 {
                        removal?()
                    } else {
                        offset = .zero
                    }
                }
            )
        .onTapGesture {
            isShowingAnswer.toggle()
        }

    }
}

ContentView.swift

import Combine
import SwiftUI


extension View {
    func stacked(at position: Int, in total: Int) -> some View {
        let offset = Double(total - position)
        return self.offset(y: offset * 10)
    }
}

struct ContentView: View {
    @Environment(\.accessibilityDifferentiateWithoutColor) var accessibilityDifferentiateWithoutColor
    // Environment info on active status of app
    @Environment(\.scenePhase) var scenePhase
    // To set if app is active
    @State private var isActive = true
    
    // Create a stack of identical cards using 'repeating'
    @State private var cards = Array<Card>(repeating: .example, count: 10)
    
    // 'static let' belongs to the blueprint, so it's born first!
    static let defaultDuration = 10
    // Access static constant for default duration value
    @State private var timeRemaining = ContentView.defaultDuration
    
    let timer = Timer.publish(
        every: 1,
        on: .main,
        in: .common).autoconnect()
    var body: some View {
        ZStack {
            // Add background image
            Image(.background)
                .resizable()
                .ignoresSafeArea()
            VStack {
                Text("Time: \(timeRemaining)")
                    .font(.largeTitle)
                    .foregroundStyle(.white)
                    .padding(.horizontal, 20)
                    .padding(.vertical, 5)
                    .background(.black.opacity(0.75))
                    .clipShape(.capsule)
                
                ZStack {
                    ForEach(0..<cards.count, id: \.self) { index in
                        CardView(card: cards[index]) {
                            withAnimation {
                                removeCard(at: index)
                            }
                        }
                        .stacked(at: index, in: cards.count)
                    }
                }
                .allowsHitTesting(timeRemaining > 0)
                if cards.isEmpty {
                    Button("Start Again", action: resetCards)
                        .padding()
                        .background(.white)
                        .foregroundStyle(.black)
                        .clipShape(.capsule)
                }
            }
            if accessibilityDifferentiateWithoutColor {
                VStack {
                    Spacer()
                    
                    HStack {
                        Image(systemName: "xmark.circle")
                            .padding()
                            .background(.black.opacity(0.7))
                            .clipShape(.circle)
                        Spacer()
                        Image(systemName: "checkmark.circle")
                            .padding()
                            .background(.black.opacity(0.7))
                            .clipShape(.circle)
                    }
                    .foregroundStyle(.white)
                    .font(.largeTitle)
                    .padding()
                }
            }

        }
        .onReceive(timer) { time in
            // Check if app is active (in the foreground)
            guard isActive else { return }

            // Condition to prevent straying into negative numbers
            if timeRemaining > 0 {
                // Subtract one second
                timeRemaining -= 1
            }
        }
        .onChange(of: scenePhase) {
            if scenePhase == .active {
                if cards.isEmpty {
                    isActive = true
                }
            } else {
                isActive = false
            }
        }
    }
    func removeCard(at index: Int) {
        cards.remove(at: index)
        if cards.isEmpty {
            isActive = false
        }
    }
    func resetCards() {
        cards = Array<Card>(repeating: .example, count: 10)
        // Access static constant for default duration value
        timeRemaining = ContentView.defaultDuration
        isActive = true
    }
}

Ai Insight

QUESTION

I have an interesting initialization and variable issue. I have this line to set the duration in seconds: 

@State private var timeRemaining = 10

However, I'd like to be able to use another variable to assign that value (of 10 in this example).This is because the timeRemaining is also reset in another part of the code, and I want to have a single source of value.

If I do something like: 

@State private var duration = 10
@State private var timeRemaining= duration

...it fails the initialization step with an error. How can I work around this?

ANSWER

You have run into one of the most classic catch-22s in Swift initialization.

The compiler is stopping you because of a strict safety rule: You cannot use one property of a struct to initialize another property before the struct is fully born. When Swift is building your view, duration and timeRemaining are being created at the exact same millisecond. Because duration doesn't officially exist yet, Swift won't let timeRemaining look at it for its initial value.

Fortunately, there are a few clean ways to work around this depending on how you want to manage your app's settings.

Solution 1: Use a Custom Initializer (The Direct Workaround)

If you want duration to be a variable that can be changed, you can write a custom init() for your view. Inside the initializer, the struct is being born, which gives you a chance to pass the value across.

When dealing with @State variables, you have to use a slightly special syntax inside the initializer - you initialize their underlying storage by adding an underscore (_) before the variable name: 

struct TimerView: View {
    @State private var duration: Int
    @State private var timeRemaining: Int

    // Custom Initializer
    init(initialDuration: Int = 10) {
        // 1. Set the duration
        _duration = State(initialValue: initialDuration)
        // 2. Safely copy it to timeRemaining during birth!
        _timeRemaining = State(initialValue: initialDuration)
    }

    var body: some View {
        Text("\(timeRemaining) seconds left")
    }
}

Solution 2: The Modern SwiftUI Way (.onAppear)

If duration is just a constant default number that never changes, you don't actually need it to be a @State variable. You can make it a plain static constant, or you can initialize timeRemaining to 0 and load the real duration the split-second the view appears on screen.

This is often preferred in SwiftUI because it avoids messing with complex custom initializers: 

struct TimerView: View {
    @State private var duration = 10
    // 1. Start at 0 temporarily
    @State private var timeRemaining = 0 

    var body: some View {
        VStack {
            Text("\(timeRemaining) seconds left")
        }
        // 2. The moment the view hits the screen, grab the source value
        .onAppear {
            timeRemaining = duration
        }
    }
}

NOTE: .onAppear will not trigger when the app is brought back to the foreground. It fires exactly once when your SwiftUI view is physically constructed and slid onto the screen.

Solution 3: The Plain Constant Approach

If your duration is truly a hardcoded default that you just want to reference in multiple places (like your timer setup and your reset button), you can drop the @State modifier from duration entirely and mark it as a static let.

Static constants belong to the type itself, not the individual instance, meaning they are fully alive before the struct is initialized: 

struct TimerView: View {
    // 'static let' belongs to the blueprint, so it's born first!
    static let defaultDuration = 10 
    
    @State private var timeRemaining = TimerView.defaultDuration

    var body: some View {
        Button("Reset Timer") {
            // You can easily read it here to reset your value!
            timeRemaining = TimerView.defaultDuration 
        }
    }
}

Summary Recommendation

  • Go with Solution 3 (static let) if your default time is a fixed number that never changes during runtime. It is the cleanest and requires the least code.
  • Go with Solution 2 (.onAppear) if duration is a dynamic value coming from somewhere else (like a user settings screen) that you need to capture right as the timer view loads.
