VimTips - Vim cheatsheet, tips and tricks

Mapping <esc> to 'jk'

Depending on systems, one of the following should be true: ^[ NOT TYPED <-- it is the character for <esc> (but to type it in a macro: <cntl>v<esc>) ^[ NOT TYPED <-- it is the character for <esc> (but to type it in a macro: <cntl>q<esc>)

Set jk to replace escape key for command/normal mode in the vimrc config file:

inoremap jk <esc>

...or inline as:

:inoremap jk <esc> :inoremap jk <ctrl>v<esc> :inoremap jk <ctrl>q<esc>

Copy/Paste

To use visual to select, and copy the whole file ggVGy <-- yank whole text :w !pbcopy <-- copy selection to system clipboard (seems to copy whole file) Or to copy whole text: :%w !pbcopy <-- copy all file text without selection To paste from system clipboard: :r !pbpaste Or <command>v <-- macos `<cntl>v <-- presumably other OS's

Copy/Paste to/from System Clipboard

In 'normal' mode: "+y <-- copy to system clipboard. "+p <-- paste to system clipboard.

In 'insert' mode: <ctrl>r+ <-- paste while in insert mode (from system clipboard) <ctrl>r" <-- paste while in insert mode (from Vim unnamed clipboard or register)

Searching

:set hlsearch <-- highlight search on (highlighted search terms) :noh <-- to turn off highlight search :set nohlsearch <-- turn off highlights permanently :nohlsearch <-- command not option; highlight will return on next search :set hls? <-- hightlight searched words on or off? :set hls <-- highlight searches on (have to turn off to clear) :set nohls <-- highlighting off

:set is or :set incsearch <-- incremental search on (to hope to next matched string) :set nois or :set noincsearch <-- off :set is? <-- shows current noincsearh or incsearch (incremental search option) status

f[char] <-- search/move cursor to next [char] on the line. 2f[char] <-- search/move cursor to second occurance of [char] on the line. F[char] <-- search/move cursor back next [char] on the line. ; <-- repeat to next searched char on the line. , <-- move back to the previous [char] on the line. t[char] <-- search/move cursor one character to the left of the [char] ('Til') T[char] <-- search/move cursor back one character to the right of the [char] ('Til')

% <-- jumps to matching bracket when cursor is on a bracket.

/ <-- search (string followed by return/enter) /and<CR> <-- search for 'and'. use :set nohls to hide highlighting if on. n <-- find next N (<shift>n) <-- find previous ? <-- search backwords for string ?and <-- search backwards for 'and'

* <-- search for next entire repeat of word under cursor, * again or n & N to move again = <-- like the asterisk, but in reverse direction.

Mapping 'highlight search' in your Vimrc config

Add to .vimrc for F4 toggle option: (press f4 to toggle highlighting on/off, and show current value.) :noremap <f4> :set hlsearch! hlsearch?<CR> Same thing but mapped to the Backspace/Delete key: :noremap <BS> :set hlsearch! hlsearch?<CR> Using Spacebar and clear messages already displayed: :nnoremap <silent> <space> :nohlsearch<bar>:echo<cr>

Substitutions

:s/old/new/ <-- search and replace on current line (first instance) :s/old/new/g <-- search and replace all on current line :s/old/new/gc <-- search and replace on current line (with confirmation) :[range]s/old/new/g <-- replace on line given in [range]

:259s/old/new/g <-- change all on line 259 :1,5s/old/new/g <-- change all between lines 1 to 5

:.,$/old/new/g <-- change from current cursor position to end of file :%s/old/new/g <-- entire file :%s/old/new/gc <-- entire file with confirmation

:/<start pattern>/,/<end pattern>/s/old/new/g <-- change between patterns

:/local/,166s/old/new/g <-- change from 'local' search pattern through line 166.

:s/\/old\/dir\/new\/dir/ <-- change /old/dir to /new/dir :s=/old/dir=/new/dir= <-- same as before, but using arbitrary separaters to avoid backslashes. Quick clarification: these next two are alternatives based on system; one should work! :s/ /<cntl>q<return>/g <-- Replaces spaces with newlines on a line :s/ /<cntl>v<return>/g <-- Replaces spaces with newlines on a line The lines above should appear as :s/ /^M/g if prior to <enter> all is well

:g/^/+1 d3 <-- Deletes 3 lines repeating every 4th (use d<number> to taste)

Summary: :[range]s/{pattern}/{string}/[flags] format :%s/[pattern]/[string]/g global substitution

Macros

Depending on systems, one of the following should be true: ^[ NOT TYPED <-- it is the character for <esc> (but to type it in a macro: <cntl>v<esc>) ^[ NOT TYPED <-- it is the character for <esc> (but to type it in a macro: <cntl>q<esc>)

q[letter] {keystrokes} q <-- to make a macro at a letter register @[letter] <-- to recall macro :reg [letter] <-- to inspect macro q[capital letter] {keystrokes} q <-- to append to a macro

To edit register/macro:

On an empty line: "[letter]p <-- for example, for register 'a' "ap This will paste the register contents to the Vim session. Then edit as in vim. Edit, then to return the string/pattern to the register: 0"[letter]y\( <-- for example, for register 'a' 0"ay\) The macro will be updated: :reg [letter] to check.

Hardcoding Macros/Registers

The history, registers (and macros) are in the .viminfo file. Explicit macros can be written to the .vimrc file directly, especially if you want them read into registers every time. ==== Syntax for storing a macro is ===== let @[letter] = '0:s/"/'/g' (where [letter] is literally a letter!) let @[letter] = 'inote: ^[' (where [letter] is literally a letter!) note!! to type the ^[ above (a literal <esc>) type: <cntl>v <esc> or cntl>q depending on system. Folding

zo to open folded lines zO to open all nested folded lines zc to close folded lines zC to close all nested folded lines za to toggle open/close folded lines zA to toggle open/close all nested folded lines zr to open one level of all folded lines zR to open ALL levels of lines zm to close one level of all folded lines zM to close ALL levels of lines zn disable folding zN enable folding (revert to 'as was') zi toggle disable/enable folding zf create zd delete fold markers zE eliminate folds in entire document zj next fold (maybe set in vimrc) zk last fold (Not default keys - set in vimrc)

:set foldenable <-- enable folding :set nofoldenable <-- disable folding zi <-- toggle enable folding :86,136 foldopen <-- open all folds in this range :86,136 foldclose <-- close all folds in this range :85,$foldclose <-- close all folds from line 85 to the end of the document

[z <-- jump to top line of this fold ]z <-- jump to bottom line of this fold

:%foldopen <-- open all folds in buffer :%foldo <-- open all folds in buffer :%foldclose <-- close all folds in buffer :%foldc <-- close all folds in buffer

:% foldopen – opens all the folds in the document :% foldclose – close all the folds in the document set foldmethod=manual <-- fold methods set fdm=expr <-- expression fold set fdm=diff <-- difference fold set fdm=indent <-- indent fold set fdm=manual <-- manual fold set fdm=marker <-- marker fold set fdm=syntax <-- syntax fold

Recipes zf10j <-- will fold the current line and the 10 lines below it. zk10j <-- will fold the current line and the 10 lines above it :148,158fold <-- will fold lines 148 to 158 :148,158 fold <-- shows you can add a space (but you don't need to) :148,158 fo <-- fo is a shortcut for fold :,+10 fold <-- will fold from where the line the cursor is on to 10 lines below it :,-10 fold <-- will fold from where the line the cursor is on to 10 lines above it zfa{ <-- will fold the text between the two curly bracess {} va{zf <-- va{ will visually select the text between curly braces {} and zf will fold the selection

Vimdiff

To launch: vimdiff <file1> <file2> <-- Launch into Vim Diff vim -d <file1> <file2> <-- Launch into Vim Diff vim -do <file1> <file2> <-- Launch into Vim Diff with horizonal split

To update 'diff' analysis :diffupdate :diffu

To invoke diff within Vim session: :vs <file2> <-- veritcal split in Diff mode :vert <file2> <-- veritical split in Diff mode :split <file2> <-- horizontal split in Diff mode :diffsplit <file2> <--horizontal split vert diffsplit <file2> <--vertical split

:windo diffthis <-- invoke diff mode :windo diffoff <-turn off diff mode

Pushing/Pulling changes: diffput <-- pushes changes to 'other' file (window) diffget <-- obtains/gets changes from 'other' file (window)

Alternates for the above: dp <-- pushes changes to 'other' file (window) do <-- obtains/gets changes from 'other' file (window)

[c <-- jump to next difference ]c <-- jump to previous difference
