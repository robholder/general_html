Swift - String Formatting

If you are writing modern Swift, the old String(format:) approach has been largely superseded for user-facing text.

The modern replacement is called String Interpolation Specifiers (introduced alongside SwiftUI's native localization updates). Instead of wrapping your text inside a clunky String(format: ...) initializer, you can now embed formatting rules directly inside standard SwiftUI string interpolation \( ).

This keeps your code cleaner, highly readable, and perfectly compatible with SwiftUI’s automatic language translation engines.

Here is how your notes should look to reflect the modern workflow:

The Modern Workflow Shift

1. Floating-Point Precision (The Modern Way)

Instead of using String(format: "%.2f", pi), you pass a specifier string directly inside the interpolation parenthesis. 

let pi = 3.14159

// ❌ OLD (Superseded)
Text(String(format: "Value: %.2f", pi))

//   MODERN WAY
Text("Value: \(pi, specifier: "%.2f")") // Output: "Value: 3.14"

2. Zero-Padding & Lead Zeros (The Modern Way)

This applies directly to your dice simulator project! You can ditch String(format:) entirely when computing your image names or text displays. 

let sides = 6

// ❌ OLD (Superseded)
let imageName = String(format: "dice_sides_%02d", sides)

//   MODERN WAY
let imageName = "dice_sides_\(sides, specifier: "%02d")" // Output: "dice_sides_06"

When should you still use the old String(format:)?

While SwiftUI handles UI text beautifully with specifiers and .formatted(), the classic String(format:) approach is still used in one specific scenario: Pure Data Logging & Backend Logic.

If you are formatting text outside of a user interface view - like saving a formatted string to a database, printing raw debug code to the Xcode console via print(), or generating file names on a local disk - String(format:) remains a valid tool because those areas don't have access to SwiftUI's view rendering engine.

Rule of Thumb for Your Notes:

  • If a user is going to see it on a screen, use \(value, specifier: "...") or .formatted().
  • If it's just for the computer's eyes (logs, file paths, console strings), standard String(format:) is perfectly fine.
