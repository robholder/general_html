Sort & Sorted

Remember: Writing return items.sort() is a common pitfall because it returns None, not the sorted list.

Understanding the difference between sort() and sorted() is key, especially when writing functions that return sorted lists. Here’s a breakdown:

list.sort()

  • How It Works: The .sort() method sorts a list in place; it modifies the original list instead of creating a new one.
  • Return Value: It returns None. This is a deliberate design choice in Python so you don’t accidentally rely on its output.
  • Usage Example: 
names = ["Zoe", "Alice", "Bob"]
names.sort()  # Sorts in place; 'names' is modified
print(names)  # Output: ['Alice', 'Bob', 'Zoe']
  • Caution in Functions: If you write something like this: 
def get_sorted_names(items):
    return items.sort()  # BAD: items.sort() returns None!

Calling get_sorted_names(["Zoe", "Alice", "Bob"]) would return None because sort() doesn't return the sorted list.

Instead, you would do: 

def get_sorted_names(items):
    items.sort()  # Sort in place
    return items  # Now return the sorted list

# Or avoid modifying the original list:
def get_sorted_names_copy(items):
    new_items = items.copy()
    new_items.sort()
    return new_items

sorted()

  • How It Works: The sorted() function takes any iterable and returns a new sorted list. It does not change the original iterable.
  • Return Value: It returns the sorted list, so you can immediately use it (or return it from a function).
  • Usage Example: 
names = ["Zoe", "Alice", "Bob"]
sorted_names = sorted(names)  # 'names' remains unchanged
print(sorted_names)  # Output: ['Alice', 'Bob', 'Zoe']
  • Usage in Functions: When you want your function to return a sorted list without altering the input, use sorted(): 
def get_sorted_names(items):
    return sorted(items)

# Calling it:
sorted_list = get_sorted_names(["Zoe", "Alice", "Bob"])
print(sorted_list)  # Output: ['Alice', 'Bob', 'Zoe']

When to Use Which

  • Use .sort() when:
    • You want to sort the list in place (no need for an extra list).
    • You’re not going to use the original unsorted list.
    • Inside a function, if you choose to sort in place, do not return the direct result of .sort(). Instead, perform the sort on a copy or sort first and then return the list.
  • Use sorted() when:
    • You need to preserve the original list.
    • You’re writing a function that should return a sorted version without side effects.
    • You want a clean, one-line return of a sorted list.

Quick Function Examples

Using sorted(): 

def get_sorted_items(items):
    return sorted(items)

# Usage:
print(get_sorted_items(["banana", "apple", "cherry"]))

Using .sort() properly: 

def get_sorted_items_inplace(items):
    items.sort()  # sorts the list in place
    return items

# Usage:
my_list = ["banana", "apple", "cherry"]
sorted_list = get_sorted_items_inplace(my_list)
print(sorted_list)
