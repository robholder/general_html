Git Diff Between Branches

QUESTION

If I have a branch in a repo, how do I tell what is different from the 'main' branch?

ANSWER

To see the differences between your current branch and main, you can use several variations of the git diff command depending on how much detail you want.

1. The "Quick Summary" (File Names Only)

If you just want to see which files have changed without scrolling through lines of code: 

git diff main --stat

This shows a list of filenames and how many lines were added or deleted in each.

2. The "Full Review" (Line-by-Line)

To see every single change in your current branch compared to main: 

git diff main

Since you are a Vim user, you can pipe this to Vim to read it more comfortably: git diff main | vim -

3. The "Triple Dot" (The Best Practice)

This is the most accurate way to see only the changes you made on your branch since it diverged from main: 

git diff main...

Two dots (main..): Shows the difference between the current state of both.

Three dots (main...): Shows the changes on your branch starting from the moment you branched off main, ignoring any new commits that were added to main since then.

4. The "Visual" Way (Vimdiff)

If you want to see the differences side-by-side using your Vim skills: 

git difftool main

This will launch vimdiff (or whatever you set as your diff.tool) for each modified file.

5. Reviewing Commit History

If you want to see the commit messages that exist on your branch but are missing from main: 

git log main..HEAD --oneline
  • Pro-Tip: If you haven't run git fetch lately, your local main might be outdated. Run git fetch origin first, then compare against origin/main to see the difference between your work and the actual "truth" on GitHub.

QUERY

Would you like to see how to limit the diff to just one specific file or directory?

ANSWER

To limit the diff to a specific file or directory, you simply append the path to the end of your command. Git uses the -- separator to distinguish between branch names and file paths.

1. Diff a Specific File

If you only want to see changes in index.html compared to main: 

git diff main -- index.html

2. Diff an Entire Directory

To see changes for everything inside your src/ folder: 

git diff main -- src/

3. Using Wildcards (Globbing)

If you only want to see changes in your CSS files: 

git diff main -- "*.css"

(Note: Use quotes around the wildcard so your shell doesn't expand it before Git sees it.)

4. The "Vim" Shortcut

Since you're a Vim user, you can use the difftool with a specific file to open it in vimdiff immediately: 

git difftool main -- path/to/file.txt

Why use the -- separator?

While often optional, using the -- (double dash) is a best practice. It tells Git: "Everything after this is a file path, not a branch name or an option." This prevents errors if you ever happen to have a file and a branch with the same name.

  • One final tip: If you want to see which files were changed in a specific directory without the full code diff, combine the path with the stat flag: 
    git diff main --stat -- src/

QUERY

Would you like to know how to save these specific changes into a patch file that you can email or move to another machine?

ANSWER

To save your changes as a patch file, you use the git diff command and redirect the output to a file. This is perfect for sharing work-in-progress without pushing to a branch.

1. Save Changes to a Patch File

To save all differences between your current branch and main into a single file: 

git diff main > my_changes.patch

2. Save a Patch for Specific Files/Directories

If you only want to "bundle up" changes from your src/ folder: 

git diff main -- src/ > feature_logic.patch

3. How to Apply the Patch (On another machine/branch)

When you (or a teammate) want to take that file and apply those changes to a different local repo, use the git apply command: 

git apply my_changes.patch
  • Note: This applies the code changes to the working directory but does not create a commit. You still get to review them before running git add and git commit.

4. The "Vim" Way: Formatting for Email

If you want to keep the commit messages, author info, and timestamps intact (so the recipient can simply "import" your commits), use git format-patch: 

git format-patch main..HEAD

This will generate a series of .patch files (one for each commit) in your current directory. These are specially formatted to be readable by Vim and compatible with git am (Apply Mailbox).

  • Pro-Tip: Testing the Patch

Before applying a patch, you can check if it will cause any conflicts without actually changing your files: 

git apply --check my_changes.patch

If it returns nothing, the patch will apply cleanly!
