Hacking With Swift - 218 - Challenge 2

Challenge 2

  • Fix the list rows in iExpense so they read out the name and value in one single VoiceOver label, and their type in a hint.

Ai Insight

This is a classic SwiftUI accessibility challenge. By default, VoiceOver treats separate Text views inside a row as distinct focal points, which makes the user swipe multiple times just to read a single expense item (e.g., Swipe 1: "Business Lunch", Swipe 2: "$42.50").

To solve Paul's challenge, we want to combine them so VoiceOver treats the entire row as one single element, reads the name and value right away, and then adds the expense type (Personal/Business) as a "hint" if the user pauses.

Since you've cleanly separated your row layout into an ExpenseRow(item: item) view component, that is exactly where we need to apply the accessibility modifiers!

The Solution

You need to apply three specific accessibility modifiers to the layout inside your row:

  • .accessibilityElement(children: .ignore) – Tell VoiceOver to stop looking at the individual internal Text pieces.
  • .accessibilityLabel(...) – Provide our custom combined string ("Name, Amount").
  • .accessibilityHint(...) – Provide the extra context ("Type: Business").

Assuming your ExpenseRow looks something like a standard HStack, here is how you implement it: 

struct ExpenseRow: View {
    let item: ExpenseItem

    var body: some View {
        HStack {
            VStack(alignment: .leading) {
                Text(item.name)
                    .font(.headline)
                Text(item.type)
                    .font(.subheadline)
                    .foregroundStyle(.secondary)
            }

            Spacer()
            
            // Format your amount currency properly
            Text(item.amount, format: .currency(code: Locale.current.currency?.identifier ?? "USD"))
        }
        // --- ACCESSIBILITY MODIFIERS START HERE ---
        // 1. Group the row components into a single VoiceOver element
        .accessibilityElement(children: .ignore)
        
        // 2. Read out the name and amount cleanly in one breath
        .accessibilityLabel("\(item.name), \(item.amount, format: .currency(code: Locale.current.currency?.identifier ?? "USD"))")
        
        // 3. Add the classification category as a supplementary hint
        .accessibilityHint("Category: \(item.type)")
    }
}

Why This Specific Order Matters

  • children: .ignore: Without this, VoiceOver might read out your custom label, but then still let the user swipe onto the internal name text or currency text. This completely clears out the inner elements so only your custom label exists.
  • String Interpolation with Formatting: Notice that inside .accessibilityLabel, we pass the exact same .currency format configuration used in your visual text view. If you just pass \(item.amount), VoiceOver will awkwardly read out literal decimals (e.g., "Forty-two point five") instead of a natural financial sentence (e.g., "Forty-two dollars and fifty cents").
  • The Hint: Hints are spoken with a slight delay after the label. This means power-users scrolling quickly down their expense list will only hear the names and prices, avoiding unnecessary information fatigue unless they pause on a specific row!
