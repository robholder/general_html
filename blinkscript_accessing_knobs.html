Accessing User Knobs on a BlinkScript Node

Nuke BlinkScript knob naming convention

  • Kernel Name + Parameter Display Name: Nuke prepends the name of the kernel to the parameter's display name to form the actual knob name used in scripting and expressions.
  • Unsupported Characters Removed: Before appending the display name, Nuke removes unsupported characters (like spaces) from the display name.
  • Example: If your kernel is named SimpleKernel and your parameter is defined as defineParam(spreadFloat, "Spread", 1.0f);, the knob's scripting name will be SimpleKernel_Spread.

Python code to update knobs

With this knowledge, you can now adjust User Knobs with Python code:

This snippet addresses a BlinkScript called "SimpleKernel" and takes a default float slider knob called "spreadSlider" but LABELED "Spread", and sets the limits on the slider from 0 to 1, to 1 to 20: 

node = nuke.selectedNode()
if node.Class() == 'BlinkScript':
    # Use the combined kernel name and parameter label
    knob = node.knob('SimpleKernel_Spread') # <<--- CORRECTED KNOB NAME (you'd expect 'spreadSlider')

    if knob:
        # Set the min and max values for the slider
        knob.setRange(1.0, 20.0) # Set min to 1.0, max to 20.0

        # Optional: Set the slider's step value if needed
        # knob.setIncrement(1.0)
        
        nuke.message("Slider range set successfully!")
    else:
        nuke.message("Knob 'SimpleKernel_Spread' not found on the selected BlinkScript node.")
else:
    nuke.message("Please select a BlinkScript node.")
