Hacking With Swift - 166 - Challenge 3 - Adding Personal/Business/All filter

QUESTION

I now need to add this to the project:

  • "Add a filter option to show all expenses, just personal expenses, or just business expenses."
    • Should this be another Picker style menu item?
    • How should I implement this in the code?

ANSWER

Yes, a Picker style menu item right inside your existing "Sort" or a new "Filter" menu is the perfect UX choice for this. It keeps the navigation bar clean while giving the user explicit control.

To implement this, we will use the exact same View Extraction strategy we just set up. Since ExpenseListView is already designed to re-configure its database query on the fly, we just need to pass the filter selection from the parent down to the child.

However, we are going to make one massive structural improvement: We will let the database handle the filtering using a SwiftData #Predicate, instead of fetching everything and filtering using a standard Swift array helper (.filter { ... }).

Why use a #Predicate?

In your previous code, you fetched all items from the database using @Query var items, and then used computed properties (personalExpenses and businessExpenses) to split them up.

While that works fine for a few items, if a user has 2,000 expenses, your app has to waste memory loading all 2,000 items into RAM just to display the 5 business expenses. By passing a #Predicate to the @Query, SwiftData asks the underlying SQLite database to only hand over the matching items. It is incredibly fast and memory-efficient.

The Refactoring Steps

Here is how you adjust your code to support the new filtering option.

Step 1: Update the Parent (ContentView)

We need to add a new @State property to track the active filter string ("All", "Personal", or "Business") and add the Picker UI to the toolbar. 

struct ContentView: View {
    @State private var sortOrder = [
        SortDescriptor(\ExpenseItem.name),
        SortDescriptor(\ExpenseItem.amount)
    ]
    
    // STEP 1a: Add state to track the user's filtering choice
    @State private var filterType = "All" 
    
    var body: some View {
        NavigationStack {
            // STEP 1b: Pass BOTH the sortOrder and the filterType down to the child
            ExpenseListView(sortOrder: sortOrder, filterType: filterType)
                .navigationTitle("iExpense")
                .toolbar {
                    NavigationLink {
                        AddView()
                    } label: {
                        Label("Add Expense", systemImage: "plus")
                    }
                    
                    // STEP 1c: Add a new Menu or expand your current one to host the Filter Picker
                    Menu("Filter", systemImage: "line.3.horizontal.decrease.circle") {
                        Picker("Filter", selection: $filterType) {
                            Text("Show All").tag("All")
                            Text("Personal Only").tag("Personal")
                            Text("Business Only").tag("Business")
                        }
                    }
                    
                    Menu("Sort", systemImage: "arrow.up.arrow.down") {
                        Picker("Sort", selection: $sortOrder) {
                            Text("Sort by Name").tag([SortDescriptor(\ExpenseItem.name), SortDescriptor(\ExpenseItem.amount)])
                            Text("Sort by Amount").tag([SortDescriptor(\ExpenseItem.amount), SortDescriptor(\ExpenseItem.name)])
                        }
                    }
                }
        }
    }
}

Step 2: Update the Child (ExpenseListView)

We change the child view's initializer to accept the filterType. Inside the init, we build a SwiftData #Predicate based on what the user selected, and inject both the predicate and the sort descriptors into the @Query wrapper.

We can also get rid of the computed array filters entirely, simplifying our List body significantly! 

struct ExpenseListView: View {
    @Environment(\.modelContext) var modelContext
    
    // The query now dynamically holds filtered and sorted data natively
    @Query var items: [ExpenseItem]
    
    // STEP 2a: Accept both parameters in the initializer
    init(sortOrder: [SortDescriptor<ExpenseItem>], filterType: String) {
        
        // STEP 2b: Build the Predicate database rule
        let predicate = #Predicate<ExpenseItem> { item in
            if filterType == "All" {
                return true // Matches everything
            } else {
                return item.type == filterType // Matches only "Personal" or "Business"
            }
        }
        
        // STEP 2c: Inject both rules into the underlying Query wrapper
        _items = Query(filter: predicate, sort: sortOrder)
    }
    
    var body: some View {
        // STEP 2d: The List is now beautifully simple. No manual array filtering needed!
        List {
            ForEach(items) { item in
                ExpenseRow(item: item)
            }
            .onDelete(perform: removeItems)
        }
    }
    
    func removeItems(at offsets: IndexSet) {
        for offset in offsets {
            let item = items[offset]
            modelContext.delete(item)
        }
    }
}

Because we changed the child view's initializer signatures, update your preview blocks at the bottom of the file so they pass default placeholder values:

Summary of why this pattern scales

Notice how easy it was to drop this feature in. Because you took the time earlier to split your view into a Parent (ContentView) and a Child (ExpenseListView), adding filtering didn't require breaking your view hierarchy. You simply extended the pipeline you already built to pass one extra piece of configuration down to the database engine.

Unchanged. 

import SwiftData
import SwiftUI

struct AddView: View {
    
    @Environment(\.dismiss) var dismiss
    @Environment(\.modelContext) var modelContext
    
    @State private var name = ""
    @State private var type = "Personal"
    @State private var amount = 0.0
    
    let types = ["Business", "Personal"]
    
    var body: some View {
        NavigationStack {
            Form {
                TextField("Expense item", text: $name)
                
                Picker("Type", selection: $type) {
                    ForEach(types, id: \.self) {
                        Text($0)
                    }
                }
                TextField("Amount", value: $amount, format: .currency(code: "USD"))
                    .keyboardType(.decimalPad)
            }
            .navigationTitle("Add new expense")
            
            .toolbar {
                ToolbarItem(placement: .topBarTrailing) {
                    Button("Save") {
                        let item = ExpenseItem(name: name, type: type, amount: amount)
                        modelContext.insert(item)
                        dismiss()
                    }
                }
                ToolbarItem(placement: .topBarLeading){
                    Button("Cancel") {
                        dismiss()
                    }
                }
            }
            .navigationBarBackButtonHidden(true)
        }
    }
}

ContentView.swift

import SwiftData
import SwiftUI

@Model
class ExpenseItem {
    
    var name: String
    var type: String
    var amount: Double
    
    init(name: String, type: String, amount: Double) {
        self.name = name
        self.type = type
        self.amount = amount
    }
}

struct ExpenseRow: View {
    let item: ExpenseItem
    
    var amountColor: Color {
        switch item.amount {
        case ..<10: return .primary
        case ..<100: return .green
        default: return .red
        }
    }
    
    var body: some View {
        HStack {
            VStack(alignment: .leading) {
                Text(item.name)
                    .font(.headline)
                Text(item.type)
                    .font(.subheadline)
                    .foregroundStyle(.secondary)
            }
            
            Spacer()
            
            Text(item.amount, format: .currency(code: Locale.current.currency?.identifier ?? "USD"))
                .foregroundStyle(amountColor)
        }
    }
}

struct ContentView: View {
    @State private var sortOrder = [
        SortDescriptor(\ExpenseItem.name),
        SortDescriptor(\ExpenseItem.amount)
    ]
    
    // STEP 1a: Add state to track the user's filtering choice
    @State private var filterType = "All"

    var body: some View {
        NavigationStack {
            // STEP 1b: Pass BOTH the sortOrder and the filterType down to the child
            ExpenseListView(sortOrder: sortOrder, filterType: filterType)
                .navigationTitle("iExpense")
                .toolbar {
                    NavigationLink {
                        AddView()
                    } label: {
                        Label("Add Expense", systemImage: "plus")
                    }
                    // STEP 1c: Add a new Menu or expand your current one to host the Filter Picker
                    Menu("Filter", systemImage: "line.3.horizontal.decrease.circle") {
                        Picker("Filter", selection: $filterType) {
                            Text("Show All").tag("All")
                            Text("Personal Only").tag("Personal")
                            Text("Business Only").tag("Business")
                        }
                    }
                    Menu("Sort", systemImage: "arrow.up.arrow.down") {
                        Picker("Sort", selection: $sortOrder) {
                            Text("Sort by Name")
                                .tag([
                                    SortDescriptor(\ExpenseItem.name),
                                    SortDescriptor(\ExpenseItem.amount),
                                ])
                            Text("Sort by Amount")
                                .tag([
                                    SortDescriptor(\ExpenseItem.amount),
                                    SortDescriptor(\ExpenseItem.name)
                                ])
                        }
                    }
                }
        }
    }
}

#Preview("Main Dashboard") {
    ContentView()
        .modelContainer(for: ExpenseItem.self, inMemory: true)
}
