WezTerm - Zsh, but Terminal - bash

Mojave

Terminal on macOS Mojave doesn't seem to fare well with Starship and some of the more graphical aspects of a modern setup.

Mojave's native shell is bash by default.

You can easily run different shells in each application without changing your system's global default. On macOS Mojave, Bash is the system default, so you only need to override the settings for WezTerm.

1. Configure WezTerm to use Zsh

You can force WezTerm to launch Zsh by editing its configuration file.

  • Open or create your WezTerm config file, usually located at ~/.wezterm.lua.
  • Add the following lines to set the default shell: 
    local wezterm = require 'wezterm'
local config = {}

-- Set the default shell to zsh
config.default_prog = { '/bin/zsh', '-l' }

return config

The -l flag ensures it starts as a login shell, which is necessary for loading your ~/.zshrc and Starship settings properly.

2. Configure macOS Terminal to use Bash

Since Mojave already defaults to Bash, it may already be working as you wish. However, if you've changed your global shell previously, you can lock the Terminal app specifically to Bash:

  • Open Terminal.
  • Go to the menu bar: Terminal > Preferences (or Cmd + ,).
  • Click the General tab.
  • Find the setting "Shells open with:".
  • Select "Command (complete path)" and type: /bin/bash.
    • The default radio button won't work, it has to be hard coded.
  • Restart Terminal.

3. Separation of Configs

To keep your setups clean and prevent Starship or other Zsh tools from slowing down your Bash experience in the native Terminal:

  • For Zsh (WezTerm): Put all your Starship and fancy logic in ~/.zshrc.
  • For Bash (Terminal): Put your basic macOS settings in ~/.bash_profile.

By default, Bash will ignore your .zshrc, and Zsh will ignore your .bash_profile, keeping the two environments completely independent.

Common Aliases (Conditionals)

To share aliases across both Bash and Zsh while keeping your visual themes separate, the most reliable method is to use a standalone alias file that both shells "source" during startup.

1. Create a Common Alias File

Create a new file named .aliases in your home directory. This file will store all the commands you want to use in both WezTerm and the macOS Terminal app. 

touch ~/.aliases

Add your common aliases here (e.g., alias gs='git status'). Since the syntax for simple aliases is identical in Bash and Zsh, they will work in both environments.

2. Update Your Shell Configs

You must tell each shell to load this file when it starts up.

For Zsh (WezTerm):

Add the following to your ~/.zshrc: 

# Load common aliases
if [ -f ~/.aliases ]; then
  source ~/.aliases
fi

# Keep Zsh-only visual themes (like Starship) here
eval "$(starship init zsh)"

For Bash (macOS Terminal):

Add the following to your ~/.bash_profile: 

# Load common aliases
if [ -f ~/.aliases ]; then
  . ~/.aliases
fi

# Bash will ignore the Zsh-specific Starship config above

3. Handle Shell-Specific Logic

If you ever need a command to behave differently depending on the shell (e.g., a specific plugin only for Zsh), you can use a conditional check inside your .aliases file: 

if [ -n "$ZSH_VERSION" ]; then
    # Commands only for Zsh/WezTerm
    alias reload="source ~/.zshrc"
elif [ -n "$BASH_VERSION" ]; then
    # Commands only for Bash/Terminal.app
    alias reload="source ~/.bash_profile"
fi

Why this works

  • Visual Separation: Your ~/.zshrc contains your "heavy" visual logic (like Starship), which the Terminal app will never see because it only reads ~/.bash_profile.
  • Consistency: You only have to edit one file (~/.aliases) when you want to add a new shortcut that works everywhere.
  • Performance: Terminal app remains fast because it doesn't try to load complex Zsh plugins or themes.
