Hacking With Swift - 260 - Challenge 1

Challenge 1

  • When adding a card, the text fields keep their current text. Fix that so that the textfields clear themselves after a card is added.

My Solution

My solution is to amend the function (the one that's called when a user prompt and answer is added) with two lines to revert the variables to empty strings like this: 

func addCard() {
    let trimmedPrompt = newPrompt.trimmingCharacters(in: .whitespaces)
    let trimmedAnswer = newAnswer.trimmingCharacters(in: .whitespaces)
    guard trimmedPrompt.isEmpty == false && trimmedAnswer.isEmpty == false else { return }
    
    let card = Card(prompt: trimmedPrompt, answer: trimmedAnswer)
    cards.insert(card, at: 0)
    saveData()
    // set variables to empty strings
    newPrompt = ""
    newAnswer = ""
}

Ai Insight

QUESTION

Is my solution a good candidate for Challenge 1?

ANSWER

That is 100% the correct and intended solution for this challenge.

By setting newPrompt = "" and newAnswer = "", you are modifying the @State variables bound to your text fields. Because SwiftUI is a declarative framework, it notices that the state changed, re-renders the view, and instantly empties out those text fields for the next entry.

One Minor Tip (To Make It Even Cleaner)

Your code is perfect as it is, but since you've already gone through the trouble of trimming the whitespace to validate the input, you might want to use those same trimmed versions to clear out any accidental spaces a user typed before hitting save.

Right now, if a user types " " (just spaces) into the field, your guard statement correctly catches it and exits early (return). However, because it exits before it reaches your new clearing code, those annoying empty spaces will stay stuck in the text field.

If you move the clearing logic to the very end of the function like you did, but insert the trimmed card, it works beautifully. Alternatively, you can clear the text fields even if the guard fails, or just clear them right after the guard passes:

Here's the given solution (unchanged logic) but with comments for contemplation: 

func addCard() {
    let trimmedPrompt = newPrompt.trimmingCharacters(in: .whitespaces)
    let trimmedAnswer = newAnswer.trimmingCharacters(in: .whitespaces)
    
    // If they typed nothing but spaces, this bails out, 
    // but the text field still has those ugly spaces in it.
    guard trimmedPrompt.isEmpty == false && trimmedAnswer.isEmpty == false else { return }
    
    // 1. Create and insert the clean, trimmed data
    let card = Card(prompt: trimmedPrompt, answer: trimmedAnswer)
    cards.insert(card, at: 0)
    saveData()
    
    // 2. Clear the fields completely (wiping away any stray whitespaces)
    newPrompt = ""
    newAnswer = ""
    
    // 3. Optional: Dismiss the keyboard or close the sheet if needed!
}

The Edge Case: What Happens When The User Types Only Spaces?

If a user opens your app, types three spaces (" ") into the text field, and hits "Save", here is exactly what your code does step-by-step:

  • trimmedPrompt strips the spaces and becomes completely empty ("").
  • The guard statement checks if it's empty. It sees that it is empty, so it triggers the else { return } block.
  • The function exits immediately right there at the guard line.

Because the function exits early, it never reaches the bottom two lines (newPrompt = "" and newAnswer = "").

As a result, those three spaces stay stuck in the text box on the user's screen. They have to manually backspace them out to try again.

The Fix

If you want the text fields to clear out even if the user types invalid spaces, you can reset the strings before the guard statement exits, or handle it in an else block: 

func addCard() {
    let trimmedPrompt = newPrompt.trimmingCharacters(in: .whitespaces)
    let trimmedAnswer = newAnswer.trimmingCharacters(in: .whitespaces)
    
    // If validation fails, clear the text fields anyway so they don't stay full of spaces
    guard trimmedPrompt.isEmpty == false && trimmedAnswer.isEmpty == false else {
        newPrompt = ""
        newAnswer = ""
        return 
    }
    
    let card = Card(prompt: trimmedPrompt, answer: trimmedAnswer)
    cards.insert(card, at: 0)
    saveData()
    
    newPrompt = ""
    newAnswer = ""
}

Your original code is absolutely perfect for 99% of normal use cases! This is just a minor "defensive programming" tweak for when users accidentally mash the spacebar.
