Git - No Tracking Information (Error)

When working with git branches (that you may not have worked with before), you may get a message like this: 

❯ git pull
There is no tracking information for the current branch.
Please specify which branch you want to merge with.
See git-pull(1) for details.

    git pull <remote> <branch>

If you wish to set tracking information for this branch you can do so with:

    git branch --set-upstream-to=origin/<branch> macbook-intel

This message means that your current local Git branch is not linked to a corresponding remote branch, so Git doesn't know which remote branch to fetch and merge changes from automatically.

Here is a breakdown of the message and how to resolve it:

What the message means:

  • "There is no tracking information for the current branch":
    • Your local branch doesn't have an "upstream" branch specified. An upstream branch is the default remote branch that Git should communicate with when you use commands like git pull or git push.
  • "Please specify which branch you want to merge with":
    • Because there's no default link, Git is asking you to manually provide the remote and branch names every time, as shown in the example:
    • git pull <remote> <branch>.

How to resolve it:

The message provides a specific command to set up the tracking information so that future git pull and git push commands work seamlessly.

You should run the command suggested in the message to link your current local branch to the remote branch named macbook-intel on the origin remote: 

git branch --set-upstream-to=origin/macbook-intel macbook-intel

After running this command, your current branch will be configured to track origin/macbook-intel, and you can simply use git pull (and git push) without any extra arguments in the future.

Alternative: Pull once without setting upstream

If you only want to pull from that specific branch this one time without configuring the tracking settings for future use, you can use the suggested explicit command: 

git pull origin macbook-intel

For most workflows, however, setting the upstream is the preferred approach for convenience.

More Detail

This error means your local branch is not linked to a corresponding branch on the remote server (like GitHub or GitLab).

Without this link, known as tracking information or an upstream, Git does not know which remote branch to fetch and merge from when you simply type git pull.

Why this happened

  • New Local Branch: You created a branch locally but haven't told Git which remote branch it should "watch".
  • Missing Flags: When you first pushed the branch, you might have used git push instead of git push -u origin <branch>, which would have set up the tracking automatically.
  • Manual Creation: You manually created the branch rather than checking it out directly from a remote.

How to fix it

You can resolve this using one of the following methods:

  • Set the tracking link (Recommended):
    • Run the command suggested in your terminal to link your current branch to a remote one:
    • git branch --set-upstream-to=origin/<remote_branch_name>
    • Replace with the name of the branch on the server (usually the same as your local name).
  • Pull manually:
    • If you don't want to set a permanent link, you must specify the remote and branch every time:
    • git pull origin <branch_name>
  • Future-proofing with your first push:
    • Next time you push a new branch, use the -u flag to set the upstream relationship immediately:
    • git push -u origin <branch_name>
