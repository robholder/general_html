Setting up Venv with Git Control

Python 3.3 or later required

When working with a Virtual Environment, you don't want to add the packages (Pip installs) to the repo - the requirements file will handle that.

You just want the repo to hold the project files:

  • Mkdir for project: 
    mkdir myproject_dir
cd !$
  • Create Virtual environment (often named 'env' for convenience): 
    # Create virtual environment (venv):
python3 -m venv env
# Activate:
source env/bin/activate
  • Install required packages: 
    pip install pandas
  • Deactivate the virtual environment: 
    deactivate
  • Initialize the repo: 
    git init
  • Create a .gitignore file with code to ignore the venv directory: 
    echo 'env' > .gitignore
  • Create a requirements.txt file: 
    pip freeze > requirements.txt
  • NOTE: There is an alternate library called Pipreqs, which can sometimes do a better job of making a requirements.txt file than pip freeze.
  • When you need to clone and repopulate with the correct packages from the requirements.txt file: 
    pip install -r requirements.txt
  • Add requirements.txt file to the repo (in addition to any project files you create!): 
    # With other files:
git add .
# Or directly:
git add requirements.txt
  • Commit and push: 
    git commit -m "adding latest edits"

Alternative Venv name

Another popular name for a venv is ".venv" so that the virtual environment folder is invisible: 

python3 -m venv .venv
# And to activate:
source .venv/bin/activate
# To deactivate normally:
deactivate
