SSH Remote Login

QUESTION

Can i access my MacOS device from another computer via internet and use the shell environment and files there?

ANSWER

Yes, you can securely access your macOS device's shell environment and files from another computer over the internet using its built-in Remote Login feature, which is based on the Secure Shell (SSH) protocol.

Method 1: Using Built-in Remote Login (SSH/SFTP)

This method provides command-line access (shell) and secure file transfer (SFTP).

On your macOS device (the host):

  • Open System Settings (from the Apple menu, select System Settings, then click General in the sidebar).
  • Click on Sharing, then click the information icon (i) next to Remote Login.
  • Turn on the Remote Login service toggle.
  • In "Allow access for", choose whether to allow access for "All users" or "Only these users", and select the specific users who can log in remotely.
  • If you need to transfer files, ensure "Allow full disk access for remote users" is enabled, if required.
  • Note the SSH command displayed, which will show your username and the Mac's local network IP address (e.g., ssh username@10.0.1.2).

To connect over the internet:

  • Network Configuration: Your Mac is likely behind a router and assigned a local (private) IP address. To access it over the internet, you will need to configure port forwarding on your router to direct incoming traffic on port 22 (the default SSH port) to your Mac's internal IP address.
  • Connecting: On the other computer (Mac, Windows, or Linux):
    • Open a Terminal app or an SSH client (like PuTTY on Windows).
    • Use the public IP address of your router (which you can find by searching online for "What's my IP") in the SSH command format: ssh username@your_router_public_ip.
    • Enter your password when prompted to establish a secure connection.
    • You can now use the shell environment and transfer files using SFTP clients.

Method 2: Using a VPN or Third-Party Software

For simpler and more secure internet access without complex router configuration, you can use a Virtual Private Network (VPN) service or a dedicated third-party remote access application.

  • VPN: A VPN creates a secure, private connection between your computers over the internet, making them behave as if they are on the same local network. This allows you to connect using the simpler local IP address mentioned in Method 1. Tools like Tailscale can simplify this.
  • Third-Party Tools: Software like Chrome Remote Desktop, AnyDesk, or Splashtop offer user-friendly, cross-platform solutions for both graphical control and file transfer over the internet, often without needing port forwarding.

Security Note: Exposing your Mac directly to the internet via port forwarding increases security risks. Using strong passwords or, preferably, SSH keys for authentication is highly recommended for enhanced security.

Login from Linux to MacOS

QUESTION

If I am on a Linux machine on a separate network, and ssh into a MacOS terminal on another machine, can I copy files back and forth?

ANSWER

Yes, you absolutely can copy files back and forth between a Linux machine and a macOS machine over the internet, provided you have SSH access set up correctly. The SSH protocol includes built-in command-line tools for secure file transfer: scp (Secure Copy Protocol) and sftp (SSH File Transfer Protocol).

Prerequisites

  • Remote Login enabled on Mac: Ensure you have enabled Remote Login in your macOS System Settings to allow SSH connections.
  • Network Access: Both computers must be connected to the internet, and the necessary network configurations (like port forwarding on the macOS machine's router) must be in place, as standard SSH operates on port 22.
  • Credentials: You need a valid username and password (or SSH keys for passwordless authentication) for the macOS machine.

File Transfer Methods

You can perform file transfers from either the Linux machine (initiating a pull from or push to the Mac) or from the Mac side.

  1. Using scp (Secure Copy)
    • scp is a command-line tool that uses the SSH protocol and is ideal for quick file or directory transfers.
  2. Copy a file from the Linux machine (local) to the Mac (remote): 
scp /path/to/local/file username@mac_public_ip:/path/to/remote/directory/

Use code with caution.

Example: scp important_doc.txt johnny@203.0.113.15:/Users/johnny/Documents/

  • Copy a file from the Mac (remote) to the Linux machine (local): 
scp username@mac_public_ip:/path/to/remote/file /path/to/local/directory/

Use code with caution.

Example: scp johnny@203.0.113.15:/Users/johnny/Desktop/screenshot.png /home/linuxuser/Downloads/

  • Copy an entire directory: Use the -r (recursive) flag. 
scp -r /path/to/local/dir username@mac_public_ip:/path/to/remote/directory/

Use code with caution.

  1. Using sftp (SSH File Transfer Protocol)
    • sftp provides an interactive command-line interface that functions similarly to an old-school FTP client but over a secure SSH connection.
  2. From the Linux terminal, start an sftp session: 
sftp username@mac_public_ip

Use code with caution.

  • Once connected (after entering your password), you can use sftp commands:
  • ls: List remote files
  • lls: List local files
  • cd remote_dir: Change remote directory
  • lcd local_dir: Change local directory
  • get remote_file: Download a file from the Mac to the Linux machine
  • put local_file: Upload a file from the Linux machine to the Mac
  • quit: Exit the sftp session

Alternative: Graphical Clients

If you prefer a graphical user interface over command-line tools, you can use cross-platform SFTP clients such as FileZilla (available for both Linux and Mac). These clients allow you to drag and drop files between the two machines securely.

Transferring Files

QUESTION

Would this approach be good for working on repos on the MacOS and transferring files to execute to the Linux machine?

ANSWER

Yes, using SSH-based methods (ssh, scp, and sftp) is a standard and highly effective approach for managing files and repositories remotely. It is suitable for your workflow of developing on the macOS device and executing files on the Linux machine.

Here is a breakdown of why this approach works well and how to leverage it effectively:

Why this approach is good:

  • Security: SSH is built on strong encryption, ensuring that your code, credentials, and data transfers are secure over the public internet.
  • Efficiency for Developers: Command-line tools like scp and rsync are fast and scriptable, which is crucial for automation in a development workflow.
  • Direct Shell Access: You can directly SSH into the macOS machine to manage Git operations (commits, pushes, pulls) and perform local builds before transferring the outputs.
  • Cross-Platform Compatibility: Both macOS and Linux have robust, native support for all SSH tools.

How to adapt the workflow:

You have two primary options depending on which machine holds the definitive "source" of the code.

Option 1: Mac is the Development Machine (Recommended)

This is the scenario you described: you develop, commit, and build on the macOS machine, then transfer the final output (e.g., compiled binaries, final scripts, container images) to the Linux machine for execution or testing.

  • Develop & Commit: Work within the repository on your Mac's filesystem. Use your local SSH session to manage Git commands.
  • Build (if necessary): Compile your code on the Mac.
  • Transfer Files: Use scp or, even better, rsync to transfer only the necessary files to the Linux machine.
    • rsync is more efficient than scp for frequent updates because it only copies the changes (deltas) between files, not the entire file each time.

Example rsync command: 

rsync -avz /path/to/mac/build/folder/ username@linux_ip:/path/to/execution/folder/

Use code with caution.

  • Execute: Use the SSH session to execute the files on the Linux machine: 
ssh username@linux_ip './start_script.sh'

Use code with caution.

Option 2: Linux is the Development Machine

Alternatively, you could keep the main repo on the Linux machine and SSH into the Linux machine from the Mac to do all your work remotely.

  • You would use a feature called SSHFS (SSH Filesystem) to mount the remote Linux directory onto your local Mac desktop as if it were a local drive. This lets you use your favorite local macOS editor while the files physically reside on the Linux machine.
