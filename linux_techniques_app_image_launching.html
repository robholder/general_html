To Launch AppImage from Shell Directly

The simplest way to do this is by creating a symbolic link (symlink) to the AppImage file in a directory that is already in your system's PATH.

What is PATH? 💻

Your PATH is an environment variable that tells your shell (like Bash or Zsh) where to look for executable files when you type a command. By default, it includes directories like /usr/local/bin and /usr/bin. When you place a symlink to your AppImage in one of these directories, you can then run it from any location just by typing its name, like you would with any other installed program.

Steps to create a symlink:

Open your terminal and navigate to the directory where your AppImage is stored: 

cd /opt

Make sure the AppImage is executable (if it isn't already): 

chmod +x Inkscape-*.AppImage

(Note: Replace Inkscape-*.AppImage with the actual filename of your AppImage).

Create the symbolic link. You'll likely need root permissions for this, since you're writing to a system directory like /usr/local/bin: 

sudo ln -s /opt/Inkscape-*.AppImage /usr/local/bin/inkscape

This command creates a link named inkscape in the /usr/local/bin directory that points to your AppImage file in /opt.

Now, you can simply type inkscape in your terminal from any location to launch the program.

Alternative (with a desktop entry)

If you'd like the program to also appear in your application menu, you can create a .desktop file. This file provides the graphical environment with information about the program, including its name, icon, and the command to run it.

Create a new file in ~/.local/share/applications/ (this location is for user-specific applications): 

nano ~/.local/share/applications/inkscape.desktop

Paste the following content into the file. Make sure to replace /opt/Inkscape-*.AppImage with the correct path to your file. 

[Desktop Entry]
Name=Inkscape
Exec=/opt/Inkscape-*.AppImage
Icon=inkscape
Type=Application
Comment=Draw freely
Categories=Graphics;VectorGraphics;

(You might need to find the correct icon name if the default inkscape doesn't work, but it usually does.)

Save and exit the text editor. You may need to log out and back in for the changes to take effect in your application menu.

In Tcsh

The methods described will work in tcsh, but with a few minor changes due to differences in syntax for setting environment variables.

Symbolic Link

Creating the symbolic link itself is a standard Linux command and isn't shell-specific. The ln command works the same in tcsh as it does in bash: 

sudo ln -s /opt/Inkscape-*.AppImage /usr/local/bin/inkscape

The difference comes when you want to modify your PATH variable to include a new directory.

Modifying PATH in tcsh

In tcsh, environment variables are set using setenv instead of export. Also, PATH is handled a bit differently. It's often manipulated as an array of directories rather than a colon-separated string. The tcsh shell actually maintains two variables, PATH (the environment variable) and path (an internal shell variable), and they are kept in sync.

To add a directory (like ~/bin) to your PATH in tcsh, you would use this syntax in your ~/.tcshrc or ~/.login file: 

set path = ( $path ~/bin )

This appends the ~/bin directory to your existing path array.

Desktop Entry File

The .desktop file is a standard defined by the FreeDesktop.org project, not a shell-specific feature. Its syntax is the same regardless of whether you're using tcsh, bash, or any other shell. The file simply provides metadata about the application, including the path to its executable. When a desktop environment (like GNOME, KDE, or XFCE) needs to launch the application, it reads the Exec line from the .desktop file and executes that command. The shell you're using in your terminal doesn't factor into this process.

So, the .desktop file you would create for Inkscape remains identical to the one described previously: 

[Desktop Entry]
Name=Inkscape
Exec=/opt/Inkscape-*.AppImage
Icon=inkscape
Type=Application
Comment=Draw freely
Categories=Graphics;VectorGraphics;
