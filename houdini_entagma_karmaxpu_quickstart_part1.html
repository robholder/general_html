Intro To Solaris

YouTube - Intro To Solaris

Solaris Desktop

Switch the 'Desktop' to the Solaris desktop from the Build menu.

While using this setup, it's like working in regular Obj space, but now objects, materials, lights and rendering are all together.

Importing Geometry To Solaris & Setting Up A Basic Scene

SOP Import

First, to bring in object (from Obj context):

  • Drop a SOP Import node:
    • In the SOP Path put the path to the object to import (usually an 'OUT' null).

Now we should see the geometry in the viewport.

Material Library

Next, connected from the SOP Import:

  • Create a Material Library node.

Lights

Connected again, add any lights you need.

  • <cntl> LMB on the shelf lights to drop them at the end of the selected node in the network.

Camera

As with the light[s]:

  • Select the last node in the chain, and <cntl> LMB on the camera tool on the LOP Lights and Cameras shelf:
    • This will drop a camera, and also the viewport will be viewing through it (and already a locked view so edits in the viewport will translate the camera too.

Rendering

We'll need two nodes to complete the rendering setup.

From the selected Camera node:

  • Drop a Karma Render Settings node.
  • Add a USD Render ROP.

These two nodes set up the rendering - settings for all the parameters for the render, and the ROP to actually perform the render submission.

Domelight Settings

YouTube - Domelight Settings

In the Viewport, set the interactive render engine to Karma (in the first dropdown at the top-right of the Viewport).

You can change the Karma Render Settings node's parameters and see the changes in a cut down form in the Viewport.

There is an icon in the center representing the Environment (Dome) Light. Click on the Light icon on the right panel of the Viewport to toggle it off.

Setting the HDRI Map for the Dome Light

Select the Environment Light node:

  • In the parameters:
    • Texture - navigate to an HDRI map using the file browser (arrow on a file icon) at the end of the path field.

Assigning & Creating A Material Using MaterialX

YouTube - Assigning & Creating A Material Using MaterialX

I used the Karma Material Builder for this stage. I didn't delete any existing nodes (as was done in the tutorial).
  • Jump inside the Material Library node (i or Double-LMB):
    • Drop a USD MaterialX Subnet:
      • It will have a default name like mtlxsubnet:
        • Feel free to rename.
      • Inside this node-network:
        • Delete all but the two nodes on the right:
          • surface_output
          • displacement_output
        • Drop a MtlX Standard Surface node:
          • Wire the out output into the subout on the surface_output node.

Assigning to geometry

Back up at the SOP level:

  • Select the Material Library:
    • In the Material VOP field, navigate to the USD MaterialX Subnet (either 'mtlxsubnet' or whatever it was named).
    • Check the Assign to Geometry checkbox.
    • In the Geometry Path:
      • All Geometry Primitives
      • (Or path to specific geometry.)

Setting up the Material Properties

Go back into the Material Library node:

  • Select the mtlxstandard_surface1 node (the 'MtlX Standard Surface'):
    • Base tab:
      • Base Color to a warm yellow copper color.
      • Dial the Metalness to 1.
    • Specular tab:
      • Dial the Specular Roughness to 0.

Blending Colors (DIY Fresnel Falloff)

YouTube - Blending Colors (DIY Fresnel Falloff)

Next we want to have an viewing angle dependent blend of color shades (a Fresnel falloff or blend).

  • Drop a Facing Ratio node:
    • This node needs two inputs:
      • Camera ray:
        • Drop a Karma Ray Import:
          • (See info box below: In some versions of Houdini, this node isn't available at every level, and you may have to move up one level to create it and then cut and paste to inside the correct level.)
        • Set the Signature to Vector 3
        • Set the Variable Name to ray:direction ('Ray Direction (vector)' in the dropdown).
        • Plug the out into the in1 input of the 'facing ratio' node.
        • The ray is actually the opposite direction to that of what we need:
        • (I didn't find any difference with this direction change - it looked the same.)
          • Drop a MtlX Multiply node between the nodes:
            • Signature to Vector 3
            • Set the input 2 fields to -1, -1, -1
      • Surface Normal:
        • Drop a MtlX Normal node:
          • connect the out into the in2 input of the 'facing ratio' node.

Quirks In MaterialX Subnets (plz help!)

YouTube - Quirks In MaterialX Subnets (plz help!)

Answer in YouTube comments:

The reason why you can't see the "kma" nodes is because they are houdini specific nodes by sidefx, not part of the materialX definition. So access them you need to put down the "Karma materialX subnet" and NOT the "USD materialX subnet" which is strictly following the materialX definition. Same way you get access to the volume and the hair shader, you have to create the karma materialX subnet, its the same thing + sidefx nodes.

To see in the Viewport

  • Plug the out from the 'facing ratio' node into the base_color input of the mtlxstandard_surface1 node.

We'll see a subtle facing ratio gradient (if we turn down the metalness temporarily). (NOTE: The copper color will not be there anymore - the facing ratio is now driving the color 100%.)

  • Between the facing ratio node and the surface output, drop an MtlX Color Ramp node:
    • Set up the ramp with the color range we want on the surface.
  • If you want to invert the mapping, on the facing ratio node:
    • Check the Invert checkbox.
  • While on the facing ratio node, you can adjust:
    • Bias (quite drastic changes may be necessary to get the blend looking good)
    • Gain (this will smooth the transition with higher numbers)
    • Gamma

Using the Facing Ratio for Roughness

To re-use the facing ratio:

  • Fork an MtlX Remap node from the out of the facing ratio node (to its in):
    • Inlow to 0
    • Inhigh to 1
    • Outlow to 0
    • Outhigh to 0.5
  • Plug this into our specular_roughness:
    • out to specular_roughness
  • This should give the surface looking straight at us a less reflective look.

Setting Up A Background Grid

YouTube - Setting Up A Background Grid

Creating a Backdrop

Drop a free-floating Grid at the Stage level:

  • This is actually a SOP Create node in type.
  • Jump into the node:
    • Add a UV Texture node:
      • Set to Projection Axis to Y Axis
      • Set the Angle to 30

Back at the Stage level:

  • Create another Material Library connected to the grid:
    • Inside:
      • Drop a USD MaterialX Subnet and rename to BG.
      • Delete all but the two outputs (like the first section):
        • Drop a MtlX Standard Surface:
          • Connect the out to the subout
          • Base tab:
            • Set the color to a blue-green (0.4, 1, 0.7)
          • Specular tab:
            • Set the Specular to zero.

Back to Stage:

  • After the first Material Library node:
    • Drop a Merge node:
      • Join the two streams.
  • Insert a Transform after the Grid's Material Library node:
    • Delete the string in the Primitives field.
    • Using the dropdown, set to All Geometry Primitives
    • Set Translate to 0, 0, -8
    • Set Rotate to 90, 0, 0
    • Uniform Scale to 1.7
  • In the Material Library node parameters:
    • Material VOP field, navigate to the BG
    • Check 'On' Assign to Geometry
    • Geometry Path to All Geometry Primitives

Setting up a Color Ramp on the Backdrop

Inside the MtlX Standard Surface (BG) node:

  • Drop a MtlX Geometry Property Value node:
    • Set the Signature to Vector 2
    • Set the Geomprop to st
      • To use the uv attribute, we have to be careful - in USD world, uv becomes st
  • Add and connect a MtlX Separate2 node:
    • This actually outputs a black to white ramp at this stage.
  • Drop a MtlX Color Ramp node:
    • Connect to the base_color input on the standard surface node.
    • Set a ramp of color (tutorial uses a yellow to cyan)

Setting Up KarmaXPU For Rendering

YouTube - Setting Up KarmaXPU For Rendering

On the Karma Render Settings:

  • Output Picture to a render path on disk.
  • Try the Rendering Engine XPU option
  • Path Traced Samples to 128

Quick note on Advanced tab:

  • Limits tab:
    • Color Limit is how bright the render can go:
      • low values make the image potentially dull

USD Render ROP

On the USD Render ROP:

  • set Valid Frame Range to Render Current Frame

To render:

  • Render to MPlay
  • We can save as from the finished render (in MPlay)

Applying OCIO Color Space (ACESCG to srgb)

YouTube - Applying OCIO Color Space (ACESCG to srgb)

If you have ACES setup in Houdini, in a compositor (like DaVinci Resolve), we'd have to have an OCIO node set to:

  • Source Space - ACEScg
  • Output Space - out_srgb
