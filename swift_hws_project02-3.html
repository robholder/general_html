Hacking With Swift - Project 2 part three - Guess the Flag

Guess the Flag: Wrap up

You’ll use VStack, HStack, and ZStack in almost every project you make, and you’ll find you can quickly build complex layouts by combining them together.

Many people find SwiftUI’s way of showing alerts a little odd at first: creating it, adding a condition, then simply triggering that condition at some point in the future seems like a lot more work than just asking the alert to show itself. But like I said, it’s important that our views always be a reflection of our program state, and that rules out us just showing alerts whenever we want to.

Challenge

One of the best ways to learn is to write your own code as often as possible, so here are three ways you should try extending this app to make sure you fully understand what’s going on:

  • Add an @State property to store the user’s score, modify it when they get an answer right or wrong, then display it in the alert and in the score label.
  • When someone chooses the wrong flag, tell them their mistake in your alert message – something like “Wrong! That’s the flag of France,” for example.
  • Make the game show only 8 questions, at which point they see a final alert judging their score and can restart the game.
  • Hacking with Swift+ subscribers can get a complete video solution for this checkpoint here: Solution to Guess the Flag. If you don’t already subscribe, you can start a free trial today.

Note: That last one takes a little more thinking than the others. A good place to start would be to add a second alert() modifier watching a different Boolean property, then connect its button to a reset() method to set the game back to its initial state.

AI Suggestions

This looks like a solid implementation of the "Guess the Flag" game! Your logic is sound, and the layout is clean. However, in SwiftUI, as your views grow, the code can become a "pyramid of doom" with nested stacks.

Here are a few ways to make this cleaner, more "Swifty," and easier to maintain.

1. Extract Subviews

The biggest improvement you can make is extracting the Flag image into its own view. This removes the repetitive styling (shadow, corner radius) from your main body and makes it reusable. 

struct FlagImage: View {
    let name: String

    var body: some View {
        Image(name)
            .clipShape(.rect(cornerRadius: 5))
            .shadow(radius: 8)
    }
}

2. Use Custom View Modifiers

You have a specific style for your "Title" and "Score" text. You can create a custom modifier to keep those styles consistent. 

struct WhiteTitle: ViewModifier {
    func body(content: Content) -> some View {
        content
            .font(.largeTitle.bold())
            .foregroundStyle(.white)
    }
}

extension View {
    func whiteTitleStyle() -> some View {
        modifier(WhiteTitle())
    }
}

3. Refine the Logic

Constants: Instead of hardcoding 0...2 or tries > 3, use a constant or a computed property. It makes the "rules" of your game easier to change in one place.

Alerts: You have two alerts that look very similar. While having two is fine, you could technically use a single alert that changes its button text based on the tries count.

The "Cleaned Up" Version

Here is how the code looks with those changes applied: 

import SwiftUI

struct ContentView: View {
    @State private var countries = ["Estonia", "France", "Germany", "Ireland", "Italy", "Monaco", "Nigeria", "Poland", "Spain", "UK", "Ukraine", "US"].shuffled()
    @State private var correctAnswer = Int.random(in: 0...2)

    @State private var score = 0
    @State private var scoreTitle = ""
    @State private var showingScore = false
    @State private var tries = 0
    @State private var complete = false
    
    let maxTries = 8 // Easier to adjust the game length here

    var body: some View {
        ZStack {
            RadialGradient(stops: [
                .init(color: Color(red: 0.1, green: 0.2, blue: 0.45), location: 0.3),
                .init(color: Color(red: 0.76, green: 0.15, blue: 0.26), location: 0.3),
            ], center: .top, startRadius: 200, endRadius: 700)
            .ignoresSafeArea()

            VStack {
                Spacer()

                Text("Guess the Flag")
                    .whiteTitleStyle()
                
                VStack(spacing: 15) {
                    VStack {
                        Text("Tap the flag of")
                            .foregroundStyle(.secondary)
                            .font(.subheadline.weight(.heavy))
                        
                        Text(countries[correctAnswer])
                            .font(.largeTitle.weight(.semibold))
                    }
                    
                    ForEach(0..<3) { number in
                        Button {
                            flagTapped(number)
                        } label: {
                            FlagImage(name: countries[number])
                        }
                    }
                }
                .frame(maxWidth: .infinity)
                .padding(.vertical, 20)
                .background(.regularMaterial)
                .clipShape(.rect(cornerRadius: 20))
                
                Spacer()
                Spacer()
                
                Text("Score: \(score)")
                    .whiteTitleStyle()
                
                Spacer()
            }
            .padding()
        }
        // Unified Alert logic (optional, but keeps the body shorter)
        .alert(scoreTitle, isPresented: $showingScore) {
            Button("Continue", action: askQuestion)
        } message: {
            Text("Your score is \(score)")
        }
        .alert("Game Over", isPresented: $complete) {
            Button("Reset", action: reset)
        } message: {
            Text("Your final score is \(score) / \(maxTries)")
        }
    }

    func flagTapped(_ number: Int) {
        tries += 1
        
        if number == correctAnswer {
            scoreTitle = "Correct"
            score += 1
        } else {
            scoreTitle = "Wrong! That's the flag of \(countries[number])"
        }

        if tries >= maxTries {
            complete = true
        } else {
            showingScore = true
        }
    }

    func askQuestion() {
        countries.shuffle()
        correctAnswer = Int.random(in: 0...2)
    }

    func reset() {
        tries = 0
        score = 0
        askQuestion()
    }
}

// Reusable components kept outside the main view
struct FlagImage: View {
    var name: String
    var body: some View {
        Image(name)
            .clipShape(.rect(cornerRadius: 5))
            .shadow(radius: 8)
    }
}

struct WhiteTitle: ViewModifier {
    func body(content: Content) -> some View {
        content
            .font(.title.bold())
            .foregroundStyle(.white)
    }
}

extension View {
    func whiteTitleStyle() -> some View {
        modifier(WhiteTitle())
    }
}

    ContentView()
}

Why this is "Better":

  • Readability: The body property is now much shorter. You can see the structure of the UI without getting bogged down in image styling or font weights.
  • Scalability: If you decide you want the flags to be Capsule shaped later, you only have to change it once in FlagImage instead of three times in a loop.
  • Encapsulation: The WhiteTitle modifier ensures that every time you display a score or a header, it looks exactly the same.
