Hacking With Swift - 88 - Animation: Wrap up

This technique project started off easier, took a few twists and turns, and progressed into more advanced animations, but I hope it’s given you an idea of just how powerful - and how flexible! - SwiftUI’s animation system is.

As I’ve said previously, animation is about both making your app look great and also adding extra meaning. So, rather than making a view disappear abruptly, can you add a transition to help the user understand something is changing?

Also, don’t forget what it looks like to be playful in your user interface. My all-time #1 favorite iOS animation is one that Apple ditched when they moved to iOS 7, and it was the animation for deleting passes in the Wallet app - a metal shredder appeared and cut your pass into a dozen strips that then dropped away. It only took a fraction of a second more than the current animation, but it was beautiful and fun too!

Challenge One of the best ways to learn is to write your own code as often as possible, so here are three ways you should try extending this app to make sure you fully understand what’s going on.

Go back to the Guess the Flag project and add some animation:

  • When you tap a flag, make it spin around 360 degrees on the Y axis.
  • Make the other two buttons fade out to 25% opacity.
  • Add a third effect of your choosing to the two flags the user didn’t choose - maybe make them scale down? Or flip in a different direction? Experiment!

These challenges aren’t easy. They take only a few lines of code, but you’ll need to think carefully about what modifiers you use to get the exact animations you want. Try adding an @State property to track which flag the user tapped on, then using that inside conditional modifiers for rotation, fading, and whatever you decide for the third challenge.

Flag Rotation

The "Angle" Fix

Instead of checking which index was tapped, let's give every button its own rotation state. The most effective way to do this within a ForEach without creating a mess of variables is to use an array of doubles. 

// 1. Create an array to hold the rotation for each of the 3 buttons
@State private var rotations = [0.0, 0.0, 0.0]

// ... inside your body ...

ForEach(0..<3) { number in
    Button {
        withAnimation(.spring(duration: 1, bounce: 0.5)) {
            // 2. Add 360 to ONLY this button's rotation index
            rotations[number] += 360
        }
        flagTapped(number)
    } label: {
        FlagImage(name: countries[number])
    }
    // 3. Bind the rotation directly to that button's specific value
    .rotation3DEffect(.degrees(rotations[number]), axis: (x: 0, y: 1, z: 0))
}

Why this works

  • Unique Identity: Each button now has its own "bank" of degrees. Button 0 doesn't care what Button 1 is doing.
  • Infinite Taps: Because we use += 360, every single tap increases the number. 0 -> 360 -> 720. SwiftUI sees a change every time, so it triggers the animation every time.
  • No Ghost Spins: Since you aren't changing a shared selectedNumber variable, the other buttons have no reason to react or "un-spin."

Resetting the Game

When you move to the next round (in your askQuestion function), you have a choice:

  • Do nothing: The buttons will just stay at whatever rotation they ended at. When clicked again, they'll spin from 360 to 720. This is usually fine!
  • Hard Reset: If you want them all to point back to 0 without animating, you can do:

rotations = [0.0, 0.0, 0.0] (Do this outside of a withAnimation block so they snap back invisibly).

Summary of the logic evolution:

  • Global Variable: All buttons share one number (Everyone spins).
  • Ternary Index: Shared index, but filtered (Previous button "un-spins").
  • State Array: Every button has its own number (Only the clicked button spins, every time).

import SwiftUI

struct ContentView: View {
    @State private var countries = ["Estonia", "France", "Germany", "Ireland", "Italy", "Monaco", "Nigeria", "Poland", "Spain", "UK", "Ukraine", "US"].shuffled()
    
    @State private var correctAnswer = Int.random(in: 0...2)

    @State private var score = 0
    @State private var scoreTitle = ""
    @State private var showingScore = false
    @State private var tries = 0
    @State private var complete = false
    
    @State private var rotations = [0.0, 0.0, 0.0]
    
    let maxTries = 8 // Easier to adjust the game length here

    var body: some View {
        ZStack {
            RadialGradient(stops: [
                .init(color: Color(red: 0.1, green: 0.2, blue: 0.45), location: 0.3),
                .init(color: Color(red: 0.76, green: 0.15, blue: 0.26), location: 0.3),
            ], center: .top, startRadius: 200, endRadius: 700)
            .ignoresSafeArea()

            VStack {
                Spacer()

                Text("Guess the Flag")
                    .whiteTitleStyle()
                
                VStack(spacing: 15) {
                    VStack {
                        Text("Tap the flag of")
                            .foregroundStyle(.secondary)
                            .font(.subheadline.weight(.heavy))
                        
                        Text(countries[correctAnswer])
                            .font(.largeTitle.weight(.semibold))
                    }
                    
                    ForEach(0..<3) { number in
                        Button {
                            // Spin 360 when clicked
                            withAnimation(.spring(duration: 1, bounce: 0.5)) {
                                rotations[number] += 360
                            }
                            flagTapped(number)
                            // Reset rotations
                            rotations = [0.0, 0.0, 0.0]

                        } label: {
                            FlagImage(name: countries[number])
                        }
                        // Actual rotation animation specs
                        .rotation3DEffect(.degrees(rotations[number]), axis: (x: 1, y: 0, z: 0))
                    }
                }
                .frame(maxWidth: .infinity)
                .padding(.vertical, 20)
                .background(.regularMaterial)
                .clipShape(.rect(cornerRadius: 20))
                
                Spacer()
                Spacer()
                
                Text("Score: \(score)")
                    .whiteTitleStyle()
                
                Spacer()
            }
            .padding()
        }
        // Unified Alert logic (optional, but keeps the body shorter)
        .alert(scoreTitle, isPresented: $showingScore) {
            Button("Continue", action: askQuestion)
        } message: {
            Text("Your score is \(score)")
        }
        .alert("Game Over", isPresented: $complete) {
            Button("Reset", action: reset)
        } message: {
            Text("Your final score is \(score) / \(maxTries)")
        }
    }

    func flagTapped(_ number: Int) {
        tries += 1
        
        if number == correctAnswer {
            scoreTitle = "Correct"
            score += 1
        } else {
            scoreTitle = "Wrong! That's the flag of \(countries[number])"
        }

        if tries >= maxTries {
            complete = true
        } else {
            showingScore = true
        }
    }

    func askQuestion() {
        countries.shuffle()
        correctAnswer = Int.random(in: 0...2)
    }

    func reset() {
        tries = 0
        score = 0
        askQuestion()
    }
}

// Reusable components kept outside the main view
struct FlagImage: View {
    var name: String
    var body: some View {
        Image(name)
            .clipShape(.rect(cornerRadius: 5))
            .shadow(radius: 8)
    }
}

struct WhiteTitle: ViewModifier {
    func body(content: Content) -> some View {
        content
            .font(.title.bold())
            .foregroundStyle(.white)
    }
}

extension View {
    func whiteTitleStyle() -> some View {
        modifier(WhiteTitle())
    }
}

Add Fading and Shrinking Non-Picks

1. Add the state variable

Add this near your other @State properties: 

@State private var selectedNumber = -1 // -1 means nothing is tapped yet

2. Update the Button logic

Modify your Button action to set that number. Note that I removed the rotations = [0,0,0] reset from inside the button—resetting there would snap the flag back to 0 immediately, killing the 1-second animation you just started! 

Button {
    withAnimation(.spring(duration: 1, bounce: 0.5)) {
        rotations[number] += 360
        selectedNumber = number // Track which one was clicked
    }
    
    // Slight delay so the user sees the animation before the alert pops up
    DispatchQueue.main.asyncAfter(deadline: .now() + 0.5) {
        flagTapped(number)
    }
} label: {
    FlagImage(name: countries[number])
}

3. Add the Opacity Ternary

Now, apply the opacity modifier to the button. The logic is: "If nothing is selected, opacity is 1. If something is selected, and it's NOT me, opacity is 0.25." 

.rotation3DEffect(.degrees(rotations[number]), axis: (x: 1, y: 0, z: 0))
// New Opacity Logic
.opacity(selectedNumber == -1 || selectedNumber == number ? 1.0 : 0.25)
// Bonus: Scale down the unselected ones for a better "pop" effect
.scaleEffect(selectedNumber == -1 || selectedNumber == number ? 1.0 : 0.85)
// Ensure the opacity change is animated
.animation(.default, value: selectedNumber)

4. Reset in askQuestion

Finally, you need to "turn the lights back on" for the next round. Update your askQuestion function: 

func askQuestion() {
    countries.shuffle()
    correctAnswer = Int.random(in: 0...2)
    selectedNumber = -1       // Reset selection
    rotations = [0.0, 0.0, 0.0] // Reset rotation values safely here
}

Saturation

If you want to make the selected flag really "pop," try adding a Saturation modifier using that same selectedNumber logic. It makes the "wrong" flags turn grayscale while the "correct" one stays vibrant: 

.saturation(selectedNumber == -1 || selectedNumber == number ? 1.0 : 0.0)

It adds a really professional, "polished app" feel to the interaction!
