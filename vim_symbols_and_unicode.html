Get Unicode Values

Finding the value of the character under the cursor:

  • ga → shows the decimal, hexadecimal and octal value of the character under the cursor. If there are composing characters these are shown too. (If the message is truncated, use :messages).
  • :ascii → same as ga
  • g8 → shows the bytes used in a UTF-8 character, also the composing characters, as hex numbers.

Example

For →:

  • ga:
    • <→> 8594, Hex 2192, Oct 20622, Digr ->
  • The Hex number is the unicode:
    • <ctrl>vu2192 will create → in Insert mode.

You can map this to be able to extract just the hex code to the '+' clipboard: 

nnoremap <leader>ch :let @+ = matchstr(execute('normal! ga'), 'Hex \zs[0-9A-F]*')<CR>

Digraphs vs Unicode

In Vim, digraphs and Unicode are both ways to input special characters, but they work differently. Here’s the breakdown:

Digraphs

🔹 A shortcut system to type special characters using two-key combinations. 🔹 Vim has predefined digraphs, like :digraphs showing available pairs. 🔹 Example:

  • Typing Ctrl+K then o then / → Inserts ø (the Scandinavian Ø character).
  • Ctrl+K then A then E → Inserts Æ.

✅ Useful when:

  • You don’t remember Unicode codes.
  • You want quick shortcuts for specific characters.

Unicode

🔹 Directly inserts Unicode characters using their code points. 🔹 Works by typing the sequence: 

Ctrl+V u <Unicode Hex Code>

🔹 Example:

  • Ctrl+V u 00F8 → Inserts ø (Unicode U+00F8).
  • Ctrl+V u 2665 → Inserts ♥ (Unicode U+2665).

✅ Useful when:

  • You need precise control over Unicode characters.
  • You’re working with non-standard characters not included in digraphs.

Key Differences

Feature Digraphs Unicode
Input Method Two-key sequences Hexadecimal Unicode codes
Ease of Use Easy & built-in Requires knowing the code point
Scope Limited to predefined digraphs Supports all Unicode characters

If you need something quick, use digraphs. If you need full control, use Unicode. 🚀

Useful Emojis

Here’s a list of useful emojis that might work well in Vim:

Symbols for Marking Lines & Highlights

  • ✅ ✔️ (Checkmark) → Mark completed tasks (u2705)
  • ⚠️ (Warning) → Flag important sections
  • ❌ (Cross) → Indicate errors or deletions
  • ⭐ (Star) → Highlight key lines
  • 🔥 (Fire) → Track hot spots in code
  • → (Arrow) → Provide comment

Debugging & Coding

  • 🐛 (Bug) → Mark problematic lines
  • 🔧 (Wrench) → Used for areas needing fixes
  • 🛠️ (Tools) → Notes for refactoring
  • 🚀 (Rocket) → Optimized, fast code sections
  • 🎯 (Target) → Important logic points

Organization & Commenting

  • 📌 (Pin) → Keep track of important notes
  • 📖 (Book) → Reference documentation
  • 📝 (Memo) → General comments and notes
  • ⏳ (Hourglass) → Areas with slow performance
  • 🚧 (Construction) → Work-in-progress sections

How to Use Emojis in Vim

You can insert them using:

  • Copy/Paste directly into your Vim document (CTRL-V works in most terminals).
  • Use Unicode: 
Ctrl+V u 1F680  " 🚀 Rocket

Here’s how you can integrate emojis into Vim for comments, status lines, and custom mappings:

Emojis in Comments (Great for Marking Code)

You can use emojis in comments to highlight important sections: 

// ✅ Done: Implemented the search feature
// 🛠️ TODO: Optimize the sorting algorithm
// 🐛 Bug: Fix array index issue

This makes comments visually distinct and easy to scan.

Emojis in Status Line (Custom Vim Powerline)

If you're using Vim's status line, you can add emojis to make it stand out: 

set statusline=%f\ 🔥\ %y\ 📖\ %m

🔹 Displays the filename (%f), file type (%y), and 🔥 for modified files (%m). 🔹 Try adding 🎯 (Target), ⚠️ (Warning), or 🚀 (Performance Boost).

Emojis in Custom Mappings

You can bind shortcuts with emojis to make keybindings more intuitive: 

nnoremap <leader>b :echo "🐛 Bug Fix Reminder!"<CR>
nnoremap <leader>c :echo "✅ Code Cleaned!"<CR>

Whenever you press b, Vim will display "🐛 Bug Fix Reminder!"—fun, right? 😃

Using Emojis in Text Editing

If your terminal supports Unicode, you can insert emojis directly into text files using: 

Ctrl+V u 1F680  " 🚀 Rocket emoji
Ctrl+V u 1F4DD  " 📝 Memo emoji

Emoji-enhance Git Commits

Emoji-enhanced Git commits make commit messages more visually engaging and help quickly categorize changes. Here’s how to do it:

Use Emojis in Your Commit Messages

Just include emojis directly in your commits: 

git commit -m "🚀 Improved performance in rendering engine"
git commit -m "🐛 Fixed memory leak issue"
git commit -m "🛠️ Refactored helper functions"
git commit -m "📖 Updated README documentation"

✅ This makes commit logs easy to scan at a glance!

Standardized Emoji Commit Conventions

Many developers use a standard emoji guide for common commit types. Here’s a cheat sheet:

Emoji Commit Purpose
🚀 Performance improvements
🐛 Bug fixes
🔧 Fixes in tools or scripts
📖 Documentation updates
🎨 UI/UX improvements
🚧 Work in progress
⚡ Optimizations

Automate Emoji Usage with Git Hooks

You can set up Git hooks to auto-insert emojis based on keywords: 

#!/bin/sh
commit_msg=$(cat "$1")
case "$commit_msg" in
    *fix*) commit_msg="🐛 $commit_msg" ;;
    *docs*) commit_msg="📖 $commit_msg" ;;
    *optimize*) commit_msg="⚡ $commit_msg" ;;
esac
echo "$commit_msg" > "$1"

✅ This automatically adds relevant emojis to commit messages based on keywords like fix, docs, optimize!

Searching for Emojis in Git Logs

You can filter your Git history based on emojis: 

git log --grep="🐛"

This finds only bug fixes, making issue tracking super easy!

Superscript Numbers

Because Vim and Vimwiki handle standard plain text, you can achieve superscript by using Unicode characters.

Unicode has dedicated, built-in characters for the numbers 0–9 in superscript form. To write 10 39 , you just need the standard 1 and 0, followed by the Unicode characters for superscript 3 (³) and superscript 9 (⁹), resulting in: 10³⁹

Here are the three best ways to type these directly into Vim without breaking your note-taking flow.

Method 1: The Quickest Way (Vim Digraphs)

Vim has a built-in feature called Digraphs specifically designed for entering special characters. You enter a shortcut key, type a two-letter code, and Vim transforms it.

  • In Insert Mode, press <Ctrl> + k. (This tells Vim a digraph is coming).
  • Type the two-letter code for the number you want:
    • For ³ type: 3S
    • For ⁹ type: 9S

So, to type 10³⁹, you would type: 1 0 <Ctrl>+k 3S <Ctrl>+k 9S.

Tip: You can see Vim's entire built-in list of superscript and subscript digraphs by running the command :digraphs in normal mode. Almost all superscripts use the number followed by a capital S.

Method 2: Insert by Unicode Hex Code

If you don't want to memorize digraphs, you can insert them directly via their universal Unicode hex points using <Ctrl> + v.

In Insert Mode:

  • Press <Ctrl> + v, then type u
  • Type the 4-digit hex code for the number.

Here are the hex codes for your specific numbers:

  • ³ (Superscript 3) = 00B3
  • ⁹ (Superscript 9) = 2079

So you would type: 1 0 <Ctrl>+v u00B3 <Ctrl>+v u2079.

Method 3: The Vimwiki/Markdown Way (Best for HTML Exports)

If you use Vimwiki with Markdown syntax (or use Vimwiki's default syntax and export your notes to HTML later), you can use standard HTML or Markdown formatting.

Instead of hunt-and-pecking for special characters, you wrap the exponent in <sup> tags: 

10<sup>39</sup>

Pros: It is incredibly easy to type and works perfectly if you ever export your Vimwiki notes to a browser, PDF, or a static website.

  • Cons: In the raw text file inside Vim, it will look like 10<sup>39</sup> rather than the clean 10³⁹ (though some Vim syntax-highlighting plugins will visually conceal the tags to make it look like a true superscript).
