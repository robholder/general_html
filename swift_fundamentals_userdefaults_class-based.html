Swift - Class Based UserDefaults

Blueprint: Class-Based UserDefaults Management with @Observable

This reference guide isolates the architectural pattern for managing lightweight persistent data (such as a Favorites feature) globally outside of your views.

This technique maintains a high-performance Runtime Layer (Set) for instantaneous user interactions, paired with a reliable Storage Layer (UserDefaults) for background persistence.

Complete Blueprint Code (Copy-Paste Ready)

This class completely encapsulates your data logic. Your views simply call add(), remove(), or contains(), unaware of how the data is being saved under the hood. 

import SwiftUI

// 1. Centralized storage key to prevent typo-driven bugs
enum StorageKeys: String {
    case favoritesList
}

// 2. A placeholder model representing your data item
struct CustomItem: Identifiable {
    let id: String
    let name: String
}

@Observable
class StorageManager {
    // High-performance runtime collection (O(1) lookups)
    private var itemIDs: Set<String>
    
    // Resolve the string value from our enum key
    private let storageKey = StorageKeys.favoritesList.rawValue
    
    init() {
        // READ STEP: Attempt to read the array from disk
        if let savedArray = UserDefaults.standard.stringArray(forKey: storageKey) {
            // Convert storage Array back into runtime Set
            self.itemIDs = Set(savedArray)
        } else {
            // Fallback initialization if no history exists
            self.itemIDs = []
        }
    }
    
    /// Checks if an item is stored (Runs in constant time).
    func contains(_ item: CustomItem) -> Bool {
        itemIDs.contains(item.id)
    }
    
    /// Appends an item to the collection and syncs to disk.
    func add(_ item: CustomItem) {
        itemIDs.insert(item.id)
        save()
    }
    
    /// Removes an item from the collection and syncs to disk.
    func remove(_ item: CustomItem) {
        itemIDs.remove(item.id)
        save()
    }
    
    /// Encapsulated persistence engine.
    private func save() {
        // WRITE STEP: Cast Set into an Array for UserDefaults serialization
        UserDefaults.standard.set(Array(itemIDs), forKey: storageKey)
    }
}

View Integration Blueprint

To make this manager accessible across your entire app layout, initialize it once in your main App entry point, inject it into your root view environment, and pull it down wherever needed.

1. Root Level Injection

@main
struct YourApp: App {
    // Instantiate the manager once for the lifetime of the application
    @State private var storageManager = StorageManager()
    
    var body: some Scene {
        WindowGroup {
            MainContentView()
                .environment(storageManager) // Inject into the environment
        }
    }
}

2. Implementation inside a View

struct ItemRowView: View {
    let item: CustomItem
    
    // Read the shared instance from the environment
    @Environment(StorageManager.self) private var storageManager
    
    var body: some View {
        HStack {
            Text(item.name)
            Spacer()
            
            Button {
                if storageManager.contains(item) {
                    storageManager.remove(item)
                } else {
                    storageManager.add(item)
                }
            } label: {
                Image(systemName: storageManager.contains(item) ? "heart.fill" : "heart")
                    .foregroundStyle(.red)
            }
        }
    }
}

3 Core Mechanics to Keep in Mind

  • The Set-to-Array Transformer: UserDefaults cannot natively understand or serialize a Swift Set. Because of this, we utilize a data transformer setup: we run our app logic inside a high-speed Set container, but explicitly cast it down to an Array (Array(itemIDs)) the exact moment it writes to disk.
  • Elimination of View Redraw Overhead: Unlike @AppStorage, which causes a view to invalidate and redraw the moment any key updates, an @Observable class allows SwiftUI to look deep inside your model properties. Your views will now only redraw if they explicitly evaluate properties targeted by a state change.
  • Implicit Main-Thread Sync: Calls to UserDefaults.standard.set() execute immediately on an in-memory dictionary before flushing asynchronously to disk. You do not need to wrap these operations in background tasks or manual threads for small item tracking arrays.
