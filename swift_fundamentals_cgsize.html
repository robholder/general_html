Swift - CGSize

The CGSize Structural Blueprint in Swift

When building user interfaces and handling gestures in SwiftUI, you frequently need to manage dimensions, scaling, and movement vectors. The foundational tool for managing two-dimensional dimensions in Apple's frameworks is CGSize.

This reference guide breaks down how CGSize operates, why its native shortcuts exist, and how to extend it to build a clean, production-grade design system.

1. What is CGSize?

CGSize is a structure defined within the CoreGraphics framework. It represents a two-dimensional size component, holding a primitive width and height. 

public struct CGSize {
    public var width: CGFloat
    public var height: CGFloat
}

The Common Misconception: Scalar vs. Struct

A common point of confusion when starting out is attempting to assign a single number to a size variable: 

// ❌ COMPILER ERROR: Cannot assign value of type 'Int' to type 'CGSize'
offset = 0

Because 0 is a single number (a scalar), Swift cannot infer your intent. A size requires explicit instructions for both the horizontal axis (width) and the vertical axis (height).

2. Native Static Shortcuts: The Power of .zero

To simplify layout math and state resets, Apple provides a built-in static property called CGSize.zero.

Writing .zero is an absolute shortcut for creating an instance where both properties are completely cleared out: 

// These two declarations are identical:
@State private var offset = CGSize(width: 0, height: 0)
@State private var offset = CGSize.zero

Type Inference in Practice

Because SwiftUI relies heavily on type inference, you only need to explicitly write out CGSize.zero during initialization. Once the compiler knows the type of the variable, you can drop the type name entirely and use implicit member lookup (the leading dot syntax): 

.gesture(
    DragGesture()
        .onChanged { gesture in
            offset = gesture.translation // Tracking user movement
        }
        .onEnded { _ in
            // Resetting the card back to the exact screen center
            offset = .zero // Swift knows this means CGSize.zero
        }
)

3. Advanced Built-In Utilities

While CGSize is purposefully kept lightweight, it includes a few specific constructors and serialization tools for data persistence:

Uniform Squares

When drawing layout shapes that require identical width and height proportions, you can take advantage of Swift's uniform initializer behavior in contexts that bridge structural literals, or initialize them directly: 

let squareFrame = CGSize(width: 50, height: 50)

CoreGraphics Serialization

If you need to save a user's interface state or a drag position to local persistence (like UserDefaults), you cannot save a raw struct directly. CGSize provides static capability to transform the data to and from CoreFoundation dictionaries: 

// 1. Exporting a size to a dictionary structure
let dictionaryData = currentOffset.dictionaryRepresentation

// 2. Re-initializing a size from a stored dictionary structure
if let savedDict = UserDefaults.standard.dictionary(forKey: "SavedBounds") {
    let restoredSize = CGSize(dictionaryRepresentation: savedDict as CFDictionary)
}

4. Professional Best Practice: Custom Static Extensions

As an application grows, hardcoding explicit dimension values throughout your view hierarchy introduces structural duplication and layout bugs.

The gold-standard approach in professional iOS development is to write a CGSize extension. This allows you to append your own semantic, project-wide design templates that behave exactly like Apple's native .zero shortcut. 

import CoreGraphics

extension CGSize {
    // Standard Interactive Card Blueprint
    static let businessCard = CGSize(width: 450, height: 250)
    
    // UI Element Templates
    static let profileAvatar = CGSize(width: 60, height: 60)
    static let standardButton = CGSize(width: 200, height: 44)
}

Consuming Custom Extensions in SwiftUI

Once declared, your custom size tokens are globally accessible across your entire development workspace. This keeps your layout declarations incredibly clean and readable: 

struct CardView: View {
    let card: Card
    
    var body: some View {
        VStack {
            Text(card.prompt)
        }
        // Utilizing custom static shortcuts for readable frame layouts
        .frame(width: CGSize.businessCard.width, height: CGSize.businessCard.height)
    }
}

Quick Reference Summary

  • CGSize isolates structural spatial dimensions (width and height) using floating-point coordinates.
  • CGSize.zero acts as a global template representing a size with zero dimensions, used heavily to clear animations or reset gesture modifications.
  • Leading Dot Syntax (.zero) is unlocked via Type Inference whenever the compiler already knows a parameter expects a CGSize.
  • Use extension CGSize to declare custom static properties for your layout system, eliminating magic numbers and unifying your UI architecture.
