Git

Restore

Revert takes a previous commit and re-commits as the latest version.
Reset deletes to a previous commit. (If a filename is given it is unstaged.)
Restore 'checks-out' a file from a previous commit.

git restore --source <commit-hash|HEAD-ref> <file-path>

# or

git restore -s HEAD~<#> <file-path>

Example

D (myfile.txt)
|
C
|
B (myfile.txt)
|
A

Starting status:

working stage/indexed commit
D D D 


git restore -s HEAD~2 myfile.txt

After running restore (two commits back)

working stage/indexed commit
B D D

The myfile.txt file remains the same in commit D (at HEAD) and in the staged area.

git restore myfile.txt - would restore the file to unmodified
