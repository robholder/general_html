Python - Quick Action - Remove Square Brackets

Existing 'bin' Script

#!/usr/bin/env python3
import os
import re

# 1. Set the path to the folder containing your files
folder_path = "."

# 2. Compile the regex pattern to find [anything inside brackets]
bracket_pattern = re.compile(r'\[[^\]]*\]')

# 3. Read the actual files from the disk
files = os.listdir(folder_path)

print("Starting file renaming process...")

for filename in files:
    # Skip folders, only look at files
    full_old_path = os.path.join(folder_path, filename)
    if not os.path.isfile(full_old_path):
        continue

    # Check if the filename contains brackets
    if '[' in filename and ']' in filename:
        # Remove the bracketed portion
        cleaned_name = bracket_pattern.sub('', filename)

        # FIX 1: Convert any multiple spaces into a single space
        cleaned_name = re.sub(r'\s+', ' ', cleaned_name)

        # FIX 2: Look for a space right before a dot and remove it (e.g., " .mp3" -> ".mp3")
        cleaned_name = re.sub(r'\s+\.', '.', cleaned_name)

        # Wipe out any stray leading/trailing spaces at the very ends of the string
        cleaned_name = cleaned_name.strip()

        # Create the full path for the new file name
        full_new_path = os.path.join(folder_path, cleaned_name)

        # Physically rename the file on your hard drive!
        os.rename(full_old_path, full_new_path)
        print(f"Renamed: '{filename}' -> '{cleaned_name}'")

print("Finished!")

Ai Insight

Creating Quick Action Version

Adapting the existing script into a Quick Action is incredibly straightforward. Since you are already passing file selection details dynamically from Finder, we just need to swap out your hardcoded folder_path = "." and os.listdir() steps.

Instead, Python will read the files you have highlighted right from the command line using sys.argv.

Here is how to set it up, along with the updated, bulletproof script.

Step 1: Create the Quick Action in Automator

  • Open Automator on your Mac and select New Document > Quick Action.
  • Configure the top adjustment settings exactly like this so it knows to target files:
    • Workflow receives current: files or folders
    • in: Finder.app
  • In the left-hand actions library, search for "Run Shell Script" and drag it into the main workflow window.
  • Configure the settings inside the Run Shell Script box:
    • Shell: /usr/bin/python3
    • Pass input: as arguments (This is critical; it changes how Python receives the file selections).

Step 2: The Adapted Python Script

Clear out any template code inside that Automator text box and paste this updated version:

Note: The pattern is adjusted to remove any trailing underscores after the closing square bracket.

We can modify it to look for \]_?. The ? symbol in regex means "optional" (0 or 1 matches). This tells the script: "Find the closing bracket, and if there happens to be an underscore immediately following it, grab that too and wipe them both out together." 

import os
import re
import sys

# 1. UPDATED REGEX: Matches [anything inside] followed by an optional underscore (_)
bracket_pattern = re.compile(r'\[[^\]]*\]_?')

# 2. Process each file passed from the Finder selection
for full_old_path in sys.argv[1:]:
    
    # Extract the directory path and the individual filename
    folder_path = os.path.dirname(full_old_path)
    filename = os.path.basename(full_old_path)
    
    # Skip if it's a directory or if it doesn't contain brackets
    if not os.path.isfile(full_old_path):
        continue
        
    if '[' in filename and ']' in filename:
        # Remove the bracketed portion (and trailing underscore if it exists)
        cleaned_name = bracket_pattern.sub('', filename)

        # FIX 1: Convert any multiple spaces into a single space
        cleaned_name = re.sub(r'\s+', ' ', cleaned_name)

        # FIX 2: Look for a space right before a dot and remove it (e.g., " .mp3" -> ".mp3")
        cleaned_name = re.sub(r'\s+\.', '.', cleaned_name)

        # Wipe out any stray leading/trailing spaces at the very ends of the string
        cleaned_name = cleaned_name.strip()

        # 3. Safety Check: Make sure we aren't creating an empty filename 
        name_part, ext_part = os.path.splitext(cleaned_name)
        if not name_part.strip():
            continue 

        # Create the full path for the new file name
        full_new_path = os.path.join(folder_path, cleaned_name)

        # 4. Handle filename collisions (safeguard if file already exists)
        counter = 1
        while os.path.exists(full_new_path):
            new_name = f"{name_part}_{counter}{ext_part}"
            full_new_path = os.path.join(folder_path, new_name)
            counter += 1

        # Physically rename the file on your hard drive!
        os.rename(full_old_path, full_new_path)

Step 3: Save and Run

  • Press Cmd + S to save the Quick Action.
  • Name it something descriptive, complete with your style of header symbols, like: 🏷️Strip Brackets from Name.
  • Open Finder, select one or multiple files that have brackets in their names (e.g., My Video [1080p].mp4), right-click, choose Quick Actions, and trigger your new script.

What Changed in the Adaptation?

  • Dynamic Scoping: Instead of searching an entire static folder using os.listdir('.'), the script handles only the specific item paths you visually select in your Finder window. This means you can run it on a single file or highlight 50 files across different folders simultaneously.
  • Collision Safeguard: I added a safety loop at the bottom. If you have Song [Mix A].mp3 and Song [Mix B].mp3 in the same folder, stripping the brackets would normally cause them both to try to become Song.mp3, which would crash the script or overwrite data. The loop automatically steps in and creates Song_1.mp3 instead!
