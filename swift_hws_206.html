Hacking With Swift - 206 - Challenge 2

Challenge 2

Our app silently fails when errors occur during biometric authentication, so add code to show those errors in an alert.

Ai Insight

QUESTION

I've temporarily disabled the Face ID authentication, as it's very slow to run the app in the simulator. Once enabled, my next challenge in Paul Hudson's course is to do this:

"Our app silently fails when errors occur during biometric authentication, so add code to show those errors in an alert."

What kind of errors am I expecting? And how to implement the solution?

ANSWER

When testing biometric authentication in the iOS Simulator, it can definitely slow down your momentum, so temporarily bypassing it is a smart move!

When you do flip that authentication switch back on, dealing with biometric errors is a crucial step. The LAContext API doesn't just return a simple "yes or no"—it throws highly specific errors when things go sideways.

Here is a breakdown of what errors to expect and how to implement a clean alert system to handle them.

What Kind of Errors to Expect

The evaluatePolicy method returns errors wrapped in a native type called LAError. The most common errors you will encounter fall into two main categories:

1. User-Driven Failures

  • .authenticationFailed: The user scanned their face or finger, but the device didn't recognize it (e.g., a mismatched fingerprint or face look).
  • .userCancel: The user explicitly tapped the "Cancel" button inside the system Face ID/Touch ID popup.
  • .userFallback: The user tapped the "Enter Password" fallback button inside the biometric popup.

2. Device Configuration Failures

  • .biometryNotAvailable: The device doesn't have biometric hardware (like an old iPhone or a simulator that hasn't had Face ID turned on in the top menu).
  • .biometryNotEnrolled: The device supports biometrics, but the user hasn't set up any faces or fingerprints in their iOS Settings.
  • .biometryLockout: The user failed too many biometric attempts in a row, and the system has completely locked them out (requiring the device passcode to re-enable it).

How to Implement the Solution

To cleanly surface these errors to the user via a SwiftUI .alert(), we need to track two new states in our ViewModel: a boolean to trigger the alert, and a string to hold the specific error message.

Step 1: Update the ViewModel

Open your ContentView-ViewModel.swift file. Update your properties and modify the authenticate() function to catch the incoming error, parse its type, and pass a friendly message back to the main thread: 

@Observable
class ViewModel {
    // --- Existing properties ---
    private(set) var locations: [Location]
    var selectedPlace: Location?
    var isUnlocked = false
    
    // --- NEW: Error Alert States ---
    var showAuthAlert = false
    var authAlertMessage = ""
    
    // ... your init and existing functions ...

    func authenticate() {
        let context = LAContext()
        var error: NSError?
        
        if context.canEvaluatePolicy(.deviceOwnerAuthenticationWithBiometrics, error: &error) {
            let reason = "Please authenticate yourself to unlock your places"
            
            context.evaluatePolicy(.deviceOwnerAuthenticationWithBiometrics, localizedReason: reason) { success, authenticationError in
                if success {
                    Task { @MainActor in
                        self.isUnlocked = true
                    }
                } else if let laError = authenticationError as? LAError {
                    // Catch and parse the biometric error
                    Task { @MainActor in
                        self.authAlertMessage = self.errorMessage(for: laError)
                        self.showAuthAlert = true
                    }
                }
            }
        } else if let nsError = error as? LAError {
            // Catch errors if the device can't even evaluate biometrics at all
            self.authAlertMessage = self.errorMessage(for: nsError)
            self.showAuthAlert = true
        } else {
            // General fallback error
            self.authAlertMessage = "Biometric authentication is not supported on this device."
            self.showAuthAlert = true
        }
    }
    
    // HELPER: Convert confusing system codes into plain English rules
    private func errorMessage(for error: LAError) -> String {
        switch error.code {
        case .authenticationFailed:
            return "Identity could not be verified. Please try again."
        case .userCancel:
            return "Authentication was cancelled."
        case .userFallback:
            return "Fallback password option selected."
        case .biometryNotAvailable:
            return "Biometric authentication is not available on this device."
        case .biometryNotEnrolled:
            return "No Face ID or Touch ID profiles are enrolled on this device. Please check your system Settings."
        case .biometryLockout:
            return "Too many failed attempts. Face ID is locked out. Please lock and unlock your device with your passcode to reset."
        default: // <-- Change this to a standard default case to satisfy the compiler
            return "An unknown authentication error occurred: \(error.localizedDescription)"
        }
    }
}

Step 2: Attach the Alert Modifier to ContentView

Now, head back into your ContentView.swift. Go to the very bottom block where your commented-out "Unlock Places" button lives, and attach an .alert modifier bound directly to your new view-model states: 

// ... at the very bottom of your ContentView body block:
} else {
    Button("Unlock Places", action: viewModel.authenticate)
        .padding()
        .background(.blue)
        .foregroundStyle(.white)
        .clipShape(.capsule)
        // MONITOR THE ERROR STATES HERE
        .alert("Authentication Failed", isPresented: $viewModel.showAuthAlert) {
            Button("OK", role: .cancel) { }
        } message: {
            Text(viewModel.authAlertMessage)
        }
}

Pro-Tip for Simulator Testing 💡

When you turn authentication back on, you don't have to suffer through slow manual inputs! You can test all of these error states using the Simulator hardware tools:

  • Launch your app in the Simulator.
  • In the Mac menu bar at the top of the screen, navigate to Features ➔ Face ID (or Touch ID).
  • Check Enrolled to simulate a device that has a setup profile.
  • Hit your "Unlock Places" button. When the Face ID prompt pops up, go back to Features ➔ Face ID and toggle Matching Face to simulate success, or Non-Matching Face to instantly trigger your new .authenticationFailed alert!

import CoreLocation
import Foundation
import LocalAuthentication
import MapKit
import SwiftUI

// Make extension to keep ViewModel easy to call.
// Just call it within the ContentView in this case.
extension ContentView {
    @Observable
    class ViewModel {
        // 1. Define custom map style choices
        enum MapStyleOption: String, CaseIterable, Identifiable {
            case standard = "Standard"
            case imagery = "Satellite"
            case hybrid = "Hybrid"
            
            var id: String { self.rawValue }
            
            var nativeStyle: MapStyle {
                switch self {
                case .standard: return .standard
                case .imagery:  return .imagery
                case .hybrid:   return .hybrid
                }
            }
        }
        
        // 2. Track the active style selection
        var selectedStyle: MapStyleOption = .standard
        
        // 'private(set)' means other parts of the code can read it, but can't set it
        private(set) var locations: [Location]
        var selectedPlace: Location?
        var isUnlocked = false
        
        // --- NEW: Error Alert States ---
        var showAuthAlert = false
        var authAlertMessage = ""
        
        let savePath = URL.documentsDirectory.appending(path: "SavedPlaces")
        
        init() {
            do {
                let data = try Data(contentsOf: savePath)
                locations = try JSONDecoder().decode([Location].self, from: data)
            } catch {
                locations = []
            }
        }
        
        func save() {
            do {
                let data = try JSONEncoder().encode(locations)
                try data.write(to: savePath, options: [.atomic, .completeFileProtection])
            } catch {
                print("Unable to save data.")
            }
        }
        func addLocation(at point: CLLocationCoordinate2D) {
            let newLocation = Location(
                id: UUID(),
                name: "New location",
                description: "",
                latitude: point.latitude,
                longitude: point.longitude
            )
            locations.append(newLocation)
            save()
        }
        
        func update(location: Location) {
            guard let selectedPlace else { return }
            
            if let index = locations.firstIndex(of: selectedPlace) {
                locations[index] = location
                save()
            }
        }
        
        func authenticate() {
            let context = LAContext()
            var error: NSError?
            
            if context.canEvaluatePolicy(.deviceOwnerAuthenticationWithBiometrics, error: &error) {
                let reason = "Please authenticate yourself to unlock your places"
                
                context.evaluatePolicy(.deviceOwnerAuthenticationWithBiometrics, localizedReason: reason) { success, authenticationError in
                    if success {
                        Task { @MainActor in
                            self.isUnlocked = true
                        }

                        // Catch and parse the biometric error
                        Task { @MainActor in
                            self.authAlertMessage = self.errorMessage(for: laError)
                            self.showAuthAlert = true
                        }
                    }
                }
            } else if let nsError = error as? LAError {
                // Catch errors if the device can't even evaluate biometrics at all
                self.authAlertMessage = self.errorMessage(for: nsError)
                self.showAuthAlert = true
            } else {
                // General fallback error
                self.authAlertMessage = "Biometric authentication is not supported on this device."
                self.showAuthAlert = true
            }
        }
        
        // HELPER: Convert confusing system codes into plain English rules
        private func errorMessage(for error: LAError) -> String {
            switch error.code {
            case .authenticationFailed:
                return "Identity could not be verified. Please try again."
            case .userCancel:
                return "Authentication was cancelled."
            case .userFallback:
                return "Fallback password option selected."
            case .biometryNotAvailable:
                return "Biometric authentication is not available on this device."
            case .biometryNotEnrolled:
                return "No Face ID or Touch ID profiles are enrolled on this device. Please check your system Settings."
            case .biometryLockout:
                return "Too many failed attempts. Face ID is locked out. Please lock and unlock your device with your passcode to reset."
            default: // <-- Change this to a standard default case to satisfy the compiler
                return "An unknown authentication error occurred: \(error.localizedDescription)"
            }
        }
        
        // Challenge zero - option dialog
        func delete(location: Location) {
            if let index = locations.firstIndex(of: location) {
                locations.remove(at: index)
                save() // Keep your local storage synchronized!
            }
        }
        
    }

ContentView.swift

import MapKit
import SwiftUI

struct ContentView: View {
    let startPosition = MapCameraPosition.region(
        MKCoordinateRegion(
            center: CLLocationCoordinate2D(
                latitude: 56,
                longitude: -3),
            span: MKCoordinateSpan(
                latitudeDelta: 10,
                longitudeDelta: 10)
        )
    )
    
    @State private var viewModel = ViewModel()
    @State private var showingOptions = false // Controls the dialog menu
    @State private var placeToEdit: Location? // Controls the final edit sheet
    
    var body: some View {
        if viewModel.isUnlocked {
            ZStack { // 1. Changed to a standard neutral ZStack
                MapReader { proxy in
                    Map(initialPosition: startPosition) {
                        ForEach(viewModel.locations) { location in
                            Annotation(
                                location.name,
                                coordinate: location.coordinates) {
                                    Image(systemName: "star.circle")
                                        .resizable()
                                        .foregroundStyle(.red)
                                        .frame(width: 44, height: 44)
                                        .background(.white)
                                        .clipShape(.circle)
                                        .simultaneousGesture(LongPressGesture(minimumDuration: 0.2)
                                            .onEnded { _ in
                                                // 1. Lock in the active location choice
                                                viewModel.selectedPlace = location
                                                // 2. Trigger the options dialog menu
                                                showingOptions.toggle()
                                            }
                                        )
                                }
                        }
                    }
                    // --- FIXED: Only one dynamic map style listener remains ---
                    .mapStyle(viewModel.selectedStyle.nativeStyle)
                    .mapControls {
                        // Leaving this blank completely hides all native controls!
                    }
                    .onTapGesture { position in
                        if let coordinate = proxy.convert(position, from: .local) {
                            // Run the addLocation function in the viewModel
                            viewModel.addLocation(at: coordinate)
                        }
                    }
                    // MARK: - 1. The Options Menu Dialog
                    .confirmationDialog(
                        "Manage Location",
                        isPresented: $showingOptions,
                        titleVisibility: .visible,
                        presenting: viewModel.selectedPlace
                    ) { place in
                        Button("Edit Details") {
                            placeToEdit = place // Hands off to the sheet
                        }
                        
                        Button("Delete Pin", role: .destructive) {
                            viewModel.delete(location: place)
                            viewModel.selectedPlace = nil // Reset state cleanup
                        }
                        
                        Button("Cancel", role: .cancel) {
                            viewModel.selectedPlace = nil // Reset state cleanup
                        }
                    } message: { place in
                        Text("What would you like to do with \(place.name)?")
                    }
                    // MARK: - 2. The Final Edit Sheet
                    .sheet(item: $placeToEdit) { place in
                        EditView(location: place) {
                            viewModel.update(location: $0)
                            viewModel.selectedPlace = nil // Final pipeline cleanup
                        }
                    }
                }
                // 3. NEW FLOATING UTILITY COLUMN
                // We push this container up and out to the top-right using Spacers
                VStack {
                    HStack {
                        Spacer() // Pushes items to the right side
                        
                        VStack(spacing: 12) { // Vertically stacks the controls with a clean gap
                            
                            // A. NATIVE COMPASS CONTROL (Now behaves as an explicit view element!)
                            MapCompass()
                                .mapControlVisibility(.automatic) // Only displays if the map is actually rotated
                            
                            // B. YOUR CUSTOM STYLE MENU BUTTON
                            Menu {
                                Picker("Map Style", selection: $viewModel.selectedStyle) {
                                    ForEach(ViewModel.MapStyleOption.allCases) { option in
                                        Text(option.rawValue).tag(option)
                                    }
                                }
                            } label: {
                                Image(systemName: "map.circle.fill")
                                    .font(.system(size: 44))
                                    .foregroundStyle(.primary)
                                    .background(.white)
                                    .clipShape(.circle)
                                    .shadow(radius: 4)
                            }
                        }
                        .padding(.trailing, 12) // Safe padding distance away from the right screen edge
                        .padding(.top, 60)      // Safe padding distance down to avoid hitting the iPhone notch / status bar
                    }
                    Spacer() // Pushes the entire row assembly to the top of the screen
                }
            }
        } else {
            Button("Unlock Places", action: viewModel.authenticate)
                .padding()
                .background(.blue)
                .foregroundStyle(.white)
                .clipShape(.capsule)
                // MONITOR THE ERROR STATES HERE
                .alert("Authentication Failed", isPresented: $viewModel.showAuthAlert) {
                    Button("OK", role: .cancel) { }
                } message: {
                    Text(viewModel.authAlertMessage)
                }
        }
    }
}
