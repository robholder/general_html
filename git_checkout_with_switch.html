Git - Checkout Versions Using 'Switch'

The modern way to navigate your history is by using the git switch command.

In 2019, the Git team realized that git checkout was doing too many unrelated jobs (switching branches, restoring files, undoing changes). To make things clearer, they split those responsibilities into two new commands:

  • git restore and git switch.

How To Use Git Switch Versions

When you want to jump to a specific commit or tag to see your code at that stage, you use the --detach flag. This is required because, unlike a branch, a commit is a fixed point in time.

1. Find your target

First, get your list of commits:

  • git log --oneline -n 10

2. Make the jump

git switch --detach <hash-or-tag>

3. Return to the present

When you're done looking at the old version and want to go back to your latest work:

  • git switch main (or master)

If you have changes you want to ignore

1. The "Reset and Switch" (The Nuclear Option)

If you want to erase all tracked changes and jump to main, run these two commands: 

git reset --hard HEAD
git switch main
  • git reset --hard HEAD: This wipes out all changes to tracked files. They are gone forever, so make sure you truly don't want them!
  • git switch main: Now that your working directory is "clean," Git will let you move freely.

2. The "Force Switch" (The Faster Way)

In newer versions of Git, you can combine these steps into one command: 

git switch -f main

The -f (or --force) flag tells Git: "I don't care if I have uncommitted changes, throw them away and switch branches anyway."

3. Cleaning up "Untracked" Files

The commands above only affect files Git already knows about. If you created new files that were never staged or committed, they will still be sitting there. To delete those too, use: 

git clean -fd
  • -f: Force delete.
  • -d: Remove directories as well.

Summary Table: Which command do I need?

If you want to... Use this command
Switch and discard tracked changes git switch -f main
Discard changes but stay on current branch git reset --hard HEAD
Delete new/untracked files and folders git clean -fd
"Nuke" everything and start over on main git reset --hard && git clean -fd && git switch main

Warning: These commands are destructive. Unlike a text editor, there is no "Undo" for a git reset --hard or a git clean. If there is even a 1% chance you might want those changes later, use git stash instead of deleting them!

Why is this better than git checkout?

  • Safety: git switch was designed specifically for moving between points in time. It has better safety checks to ensure you don't accidentally lose local changes when moving around.
  • Clarity: When you use checkout, Git has to guess if you mean "I want to switch to a branch" or "I want to undo changes to a file." switch only does one thing: it moves your HEAD.
  • The "Detached" Message: Just like checkout, switch will tell you that you've entered a "detached HEAD" state. Since you're just examining old tutorial stages, this is exactly what you want.

Pro-Tip: The "Previous" Shortcut

If you are jumping back and forth between your current work and an old stage, you don't need to keep typing the branch name. Use the dash shortcut:

  • git switch -

This works just like the "Last Channel" button on a TV remote—it will toggle you back to wherever you were just standing.

Comparison Table

Goal Old Command Modern Command
Switch to a branch git checkout main git switch main
Look at a specific commit git checkout <hash> git switch --detach <hash>
Create and switch to a branch git checkout -b new-idea git switch -c new-idea
Undo changes to a file git checkout -- file.txt git restore file.txt
