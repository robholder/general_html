Navigation & Modes

Motion command reference

Link: (Mostly) Taken from Daniel Miessler's Blog

  • j --> move down one line
  • k --> move up one line
  • h --> move left one character
  • l --> move right one character
  • 0 --> move to the beginning of the line
  • g0 --> move to the beginning of the (wrapped) line
  • $ --> move to the end of the line
  • g$ --> move to the end of the (wrapped) line
  • ^ --> move to the first non-blank of the line
  • g_ --> move to the last non-blank or the line
  • w --> move forward one word
  • W --> move forward one word (by whitespace boundaries)
  • b --> move back one word
  • B --> move back one word (by whitespace boundaries)
  • e --> move to the end of your word
  • E --> move to the end of your word (by whitespace boundaries)
  • ) --> move forward one sentence
  • } --> move forward one paragraph
  • :<line_number> --> move to a given line number (e.g. :10 --> line 10)
  • <line_number> G --> move to a given line number (e.g. 10G --> line 10) Notice there's no :
  • <number>% --> This will jump to the percentage of the buffer (e.g. 66% --> 2/3 of the buffer) Notice there's no :
  • #| --> move to the # position on the line (e.g. 10| takes the cursor to the 10th character position on the current line)
  • H --> move to the top of the screen
  • M --> move to the middle of the screen
  • L --> move to the bottom of the screen
  • ^E --> scroll up one line (hold for scrolling up)
  • ^Y --> scroll down one line (hold for scrolling up)
  • gg --> go to the top of the file
  • G --> go to the bottom of the file
  • ^U --> move up half a page
  • ^D --> move down half a page
  • ^F --> move down a page
  • ^B --> move up a page
  • Ctrl-i --> jump to your previous navigation location
  • Ctrl-o --> jump back to where you were
  • gf --> open a file if the cursor sits on a file path in the text
  • gi --> enter 'insert' mode at the last place text was inserted. (To continue or add to the last edit)

Modes

  • Normal Mode or Command Mode
  • Insert Mode
  • Visual Mode
  • Command-line mode
  • Replace Mode
  • Select Mode
  • Operator Pending Mode
  • Ex Mode

Mode Breakdown

Hereâ€™s a breakdown of the primary modes in Vim and what each is used for:

Normal Mode

  • Purpose: This is the default mode when you open Vim. It's used for navigating and manipulating text.
  • Key Functions:
    • Navigation: Move the cursor around (h, j, k, l).
    • Editing: Delete, yank (copy), paste, change text (d, y, p, c).
    • Search: Search for text within the file (/, ?).

Insert Mode

  • Purpose: Used for inserting and editing text.
  • Key Functions:
    • Entered by pressing
      • i (insert before cursor)
      • a (append after cursor)
      • I (insert at beginning of line)
      • A (append at end of line)
      • o (open new line below) or O (open new line above).
    • Allows you to type and edit text as you would in a regular text editor.

Visual Mode

  • Purpose: Used for selecting and highlighting text.
  • Key Functions:
    • Entered by pressing v (visual character mode)
    • V (visual line mode), or <ctrl> v (visual block mode).
    • Once text is selected, you can perform operations on the selection, such as deleting, yanking, changing, or formatting.

Select Mode

  • RARELY USED
  • Purpose: Similar to Visual mode, but behaves like selection in other text editors.
  • Key Functions:
    • Entered by the same mappings as Visual mode
    • Then press <ctrl> g
    • Selected text will now be replaced by typing new text.
    • The selection behaves similarly to typical text editors, where typing replaces the selected text, and actions like Backspace will modify the selection.

Command-Line Mode

  • Purpose: Used for executing Ex commands, such as saving, quitting, searching, and running external commands.
  • Key Functions:
    • Entered by pressing : (for most commands)
    • / (forward search) or ? (backward search).
    • Common commands include :w (write/save), :q (quit), :wq (write and quit), and :! (run external commands).

Replace Mode

  • Purpose: Used for replacing existing text with new text.
  • Key Functions:
    • Entered by pressing R.
    • Every character you type replaces the existing character under the cursor.

Operator-Pending Mode

  • Purpose: Temporarily active while awaiting further input for an operator command.
  • Key Functions:
    • Entered automatically when you start an operator command (e.g., d, y, c).
    • Awaits movement command to complete the operation (e.g., dw to delete a word).
    • Each mode has its specific purpose and set of commands, making Vim a powerful and efficient text editor once you get the hang of it.

Ex Mode

  • Purpose: Ex mode is used for executing Ex commands without returning to normal mode after each command. It's useful for performing a series of commands or scripting within Vim.
  • Key Functions:
    • Entering Ex Mode: You can enter Ex mode by pressing Q in normal mode.
    • Exiting Ex Mode: You can exit Ex mode by typing visual or vi and pressing Enter, or by pressing Ctrl-C.

Common Ex Commands

While in Ex mode, you can use any Ex command, just like you would after typing : in normal mode. Here are a few examples:

  • :w --> Write/save the file.
  • :q --> Quit Vim.
  • :wq --> Write and quit.
  • :s/foo/bar/g --> Substitute foo with bar globally.
  • :e filename --> Open a new file.

Example Workflow in Ex Mode

  • Enter Ex Mode:
    • Press Q in normal mode.
  • Execute Commands:
    • Type commands like :w, :q, :s/foo/bar/g etc.
  • Exit Ex Mode:
    • Type visual or vi and press <enter>, or press <ctrl> c.

Ex mode is especially useful for running a sequence of commands without repeatedly entering the command-line mode.

Ex commands can be stored in a file and referenced on launch of a file in Vim, or 'sourced'. The extension should be '.ex'. It's best practice to omit the : at the start of each command if storing in a file.
