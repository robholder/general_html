rsync - Two Macs

Ethernet

Connecting two Macs directly via Ethernet is one of the fastest, most reliable ways to transfer massive amounts of data or run automated backups.

Here is a complete, step-by-step guide to setting up your connection, mounting the volumes, navigating them in the Terminal, automating backups with rsync, and optimizing performance.

Step 1: Physical Setup & Network Configuration

  • Connect both Macs using an Ethernet cable.
    • (Modern Macs automatically detect direct connections, so a standard Cat5e or Cat6 cable works perfectly - no crossover cable needed).
  • On both Macs, go to System Settings > Network.
  • Look at the Ethernet status.
    • They should automatically assign themselves local IP addresses (Self-Assigned IPs) and show a green dot or a "Connected" status.
💡 Tip: To ensure the Macs talk directly over Ethernet rather than your slower Wi-Fi network, make sure Ethernet is at the top of your network service order, or temporarily turn off Wi-Fi on both machines while setting this up.

Step 2: Enable File Sharing on the Target Mac (Mac Mini)

Before you can connect, the "host" Mac (your Mac Mini) needs to allow incoming connections:

  • On the Mac Mini, go to System Settings > General > Sharing.
  • Turn on File Sharing and click the small "i" (info) icon next to it.
  • Note the exact local hostname listed at the bottom (e.g., macmini.local).
  • Ensure the folders you want to share are listed under Shared Folders, and your user account has Read & Write permissions.

Step 3: Mount the Volume via SMB (Command+K)

Now, move to your main MacBook:

  • Click on your desktop to focus on Finder, then press <Cmd> + k (or go to Go > Connect to Server in the menu bar).
  • In the Server Address field, type: smb://macmini.local (adjust for the name of the mac!) and click Connect.
  • A prompt might ask for a username and password. Enter the login credentials for the Mac Mini.
  • A pop-up will appear asking you to choose the volume/mount folder you want to mount (e.g., Macintosh HD, User, or a specific shared folder). Select it and click OK.

The volume will now appear in your Finder sidebar under Locations, and on your Desktop (if you have "Connected Servers" visible in Finder settings).

Step 4: Accessing the Connected Volume in Terminal

In macOS, all external drives, network shares, and mounted volumes are located in a hidden root directory called /Volumes.

To find and navigate your mounted Mac Mini in the Terminal:

  • Open Terminal.
  • Type the following command to see all mounted drives: 
ls -la /Volumes
  • You will see your Mac Mini's shared folder listed there. To enter it, use the cd command (wrap the path in quotes if the folder name has spaces): 
cd "/Volumes/Your Mounted Folder Name"

Step 5: Setting Up the rsync Automation & Alias

Since you are running rsync over a mounted network volume, you don't actually need to pass the network username and SSH address (<username>@<MACHINE-1-NAME>.local) inside the rsync command itself. Because the drive is already mounted via SMB, the Mac treats it like a local hard drive located in /Volumes.

Here is how to structure your rsync command, along with a breakdown of your requested syntax:

The rsync Command Explained

# Dry run:
rsync −avzn −−delete /Source/Directory/ /Destination/Directory/

rsync −avz −−delete /Source/Directory/ /Destination/Directory/
  • -a (archive mode): Preserves permissions, modification times, and symlinks.
  • -v (verbose): Shows the files being transferred in real-time.
  • -z (compress): Compresses data during transfer (great for networks).
  • --delete: Caution! This deletes files in the destination directory if they no longer exist in the source directory. It keeps both folders perfectly mirrored.

Dry Run

The -n flag (also written as --dry-run) is one of the most important safety features in rsync.

When you include -n in your command, rsync performs a trial run that simulates exactly what would happen if you ran the command for real, without making any actual changes to your files. It won't copy, move, or delete a single byte.

Why You Should Use It

It is highly recommended to use -n whenever you are testing a new rsync command, especially when using the dangerous --delete flag. It lets you catch mistakes before you accidentally wipe out or overwrite data on your destination drive.

What to Look For in the Output

When you run a dry run, rsync will output a list of files.

  • If you see deleting some_file.mp4, it means that file would be deleted if you ran it for real.
  • At the very bottom of the output, you will see a summary like:
    • DRY RUN (a clean run would have modified X files)

Once you review the list and are 100% confident that rsync is targeting the correct files, simply remove the n from your flags to execute the actual transfer.

Creating the Permanent Alias

  • Open your shell configuration file (modern macOS uses Zsh by default) by running: 
alias sync-to-m1="rsync -avz --delete /path/to/local/tutorials/ \"/Volumes/Your Shared Folder Name/path/to/remote/tutorials/\""
  • Reload your Terminal profiles to activate the alias. 
source ~/.zshrc

Now, whenever you type sync-to-m1 in Terminal, your sync will run instantly.

Step 6: Pro-Tips for Power Users

How to Auto-Connect at Login

If you don't want to press <Cmd> + k every time you boot your Mac, you can force macOS to mount the Mac Mini automatically when you log in:

  • Go to System Settings > General > Login Items.
  • Under the Open at Login section, click the + (plus) icon.
  • In the Finder sidebar, look under Locations, select your mounted Mac Mini folder, and click Open.
  • Now, macOS will automatically attempt to mount this share every time you log in (as long as the Ethernet cable is plugged in).

Turn Off Spotlight on the Mounted Drive (Highly Recommended)

When you connect a network share, macOS will often try to index the entire drive using Spotlight. This can cause massive lag, high CPU usage, and slow down your rsync file transfers.

  • Go to System Settings > Siri & Spotlight.
  • Scroll all the way to the bottom and click Spotlight Privacy.
  • Drag and drop your mounted Mac Mini volume from your desktop (or Finder) directly into this list, or click the + icon to browse and add it.
  • Click Done. Spotlight will now ignore that directory completely.

Alias To Mount, Sync, And Unmount

A single, reliable Terminal command that:

  • mounts your Mac mini
  • runs your rsync sync
  • unmounts cleanly

...and does it without requiring Finder to “wake up” the network share first.

Below is a polished, ready‑to‑use script.

Your Mount → Sync → Unmount Script

Replace my_user with the actual username on the Mac mini. 

#!/bin/zsh

# --- CONFIG ---
REMOTE_HOST="macmini.local"
REMOTE_USER="my_user_name"
REMOTE_SHARE="my_user_name"
MOUNT_POINT="$HOME/mnt/macmini"

LOCAL_SYNC_DIR="$HOME/Desktop/xcode_projects"
REMOTE_SYNC_DIR="$MOUNT_POINT/Desktop/xcode_projects"

# --- ENSURE MOUNT POINT EXISTS ---
mkdir -p "$MOUNT_POINT"

echo "Checking for Finder auto-mounts..."

# --- FIND AND UNMOUNT ANY EXISTING /Volumes AUTO-MOUNTS ---
AUTO_MOUNT=$(mount | grep "smbfs" | grep "/Volumes/$REMOTE_USER" | awk '{print $3}')

if [[ -n "$AUTO_MOUNT" ]]; then
    echo "Auto-mounted share found at $AUTO_MOUNT — unmounting..."
    umount "$AUTO_MOUNT" 2>/dev/null || umount -f "$AUTO_MOUNT"
fi

# --- ENSURE OUR MOUNT POINT IS NOT ALREADY MOUNTED ---
if mount | grep -q "$MOUNT_POINT"; then
    echo "Unmounting stale mount at $MOUNT_POINT..."
    umount "$MOUNT_POINT" 2>/dev/null || umount -f "$MOUNT_POINT"
fi

# --- MOUNT REMOTE SHARE ---
echo "Mounting remote share..."

# Use AppleScript to mount via Finder (utilizes your saved Keychain password)
osascript -e "mount volume \"smb://$REMOTE_HOST/$REMOTE_SHARE\""

# Give macOS a couple of seconds to finalize the network mount
sleep 2

# Verify mount succeeded (Finder mounts inside /Volumes by default)
# We update the MOUNT_POINT dynamically to match where Finder put it
MOUNT_POINT="/Volumes/$REMOTE_SHARE"

if [ ! -d "$MOUNT_POINT" ]; then
    echo "ERROR: Mount failed."
    exit 1
fi

# Update your sync dir path to match the new Finder mount point
REMOTE_SYNC_DIR="$MOUNT_POINT/Desktop/xcode_projects"

# --- RUN SYNC ---
echo "Running rsync..."
## Dummy run (-n)
#rsync -avzn --delete "$LOCAL_SYNC_DIR/" "$REMOTE_SYNC_DIR/"
## Full run
rsync -avz --delete "$LOCAL_SYNC_DIR/" "$REMOTE_SYNC_DIR/"

# --- UNMOUNT CLEANLY ---
echo "Unmounting..."
umount "$MOUNT_POINT" 2>/dev/null || umount -f "$MOUNT_POINT"

if mount | grep -q "$MOUNT_POINT"; then
    echo "WARNING: Unmount failed."
else
    echo "Unmount successful."
fi

echo "Done."

How This Maps to Your Existing Alias

Your alias: 

rsync -avz --delete ~/Desktop/xcode_projects /Volumes/my_user/Desktop/xcode_projects

The script above does the same thing, but:

  • mounts the Mac mini at a predictable path (~/mnt/macmini)
  • syncs to that path
  • unmounts afterward

No Finder navigation required.

If You Want a One‑Word Command

Save the script as: 

~/bin/xcsync

Make it executable: 

chmod +x ~/bin/xcsync

Then add this alias to your shell config: 

alias xcsync="~/bin/xcsync"

Now you can run: 

xcsync

...and it will mount → sync → unmount automatically.

Finder Desktop Volume Icons

QUESTION

I have many icons showing another mac on the same network mounted on my macbook's desktop. Even if I eject them, the icons remain. How do I clear them?

ANSWER

That is a classic Finder ghosting glitch. It’s incredibly annoying to see your desktop cluttered with "phantom" network drives that refuse to leave even after you've politely asked them to eject.

Don't worry, those icons aren't permanent. Here are the quickest ways to clear them out and get your desktop back.

1. The Quickest Fix: Relaunch Finder

Most of the time, the network shares are actually disconnected, but Finder’s visual interface hasn't refreshed to reflect that. Forcing Finder to restart usually sweeps away the ghosts.

  • Press <Option> + <Command> + <Esc> on your keyboard (this opens the Force Quit menu).
  • Scroll down to the bottom of the list and select Finder.
  • Click the Relaunch button.

Your desktop icons will vanish for a second and reappear - minus the ghost network icons.

2. The Terminal Nuclear Option (If they still won't leave)

If a simple relaunch didn't work, macOS might still have a background process clinging to those network mounts. You can force them to unmount using a quick Terminal command.

  • Open the Terminal app (Press <Command> + <Space>, type "Terminal", and hit Enter).
  • Copy and paste the following command, then press Enter: 
killall Finder && diskutil unmount ~/Desktop/*

Note: If it says "forced unmount failed," you can try the ultimate clearing command below, which safely forces all stuck network drives to disconnect: 

sudo umount -a -t afp,smb

(If you use sudo, it will ask for your Mac's login password. You won't see characters moving as you type; just type it blindly and hit Enter.)

3. Preventative Measure: Hide Network Servers from the Desktop

If you frequently connect to this other Mac, those icons are going to keep popping up. If you'd rather just access them via the Finder sidebar instead of having them clutter your desktop, you can turn them off permanently:

  • Click on your desktop so Finder is the active app in the top-left corner of your screen.
  • Click Finder in the menu bar and select Settings (or Preferences on older macOS versions).
  • Under the General tab, look for the section: "Show these items on the desktop:"
  • Uncheck the box next to Connected servers.
    💡 Tip: You will still be able to access the other Mac anytime by opening a Finder window and looking under the Locations section in the left sidebar.
