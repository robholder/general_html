WezTerm - Finding Fonts

It is a classic "Developer's Dilemma"—you’ve installed the font, but now you have to guess if it's "Fira Code," "FiraCode," or "Fira Code Nerd Font."

Luckily, WezTerm has a built-in "tell-all" command that reveals exactly what it sees on your system.

The "Source of Truth" Command

Open your terminal and run the following command. It lists every font family name that WezTerm recognizes: 

wezterm ls-fonts --list-system

If you have a lot of fonts, you can filter the list to find your new Fira Code installation specifically: 

wezterm ls-fonts --list-system | grep -i "Fira"

What to look for

In the output, look for the line that says family: "Font Name". That exact string inside the quotes is what you need to paste into your wezterm.lua.

  • Standard Install: It is usually just "Fira Code".
  • Nerd Font Version: If you installed the patched version via Homebrew, it is likely "FiraCode Nerd Font".

Pro-Tip: Testing via CLI

If you want to be 100% sure the font works before messing with your config file, you can tell WezTerm to start a temporary instance using that font: 

wezterm start --font-config 'font={family="Fira Code", weight="Bold"}'

If the window opens and looks right, you've found your string.

Updated Config Example

Once you've confirmed the name, your table would look like this: 

local my_fonts = {
  { family = "Fira Code", weight = 700, size = 12 }, -- Adjust based on your 'ls-fonts' result
  { family = "Agave Nerd Font", weight = 700, size = 14},
  -- ... the rest of your list
}

Fallbacks

In WezTerm, fallbacks work like a "search list." If your primary font doesn't have a specific character (like a GitHub icon or a specific emoji), WezTerm will look at the next font in your list.

You use the wezterm.font_with_fallback function to achieve this.

The Basic Implementation

Instead of just giving WezTerm a single font name, you provide a list. Here is how you would modify your config to use Fira Code as your main font, but keep a Nerd Font as a backup for icons: 

local wezterm = require 'wezterm'
local config = wezterm.config_builder()

config.font = wezterm.font_with_fallback({
  -- 1. Your preferred main text font
  { family = "Fira Code", weight = "Bold" },

  -- 2. Your "Icon" fallback (use the name you found via ls-fonts)
  { family = "JetBrainsMono Nerd Font", weight = "Bold" },

  -- 3. Emoji fallback (optional, WezTerm has built-ins, but you can be explicit)
  { family = "Apple Color Emoji" }, 
})

return config

Advanced: Handling Different Sizes

Sometimes a fallback font looks tiny or huge compared to your main font. You can use the scale property to balance them out: 

config.font = wezterm.font_with_fallback({
  { family = "Fira Code", size = 12 },
  -- If the icons in Agave look too small next to Fira, boost them by 20%
  { family = "Agave Nerd Font", scale = 1.2 },
})

Why this is better than "Nerd Fonts" alone

When you install a "Nerd Font" (like font-fira-code-nerd-font), the icons are "patched" into the font. This can sometimes lead to:

  • Blurry icons if the patching wasn't perfect.
  • Old versions of icons if the patch is out of date.

By using fallback, you can use the official, clean Fira Code for your text and a dedicated symbols font (like Symbols Nerd Font) for your icons. WezTerm will stitch them together perfectly.

Cursive Fonts

This is a popular aesthetic—often called the "Operator Mono" look. It keeps your main code block clean and readable in a sturdy font like Fira Code, but renders keywords (like function, return, or import) in a stylish cursive or distinct font.

In WezTerm, you achieve this using font_rules.

The "Cursive Italics" Setup

You’ll need a font that actually has a "cursive" style (like Agave, Victor Mono, or Cartograph CF). Here is how you configure your wezterm.lua to swap fonts based on the text style: 

local wezterm = require 'wezterm'
local config = wezterm.config_builder()

-- 1. Define your standard font (Regular/Bold)
config.font = wezterm.font_with_fallback({
  { family = "Fira Code", weight = 700 }, -- Your main workhorse
  { family = "Symbols Nerd Font" },       -- Icons fallback
})

-- 2. Define the "Rule" for Italics
config.font_rules = {
  {
    -- This rule matches whenever the terminal asks for Italic text
    italic = true,
    font = wezterm.font_with_fallback({
      -- Replace "Agave Nerd Font" with your favorite cursive font
      { family = "Agave Nerd Font", italic = true, weight = 700 },
    }),
  },
}

return config

Pro-Tip: Visual Balance

Cursive fonts are often naturally smaller or thinner than blocky fonts like Fira Code. If your italics look "shrunken," you can use the scale property we discussed earlier inside the rule: 

font = wezterm.font_with_fallback({
  { family = "Agave Nerd Font", italic = true, scale = 1.1 },
}),

How to verify it's working

Most modern color schemes (like Catppuccin or Tokyo Night) automatically italicize keywords in Neovim or VS Code. If you want to test it manually in your terminal right now, run this command: 

echo -e "\e[3mThis should be in your Cursive Font\e[0m"

Common Cursive Fonts to Try

If you haven't picked a cursive font yet, these are the community favorites for this specific setup:

  • Victor Mono: Famous for its very distinct, "handwritten" italics.
  • JetBrains Mono: Has a very clean, professional italic style.
  • Recursive: A variable font that can go from "Linear" to "Casual" (very curvy).

Setup

This setup uses Fira Code for your everyday typing, Victor Mono for that stylish cursive touch on italics, and ensures your Nerd Font icons are always there as a backup.

I’ve included the scale and weight tweaks to make sure the transition between the two fonts feels smooth and balanced. 

local wezterm = require 'wezterm'
local config = wezterm.config_builder()

-- 1. Main Font Configuration (Regular and Bold text)
config.font = wezterm.font_with_fallback({
  -- Your main workhorse
  { family = "Fira Code", weight = 700 }, 
  
  -- The "Safety Net" for icons (adjust if your ls-fonts name differs)
  { family = "Symbols Nerd Font Mono" },
  { family = "JetBrainsMono Nerd Font", scale = 1.0 },
})

config.font_size = 13.0

-- 2. Custom Rules for Italics (The "Operator Mono" Look)
config.font_rules = {
  {
    -- When the terminal asks for Italic (usually keywords in code)
    italic = true,
    font = wezterm.font_with_fallback({
      -- Victor Mono is famous for its cursive italics
      -- We add 'italic = true' here to ensure we get the script version
      { 
        family = "Victor Mono", 
        italic = true, 
        weight = "Medium",
        scale = 1.05 -- Victor Mono can feel slightly slim, this buffs it up
      },
      -- Fallback to your main font if Victor Mono is missing a character
      { family = "Fira Code", italic = true },
    }),
  },
  {
    -- This handles Bold + Italic specifically
    italic = true,
    bold = true,
    font = wezterm.font_with_fallback({
      { family = "Victor Mono", italic = true, weight = "Bold", scale = 1.05 },
    }),
  },
}

-- 3. Optional: Ligatures
-- Fira Code is famous for these (turning -> into an actual arrow)
config.harfbuzz_features = { 'calt', 'liga', 'ss01' }

return config

Color

To really make those Victor Mono italics pop, you need a color scheme that actually uses italics for keywords (like local, function, if, etc.).

WezTerm comes with hundreds of built-in schemes. You can browse them all by running: 

wezterm show-keys --all | grep -i "color_scheme"

1. Adding the Color Scheme to your wezterm.lua

The most popular schemes for this "Cursive + Dark Mode" aesthetic are Catppuccin, Tokyo Night, and Kanagawa.

Here is how to add them to your existing config: 

local wezterm = require 'wezterm'
local config = wezterm.config_builder()

-- [Previous Font Config goes here...]

-- 4. Color Scheme Selection
-- Options: "Catppuccin Macchiato", "Tokyo Night", "Kanagawa (Glow)", "Night Owl"
config.color_scheme = 'Catppuccin Macchiato'

-- Optional: Make the window slightly transparent for a modern look
config.window_background_opacity = 0.95
config.macos_window_background_blur = 20 -- macOS only

return config

2. How to "Hot-Swap" and Preview Schemes

You don't have to keep editing your file to see what looks good. You can open the WezTerm Command Palette while the terminal is open:

  • Press CMD + SHIFT + L (or CTRL + SHIFT + L on Linux/Windows).
  • Type "color scheme".
  • Use the arrow keys to scroll through the names.
  • The terminal will live-preview the colors as you highlight them!

3. A Note on Neovim / CLI Tools

If you set a color scheme in WezTerm but your code doesn't show italics, it’s usually because your code editor (like Neovim) needs to be told to allow them.

For Neovim, you usually just need to ensure your theme's configuration has italics enabled: 

-- Example for Catppuccin in Neovim
require("catppuccin").setup({
    integrations = {
        native_lsp = {
            enabled = true,
            underlines = {
                errors = { "undercurl" },
                hints = { "undercurl" },
            },
        },
    },
    styles = {
        comments = { "italic" },
        conditionals = { "italic" },
        keywords = { "italic" },
        functions = { "italic" },
    },
})

  • The Global Override: config.font sets your "standard" typing font.

Gemini said

Your current toggle script works by using window:set_config_overrides, which effectively "papers over" your main config. To keep your cursive italics working while you cycle through fonts, you need to make sure the overrides also include the font_rules logic.

If you don't update this block, pressing <Ctrl><Shift>F will switch your main font but might "kill" the cursive rules because the override takes total priority.

The Updated Toggle Block

Replace your existing action_callback with this version. It ensures that every time you switch fonts, it reapplies the Victor Mono rule for italics: 

  {
    key = 'f', mods = 'CTRL|SHIFT',
    action = wezterm.action_callback(function(window, pane)
      if wezterm.GLOBAL.font_indices == nil then
        wezterm.GLOBAL.font_indices = {}
      end
      
      local id = tostring(window:window_id())
      local current_index = wezterm.GLOBAL.font_indices[id] or 1
      
      local next_index = current_index + 1
      if next_index > #my_fonts then
        next_index = 1
      end
      
      local next_font = my_fonts[next_index]
      
      -- Apply the overrides
      window:set_config_overrides({
        -- 1. Set the new main font from your list
        font = wezterm.font(next_font.family, { weight = next_font.weight }),
        font_size = next_font.size,
        
        -- 2. RE-APPLY the cursive rules here so they persist after the toggle
        font_rules = {
          {
            italic = true,
            font = wezterm.font_with_fallback({
              { family = "Victor Mono", italic = true, weight = "Medium", scale = 1.05 },
              { family = next_font.family, italic = true }, -- Fallback to the current toggled font
            }),
          },
          {
            italic = true,
            bold = true,
            font = wezterm.font_with_fallback({
              { family = "Victor Mono", italic = true, weight = "Bold", scale = 1.05 },
            }),
          },
        },
      })
      
      wezterm.GLOBAL.font_indices[id] = next_index
    end),
  },

What changed?

  • The Persistence: By adding font_rules inside the set_config_overrides, WezTerm knows that even though you changed the base font to "Hack" or "Agave," the rule for italics remains "Use Victor Mono."
  • Dynamic Fallback: In the italic rule, I set the second font in the fallback list to next_font.family. This means if Victor Mono is missing a specific italic character, it will fall back to the italic version of whichever font you just toggled to.

Toast Notification

This is a great quality-of-life upgrade. We’ll use window:toast_notification to display the font name. It’s subtle, disappears on its own, and saves you from having to guess which "Mono" you're currently looking at.

Here is the complete, polished block for your config.keys: 

  {
    key = 'f', mods = 'CTRL|SHIFT',
    action = wezterm.action_callback(function(window, pane)
      -- Ensure our global font tracker exists
      if wezterm.GLOBAL.font_indices == nil then
        wezterm.GLOBAL.font_indices = {}
      end
      
      local id = tostring(window:window_id())
      local current_index = wezterm.GLOBAL.font_indices[id] or 1
      
      -- Increment and wrap around
      local next_index = current_index + 1
      if next_index > #my_fonts then
        next_index = 1
      end
      
      local next_font = my_fonts[next_index]
      
      -- 1. Show a notification in the top right
      window:toast_notification("WezTerm", "Font: " .. next_font.family, nil, 2000)
      
      -- 2. Apply the overrides (Font + Cursive Rules)
      window:set_config_overrides({
        font = wezterm.font(next_font.family, { weight = next_font.weight }),
        font_size = next_font.size,
        
        -- Re-apply cursive rules so they stay active during the toggle
        font_rules = {
          {
            italic = true,
            font = wezterm.font_with_fallback({
              { family = "Victor Mono", italic = true, weight = "Medium", scale = 1.05 },
              { family = next_font.family, italic = true }, 
            }),
          },
          {
            italic = true,
            bold = true,
            font = wezterm.font_with_fallback({
              { family = "Victor Mono", italic = true, weight = "Bold", scale = 1.05 },
            }),
          },
        },
      })
      
      -- Store the index
      wezterm.GLOBAL.font_indices[id] = next_index
    end),
  },

What this adds:

  • The Toast: window:toast_notification pops up a small box. I've set the timeout to 2000ms (2 seconds), which is just enough time to read it without it becoming annoying.
  • The Logic: It uses the next_font.family string from your my_fonts table so the name is always 100% accurate to what you're seeing.
