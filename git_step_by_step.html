Initialize a Git repository

# Navigate to enclosing directory:
git init

# By default, the branch is named 'main' (used to be 'master' in the olden days), but to change:
git init --initial-branch=mybranch

# To get the current branch name:
git branch --show
# or
git branch --show-current

# To change to a new name ('main' in this example):
git checkout -b main
# or
git branch -M main

# To initialize a git repository with a name of your choice ('main' in this example):
git init --initial-branch=main

Git Configuration

Configure Git on the local machine, ready for git use: 

git config --global user.name "robholder"
git config --global user.email "robholder@gmail.com"
git config --global init.default branch main

For help with config: 

git config -h
#or
git help config

Stages of Git Workflow

  • Working files:
    • A live status; files in progress
  • Staging:
    • Awaiting commit
  • Commit:
    • Logging to the git repository as a change

Add Files

Add files to the git repository stage: 

# adds specific files:
git add index.htm

# adds all but 'dot' files and directories:
git add * 

# adds all
git add --all
# or:
git add -A

# adds all recursively (including dot files):
git add .

You can also add and commit in one command: 

git commit -a -m "my commit"

Checking the status after adding files: 

git status
>> on branch main
>>
>> no commits yet
>>
>> changes to be committed:
>>  (use "git rm --cached <file>..." to unstage)
>>      new file:   .gitingore
>>      new file:   logo.png
>>      new file:   index.htm
>>

Remove files

To delete file from the local repo (and actually delete the file):

git rm <file>

To delete file from the local repo (but not actually delete the file from the working area):

git rm --cached index.htm

Committing Files

Commit with a message: 

git commit -m "first commit = committing all files to the repository"

Adding two messages results in the commit title and then explanation field: 

git commit -m "my commit" -m "A commit to do such and such"

Git Diff

Shows differences between commits. (I think it's 'q' to quit.) 

git diff

Restore Files

To restore a file from the git repo: 

git restore "myfile.txt"

Change Name of Items

Essentially 'move' to a new name: 

git mv "oldname.png" "newname.png"
git commit -m "changed the file name of image"

Ignore Files

Add a file called .gitignore to the root of the repository:

To ignore certain files (or wildcard patterns), add the rules to the body of the file:

# Ignore all .txt files:
*.txt
# On MacOS, ignore the thumbnail hidden files:
.DS_Store

To NOT ignore a file (perhaps a file with an extension you've already flagged to ignore in the .gitignore file:

# Ignore text files:
*.txt
# Don't ignore a specifit file (by the prefix '!'):
!mylovelyhorse.txt

More help can be found at: https://github.com/github/gitignore

Git Log

Git log shows all the commits as a log of events and changes: 

git log

To show a brief summary for each commit: 

git log --oneline

See also:

  • Git - History & Log

Change Commit Message

You can change the commit message: 

git commit -m "This is a new commit message" --amend

To see a diff of commit messages: 

git log -p

Git Branches

Create a new 'branch': 

git branch NewBranch

..or to switch to a new branch as it's created: 

git switch -c NewBranch

Check branches: (notice the asterix next to the 'active' branch) 

git branch
>>  NewBranch
>> *main

Switch to another branch: 

git switch NewBranch

git branch
>> *NewBranch
>>  main

Git status will now show the branch status also

Git Merge

git switch main
git merge -m "merge NewBranch to main" NewBranch

If there is no conflict, the branches will merge, else you will have to pick the changes. The changes show up as: 

<<<<<<<< HEAD
This is one branch
=======
This is the other
>>>>>>>> UpdateText

You have to edit the options (delete between "<<<<<" or ">>>>>" and "======").

Delete A Branch

git branch -d NewBranch
>> Deleted branch NewBranch (was 14da399)

Changing The Name Of A Branch

You can change the default branch name when initializing a Git repository or rename it afterward. Here’s how:

Set a different initial branch during git init

You can specify the branch name when creating a new repository: 

git init --initial-branch=mybranch

This sets the initial branch to mybranch instead of main.

Rename an existing branch

If you already have a branch named main and want to rename it, use: 

git branch -m main mybranch

This renames main to mybranch.

Update the default branch on remote (like GitHub)

If you're pushing the renamed branch to a remote repository, update the tracking reference: 

git push -u origin mybranch

Then, set it as the default branch on platforms like GitHub and delete the old main branch: 

git push origin --delete main
