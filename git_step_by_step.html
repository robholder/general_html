Git Step-by-step

Git Configuration

Configure Git on the local machine, ready for git use: 

git config --global user.name "robholder"
git config --global user.email "robholder@gmail.com"
git config --global init.default branch main

For help with config: 

git config -h
#or
git help config

Initialize a Git repository

# Navigate to enclosing directory:
git init

Stages of Git Workflow

  • Working files:
    • A live status; files in progress
  • Staging:
    • Awaiting commit
  • Commit:
    • Logging to the git repository as a change

git status

>>

Add files

Add files to the git repository stage: 

git add index.htm
git add --all
git add -A
git add .

You can also add and commit in one command: 

git commit -a -m "my commit"

Checking the status after adding files: 

git status
>> on branch main
>>
>> no commits yet
>>
>> changes to be committed:
>>  (use "git rm --cached <file>..." to unstage)
>>      new file:   .gitingore
>>      new file:   logo.png
>>      new file:   index.htm
>>

Remove files

git rm --cached index.htm

Committing Files

Commit with a message: 

git commit -m "first commit = committing all files to the repository"

Adding two messages results in the commit title and then explanation field: 

git commit -m "my commit" -m "A commit to do such and such"

Modified Files After Commit

Git status shows if a file has been modified since the last commit: 

git status

Git Diff

Shows differences between commits. (I think it's 'q' to quit.) 

git diff

Restore Files

To restore a file from the git repo: 

git restore "myfile.txt"

Change Name of Items

Essentially 'move' to a new name: 

git mv "oldname.png" "newname.png"
git commit -m "changed the file name of image"

Ignore Files

Add file names (or wildcard patterns) to the body of the file to ignore certain files: 

# Ignore all .txt files:
*.txt

More help can be found at: https://github.com/github/gitignore

Git Log

Git log shows all the commits as a log of events and changes: 

git log

To show a brief summary for each commit: 

git log --oneline

Change Commit Message

You can change the commit message: 

git commit -m "This is a new commit message" --amend

To see a diff of commit messages: 

git log -p

Git Branches

Create a new 'branch': 

git branch NewBranch

..or to switch to a new branch as it's created: 

git switch -c NewBranch

Check branches: (notice the asterix next to the 'active' branch) 

git branch
>>  NewBranch
>> *main

Switch to another branch: 

git switch NewBranch

git branch
>> *NewBranch
>>  main

Git status will now show the branch status also

Git Merge

git switch main
git merge -m "merge NewBranch to main" NewBranch

If there is no conflict, the branches will merge, else you will have to pick the changes. The changes show up as: 

<<<<<<<< HEAD
This is one branch
=======
This is the other
>>>>>>>> UpdateText

You have to edit the options (delete between "<<<<<" or ">>>>>" and "======").

Delete A Branch

git branch -d NewBranch
>> Deleted branch NewBranch (was 14da399)
