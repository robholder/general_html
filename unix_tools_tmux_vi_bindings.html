Tmux - Vi Bindings (Regular)

You can enable vi bindings for the tmux status line (the command prompt where you rename windows) by setting the status-keys option.

While many users are familiar with mode-keys vi for navigating terminal history (Copy Mode), status-keys vi specifically targets the interactive prompt.

1. Enable Vi Bindings for Prompts

Add this line to your ~/.tmux.conf to use vi-style editing when renaming windows (triggered by Prefix + ,) or entering commands (Prefix + ) 

set -g status-keys vi

2. How to Use It

Once enabled, you can switch between Insert and Normal modes just like in Vim or a vi-style shell:

  • Enter Normal Mode: Press Esc.
  • Move Around: Use h and l to move the cursor left and right.
  • Edit Text: Use standard vi commands like w (next word), b (back a word), x (delete character), or cw (change word).
  • History: Press k or j in Normal Mode to scroll through previous tmux commands or window names.
  • Return to Insert Mode: Press i or a.
  • Confirm/Cancel: Press Enter to save the new name or Ctrl+c to cancel.

3. Comparison of Vi Modes in Tmux

It is common to enable both settings to get a consistent Vim feel across the entire multiplexer:

Setting Purpose Trigger
set -g status-keys vi Editing window names, session names, and the command prompt. Prefix + , or :
setw -g mode-keys vi Navigating, searching, and copying text from your terminal's history.  

If you'd like, I can show you how to bind a single key that prompts for a name and automatically clears the old one, or how to sync these settings with your system's $EDITOR variable.

Single Key Binding

You can streamline your workflow by creating a "clean" rename prompt and ensuring tmux automatically adopts your preferred editor's style.

1. Bind a Key to Prompt with an Empty Name

By default, the rename-window prompt (Prefix + ,) is pre-filled with the current window name. To get a blank prompt for faster typing, add this to your ~/.tmux.conf: 

# Rebind ',' to prompt for a new name without pre-filling the current one
bind-key , command-prompt -p "rename-window:" "rename-window '%%'"
  • How it works: The -I flag (which sets the initial text) is removed, leaving the field empty.
  • The %%: This is a placeholder that tmux replaces with whatever you type into the prompt.

2. Sync Settings with your $EDITOR

Tmux is actually designed to be "smart" about your preferences. It checks your VISUAL or EDITOR environment variables; if they contain the string "vi" (like vim or nvim), tmux should automatically set status-keys to vi.

To ensure this works regardless of environment or to force it, you can use these settings:

  • Check Current Mode: Run this in your terminal to see what tmux currently thinks: 
tmux show-options -g | grep key
  • Force Sync in Config: If you want to be explicit in your ~/.tmux.conf, add: 
set -g status-keys vi
setw -g mode-keys vi

3. "Pro" Rename with Project Prefix

If you want a single key to do something more specific—like naming a window for a project—you can use a custom binding like this: 

# Press Prefix + P to rename window with a "proj-" prefix automatically
bind-key P command-prompt -p "Project Name:" "rename-window 'proj-%%'"

Quick Vi Commands in the Prompt

Once status-keys vi is active and you trigger your new empty prompt, remember these shortcuts:

  • Esc: Enter Normal mode to use h/l or w/b for navigation.
  • Ctrl+c: Cancel and exit the prompt.
  • Enter: Save and apply the name.

Directory Based Naming

Here is how you can set up automatic directory-based renaming and advanced navigation for a truly "Vim-like" tmux experience.

1. Automatically Rename to Current Directory

You can configure tmux to show the folder name instead of the command. Add these to your ~/.tmux.conf:

Standard Directory Name:

set-option -g status-interval 1
set-option -g automatic-rename on
set-option -g automatic-rename-format '#{b:pane_current_path}'
  • #{b:pane_current_path} uses the "basename" (just the folder name, not the full path).

Git Project Name: If you are in a Git repo, this names the window after the repo root:

set -g automatic-rename-format '#(basename "#{?#{pane_current_path},$(git -C #{pane_current_path} rev-parse --show-toplevel 2>/dev/null || echo #{pane_current_path}),}")'

2. Jump Directly to Named Windows

You don't need to cycle through windows (Prefix + n/p). Use these shortcuts to "jump" like you do in Vim:

  • Jump by Name (Built-in): Press Prefix then ' (single quote).
    • Type the window name (even just a few unique starting letters) and press Enter.
  • Search and Jump: Press Prefix + f.
    • This opens a search prompt. Type part of the name or even text visible inside the window to find it.
  • Interactive Tree: Press Prefix + w.
    • This shows an interactive list of all windows with a live preview. Use j/k (if mode-keys vi is set) to navigate and Enter to select.

3. Quick Key Bindings for Common Windows

If you have "fixed" windows you use every day (e.g., one named work, one named logs), you can bind direct keys to them: 

# Jump to window named 'work' with Prefix + W
bind-key W select-window -t 'work'

# Jump to window named 'logs' with Prefix + L
bind-key L select-window -t 'logs'
