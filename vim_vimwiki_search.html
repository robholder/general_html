VimwikiSearch

Using :VimwikiSearch is one of the most powerful features of Vimwiki. It allows you to treat your entire personal knowledge base as a searchable database.

Instead of searching only inside your current file, :VimwikiSearch scans all text files within your active wiki and outputs the matches into a navigable window.

1. Core Command Usage

To run a basic search across your entire wiki, press : from Normal mode and execute the following command: 

:VimwikiSearch /your_search_term/
  • The Slashes (/): Wrapping your search term in /.../ tells Vimwiki to use regular expressions (regex). If you want a literal, case-insensitive string search instead, you can omit the slashes (e.g., :VimwikiSearch taylor swift).

2. Navigating the Search Results (The Quickfix & Location Lists)

NOTE: Vimwiki almost ALWAYS uses 'LOCATION LIST'.

In Vim, both the Quickfix List and the Location List are special internal text buffers designed to hold a list of file positions (a file name, line number, column, and a text snippet). They are primarily used to store compiler errors, grep results, or search results so you can quickly jump through them.

The difference between them comes down to scope and multi-tasking.

1. The Quickfix List (Global Scope)

The Quickfix list is global to your entire Vim session.

  • How it works: There is only one global Quickfix list per Vim instance. If you open four different file tabs or split windows, they all share the exact same Quickfix list.
  • The Catch: If you run a new search (:vimgrep or a global plugin search) in one window, it will overwrite the Quickfix list for all other windows.
  • Analogy: Think of it like a single global clipboard. Anyone can copy to it, but it replaces whatever was there before.

2. The Location List (Window-Local Scope)

The Location List is local to the specific window pane currently active.

  • How it works: Every single split window or tab you open can have its own individual, unique Location List.
  • The Benefit: You can run a search in your left window pane, pop open its Location List, and lock those results there. Then, you can move to your right window pane, run a completely different search, and view those results simultaneously without ruining your first search.
  • Analogy: Think of it like a sticky note attached directly to a specific monitor screen. It only pertains to what you are doing on that exact screen.

Why this matters for Vimwiki

Vimwiki defaults to using the Location List for :VimwikiSearch because it assumes you want your wiki search results safely attached to the window where you are writing notes. This allows you to keep your global Quickfix list completely free for other tasks, like tracking coding compilation errors or code-wide refactoring searches.

Quickfix & Location Lists in Vimwiki

By default, Vimwiki puts your search results into a temporary background container. To actually see and navigate through them, you use Vim’s Location List (local to your window) or Quickfix List (global), depending on how you've configured Vimwiki.

NOTE: Vimwiki almost ALWAYS uses 'LOCATION LIST'.

You can easily tell which list Vimwiki is using by looking at the small status bar at the bottom of the pane when you open the search window.

Try running a test search:

  • Type :VimwikiSearch /test/ and press Enter.
  • Type :lopen (to open the Location list window).

Look at the status line at the very bottom edge of that new split window.

  • If it is a Location List: You will see [Location] written explicitly in the status bar text.
  • If it is a Quickfix List: It will display [Quickfix] instead.

The Cheat Sheet Rules for Vimwiki

If you haven't explicitly altered your Vim configuration file, Vimwiki defaults to using the Location List for its search features (:VimwikiSearch and :VimwikiSearchTags).

If you ever want to quickly check if a list window is open without looking at the status line, you can test it directly with a navigational shortcut:

  • If typing :lnext shifts your cursor to the next search result, your results are in a Location List.
  • If typing :cnext moves your cursor, your results are in a Quickfix List.

Here are the critical long and short commands you need to manage these search layout panes:

Opening the Panel

If your results don't show up automatically, use these commands to force the results window open at the bottom of your screen:

Long

  • Location List (Long): :lopen
  • Quickfix List (Long): :copen

Short

  • Location List (Short): :lope
  • Quickfix List (Short): :cope

Closing the Panel

Long

  • Location List (Long): :lclose
  • Quickfix List (Long): :cclose

Short

  • Location List (Short): :lcl
  • Quickfix List (Short): :ccl

Jumping Between Results

Once the list is populated, you can cycle through the search matches line-by-line without even opening the panel:

Long

  • Next match:
    • :lnext or
    • :cnext
  • Previous match:
    • :lprevious or
    • :cprevious
  • First match:
    • :lfirst or
    • :cfirst
  • Last match:
    • :llast or
    • :clast

Short

  • Next match:
    • :ln or :cn
  • Previous match:
    • :lpr / :lp
    • :cpr / :cp
  • First match:
    • :lf or
    • :cf
  • Last match:
    • :ll or
    • :cla

3. Advanced Features & Use Cases

Because :VimwikiSearch natively supports Vim regular expressions, you can perform highly functional scoped searches on your markdown/wiki files.

Finding Incomplete Tasks

If you use Vimwiki’s task lists (- [ ]), you can search your entire archive to find every single open action item across all files: 

:VimwikiSearch /- \[ \]/

Once the results match, type :lope to see every single uncompleted checkbox task you've written down.

Search Anchors

  • Lines starting with a word: :VimwikiSearch /^Project/ (Finds pages where a line starts with "Project")
  • Case-insensitive flag: :VimwikiSearch /ideas\c/ (The \c forces Vim to match "ideas", "Ideas", or "IDEAS")

4. Pro-Tip: Map it to a Shortcut

Typing out :VimwikiSearch and :lopen over and over gets tedious. You can map this entire workflow to a handy shortcut in your .vimrc file.

Add these lines to your configuration to let <Leader>ws (usually \ws) search your wiki, and automatically pop open the short version of the window right away: 

" Search entire wiki for the word under your cursor and open the location list
nnoremap <Leader>ws :VimwikiSearch /<C-r><C-w>/<CR>:lope<CR>

" Fast navigation shortcuts for the results
nnoremap <C-n> :lnext<CR>
nnoremap <C-p> :lprev<CR>
