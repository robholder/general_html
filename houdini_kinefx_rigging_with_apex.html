Minimal Workflow - (FK) Setup

YouTube - 2 Minimal Workflow - FK Setup

Apparently, the logic and execution of the APEX rig, is separated, resulting in a very efficient system without the lag from performing logic and execution simultaneously. (Ik or Fk logic can be separated for example.)

The Packed Folder is the basis for this organization - clearly defining all the components to process.

In the network illustrated above, the jointdeform is used to point out that it is performing rig logic and rig execution at the same time - the APEX doesn't work like this.

APEX Autorig Component

The tutorial introduces the APEX Autorig Component node, and it is the initiation of the APEX setup.

It has two inputs:

  • Input from Packed Folder node
  • Graph or Script input (optional - as there are 'build-in' options from the Component Source dropdown).

FK Transform

The tutorial adds an APEX Autorig Component node, and chooses fktransform as the Component Source

The first thing that happens is that everything disappears from the Viewport. This will be the state until part 3.

Another thing to observe, that if we look at the Rig Tree viewer panel, we'll see a new item called Base.rig. This is the default name supplied in the APEX Autorig Component node's Rig section.

Graphs

There is a side piece of the tutorial, that shows what a 'script' is. It's a graph (much like a shader tree or VOPs graph), but it is saved as geometry.

These scripts or graphs are what define the rig logic.

You can actually see the geometry that's referenced if you use a 'built-in' graph option (like fktransform in the Autorig Component; it's a path reference to a .bgeo file - which is pure geometry.

The geometry format is purely for convenience however, the nodes are stored as points, and the attributes are stored in the same way that geometry attributes are stored.

Reading the Graphs

To see what the graphs really represent, view through the APEX Network View (another Houdini panel, dedicated to the APEX graphs).

The will be a familiar layout - again, much like VOPs or shader networks.

Minimal Workflow - Controls

YouTube - 4 Minimal Workflow - Controls

Controls

Continuing on from the second APEX Autorig Component, an APEX Configure Controls node is dropped.

We are introduced to the settings, whereby 'control' icons can be configured.

  • Control Config:
    • ^* root is the string entered ('not everything' and 'root', resulting in the 'root' being considered).
  • Use Shape Override:
    • Shape Override is set to circle_wires, although there are many options in the dropdown.
  • Use Shapeoffset:
    • Mode is set to Pre-Multiply:
      • Translate is zero
      • Rotate is 90, 0, 0
      • Scale is 15, 15, 15
  • There are several other settings (like Color and Visibility)
  • Use Limits:
    • To set transform limits on the rig controls
    • You set the parm name and then the limit values.

All this provides a control icon for the 'root' bone of the rig.

Unpack Folder (for viewing)

There is a brief aside, where an Unpack Folder is used (Forked from the chain, not part of the setup) to unpack the /Base.rig component (and now the Rig Tree shows a warning for this node, as the rig is now unpacked to a graph), and an Edit Graph node is dropped after it to see the APEX Network View pane. This allows the user editing of the graph.

When we add the controls with the APEX Configure Controls node, you can see the attributes changed using the Geometry Spreadsheet while inspecting the Unpacked Folder node.

SUPER TIP

'RMB' clicking on a Geometry Spreadsheet cell allows you to pick 'Inspect...' to see a (possibly json) view of the cell's contents.

In our case, the tutorial views the json representation of the controller settings, and we can see all the components of the circle-wire control icon (as defined in our 'APEX Configure Controls' node).

ROP Geometry

The finished rig and geometry can be saved out as an asset using a ROP Geometry node, through which, the geo can be saved as a .bgeo or .bgeo.sc file (the 'sc' being a compressed version).

These saved assets can be brought in again using a File node, and fed into a Scene Add Character node, and then into a Scene Animate node.

Component Parameters

YouTube - 5 Component Parameters

The APEX Autorig Component node has two distinct Component parameters.

  • The node's generic parameters (only 3):
    • Component:
      • The logic
    • Rig:
      • Whether it's added or updated
      • Name of the rig
    • Advanced
  • The component as chosen has its own setting/parameters:
    • These are different depending on which component is chosen.
    • NOTE: There are two buttons on the component parameters:
      • Broom - to clean away 'spare parameters' (these are all 'spare parameters')
      • Refresh - to reload parameters if the 'Component' that we are referencing changes.
    • EXAMPLE:
      • Clicking the brush will remove all the Component Parameters
      • Refreshing with the reload button will bring them all back (from the reference component graph).

This workflow is essential for Beta versions of these nodes, but might get less frequent with later builds of Houdini.

Parms

YouTube - 6 Parms

This part of the tutorial showed the Attribute Adjust Dictionary node - and showed how it's used to specify:

  • The components to affect (e.g. Group @name=root)
  • Attribute Name parms
  • Set Values (and this is a Dictionary):
    • Key:
      • promote
    • Value
      • t r s (spaces between attributes)

The Attribute Adjust Dictionary overrides any controls that have been promoted in the APEX Autorig Component nodes.

Properties and Tags

YouTube - 7 Properties and Tags

Properties

This tutorial expands on the introduction to the Attribute Adjust Dictionary, and explains that the attributes we specify are essentially data that is included with the rig components.

We see various parts of a skeleton being targeted for updating with specifications:

  • Group - @name=front_left_hip,front_left_knee,front_left_toe,frint_right_hip,front_right_knee,etc
  • Attribute Name - properties

and for Set values:

  • Number of Entries 2:
    • Key - shape
    • Value - circle_wires
    • Key - shapescale
    • Value - 1, 1, 1

This is storing data (in the form of dictionaries) to be used later to apply preferences to rig components (like the control icons in this case).

All data set here, don't show up in the Viewport for example; the data is stored read to call upon later (using the APEX Autorig Component or other APEX nodes).

This data can be seen on the Geometry Spreadsheet. They will show up under the specified Attribute Name columns.

Tags

For the editing of Tags we use the Attribute Adjust Array node.

  • Add joint/points with Operation and Add then the reference, @name=root
  • Value root
  • Operation and Add, reference @name=dome
  • Value dome

etc.

In the tutorial examples, the joints/points are often given tags of just the name of the joints/points, but some are given a tag of IK for example, to show they are Inverse Kinematic joints perhaps.

The tutorial actually gives the same joints/points multiple Tags (name of the joint/point and the 'IK' for example). By including more and more tag layers, you can display any amount of useful labeling or tagging.

Displaying Tags

Selecting the Attribute Adjust Array node, and pressing Enter in the viewport, you get the Tags displayed next to each joint/point in the skeleton.

At the base section of the Attribute Adjust Array you can colorize the tags.

  • RegEx the tag values and allocate a color:
    • Example: ^* IK and 'red', will omit all, then add the IK tag to be colored with the red swatch.
    • (Refresh the node - on and off - to see the colors update.)

It's explained that by feeding these pre-made data sets into the APEX Autorig Component nodes can make for standardized rigs even on very different rig set-ups.

APEX Autorig Components

YouTube - 8 APEX Autorig Components

Transform Driver

The tutorial shows the Attribute Adjust Array node going into the Pack Folder node along with the null nodes shown at the start. All the attributes get fed into the Pack Folder with everything else.

After the APEX Autorig Component nodes (one for fktransform the other for bonedeform), another is added set to transformdriver.

  • Yet again, this is set to Add or Update.
  • guidesource is set to Guide.skel (one of the conventions set up at the start).
  • Each instance has tabs:
    • Settings#:
      • Set's the transforms that will be driven
    • Driven#:
      • set the joint/point name(s) - root in the example.
    • Control#:
      • driver is set to Main (in this case - it's an arbitrary name given to the control)
      • Make a control (checkbox)
      • Options to promote the transforms
    • Tags#:
      • An opportunity to add Tags to the Controls this time.
    • Shape#:
      • shape to add the control icon style - and int his case, it's picking up the shape we already setup.

The tutorial shows several more setups for 'core' control (a secondary control to move everything from 'Main', and anther for limb offsets (to lower the limbs of the mechanism model.

Multi IK

This example has the usual tags and options but for IK.

The rootparent and tipparent options are interesting. They allow you to specify which other parts of the skeleton the driving icons are parented to. Being IK, the root can be set to the overall hip thing, and the tips (like the ends of the limbs) to the main root, so they get moved with the whole thing.
