Hacking With Swift - Summary: Functions

Functions

  • Functions - reuse code
  • Start with the word 'func', followed by function 'name'
  • Functions can accept parameters
  • You can add custom external parameter names
  • You can hide parameter names with '_ '
  • Parameters can have default values set at definition
  • Functions can return a value
  • When returning multiple pieces of data, you can return on the form of a tuple.

Errors

  • Functions can throw errors: you create an enum defining the errors you want to happen, throw those errors inside the function as needed, then use do, try, and catch to handle them at the call site.
