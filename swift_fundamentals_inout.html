Swift - Inout Keyword

In Swift, function parameters are constants by default. If you try to change the value of a variable passed into a normal function, Xcode will throw a compiler error.

The inout keyword explicitly changes this behavior. It tells Swift that a function is allowed to directly modify the value of a variable passed into it, and that those changes should persist after the function finishes running.

Think of it as passing a variable by reference rather than passing a copy of its value.

How Normal Parameters Work (Pass-by-Value)

Normally, when you pass a variable into a function, Swift makes a temporary copy of that data inside the function. 

func attemptToIncrement(number: Int) {
    // This line will cause a compiler error!
    // 'number' is a constant copy.
    number += 1 
}

How inout Parameters Work (Pass-by-Reference)

When you label a parameter as inout, Swift does not make a copy. Instead, it temporarily gives the function direct access to the original variable in memory.

To use an inout parameter, you must do two things:

  • Put inout right before the data type in the function definition.
  • Put an ampersand (&) right before the variable name when you call the function. The & acts as an explicit warning to you, the developer, that the variable's value is about to be mutated. 
    func realIncrement(number: inout Int) {
    number += 1 // This works perfectly!
}

var score = 10
// We pass 'score' using the ampersand (&)
realIncrement(number: &score)

print(score) // Output: 11 (The original variable was changed!)

Breaking Down the swapValues Example

Going back to the generic value-swapping function from the article: 

func swapValues<T>(_ a: inout T, _ b: inout T) {
    let temporary = a   // 1. Save the value of 'a' into a temporary box
    a = b               // 2. Overwrite 'a' with the value of 'b'
    b = temporary       // 3. Overwrite 'b' with the saved temporary value
}

Because a and b are marked as inout, the function is able to reach outside its own boundaries and scramble the contents of the actual variables you fed into it.

Here is how you would use it in practice: 

var lowScore = 5
var highScore = 99

// We pass both variables with an '&' because the function modifies them in-place
swapValues(&lowScore, &highScore)

print(lowScore)  // Output: 99
print(highScore) // Output: 5

Without the inout keyword, a swapping function would be entirely impossible to write, because the function would only be swapping temporary copies inside its own local code block, leaving your original lowScore and highScore variables completely untouched.
