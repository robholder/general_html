Hacking With Swift - Summary: Classes

Classes, while less common than structs, are valuable for data sharing and are extensively used in UIKit.

Classes vs Structs - Five Differences

  • Classes can inherit from parent classes:
    • Access to a parent's properties & methods
    • Optionally override methods
    • Marks as 'final' to prevent inheritance
  • Classes don't get a memberwise initializer:
    • Sub-class must always call parent initializer
  • Class copies all point to the same instance:
    • Changing data in one, changes all
  • Classes can have 'deinitializers' (that run when the last copy is destroyed)
  • Variable properties in a class can be changed regardless of whether the instance was created as a variable or constant.

    --- --- ---
