Hacking With Swift - 11 - How to create and use enums

Enums

Enums is short for 'enumerations'.

They are a set of strict named values.

You can use the entries in a set of enums the possible allowable values:

Example (days of the week) defined as an enum: 

enum Weekdays {
    case monday
    case tuesday
    case wednesday
    case thursday
    case friday
}

Or, in a shorthand layout: 

enum Weekdays {
    case monday , tuesday , wednesday , thursday , friday
}

Calling Values

These can then be called, as in these variable assignments

(When using the Xcode editor, these 'case' items will appear as suggestions when coding.) 

var day = Weekday.monday
day = Weekday.tuesday
day = Weekday.friday

print(day)
# >> friday

You can't change the type of item once a variable or constant is defined; and one advantage of this is that enum values can be called like this (avoiding the re-iteration of the enum name): 

var day = Weekday.monday
day = .tuesday
day = .friday

print(day)
# >> friday

Under the hood, enums are stored in a highly efficient way.

Raw Values

Enum cases can be strings by using Raw Values. When you define an enum with a String raw type, Swift can either implicitly use the case name as the string or allow you to assign custom string literals.

Single-Line Implementation

You can still define multiple string-based cases on a single line. If you don't assign a specific value, Swift automatically uses the case name as the string. 

// Implicit: 'north' is "north", 'south' is "south", etc.
enum Direction: String {
    case north, south, east, west
}

// Explicit: Some or all can have custom strings
enum Status: String {
    case loading = "In Progress", success = "Complete", failure
}

Key Behaviors

  • Implicit Values: If you define case loading, its rawValue is automatically "loading".
  • Accessing the String: Use the .rawValue property to get the actual string value (e.g., Status.loading.rawValue).
  • Initialization: You can create an enum instance from a string using Status(rawValue: "Complete"), which returns an optional.
  • Uniqueness: Every raw value in the enum must be unique.

SwiftUI Usage

String enums are particularly useful in SwiftUI for Picker labels or Text views: 

Text(Status.success.rawValue) // Displays "Complete"

Another Example

enum Breakfast: String {
    case bacon, eggs, beans
}

// Usage
let choice = Breakfast.bacon
print(choice.rawValue) // Outputs: "bacon"

If you need different display names

If you want the code name to be different from the actual string value (for example, if you want a capital letter or a space), you can assign them explicitly: 

enum Breakfast: String {
    case bacon = "Crispy Bacon", eggs = "Scrambled Eggs", beans
}

print(Breakfast.bacon.rawValue) // Outputs: "Crispy Bacon"
print(Breakfast.beans.rawValue) // Outputs: "beans" (defaults to case name)
