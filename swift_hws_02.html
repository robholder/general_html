Hacking With Swift - 02 - How to create variables and constants

  • Website Link

Summary

Swift provides two ways to store data: variables and constants. Variables, created with the “var” keyword, can change their values, while constants, created with the “let” keyword, remain fixed. It is recommended to use constants whenever possible for better code optimization and to prevent accidental value changes.

Storing Data

  • Variables - change over time:
    • use to store values that will be subject to change.
  • Constants - unchanging data
    • Set and protect values

Variables

Keyword -> var 

import Cocoa

var greeting = "Hello, playground"
  • var:
    • Create a variable storage
  • <greeting>:
    • name given to variable
  • = :
    • Set instruction
  • "<string>" :
    • value assigned

; is optional but rarely used (and can be used to indicate line break if on single line). 


# Set variable and initial value
var name = "Ted"
 
# Assign new value
name = "Rebecca"

# Assign another value
name = "Keeley"

Constants

Keyword -> let 


# 'character' cannot be changed once set:
let character = "Daphne"

character = "Eloise" !! Cannot assign to value: 'character' is a 'let' constant

Debugging 'Print'

Note that Swift's convention is to use camelCase for variables and constants. 


var playerName = "Roy"
print(playerName)

playerName = "Dani"
print(playerName)

playerName = "Sam"
print(playerName)

Play Head

On the left of the coding pane, where the line numbers appear, there should be a blue arrow 'play' icon. The code will play to the line position of the icon when the play button is pressed.

The 'print' commands will output to the pane on the right of the Xcode UI.
