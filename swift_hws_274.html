Hacking With Swift - 274 - Challenge 1

Challenge 1

  • Make views near the top of the scroll view fade out to 0 opacity - I would suggest starting at about 200 points from the top.

QUESTION

I'm trying to set the opacity of a text background color using the GeometryReader, but Xcode is complaining about the 'type'. (It says, "Cannot convert value of type 'CGRect' to expected argument type 'Double'".)

This is what I have within a GeometryReader: 

.background(colors[index % 7]).opacity(proxy.frame(in: .global))

ANSWER

Xcode is barking here because proxy.frame(in: .global) returns a CGRect - which is an entire package containing a view's origin (X and Y coordinates) and size (width and height).

The .opacity() modifier only accepts a Double between 0.0 (invisible) and 1.0 (fully opaque). You can't pass a whole geometric box into an opacity slider!

To fix this, you need to target a specific property of that frame (like .midX or .minX) and convert it into a ratio between 0.0 and 1.0.

The Fix

Assuming you want the item to fade out as it travels across the screen, you need to divide the card's position by the total width of the screen.

Here is how you update that line: 

// Assuming 'fullView' is the outer GeometryReader measuring the screen width
.background(colors[index % 7])
.opacity(Double(proxy.frame(in: .global).midX / fullView.size.width))

What is happening here?

  • proxy.frame(in: .global).midX: Instead of the whole CGRect, this grabs just a single floating-point number representing the exact center X-coordinate of your card.
  • / fullView.size.width: We divide that coordinate by the full width of the phone screen. If the card is on the far left, this equals roughly 0.0. If it's dead center, it equals 0.5. If it's on the far right, it equals 1.0.
  • Double(...): Because geometry calculations use CGFloat, we wrap the final result in a Double cast to keep the .opacity() modifier perfectly happy.

Tip for a Carousel Fade Effect: If your goal is to have the cards be brightest in the center and fade out toward both edges, you can use absolute math to measure the distance from the center line: 

.opacity(1.0 - Double(abs(proxy.frame(in: .global).midX - fullView.size.width / 2) / (fullView.size.width / 2)))

QUESTION

This is obvious now that you tell me. Thanks for the hints!

You say that the proxy.frame(in: .global) call returns a CGRect, which appears to be 4 pieces of data. I can see that we are using .midX and .midY, but what other calls are there to get all the data? Is the remaining data accessible with other property calls?

ANSWER

You’ve got it! A CGRect is essentially a standard structural blueprint that holds four primary dimensions, but Apple provides a goldmine of pre-calculated properties on top of them so you rarely have to do any manual math.

Here is the breakdown of how a CGRect organizes its data and every property call you can use to extract it.

1. The Core Data (The Original 4 Pieces)

Under the hood, a CGRect contains exactly two nested sub-structs: origin (where it starts) and size (how big it is). Each contains two pieces of data: 

let myFrame = proxy.frame(in: .global)

// 1 & 2: The Origin (Starting Point)
let x = myFrame.origin.x  // The left-most coordinate
let y = myFrame.origin.y  // The top-most coordinate

// 3 & 4: The Size (Dimensions)
let width = myFrame.size.width
let height = myFrame.size.height

2. The Built-In Property Helpers (The Shortcuts)

Instead of constantly digging into .origin.x or doing math like origin.x + size.width to find the right edge, Apple gives you direct, read-only properties on the frame itself. These are your best friends when designing responsive layouts:

Horizontal Coordinates (X-Axis)

  • .minX: The absolute left edge of the view (identical to origin.x).
  • .midX: The exact horizontal center point of the view.
  • .maxX: The absolute right edge of the view (origin.x + width).

Vertical Coordinates (Y-Axis)

  • .minY: The absolute top edge of the view (identical to origin.y).
  • .midY: The exact vertical center point of the view.
  • .maxY: The absolute bottom edge of the view (origin.y + height).

3. Bonus Geometry Shortcuts

To round out your layout toolbox, CGRect also exposes a few handy tracking shortcuts that bundle coordinates into standard type objects: 

let frame = proxy.frame(in: .global)

// Get the top-left coordinate as a single CGPoint (x, y)
let dynamicPoint = frame.origin 

// Get the width and height bundled together as a CGSize
let dynamicSize = frame.size   

// Check if a view has zero width and height
let isEmpty = frame.isEmpty    

// Find the absolute center of the frame as a CGPoint(x, y)
let centerPoint = CGPoint(x: frame.midX, y: frame.midY)

Whenever you use a GeometryReader, you can immediately chain any of these onto .frame(in:) (e.g., proxy.frame(in: .global).maxY). They make calculating relative distances, custom carousel fades, or scaling animations incredibly clean!

List of Properties

Here is the complete reference table of every property you can call on proxy.frame(in: .global) (or any coordinate space), what it returns, and exactly what it means for your layouts.

CGRect Property Reference Table

Property Call Returns Type What it measures / represents
.minX CGFloat The absolute left edge of the view.
.midX CGFloat The exact horizontal center point of the view.
.maxX CGFloat The absolute right edge of the view.
.minY CGFloat The absolute top edge of the view.
.midY CGFloat The exact vertical center point of the view.
.maxY CGFloat The absolute bottom edge of the view.
.width CGFloat The total horizontal width of the view (same as .size.width).
.height CGFloat The total vertical height of the view (same as .size.height).
.origin CGPoint A bundled (x, y) coordinate of the top-left corner.
.size CGSize A bundled (width, height) structural property of the view.
.isEmpty Bool Returns true if the view has a width or height of 0.
.isInfinite Bool Returns true if the frame layout size is infinitely large.
.isNull Bool Returns true if the frame represents an invalid/empty geometry value.

Pro-Tip: Shorthand Chain Reading

Because these properties are chained directly onto the result of the frame calculation, you will see them written in code layouts like this: 

// Check if the bottom of the card has scrolled past a marker line
if proxy.frame(in: .global).maxY > 500 { ... }

// Scale an item based purely on its current width
let scale = proxy.frame(in: .local).width / 200
