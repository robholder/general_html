To import gizmos into Nuke's Non-Commercial version, you'll need to add the gizmo's directory to your Nuke plugin path.

This involves modifying the init.py file or using the NUKE_PATH environment variable to point Nuke to the location of your gizmo files.

Here's a breakdown of the process:

  • Locate your .nuke folder: This folder is usually found in your user directory (e.g., C:/Users/YourUserName/.nuke on Windows, and /Users/<username>/.nuke on MacOS.).

Create a gizmo folder:

  • If you don't have one, create a folder named "gizmos" within your .nuke folder.
  • Place your gizmo files:
    • Copy your .gizmo files into the "gizmos" folder.
  • Modify the init.py file:
    • If you haven't already, create a file named init.py in your .nuke folder.
    • Add the gizmo folder to the plugin path: Open init.py and add the following Python code (adjust the path if your "gizmos" folder is not directly in your .nuke folder): 
import nuke, os

# Get the directory that this file is saved in
base_path = os.path.dirname(__file__)

# Add a folder to the plugin path
nuke.pluginAddPath(os.path.join(base_path, 'gizmos'))
  • Restart Nuke:
    • You'll need to restart Nuke for the changes to take effect.

Check the path with: 

print(nuke.pluginPath())
This next tip isn't relevant to MacOS Nuke.
  • Update the toolbar:
    • In Nuke, go to "Other > All plugins > Update" to load the gizmos into the node toolbar.

Alternative Method: Using the NUKE_PATH environment variable:

  • Locate your gizmo files: Ensure your .gizmo files are in a directory that you want to add to NUKE_PATH.
  • Set the NUKE_PATH environment variable: This can be done in your operating system's environment variables settings.
  • Add the gizmo directory to the NUKE_PATH: Add the path to your gizmo files to the NUKE_PATH variable.
  • Restart Nuke: Restart Nuke for the changes to take effect.

In Nuke, gizmos are custom reusable tools that function like node groups but allow for more flexibility. On macOS (or any system), enabling gizmos requires setting up the correct paths so that Nuke can locate them.

Steps to Enable Gizmos:

Locate or Create a Gizmos Folder

  • Typically, users store gizmos in a directory like ~/Documents/Nuke/Gizmos/
  • You can place .gizmo files inside this folder.

Set Up Your Nuke Path

  • Open Nuke and navigate to the init.py file (this is a startup script that helps define paths).
  • If you don’t have an init.py, create one inside ~/.nuke/
  • Add the following lines to ensure Nuke loads gizmos properly:
Tilde Expansion (~/folder) doesn't work - it concatenates in a weird way. 
import nuke
# Tilde expansion doesn't seem to work:
# ~/Documents/Nuke/Gizmos/ 
nuke.pluginAddPath('/Users/<username>/Documents/Nuke/Gizmos/')

Restart Nuke

Close and relaunch Nuke so that it can register your gizmos.

Access Gizmos

In the node graph, press Tab, type your gizmo’s name, and it should appear if registered properly. Alternatively, you can manually add them using: 

nuke.createNode("YourGizmoName")

If Nuke isn’t recognizing your gizmos, double-check the path and ensure it is correctly pointing to where your gizmos are stored.
