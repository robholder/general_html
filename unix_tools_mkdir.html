mkdir

Nested Directories

You can create nested directories in macOS Terminal using the mkdir command with the -p flag: 

mkdir -p parent_dir/child_dir/grandchild_dir

How It Works

  • -p ensures that all parent directories are created if they don’t exist.
  • Without -p, mkdir would throw an error if the parent directory doesn’t exist.

Example Usage

Suppose you want to create: 

projects/
└── my_project/
    ├── src/
    │   ├── components/
    │   ├── utils/
    ├── docs/

You can run: 

mkdir -p projects/my_project/src/components
mkdir -p projects/my_project/src/utils
mkdir -p projects/my_project/docs

This is super useful when organizing code, documents, or structured directories efficiently.
