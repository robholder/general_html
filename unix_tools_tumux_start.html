Cheat Sheet

  • Check for existing Tmux Sessions (Outside Tmux):
    • tmux ls - list sessions
  • Launch:
    • Outside Tmux:
      • tmux - new session
      • tmux new -s "<session_name>" - new session, with initial name
      • tmux a - open last accessed session
      • tmux a -t <session_name> - open particular session
  • Inside Tmux :
    • Sessions:
      • tmux ls - list sessions
      • <prefix_keys> + : (command mode), + new -s "<session_name> - create new session (from Tmux)
      • Switch between Sessions (inside Tmux):
        • <prefix_keys> + s - interactive list of sessions and windows ('up' 'down' or 'j' 'k' to choose)
        • <prefix_keys> + ( - Switch to previous session
        • <prefix_keys> + ) - Switch to next session
        • <prefix_keys> + L - (capital 'L') Switch to last session (toggles)
        • <prefix_keys> + : - command mode:
          • switch-t <#> e.g., switch-t 3 to switch directly to session number
      • <prefix_keys> + & - closes session even if multiple panes are present:
        • Confirmation dialog will appear in the status bar (does not destroy session permanently).
      • exit - closes panes, but in last remaining pane, closes and deletes the session:
        • No confirmation
    • Windows/Tabs:
      • tmux new-window - create a new window/tab
      • <prefix_keys> + c - create new window/tab
      • <prefix_keys> + , - rename window/tab
      • <prefix_keys> + : and new-window -n <name> (inside tmux)
      • tmux new-window -n <name> (in terminal)
    • Switch Windows:
      • <prefix_keys> + p - previous window
      • <prefix_keys> + n - next window
    • Panes:
      • Split Panes:
        • <prefix_keys> + % - split window; one left, one right
        • <prefix_keys> + " - split window: one up, one down
      • Navigate Panes:
        • <prefix_keys> + ↑ - up one pane
        • <prefix_keys> + ↓ - down one pane
        • <prefix_keys> + ← - left one pane
        • <prefix_keys> + → - right one pane
      • Delete Panes:
        • exit - to close each pane (no confirmation):
          • exit on last pane will close session (and delete session from list)
        • <prefix_keys> + x - exit pane (with confirmation dialog)
        • <prefix_keys> + d - to close whole session (but not delete session panes)

Sessions are your high-level projects (e.g., "Vimwiki" vs. "Dev").

3. Windows & Tabs (Inside Tmux)

Windows are the "tabs" within a single project session.

Action Shortcut
Create new window <prefix> c
Rename current window <prefix> ,
Next / Previous window <prefix> n or <prefix> p
Jump to window # <prefix> [0-9]
Choose Window <prefix> w (Use j/k to choose)
List Windows <prefix> : + lsw
Close current window * <prefix> & (With confirmation *)
  • NOTE: * Closing the last window/tab will also erase the session!

4. Panes & Splits (Inside Tmux)

Panes are the subdivisions of a single window.

Action Shortcut
Split Vertically (Side-by-side) <prefix> %
Split Horizontally (Top-Bottom) <prefix> " (or <prefix> _)
Navigate between panes <prefix> + Arrow Keys
Zoom pane (Toggle Fullscreen) <prefix> z
Close current pain * <ctrl> d
Close current pane * <prefix> x (With confirmation *)
Close current pane * exit (No confirmation *)
  • NOTE: * Closing the last pain in a window will close/erase the window!
  • NOTE: * Closing the last pane, in the last window, will close/erase the session!

Important Distinctions for your Flow:

  • Detach (d) vs. Kill (&/x/exit):
    • Always use d for your Vimwiki. It puts the session in a "closet" so you can pull it back out later.
    • The others stop your processes and lose unsaved work.
  • Prefix:
    • Remember that you must release the <prefix> keys before hitting the command key (like c or s).

Tmux

What is it?

  • Terminal Multiplexer:
    • Terminals within terminals
  • Type tmux in a terminal to start.
  • You'll see a 'footer' at the base of the window.
  • Session is persistent, even if connections to remote terminal is broken.

EXTRAS

More On Creating Named Sessions

You can name a tmux session on creation from your shell using the -s flag with the new-session command (or its alias new).

Standard Command

To start a new named session, use: 

tmux new -s "SessionName"

Replace "SessionName" with whatever you'd like to call it (e.g., "wiki", "work", or "coding").

Attach or Create (Recommended)

As you start using tmux more, you'll likely want a command that attaches to an existing session if it’s already running, or creates it if it isn't. Use the -A flag for this: 

tmux new -A -s "SessionName"
  • If it exists: It jumps you right back into that session.
  • If it doesn't: It creates a new one with that name.

Why 'new'?

In the world of CLI tools, new is a subcommand. Think of tmux as the "app" and new as the specific "action" you want that app to perform.

Without it, tmux doesn't know if you want to create something, list something, or attach to something else.

Pro-Tip: Automatic Folder Naming

If you want to quickly name a session after the folder you are currently in, you can use this snippet in your terminal: 

tmux new -s "$(basename "$PWD")"

This is a favorite for many developers because it automatically keeps your sessions organized by project name.

Jump To Specific Windows

To jump to a specific window in tmux, you have several "Vim-speed" options depending on whether you want to use a number, a name, or a visual list.

1. Jump by Number (Fastest)

Since your windows are numbered 0 and 1, use the standard shortcut:

  • Prefix + 0 → Jumps to Vimwiki
  • Prefix + 1 → Jumps to Named Window

2. Jump by Name (Search)

If you have many windows and don't want to remember numbers, use the find-window command:

  • Prefix + f
  • Type "Named" (or just "N") and hit Enter. Tmux will instantly switch to the first window matching that string.

3. The Visual List (Interactive)

If you want to see all your windows and pick one:

  • Prefix + w
  • Use your Vim keys (j and k) to move the highlight, then hit Enter to jump into that window.

4. Toggle Between Last Two (Pro Move)

Since you specifically mentioned having two windows, you can flip-flop between them instantly:

  • Prefix + l (lowercase L)
  • This takes you back to the "Last" active window. Hit it again to jump back. This is perfect for toggling between your Vimwiki and a terminal.

5. Next and Previous

  • Prefix + n (Next)
  • Prefix + p (Previous)

Summary for your ~/.tmux.conf

If you find Prefix + 0 or Prefix + 1 annoying to reach, you can bind them to something closer to your home row: 

# Example: Jump to Window 0 (Wiki) with Prefix + w
bind w select-window -t 0
# Jump to Window 1 (Terminal) with Prefix + t
bind t select-window -t 1

Fastfetch

Disable Fastfetch in Tmux: 

# Only run fastfetch if NOT inside a tmux session
if [ -z "$TMUX" ]; then
  fastfetch

Buffer/Filename from Vim

Configuring Vim to call Tmux window the name of the buffer/current filename.

Renaming tmux windows can be done manually through default shortcuts or automated via Vim configurations to keep your workspace organized by filename.

Manual Renaming (tmux Default)

rename-window "your-new-name"
  • Key Shortcuts:
  • <prefix_keys> + , (comma).
    • <Ctrl> b + ,
    • This opens a prompt in the status bar where you can type the new name and press Enter.
  • <prefix_keys> + : and edit the name
    • <Ctrl> b + : and edit the name

Automated Renaming from Vim

You can configure Vim to automatically update the tmux window name to the current filename. Add the following to your .vimrc: 

augroup tmux
  autocmd!
  if exists('$TMUX')
    " Set window name to current filename when opening or switching files
    autocmd BufReadPost,FileReadPost,BufNewFile,BufEnter * call system("tmux rename-window " . expand("%:t"))
    " Restore automatic renaming (e.g., to 'bash' or 'zsh') when leaving Vim
    autocmd VimLeave,FocusLost * call system("tmux set-window-option automatic-rename")
  endif
augroup END

This block has:

  • Dynamic Renaming:
    • Update the window name every time you open or switch to a new file:
      • Uses set-window-option automatic-rename. This tells tmux to resume its default behavior of naming the window based on the currently active process (e.g., it will correctly show zsh or ssh as soon as you exit Vim).
    • Wraps everything in an augroup with autocmd!. This is best practice; it ensures that if you source your .vimrc multiple times, the commands aren't duplicated and performance doesn't degrade.
    • Adds BufEnter, which ensures the window name updates immediately when you switch between internal Vim buffers (files), even if you never leave the terminal.

Disable Automatic Overwriting: Tmux often renames windows based on the running process. To ensure your manual or Vim-set names "stick," add this to your ~/.tmux.conf: 

set-option -g allow-rename off

Other Useful Commands

  • Renaming Sessions: Press <prefix_keys> b + $.
  • Renaming Panes: If you use tmux version 2.3+, you can rename individual panes with: 
tmux select-pane -T "pane-title"

Another Intro to the Power of Tmux

Here are the key features that make Tmux indispensable for many developers:

1. Persistent Sessions (The "Safety Net")

The most powerful feature is that your processes are decoupled from your terminal window.

  • Accidental Disconnects: If your SSH connection drops or your terminal emulator crashes, everything keeps running.
  • Detach & Reattach: You can start a long-running process (like a database migration or a big download), detach (Prefix + d), close your laptop, and reattach later to find everything exactly as you left it.

2. Context-Based Sessions

Instead of having 50 tabs in one terminal, you can create separate Sessions for different projects.

  • Project Isolation: You can have a "Backend" session with its own windows/panes and a "Frontend" session. Switching between them (Prefix + s) takes a second.
  • Named Workspaces: Start sessions with tmux new -s project_name to stay organized.

3. The "Zoom" Feature

If you have a four-pane layout but need to focus on one for a moment, you don't have to resize it.

  • Instant Maximize: Press Prefix + z to "zoom" the current pane to full screen. Press it again to snap back to your original layout.

4. Vim-Style Copy Mode

Tmux's Copy Mode allows you to navigate and search your terminal history using keyboard shortcuts instead of the mouse.

  • Search History: Enter copy mode (Prefix + [) and use / to search upward through your terminal output.
  • Vim Bindings: You can configure Tmux to use hjkl for navigation, making it feel like an extension of your text editor.

5. Scripting and Automation

You can automate your entire startup workflow using tools like tmuxinator or simple shell scripts.

  • One-Command Setup: With one command, you can launch a session that automatically opens your editor in pane 1, a server in pane 2, and a git log in pane 3.

6. Collaboration (Pair Programming)

Two or more people can attach to the same session.

  • Shared Terminal: Both users see the same screen and can type simultaneously. This is a common way to do remote pair programming without heavy screen-sharing software.

This requires that the users can access the same remote or virtual machine.

Workspace and State Manager

Since you already use Tmux daily for splitting, closing, and zooming panes, you’ve mastered the core mechanics. You are using it as a layout manager. The "missing out" feeling usually comes because Tmux is actually a workspace and state manager.

When you shift from thinking in panes to thinking in sessions, windows, and buffers, your productivity with Vim and Vimwiki will skyrocket.

Here is a concise, intermediate-to-advanced guide tailored to your daily workflow.

1. The Superpower: Sessions (Project Isolation)

Right now, you likely keep one Tmux session open and pack everything into it. If you work on a different project, or want to keep your Vimwiki completely separate from your scratchpad or servers, you should use Sessions.

Think of a session as an entirely separate virtual desktop that runs in the background.

Action Shortcut / Command When to use it
Create Named Session Outside Tmux: tmux new -s wiki Start a dedicated session just for your Vimwiki.
Detach from Session Prefix + d Leaving for the day? Detach. Your Vim and terminal states stay frozen in the background.
List & Switch Prefix + s This opens the interactive tree view you accidentally found! Use it to jump instantly between your wiki session and your dev session.
Kill Session Outside: tmux kill-session -t name Clean up an entire project workspace at once.

💡 Workflow Shift: Instead of having one massive Tmux session with 6 panes, have a wiki session (just Vimwiki), a coding session (your app code), and a sysadmin session. Switch between them instantly using Prefix + s.

QUESTION

I tried: 

tmux new -s hello

...and got: 

sessions should be nested with care, unset $TMUX to force

ANSWER

Ah, you hit a classic Tmux rite of passage!

That error message is Tmux gently putting its hands up and saying: "Whoa there, you are already inside a Tmux session. If you run that command, you are going to nest a Tmux session inside a Tmux session, which usually turns into a inception-style visual nightmare."

Because you are already active in a terminal pane managed by Tmux, running tmux new -s hello tries to spawn a whole new server instance inside your current window.

To create and use multiple sessions properly without nesting them, you have two excellent paths forward depending on whether you want to do it from inside your current session or outside of it.

Method 1: The "Inside Tmux" Way (Best Workflow)

If you are already sitting comfortably in your current Tmux session and suddenly think, "I want to spin up a new session called 'hello' for a different project," do not run the command directly in your terminal prompt. Instead, use Tmux’s Command Mode.

  • Press Prefix + : (Usually Ctrl + b + :). This will open a yellow or green command bar at the very bottom of your screen.
  • Type the following command and hit Enter: 
new -s hello
  • What happens: Tmux will instantly create the new session in the background and automatically switch your screen over to it.

Now, remember that interactive tree view shortcut you accidentally hit earlier? If you press Prefix + s, you will see both your original session and your new hello session in a list. You can use your arrow keys and hit Enter to hop back and forth between them seamlessly!

Method 2: The "Detach First" Way (The Safe Detour)

If you prefer using the standard command line to create sessions, you just need to step out of your current session first.

  • Detach from your current session by pressing Prefix + d.
  • You will be dropped back into your normal, regular computer terminal.
  • Now, run your command: 
tmux new -s hello
  • When you are done working in hello and want to leave it running in the background, press Prefix + d again to detach.

How to see what's running from the regular terminal

If you are completely detached from Tmux and sitting in your normal terminal, you can see a list of all your frozen background sessions by running: 

tmux ls

To jump back into your hello session from your normal terminal, just "attach" to it by typing: 

tmux a -t hello

2. The Middle Tier: Windows (Tabs for your Panes)

If a session is a virtual desktop, Windows are the tabs within that desktop. If your screen is getting cluttered with too many split panes, you need to spin up a new window. 

[ Session: Work ]
 ├── Window 0: Vimwiki (Full screen or simple split)
 └── Window 1: Development (Multiple split panes for logs/servers)
Action Shortcut Description
New Window Prefix + c Creates a fresh, blank screen (tab) within your current session.
Next / Prev Window Prefix + n or p Cycle through your tabs.
Select by Number Prefix + 0 to 9 Jump directly to a specific window.
Rename Window Prefix + , Rename the current tab (e.g., change "zsh" to "Logs" or "Server").

3. Power-User Pane Tricks (Moving & Swapping)

Since you already split and zoom panes, these three commands will give you total control over fluid layouts without needing to close and recreate things.

Convert a Pane into a Window (and vice versa)

  • Prefix + !: Converts your current active pane into its own brand-new, standalone window.
  • Prefix + ↱ (enter) (Join Pane): If you want to pull a window back into a split pane, you can type :join-pane -t :0 (to join it to window 0).

Rearrange on the Fly

  • Prefix + Spacebar: Cycle through Tmux's preset layouts (Even Horizontal, Even Vertical, Main Vertical, etc.). If your manual splits get messy, just bash the spacebar until it looks clean.
  • Prefix + Ctrl + o: Rotate the positions of all open panes. Perfect if you want your bottom pane to suddenly be your top pane.

4. Scrollback, Copy Mode, and Buffers

Have you ever run a command, it spit out 500 lines of error logs, and you couldn't scroll up to read it because mouse-scroll wasn't working? This is where Copy Mode comes in. Even better, it uses Vim keys by default if your config is set up for it.

Action Shortcut Description
Enter Copy Mode Prefix + [ Freezes the terminal screen and lets you move around using standard Vim keys (k/j to go up/down, Ctrl+u/Ctrl+d to page jump).
Start Selection Spacebar (or v in vim-mode) Begin highlighting text to copy.
Copy Selection Enter (or y in vim-mode) Copies the text into Tmux's internal clipboard (buffer) and exits copy mode.
Paste Selection Prefix + ] Paste whatever you just copied into any active Tmux pane.

5. Command Mode: The Escape Hatch

Just like Vim has : mode, Tmux does too. Pressing Prefix + : opens a command prompt at the bottom of the screen. This is incredibly useful for commands you don't use often enough to memorize a hotkey for.

  • Want to synchronize your panes so typing in one pane types in all of them?
    • Prefix + : + type setw synchronize-panes
  • Want to see a massive cheat sheet of every single shortcut currently mapped?
    • Prefix + ?
