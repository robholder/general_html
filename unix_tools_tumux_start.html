Tmux - Basics - Starting Tmux

  • YouTube - Learn Tmux

Cheat Sheet

  • Check for existing Tmux Sessions (Outside Tmux):
    • tmux ls - list sessions
  • Launch:
    • Outside Tmux:
      • tmux - new session
      • tmux new -s "<session_name>" - new session, with initial name
      • tmux a - open last accessed session
      • tmux a -t <session_name> - open particular session
  • Inside Tmux :
    • Sessions:
      • tmux ls - list sessions
      • <prefix_keys> then : (command mode), then new -s "<session_name> - create new session (from Tmux)
      • Switch between Sessions (inside Tmux):
        • <prefix_keys> then s - interactive list of sessions and windows ('up' 'down' or 'j' 'k' to choose)
        • <prefix_keys> then ( - Switch to previous session
        • <prefix_keys> then ) - Switch to next session
        • <prefix_keys> then L - (capital 'L') Switch to last session (toggles)
        • <prefix_keys> then : - command mode:
          • switch-t <#> e.g., switch-t 3 to switch directly to session number
      • <prefix_keys> then & - closes session even if multiple panes are present:
        • Confirmation dialog will appear in the status bar (does not destroy session permanently).
      • exit - closes panes, but in last remaining pane, closes and deletes the session:
        • No confirmation
    • Windows/Tabs:
      • tmux new-window - create a new window/tab
      • <prefix_keys> then c - create new window/tab
      • <prefix_keys> then , - rename window/tab
    • Switch Windows:
      • <prefix_keys> then p - previous window
      • <prefix_keys> then n - next window
    • Panes:
      • Split Panes:
        • <prefix_keys> then % - split window; one left, one right
        • <prefix_keys> then " - split window: one up, one down
      • Navigate Panes:
        • <prefix_keys> then ↑ - up one pane
        • <prefix_keys> then ↓ - down one pane
        • <prefix_keys> then ← - left one pane
        • <prefix_keys> then → - right one pane
      • Delete Panes:
        • exit - to close each pane (no confirmation):
          • exit on last pane will close session (and delete session from list)
        • <prefix_keys> then x - exit pane (with confirmation dialog)
        • <prefix_keys> then d - to close whole session (but not delete session panes)

Table of Commands

Here is your organized Tmux Cheat Sheet. I’ve categorized these by "Context" (where you are when you type them) and "Object" (Sessions vs. Windows vs. Panes) to make them easier to scan.

1. Management (From the Regular Shell)

Use these to start, find, or re-enter your work.

Action Command
List all active sessions tmux ls
New session (unnamed) tmux
New session (named) tmux new -s "name"
Attach to last used session tmux a
Attach to specific session tmux a -t name

2. Session Control (Inside Tmux)

Sessions are your high-level projects (e.g., "Vimwiki" vs. "Dev").

Action Command / Shortcut
Create new session <prefix> :new -s "name"
Interactive List <prefix> s (Use j/k to choose)
Toggle to last session <prefix> L (Capital L)
Next / Previous session <prefix> ) or <prefix> (
Jump to session # <prefix> :switch-t <#>
Detach (Keep alive in background) <prefix> d
Kill current session exit (in last remaining pane)

3. Windows & Tabs (Inside Tmux)

Windows are the "tabs" within a single project session.

Action Shortcut
Create new window <prefix> c
Rename current window <prefix> ,
Next / Previous window <prefix> n or <prefix> p
Jump to window # <prefix> 0-9
Close current window <prefix> & (With confirmation)

4. Panes & Splits (Inside Tmux)

Panes are the subdivisions of a single window.

Action Shortcut
Split Vertically (Side-by-side) <prefix> %
Split Horizontally (Top-Bottom) <prefix> "
Navigate between panes <prefix> + Arrow Keys
Zoom pane (Toggle Fullscreen) <prefix> z
Close current pane <prefix> x (With confirmation)
Close current pane exit (No confirmation)

Important Distinctions for your Flow:

  • Detach (d) vs. Kill (&/x/exit): Always use d for your Vimwiki. It puts the session in a "closet" so you can pull it back out later. The others stop your processes and lose unsaved work.
  • Prefix: Remember that you must release the |<prefix> keys before hitting the command key (like c or s).|

Tmux

What is it?

  • Terminal Multiplexer:
    • Terminals within terminals
  • Type tmux in a terminal to start.
  • You'll see a 'footer' at the base of the window.
  • Session is persistent, even if connections to remote terminal is broken.

Sessions

Detach From Session - Inside of Tmux

  • <prefix_key> then d:
    • Default <prefix_key> is 'b':
      • <Ctrl> b then d --> detach

The terminal will be back to normal - the tmux session is in the background.

The session will even survive a log-out of an SSH connection, and no longer connected to a server.

1: 1 window (created Wed May 31 17:44:37 2023) (attached)

To reattach to the most recent session:

  • tmux attach

Or:

  • tmux a

To attach to a specific session (target):

  • tmux attach -t <session_name>

Make a new session:

  • tmux

Create a 'Named' Session

  • tmux new -s "<session_name>"

Managing Sessions

In regular shell or tmux, you can see a list of sessions.

  • tmux list-sessions

Or:

  • tmux ls

Create session directly from a shell (OUTSIDE OF TMUX):

  • tmux

Connect to previous (most recent) session:

Windows/Tabs

Managing Windows & Tabs

The footer (status bar) shows the list of tabs open; as in this example:

  • [0] 0:first window* 1:second window- 2:third window

Creating A Window/Tab

  • tmux - first pane
    • [0] 0:bash

Inside 'tmux':

  • tmux new-window - new 'window'/'tab'
    • [0] 0:bash- 1:bash*

Or:

  • <prefix_keys> then c
    • [0] 0:bash- 1:bash*

Tabs are sometimes called whatever is running in them at the time.

Renaming Windows/Tabs

Navigate to window/tab:

  • <prefix_keys> then , (comma):
    • Then delete and type name's characters and return.

Or:

  • tmux rename-window <new_name> and return.

Note: See EXTRAS below

Switching Windows/Tabs

Previous & Next

  • <prefix_keys> then p - Previous window/tab
  • <prefix_keys> then n - Next window/tab

Deleting Window/Tabs

Quitting or killing a window/tab will stop all the running processes in that window.

To 'quit' the window/tab:

  • Navigate to particular window:
  • <prefix_keys> then &:
    • Tmux will ask for confirmation (y/n?)

Note: Windows/tabs are assigned a number in the order of creation. Deleting windows/tabs will not re-number the existing window id's.

Panes

Creating Panes

Start Tmux with tmux, then to create panes:

  • Vertical split (misnomer: one top, one bottom):
    • <prefix_keys> + <double_quote_key>
      • default is <Ctrl> b then <Shift> "
    • OR:
    • tmux split-window
    • tmux split-window -h
  • Horizontal split (one left, one right):
    • <prefix_key> then %
      • default is <Ctrl> b then 5
    • OR:
    • tmux split-window -v
Note: -h and -v seem reversed. This is because it refers to the layout of panes, not the divider.

Navigating Panes

  • <prefix_keys> & <arrow_direction>
    • <Ctrl> b and ↑
    • <Ctrl> b and ↓
    • <Ctrl> b and →
    • <Ctrl> b and ←

Deleting Panes

  • exit in each pane.

Or:

  • <prefix_keys> then x:
    • Confirm exit of pane.
    • e.g., <Ctrl> b then x
  • Last exit (last pane) will close Tmux (and leave no session in the list).
    • This permanently closes the Tmux session. It can't be re-attached.
  • Run <prefix_keys> then d to close last pane but keep 'session' in list.

Outside of Tmux: Connect to a session by reference:

  • tmux attach -t <session_number>
    • tmux attach -t 0
    • OR:
    • tmux a -t 0

Inside of Tmux: Direct Sessions Selection

  • <prefix_keys> then s
    • <Ctrl> b then s
  • ↑ - scroll up to see preview
  • ↓ - scroll down to see preview
  • Hit <Enter>/<Return> to accept

Inside of Tmux: Switching to Session

  • switch-client -t <session_name>

Or:

  • switch-client -t <Tab> - to cycle through.
    • Add the first letter or so then tab to narrow cycles.

Renaming Sessions

While in Tmux, and in a particular session:

  • tmux rename-session <my_session_name>
  • (Long names will be truncated in the status bar.)

Better method to rename:

  • <prefix_keys> then $
    • <Ctrl> b then $
  • Dialog opens in the status bar to rename the session.

EXTRAS

More On Creating Named Sessions

You can name a tmux session on creation from your shell using the -s flag with the new-session command (or its alias new).

Standard Command

To start a new named session, use: 

tmux new -s "SessionName"

Replace "SessionName" with whatever you'd like to call it (e.g., "wiki", "work", or "coding").

Attach or Create (Recommended)

As you start using tmux more, you'll likely want a command that attaches to an existing session if it’s already running, or creates it if it isn't. Use the -A flag for this: 

tmux new -A -s "SessionName"
  • If it exists: It jumps you right back into that session.
  • If it doesn't: It creates a new one with that name.

Why 'new'?

In the world of CLI tools, new is a subcommand. Think of tmux as the "app" and new as the specific "action" you want that app to perform.

Without it, tmux doesn't know if you want to create something, list something, or attach to something else.

Pro-Tip: Automatic Folder Naming

If you want to quickly name a session after the folder you are currently in, you can use this snippet in your terminal: 

tmux new -s "$(basename "$PWD")"

This is a favorite for many developers because it automatically keeps your sessions organized by project name.

Jump To Specific Windows

To jump to a specific window in tmux, you have several "Vim-speed" options depending on whether you want to use a number, a name, or a visual list.

1. Jump by Number (Fastest)

Since your windows are numbered 0 and 1, use the standard shortcut:

  • Prefix + 0 → Jumps to Vimwiki
  • Prefix + 1 → Jumps to Named Window

2. Jump by Name (Search)

If you have many windows and don't want to remember numbers, use the find-window command:

  • Prefix + f
  • Type "Named" (or just "N") and hit Enter. Tmux will instantly switch to the first window matching that string.

3. The Visual List (Interactive)

If you want to see all your windows and pick one:

  • Prefix + w
  • Use your Vim keys (j and k) to move the highlight, then hit Enter to jump into that window.

4. Toggle Between Last Two (Pro Move)

Since you specifically mentioned having two windows, you can flip-flop between them instantly:

  • Prefix + l (lowercase L)
  • This takes you back to the "Last" active window. Hit it again to jump back. This is perfect for toggling between your Vimwiki and a terminal.

5. Next and Previous

  • Prefix + n (Next)
  • Prefix + p (Previous)

Summary for your ~/.tmux.conf

If you find Prefix + 0 or Prefix + 1 annoying to reach, you can bind them to something closer to your home row: 

# Example: Jump to Window 0 (Wiki) with Prefix + w
bind w select-window -t 0
# Jump to Window 1 (Terminal) with Prefix + t
bind t select-window -t 1

Fastfetch

Disable Fastfetch in Tmux: 

# Only run fastfetch if NOT inside a tmux session
if [ -z "$TMUX" ]; then
  fastfetch
fi

Buffer/Filename from Vim

Configuring Vim to call Tmux window the name of the buffer/current filename.

Renaming tmux windows can be done manually through default shortcuts or automated via Vim configurations to keep your workspace organized by filename.

Manual Renaming (tmux Default)

rename-window "your-new-name"
  • Key Shortcuts:
  • <prefix_keys> then , (comma).
    • <Ctrl> b then ,
    • This opens a prompt in the status bar where you can type the new name and press Enter.
  • <prefix_keys> then : and edit the name
    • <Ctrl> b then : and edit the name

Automated Renaming from Vim

You can configure Vim to automatically update the tmux window name to the current filename. Add the following to your .vimrc: 

augroup tmux
  autocmd!
  if exists('$TMUX')
    " Set window name to current filename when opening or switching files
    autocmd BufReadPost,FileReadPost,BufNewFile,BufEnter * call system("tmux rename-window " . expand("%:t"))
    " Restore automatic renaming (e.g., to 'bash' or 'zsh') when leaving Vim
    autocmd VimLeave,FocusLost * call system("tmux set-window-option automatic-rename")
  endif
augroup END

This block has:

  • Dynamic Renaming:
    • Update the window name every time you open or switch to a new file:
      • Uses set-window-option automatic-rename. This tells tmux to resume its default behavior of naming the window based on the currently active process (e.g., it will correctly show zsh or ssh as soon as you exit Vim).
    • Wraps everything in an augroup with autocmd!. This is best practice; it ensures that if you source your .vimrc multiple times, the commands aren't duplicated and performance doesn't degrade.
    • Adds BufEnter, which ensures the window name updates immediately when you switch between internal Vim buffers (files), even if you never leave the terminal.

Disable Automatic Overwriting: Tmux often renames windows based on the running process. To ensure your manual or Vim-set names "stick," add this to your ~/.tmux.conf: 

set-option -g allow-rename off

Other Useful Commands

  • Renaming Sessions: Press <prefix_keys> b then $.
  • Renaming Panes: If you use tmux version 2.3+, you can rename individual panes with: 
tmux select-pane -T "pane-title"
