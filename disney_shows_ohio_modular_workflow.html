Modular Workflow Notes

Task Modules

  • sequence_127p0 <- Foundation Lighting
  • scene_Sc001 <- Keyshot Lighting
  • packet_Sc001PacketB <- Packet Keyshot Lighting
  • Shots:
    • shot_001p00 <- Shot Lighting
    • shot_002p00
    • shot_003p00
NOTE: p is a placeholder for . (point).

Internals

  • Each node contains a whole light setup.
  • Each module layers on the next

Benefits

  • Maximizes continuity
  • Allows changes at sequence/packet level:
    • Work in parallel with leads and short artists
    • Propagates
  • Allows rollback from shot level changes

Inheritance

  • Data gets inherited as you move down the stream ('overriding')
  • It's a 'pull' system

Houdini

Nodes

  • Setup:
    • Importing:
      • geo
      • usd
      • framerange
  • Sequence:
    • Light - setup up as usual
  • Shot level:
    • The light created in Sequence shows up in the Lightbox params
    • Edit as normal in the DLight style params
    • Red dots will appear to indicate overrides

Locking

Keep upstream nodes locked (Sequence, etc.) so that you continue to inherit future changes. Unlocking breaks the live-link.

Ryan North says that he unlocked the sequence node to 'steal' setups, then relocked to proceed in his packet/shot nodes. (I presume switching OFF the relevant lights in his packet.)

This would keep his live updates from the sequence, but he could make packet/shot setups using the stolen lights.

Clear Edits

  • RMB on item in lightbox, and clear all edits
