Return a Value or Update Instance Variable?

The choice between returning a value and updating an instance variable depends on the method's purpose.

Return a Value

Return a value when a method's primary job is to compute and provide a result based on the object's current state or its inputs, without changing the object itself. This is a functional approach, where you're asking the object for an answer. Think of it like a calculator: you input numbers and operations, and it returns the result without changing the calculator's internal state.

Example:

A method that calculates the total price of items in a shopping cart. The method returns the total price, but doesn't change the list of items in the cart. 

class ShoppingCart:
    def __init__(self, items):
        self.items = items

    def calculate_total(self):
        total = sum(item.price for item in self.items)
        return total

In this case, calculate_total simply gives you information. It doesn't modify the cart.

Update an Instance Variable

Update an instance variable when a method's purpose is to change the object's state. This is a procedural approach, where you're telling the object to perform an action that modifies its properties. This is often done for methods that represent an action or a side effect.

Example:

A method that adds a new item to the shopping cart. The method updates the cart's items list, which is an instance variable. 

class ShoppingCart:
    def __init__(self):
        self.items = []

    def add_item(self, item):
        self.items.append(item)
        # No return value needed, as the action is the modification itself.

The method add_item is an action that changes the object's state.

The Hybrid Approach

Sometimes, a method will do both: it will change the state AND return a value. This can be useful, but you should do it with a clear purpose. A common example is a method that performs an action and then returns a status or the new state of the object.

Example:

A method that adds an item and returns True or False based on whether the operation was successful. 

class ShoppingCart:
    def __init__(self, max_items):
        self.items = []
        self.max_items = max_items

    def add_item(self, item):
        if len(self.items) < self.max_items:
            self.items.append(item)
            return True  # Indicates success
        return False  # Indicates failure

Here, the method updates the items list if the condition is met and also returns a boolean to inform the caller about the outcome.

The Rule of Thumb

  • Is the method a "question"? (e.g., get_user_name(), calculate_area()). If so, return a value. These methods are often called getters or pure functions.
  • Is the method a "command" or "action"? (e.g., set_user_name(), add_item()). If so, update an instance variable. These methods are often called setters or mutators.

By following this pattern, you create methods with predictable behaviors, making your classes easier to understand and use.
