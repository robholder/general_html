GNU Stow - Dotfile Management

Install on MacOS

brew install stow

Verify with... 

stow --version

Basic Setup for Dotfiles

Once installed, a common workflow for managing configuration files involves:

  • Create a Directory: mkdir ~/.dotfiles.
  • Organize Configs: Move your existing configs (e.g., .zshrc) into subdirectories within ~/.dotfiles.
  • Symlink: Run stow . from inside your dotfiles directory to symlink them back to your home folder.

Remove Stow Setup

To reverse the process and "un-link" your dotfiles from your home directory, you use the Delete flag.

The Reverse Command

Navigate to your ~/.dotfiles folder and run: 

stow -D <folder_name>

...or 

stow -D .

What this does:

  • Removes the Symlink: It finds the symbolic link in your home directory (e.g., ~/.zshrc) that points to your dotfiles folder and deletes it.
  • Safety First: Stow is smart—it will not delete the actual file inside your ~/.dotfiles folder. It only breaks the connection.
  • Folder Cleanup: If Stow created any empty parent directories to house those links, it will try to clean those up as well.

How to fully "Revert" to local files:

If you want to move the files back to your home directory permanently (no longer using Stow):

  • Un-stow: stow -D wezterm
  • Move them back: cp -r ~/.dotfiles/wezterm/.config/wezterm ~/.config/
  • Verify: Run ls -la ~/.config/wezterm to ensure they are now regular files and not "arrows" pointing elsewhere.

Pro-Tip: The "Restow"

If you just made a mistake or renamed a file and want to refresh the links without deleting everything first, use: 

stow -R <folder_name>

This performs a Delete and an Install in one quick motion.
