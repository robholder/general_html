Swift - Navigation & Tab

To built a structural navigation layout in SwiftUI, we actually combine two distinct system components that work beautifully together:

  • TabView: Handles the bottom bar switching between completely separate feature screens.
  • NavigationStack: Handles the top title, navigation buttons, and drilling down into deeper content views inside a specific tab.

Here is your comprehensive, scannable reference guide for organizing this structural layout.

1. The Global Shell Layout (TabView + NavigationStack)

The golden architectural rule in SwiftUI is: TabView wraps NavigationStack. You almost never put a TabView inside a NavigationStack. Each tab gets its own independent navigation history. 

import SwiftUI


struct HomeView: View {
    var body: some View {
        ScrollView {
            Text("Main Dashboard Content")
        }
        .navigationTitle("Dashboard")
        // Options: .automatic, .large (default scroll behavior), .inline (small centered text)
        .navigationBarTitleDisplayMode(.large)
    }
}

struct ContentView: View {
    var body: some View {
        TabView {
            // TAB 1: Home Screen (Has its own Navigation Stack)
            NavigationStack {
                HomeView()
            }
            .tabItem {
                Label("Home", systemImage: "house")
            }
            
            // TAB 2: Settings Screen (Independent Navigation Stack)

            .tabItem {
                Label("Settings", systemImage: "gear")
            }
        }
    }
}

    ContentView()
}

2. NavigationStack Titles & Top Buttons

When a view is placed inside a NavigationStack, it gains access to modifiers that let it control the parent navigation bar's title and actions.

Adding the Title

Use .navigationTitle() to assign a string. You can control its visual scale using .navigationBarTitleDisplayMode().

Swap out the HomeView struct: 

struct HomeView: View {
    var body: some View {
        ScrollView {
            Text("Main Dashboard Content")
        }
        .navigationTitle("Dashboard")
        // Options: .automatic, .large (default scroll behavior), .inline (small centered text)
        .navigationBarTitleDisplayMode(.large) 
    }
}

Adding Top Bar Options (Buttons)

Use the .toolbar modifier combined with ToolbarItem or ToolbarItemGroup to place items cleanly in the navigation bar. 

struct HomeView: View {
    var body: some View {
        List {
            Text("Content Items")
        }
        .navigationTitle("Dashboard")
        .toolbar {
            // 1. Primary Action (Top Right on iOS)
            ToolbarItem(placement: .primaryAction) {
                Button {
                    print("Create new item tapped")
                } label: {
                    Image(systemName: "plus")
                }
            }
            
            // 2. Secondary/Confirmation Actions (Grouped on the Top Right)
            ToolbarItemGroup(placement: .navigationBarTrailing) {
                Button {
                    print("Notifications tapped")
                } label: {
                    Image(systemName: "bell")
                }
                
                Button {
                    print("Profile tapped")
                } label: {
                    Image(systemName: "person.circle")
                }
            }
            
            // 3. Leading Action (Top Left on iOS)
            ToolbarItem(placement: .navigationBarLeading) {
                Button("Edit") {
                    print("Edit mode toggled")
                }
            }
        }
    }
}

3. Scannable Modifier Reference

Always apply these modifiers to the content view inside the navigation stack, never to the NavigationStack container itself.

Modifier Common Placements / Arguments Purpose
.navigationTitle("...") Plain Text String Sets the primary text header of the bar.
.navigationBarTitleDisplayMode(_:) .large, .inline, .automatic Changes header layout from massive left-aligned to small centered.
.toolbarBackground(_:for:) .visibility / .automatic, .visible, .hidden Forces the navigation bar background background translucent, visible, or clear.
.toolbarColorScheme(_:for:) .dark, .light Forces status bar and bar item text colors independently of user mode.

4. Modern Drilling (Navigating to Subviews)

The modern approach to push a new view onto the screen uses navigationDestination. It decouples the tap action from the target view by matching data types. 

struct DetailItem: Identifiable, Hashable {
    let id = UUID()
    let name: String
}

struct HomeView: View {
    let items = [DetailItem(name: "Report A"), DetailItem(name: "Report B")]
    
    var body: some View {
        List(items) { item in
            // 1. NavigationLink just broadcasts the data value when tapped
            NavigationLink(item.name, value: item)
        }
        .navigationTitle("Reports")
        // 2. This interceptor catches the specific data type and returns the proper destination view
        .navigationDestination(for: DetailItem.self) { selectedItem in
            ReportDetailView(item: selectedItem)
        }
    }
}

struct ReportDetailView: View {
    let item: DetailItem
    var body: some View {
        Text("Detailed display for \(item.name)")
            .navigationTitle(item.name) // Subview gets its own title & automatic back button!
    }
}
