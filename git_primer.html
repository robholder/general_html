SSH Keys

  • Generate a public/private rsa key 
    
ssh-keygen -t rsa -b 4096 -C "email@example.com"
# Enter name of key: testkey
# The file will be in ~/.ssh/id_rsa/
# leave 'enter passphrase' blank
  • testkey.pub will be the one you upload to Github ('pub' is 'public').
  • Copy and paste the whole key string:
    • pbcopy < ~/.ssh/id_rsa/testkey.pub
    • Paste in Github account:
      • Settings:
        • SSH & GPG keys
        • New SSH key:
          • Paste key into area
  • Depending on Operating System, add ssh-agent settings (so git can find the keys).

https://docs.github.com/en/authentication/connecting-to-github-with-ssh/generating-a-new-ssh-key-and-adding-it-to-the-ssh-agent

Git in a Terminal

Git is usually installed in Linux and MacOS

Clone from a Github Repository

  • To Pull Repo from Github:
  • Get URL/SSH key from Github:
    • "Clone with SSH":
      • copy SSH key 
        git clone <SSH key>
# e.g. git clone git@github.com:robholder/general.git
cd $
  • Git will install a .git/ directory. 
    ls -la
  • Make any changes to the files.
  • Show changes/status 
    git status
  • Add files:
    • named
    • or . (all) 
      git add <file>
# or
git add .
  • Commit files:
    • add message with -a "blah" 
      git commit -m "Added my lovely file." -m "Descript string"
[[#]] Second -m is a description, the first is the commit string.

If you want to edit using a specific editor (like Vim in the terminal), set it like so in a shell: 

# Globally:
git config --global core.editor "vim"
# or locally in the current repo:
git config core.editor "vim"

This is a global config statment to use MacVim on MacOS: 

git config --global core.editor "mvim -f"
  • The -f flag makes MacVim stay in the foreground, which is necessary for Git to wait until you finish editing.

If you want to test the commit with a designated editor, try this (which commits even though there are no changes): 

git commit --allow-empty

Git Local set up

  • Creat dir
  • cd into dir
  • create README.md 
    
git init
git add README.md
git commit -m "created README" -m "description"
  • In Github:
    • Create empty new repository (same name as directory)
    • Copy SSH string 
      
git remote add origin <SHH string>
git remote -v
# shows target
# -u fixes origin ('upstream')
git push -u origin master

# in future, just type:
git push

Git Branching

  • The Master Branch is the main branch:
  • branches can be forked to develo
  • Master Branch -> commit -> commit -> Merge
    • Feature Branch -> commit -> 

git branch
  • Checkout a branch: 

git checkout -b feature-readme-instruction
#>> Switched to a new branch 'feature-readme-instruction'
  • Running 'git branch' again will list the branches
  • The 'starred' branch is the current 

git branch

#>> * feature-readme-instructions
#>>   master
(END)
  • Switch to another branch: 

git checkout master
#>> Switched to branch 'master'
#>> Your branch is up to date with 'origin/master'.
  • Switch back: 

git checkout feature-readme-instructions
  • Git Status: 

git status

#>> modified: README.md
  • Git add: 

git add README.md
git -commit -m "updated readme"
  • Git diff 

git diff feature-readme-instructions
  • Git merge 

git merge feature-readme-instructions

..or more commonly: 


git checkout feature-readme-instructions
git status
git push
git push -u origin feature-readme-instructions
  • Pull request:
  • Go to Github and click on 'Compare & Pull Request'

Undoing

  • Undoing an add:
    • git reset README.md
  • Undoing an add on last add:
    • git reset HEAD~1
  • git log:
    • will show commits
    • copy hash string and to go back:
      • git reset <hash you copied>
