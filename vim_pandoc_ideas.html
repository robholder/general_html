Pandoc

Below is a strategy that lets you add Pandoc and its integrations step by step—all in a manner that won’t “lock in” any system changes and will let you back out gracefully if you decide to try something different.

Install Pandoc Safely

On macOS: The simplest, most reversible way is to use Homebrew. Open your Terminal and run: 

brew install pandoc
  • Why Homebrew? It keeps installed packages isolated from system files, and you can always remove Pandoc with: 
brew uninstall pandoc
  • Verification: Check if it’s installed correctly by running: 
pandoc --version

Set Up a Python Environment With pypandoc (Optional)

If you’re planning to leverage Pandoc from Python—for custom scripts, automation, or web apps—it’s a good idea to do so in an isolated environment:

  • Create a Virtual Environment: 
python3 -m venv pandoc_env

source pandoc_env/bin/activate
  • Install pypandoc: 
pip install pypandoc

This wrapper allows your Python programs to invoke Pandoc easily. Even if you decide later to remove the virtual environment, nothing outside of it is affected.

Integrating LaTeX

Integrating LaTeX into your workflow lets you tap into advanced typesetting and produce documents with refined typography. Here’s how you can extend the strategy to leverage LaTeX:

Install a LaTeX Distribution

Before anything else, make sure you have a complete LaTeX distribution on your system. On macOS, a popular option is MacTeX.

  • Installation via Homebrew Cask (macOS): 
brew install --cask mactex
# This takes a loooong time...
# Needs password (sudo)
  • Verification: Confirm that commands like pdflatex or xelatex are available in your terminal by running: 
pdflatex --version

This ensures that when Pandoc generates a LaTeX file, you can compile it into a final PDF.

Uninstall LaTeX

To uninstall MacTeX that was installed via Homebrew, you can use the following command: 

brew uninstall --cask mactex

If you want to remove all associated configuration files and settings, you can use: 

brew uninstall --cask --zap mactex

Additionally, MacTeX installs TeX Live, which is stored in /usr/local/texlive. You may need to manually remove this directory if you want to free up space. You can do this by running: 

sudo rm -rf /usr/local/texlive

MacTeX also installs GUI applications in /Applications/TeX, which you can remove by dragging them to the trash.

Convert Your Vimwiki Files to LaTeX

Pandoc can serve as a bridge between your Vimwiki notes and LaTeX. Assuming your notes are in Vimwiki format (with a .wiki extension), you can convert them directly into a LaTeX document.

  • Basic Conversion Command: 
pandoc --from=vimwiki -o output.tex your_note.wiki
  • --from=vimwiki tells Pandoc to interpret your Vimwiki-specific syntax.
  • This generates a LaTeX source file (output.tex) that you can inspect or tweak further.
  • Direct PDF Generation (Using LaTeX Under the Hood): If you prefer a one-step process, the following command converts the .wiki file directly to PDF: 
    pandoc --from=vimwiki -o your_note.pdf your_note.wiki

    Under the hood, Pandoc converts your notes to LaTeX and then uses your installed LaTeX engine (e.g., pdflatex) to compile the PDF.

    Customize the LaTeX Output with Templates

    LaTeX’s power lies in its typographic control. Pandoc uses a default LaTeX template, but you can supply your own custom template to control fonts, margins, headers, and more.

    • Creating a Custom Template: Save your custom LaTeX template (let’s say as mytemplate.tex). This might include your preferred document class settings, geometry, font packages, and even custom macros for things like special formatting or math typesetting.
    • Using the Custom Template with Pandoc: When converting, specify your template: 
    pandoc --from=vimwiki --template=mytemplate.tex -o your_note.pdf your_note.wiki

    This tells Pandoc to format the output according to your custom style, giving you a typography-centric document.

    Automate Within Vim

    Keep your workflow seamless by integrating these steps directly into Vim. Add a mapping to your .vimrc that saves your current Vimwiki file, converts it using Pandoc into LaTeX (or directly into a PDF), and then (if desired) compiles it.

    • Example Vim Mapping: 
      " Convert current Vimwiki note (in 'vimwiki' syntax) to a LaTeX PDF using Pandoc
nnoremap <leader>wl :w<CR>:!pandoc --from=vimwiki --template=mytemplate.tex % -o %:r.pdf<CR>
    • <leader>wl: Choose a keybinding that feels natural.
    • % represents the current file, and %:r extracts its filename without the extension.
    • This mapping saves your file and runs the Pandoc command with your custom LaTeX template.

    Testing and Rollback

    • Incremental Testing: Start by converting a single, small note. Inspect the generated .tex file to be sure that your LaTeX code is coming out as expected. If you need to adjust your template, do so before processing your entire library.
    • Version Control: Keep your original Vimwiki notes backed up using Git or another version control system, so you can always revert if something goes awry during conversion.
    • Isolated Environments: Since you’re using Homebrew for Pandoc and MacTeX for LaTeX, uninstalling them later is straightforward if needed. Also, if you integrate through Python using pypandoc, you can do so in a virtual environment to keep things contained.

    Summary

    • Install MacTeX to get a full LaTeX environment.
    • Use Pandoc with the --from=vimwiki flag to convert your files to a LaTeX source or directly to a PDF.
    • Customize Your Output with a custom LaTeX template to achieve refined typography.
    • Automate within Vim through mappings that execute your Pandoc conversions, keeping your workflow efficient.
    • Test Incrementally and use version control to ensure you can roll back or adjust without risk.

    With these steps, you’ll have a robust, reversible strategy for converting your Vimwiki notes into typography-rich LaTeX documents. Would you like more details on creating custom LaTeX templates or setting up further automation with Python or Vim?

    What Is a LaTeX Template in Pandoc?

    A LaTeX template is a plain text file (typically with a .tex extension) that contains both raw LaTeX code and special placeholders (or variables) that Pandoc replaces with content from your source file or metadata. For example:

    • \(title\) – gets replaced with the document title.
    • \(author\) – gets replaced with the author’s name.
    • \(body\) – where the converted content goes.
    • \(date\) – gets replaced with the date.

    Pandoc comes with a default LaTeX template, but you can create your own to adjust things like margins, font sizes, header styles, section numbering, and more.

    Breaking Down a Simple Template

    Below is an oversimplified example of what a Pandoc LaTeX template might look like: 

    \documentclass[12pt]{article}
\usepackage[utf8]{inputenc}
\usepackage{geometry}
\geometry{margin=1in}

% Custom styling and packages
\usepackage{amsmath, amssymb}
\usepackage{graphicx}

% Document metadata
\title{$title$}
\author{$author$}
\date{$date$}

\begin{document}
\maketitle

$body$

\end{document}

    Key Parts:

    • Preamble: This section (before \begin{document}) loads packages and sets up document settings (font size, margins, math packages, etc.).
    • Metadata Area: Commands like \title{}, \author{}, and \date{} include placeholders (\(title\), etc.) that Pandoc replaces with your document’s metadata.
    • Document Body: \(body\) is where Pandoc inserts your converted content from the source file.

    When you compile with Pandoc and specify this template, your note’s content “flows into” the layout defined here.

    Rolling the Template into Your Workflow

    Step-by-Step Strategy:

    Create Your Custom Template:

    • Start with Pandoc’s default: You can retrieve the default template with: 
    pandoc -D latex > default.latex

    This command outputs the default LaTeX template, which you can then use as a base.

    • Modify It: Open the file in your favorite editor (e.g., Vim) and adjust it. Add packages you like or change the document class to fit the typography you’re after. Save this file as, for example, mytemplate.tex.

    Test the Template:

    • Run a conversion using a simple file. Suppose you have a Vimwiki note in .wiki format: 
      pandoc --from=vimwiki --template=mytemplate.tex -o output.pdf your_note.wiki
    • If Pandoc converts your file to LaTeX and then to PDF, inspect the PDF to see if the layout and styles match your expectations.
    • Adjust the template as needed.

    Integrate with Vim:

    • Add a mapping in your .vimrc to streamline the process. For example: 
      " Map <leader>wl to convert current Vimwiki note to PDF using your LaTeX template
nnoremap <leader>wl :w<CR>:!pandoc --from=vimwiki --template=mytemplate.tex % -o %:r.pdf<CR>
    • Now, whenever you run <leader>wl in Vim, your current note is saved and converted into a PDF with the custom typography defined in your mytemplate.tex.

    Expand Your Workflow with Python (Optional):

    • If you need additional automation or processing, you can call Pandoc from Python using pypandoc. This gives you the ability to create scripts that, for instance, batch-convert multiple notes or integrate further processing.
    • Create a small Python script that loops through your Vimwiki files and calls Pandoc with your custom template.

    Keep Things Modular and Reversible:

    • Version Control: Keep your template file under version control. This way, you can experiment and always revert changes if something doesn’t work as expected.
    • Isolated Installations: Since Pandoc is installed using Homebrew and optional Python packages in a virtual environment, you don’t risk system stability.

    Why Use Custom LaTeX Templates?

    • Fine-Tuned Typography: Customize fonts, margins, headers, and footers to get professional-looking documents without changing your content.
    • Consistent Layout: All your notes will adhere to a unified style, which is especially useful if you plan to print or share them.
    • Flexibility: Easily modify the template to adapt to different types of documents (e.g., academic papers vs casual notes) by changing the template and re-running Pandoc.

    Summary

    • LaTeX Template: A blueprint that uses LaTeX code with placeholders (like \(title\), \(body\)) for content generated by Pandoc.
    • Workflow Integration:
      • Create or modify a template (start with pandoc -D latex).
      • Test the conversion of a .wiki file to PDF using your template.
      • Integrate the command into Vim mappings to streamline the process.
      • Optionally, expand the workflow with Python scripts for additional automation.
    • Version Control & Modularity: Keep your documents and templates under version control to easily backtrack or remove changes if needed.

    By following these steps, you can build a robust workflow that leverages the fine typography of LaTeX while keeping your Vimwiki notes as your primary source.

    Pandoc's Native Formats

    Pandoc’s Native Formats: You can check which input formats Pandoc supports by running: 

    pandoc --list-input-formats

    You can change to markdown: 

    let vimwiki_list = [{'path': '~/vimwiki/', 'syntax': 'markdown', 'ext': '.md'}]

    This won’t automatically convert existing .wiki files, but it sets the stage for future notes.

    • Use a Conversion Script: There are community scripts and filters that can translate Vimwiki’s markup into standard Markdown. Run such a script on your legacy notes, and then use Pandoc on the resulting Markdown files. For example: 
      vimwiki2markdown your_note.wiki > your_note.md
pandoc --from=markdown -o output.pdf your_note.md

    (You may need to search for or write a simple converter that handles your particular Vimwiki syntax quirks.)

    You can find the vimwiki2markdown utility on its GitHub repository. It's a Vim plugin designed to convert Vimwiki files into Markdown format. The repository includes installation instructions and usage details.

    Build a Custom Pandoc Filter

    If you prefer not to perform a separate conversion step manually, you could write or use an existing Pandoc filter that pre-processes vimwiki syntax to something Pandoc can understand. This filter would act on the intermediate representation that Pandoc uses and convert vimwiki-specific constructs into Markdown or another supported format.

    • Pandoc Filters: Filters can be written in Lua (which is now supported directly in Pandoc), Python, or Haskell. For instance, a Lua filter could scan for Vimwiki-style links (like Link Title) and convert them into standard Markdown links ([Link Title](Link_Title)).
    • Experiment First: This route is more advanced, so it’s a good idea to start with a small sample file and iterate until conversion works correctly.

    Integrate Into Your Workflow

    Once you have a reliable way to go from Vimwiki syntax to a supported format, you can roll that into your automation pipeline. For example:

    • Option A: Two-Step Process (Conversion, then Pandoc):
      • Use a script or export functionality to convert your .wiki file into Markdown (e.g., your_note.md).
      • Run Pandoc on the resulting Markdown file to produce a PDF: 
    pandoc --from=markdown --template=mytemplate.tex -o your_note.pdf your_note.md
    • Option B: Custom Mapping in Vim:
      • If you’re using Vim, you could set up a mapping that calls your conversion script and then runs Pandoc. For example, in your .vimrc: 
        " Convert Vimwiki note to Markdown then to PDF using Pandoc
nnoremap <leader>wl :w<CR>:!./vimwiki2md % > %:r.md && pandoc --from=markdown --template=mytemplate.tex -o %:r.pdf %:r.md<CR>

    Here, vimwiki2md would be your script (or command) that converts a .wiki file to Markdown. You can modify this mapping to suit your workflow.

    Summary

    • Pandoc Doesn’t Natively Support Vimwiki: The error indicates that the input format vimwiki isn’t recognized.
    • Workaround Options:
      • Convert your Vimwiki files to Markdown first (via export or conversion scripts).
      • Write a Pandoc filter to pre-process Vimwiki syntax on the fly.
    • Workflow Integration: Once you have reliable conversion, integrate it into your Vim mappings or Python scripts so you can easily produce print-ready documents with LaTeX-controlled typography.

    This approach gives you the flexibility to experiment incrementally while preserving your notes and allowing for quick rollback if needed.

    Integrate Pandoc With Vimwiki

    Since you’re using Vimwiki with the .wiki syntax, you have two main options:

    A. Convert Vimwiki Files Directly

    • Pandoc’s Vimwiki Support: Recent versions of Pandoc have improved support for Vimwiki’s format. Test it by converting a sample file: 
    pandoc --from=vimwiki -o output.pdf your_note.wiki
    • Tweak as Needed: If you find quirks or conversion issues (for instance with internal links), you can write a small preprocessor or Pandoc filter to handle Vimwiki-specific syntax. Try one conversion at a time with a test file before applying it to all your notes.

    B. Convert Vimwiki to HTML Then to LaTeX/PDF

    Since Vimwiki can export to HTML, another pathway is to:

    • Export your note to HTML using Vimwiki.
    • Convert the HTML to PDF using Pandoc: 
    pandoc --from=html -o note.pdf your_note.html

    This approach works well if your Vimwiki export creates clean, semantically structured HTML.

    Automate Within Vim

    Create convenient Vim mappings to streamline conversion. For example, add something like this to your .vimrc: 

    " For Vimwiki notes using vimwiki syntax converted directly by pandoc:
nnoremap <leader>wp :w<CR>:!pandoc --from=vimwiki % -o %:r.pdf<CR>

    Explanation:

    • :w saves your current file.
    • % inserts the filename.
    • %:r extracts the “root” filename (without extension) for the naming of your output.

    Test the mapping with a few notes to ensure the conversion behaves as expected.

    Test Incrementally & Prepare to Roll Back

    • Modular Installations: By installing Pandoc with Homebrew and Python packages in virtual environments, you keep changes compartmentalized.
    • Verify Each Step: Start with one sample file. Convert from Vimwiki to your target (Markdown, HTML, or PDF) and inspect the output. Adjust your command-line options or mappings as needed.
    • Backup: Always keep a backup of your original notes before running conversions. Use version control (like Git) for your Vimwiki directory so you can always backtrack if a conversion script misbehaves.
    • Easy Uninstallation: Homebrew allows you to remove Pandoc easily, and deleting the virtual environment simply removes its directory without system-wide impact.

    Keep Documentation Handy

    • Pandoc’s User Guide: It’s extensive and helpful for troubleshooting and customizing templates.
    • Vimwiki Docs: They may also have guides for exporting and integrating with external tools.
    • Python’s pypandoc Docs: They show examples of how to call Pandoc from Python scripts.

    By following this plan, you can add Pandoc into your Vimwiki and Python workflows in small, testable steps. This modular approach lets you experiment without “breaking” your system and gives you a clear path to backtracking if a particular integration doesn’t suit your needs.

    Vimwiki Not Working

    " _I tried the test conversion; from Vimwiki to PDF, with: pandoc --from=vimwiki -o output.pdf your_note.wiki ...except with my own filenames. I got this error: Unknown input format =vimwiki_ "

    The error

    Unknown input format =vimwiki

    Indicates that your installed Pandoc doesn’t recognize vimwiki as a native input format. In other words, even though you’re working with Vimwiki notes (with a .wiki extension), Pandoc’s built-in list of input formats does not include a reader for vimwiki syntax.

    What Does This Mean?

    Pandoc’s Native Formats: You can check which input formats Pandoc supports by running: 

    pandoc --list-input-formats

    If you don’t see vimwiki listed there, then Pandoc won’t know how to handle files in that format. (Pandoc supports formats like Markdown, MediaWiki, or DokuWiki, but vimwiki is not enabled by default.)

    Error Explanation: Using --from=vimwiki tells Pandoc to parse your file using a vimwiki reader. Since it's unknown, it throws the error.
