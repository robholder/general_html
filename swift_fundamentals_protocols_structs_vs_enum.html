Swift - Protocols - Structs vs Enums

Protocols in Structs vs. Enums

In Swift, Protocols are universal contracts. A protocol declares a checklist of properties and methods that a type must implement, but it does not care whether the type adopting it is a struct, a class, or an enum.

While structs and enums can conform to the exact same protocols to gain superpowers, they satisfy those contracts differently due to how they manage memory.

1. Where Structs and Enums Overlap

You can apply the exact same protocol to both a struct and an enum. For example, if you want SwiftUI to be able to loop over an array of items dynamically, both types can conform to the system's Identifiable protocol: 

// A Struct satisfying Identifiable
struct ResortStruct: Identifiable {
    let id: String // Satisfies the protocol contract
    let name: String
}

// An Enum satisfying Identifiable
enum ResortEnum: String, Identifiable {
    case whistler, chamonix
    
    // Satisfies the protocol contract using its underlying raw string
    var id: String { self.rawValue } 
}

2. Where Structs and Enums Diverge

Because enums cannot store arbitrary data values in memory the way fields inside a struct do, their implementations deviate in two major architectural areas:

Difference A: Stored Properties vs. Computed Properties

  • Structs can satisfy a protocol property requirement using a standard stored property (a variable allocated in physical memory).
  • Enums are strictly forbidden from holding stored properties (outside of their primary raw value). Therefore, an enum must always satisfy a protocol property contract using a computed property (a variable with a dynamic getter block). 
protocol NamedItem {
    var displayName: String { get }
}

// STRUCT IMPLEMENTATION (Stored)
struct User: NamedItem {
    let displayName: String // Stored directly in memory
}

// ENUM IMPLEMENTATION (Computed)
enum Status: NamedItem {
    case active, inactive
    
    // Must compute the value dynamically based on the current case
    var displayName: String {
        switch self {
        case .active: "Active User"
        case .inactive: "Deactivated"
        }
    }
}

Difference B: State Mutation (mutating)

If a protocol requires a method that modifies the value of the object itself, it uses the mutating keyword.

  • A struct method mutates state by modifying an individual internal variable field (self.count += 1).
  • An enum method mutates state by completely swapping out the active case of the object entirely. 
protocol Toggleable {
    mutating func toggle()
}

// STRUCT MUTATION
struct SwitchStruct: Toggleable {
    var isOn = false
    
    mutating func toggle() {
        isOn.toggle() // Modifies an internal Boolean property
    }
}

// ENUM MUTATION
enum SwitchEnum: Toggleable {
    case on, off
    
    mutating func toggle() {
        // Completely overwrites the entire state with a different case
        self = (self == .on) ? .off : .on 
    }
}
