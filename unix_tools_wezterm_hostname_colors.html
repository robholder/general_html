Setting Colors Based On Hostname

In WezTerm's lua settings, is there a way to specify a color assignment based on the host name?
UNTESTED

Yes, you can use the built-in wezterm.hostname() function to dynamically assign a color scheme or specific colors based on the machine running the terminal.

Implementation in wezterm.lua

You can use a simple if/else block or a Lua table to map hostnames to your preferred themes. 

local wezterm = require 'wezterm'
local config = wezterm.config_builder()

local hostname = wezterm.hostname()

if hostname == 'work-laptop' then
    config.color_scheme = 'Tokyo Night'
elseif hostname == 'home-desktop' then
    config.color_scheme = 'Catppuccin Macchiato'
else
    config.color_scheme = 'AdventureTime'
end

return config

Important Considerations

  • Local vs. Remote: The wezterm.hostname() function returns the name of the system running the WezTerm process. If you are using wezterm ssh or a multiplexer domain, the color scheme is typically controlled by the configuration file on the remote server.
  • Dynamic SSH Coloring: If you want your local WezTerm window to change colors automatically when you SSH into a different server, you can use the window-config-reloaded event to check for a running SSH process and apply a config override.
  • DHCP Warning: If your environment uses DHCP with short leases, hostnames can occasionally change, which might cause your logic to fail if it relies on a specific string.
    Would you like a more advanced example that uses a Lua table to manage a large list of different server-to-color mappings?

Using a Lua table is much cleaner than multiple if/else statements, especially as your list of servers grows. It also allows you to define a "fallback" or default theme if a host isn't recognized.

Here is an advanced implementation for your wezterm.lua:

The Table-Based Configuration

local wezterm = require 'wezterm'
local config = wezterm.config_builder()

-- 1. Define your host-to-theme mapping
local host_themes = {
    ["work-macbook"]     = "Tokyo Night",
    ["linux-server-01"]  = "Catppuccin Mocha",
    ["raspberry-pi"]     = "Gruvbox Dark",
    ["gaming-desktop"]   = "Cyberpunk",
}

-- 2. Get the current hostname
local current_host = wezterm.hostname()

-- 3. Assign the theme (with a default fallback)
local default_theme = "Nord"
config.color_scheme = host_themes[current_host] or default_theme

-- 4. Optional: Visual indicator (Border or Tab Bar color)
-- This makes it obvious which host you are on at a glance
if current_host == "linux-server-01" then
    config.colors = {
        tab_bar = {
            background = '#4b0000', -- Dark red background for production servers
        }
    }
end

return config

Why this approach is better:

  • Lookup Speed: Lua tables are hashed, making lookups extremely fast.
  • Readability: You can see all your environments at a glance in one block of code.
  • Scalability: If you add a new machine, you only need to add one line to the host_themes list.

Pro-Tip: Handling "Partial" Hostnames

Sometimes wezterm.hostname() returns a Fully Qualified Domain Name (e.g., macbook.local instead of just macbook). You can "sanitize" the hostname to make your table lookups more reliable: 

-- Strips everything after the first dot
local clean_hostname = current_host:split('.')[1] 
config.color_scheme = host_themes[clean_hostname] or default_theme

Conditional Based Icons

Bonus: Unique Backgrounds per Host

Since you are using Nerd Fonts, you could even add a specific icon to your status bar based on the host. Would you like to see how to inject a hostname-specific icon (like a  for Mac or  for Linux) into your WezTerm status bar?

To inject a hostname-specific icon into your WezTerm status bar, you use the update-right-status event. This event allows you to run Lua code periodically and update the status area with text, colors, and Nerd Font symbols.

Implementation in wezterm.lua

You can define a table that maps hostnames to specific icons and then use window:set_right_status to display them. 

local wezterm = require 'wezterm'
local config = wezterm.config_builder()

-- 1. Define host-to-icon mapping using Nerd Font symbols
local host_icons = {
  ["work-macbook"] = wezterm.nerdfonts.fa_apple,      -- 
  ["linux-server"] = wezterm.nerdfonts.cod_terminal_linux, -- 
  ["home-pc"]      = wezterm.nerdfonts.fa_windows,    -- 
}

-- 2. Create the status bar update function
wezterm.on('update-right-status', function(window, pane)
  local hostname = wezterm.hostname()
  -- Get the icon or use a generic "host" icon as fallback
  local icon = host_icons[hostname] or wezterm.nerdfonts.fa_server
  
  window:set_right_status(wezterm.format {
    { Foreground = { Color = '#7aa2f7' } }, -- Pick a color
    { Text = icon .. '  ' .. hostname .. ' ' },
  })
end)

return config

Why This Works

  • wezterm.hostname(): Returns the name of the system running the terminal process.
  • wezterm.nerdfonts: A built-in library that lets you look up icons by name (e.g., fa_apple) without needing to remember hex codes.
  • wezterm.format: Allows you to apply colors and styles to specific parts of your status string.

Pro-Tip: OS Detection Fallback

If you have many machines and don't want to list every hostname, you can use wezterm.target_triple to detect the Operating System generally (e.g., searching for "apple" or "linux" in the string) and assign an icon based on the OS instead of the specific host.

Assigning Colors to Variables

Can I assign a color to a variable (so that I may apply that color to various items) based on host names? (And will these settings default to a color if the host name is not in the list?)

Yes, this is a very efficient way to handle theming in Lua. By assigning a "base color" to a variable based on the hostname, you can reuse that single color for your cursor, borders, tab bars, and selection highlights, ensuring a consistent "vibe" for each machine.

Here is how you can structure your wezterm.lua to define a primary theme color with a built-in fallback:

The Variable-Based Configuration

local wezterm = require 'wezterm'
local config = wezterm.config_builder()

-- 1. Define your host-to-color mapping
-- You can use Hex codes or CSS color names
local host_colors = {
    ["work-macbook"]    = "#7aa2f7", -- Soft Blue
    ["prod-server"]     = "#f7768e", -- Warning Red
    ["raspberry-pi"]    = "#9ece6a", -- Forest Green
}

-- 2. Get hostname and determine the "Accent Color"
local current_host = wezterm.hostname()
local default_color = "#bb9af7" -- Purple fallback

-- This variable now holds the color for the current machine
local accent_color = host_colors[current_host] or default_color

-- 3. Apply that variable to various UI elements
config.colors = {
    -- Change the cursor color
    cursor_bg = accent_color,
    
    -- Change the active tab indicator
    tab_bar = {
        active_tab = {
            bg_color = accent_color,
            fg_color = '#1a1b26', -- Dark text for contrast
        },
    },

    -- Change the visual selection color (with some transparency)
    selection_bg = accent_color,
}

-- 4. Apply it to the window frame/border
config.window_frame = {
    active_titlebar_bg = accent_color,
}

return config

Why this works well:

The "Or" Fallback: The line host_colors[current_host] or default_color is a Lua idiom. If the hostname isn't found in your table, it evaluates to nil, and Lua automatically moves to the second option (default_color).

  • Centralized Control: If you decide you want your "Work" machine to be Orange instead of Blue, you only change the Hex code in one place (the table), and every UI element you've linked to accent_color updates automatically.
  • Readability: It keeps your config.colors block clean because you aren't nesting logic inside it.

A Quick Tip on Visibility

When using an accent_color for things like selection_bg, the text underneath can sometimes become hard to read. You can use WezTerm's color functions to slightly dim or brighten your variable on the fly: 

-- This would make the selection background 50% transparent
selection_bg = wezterm.color.parse(accent_color):srgba(0,0,0,0.5)
