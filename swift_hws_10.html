Hacking With Swift - 10 - How to use sets for fast data lookup

In addition to Arrays and Dictionaries there is the 'Set'.

Items are not stored in any order, and there are no duplicates.

The set is created as an array, inside a Set. Notice that the Set definition contains an array of actors' names: 

let people = Set(["Denzel Washington", "Tom Cruise", "Nicolas Cage", "Samuel L Jackson"])

Printing this set will come back in any order - it's unpredictable.

Insert

In the case of Sets, items are 'inserted' rather than appended.

Also notice the similarity of defining the set with defining arrays: 

var people = Set<String>()
people.insert("Denzel Washington")
people.insert("Tom Cruise")
people.insert("Nicolas Cage")
people.insert("Samuel L Jackson")

Advantages of Sets

No Duplicates

This is useful in limiting to unique data.

Checking Presence

If you want to check if an item is in a Set, it's extremely fast to return true or false. With an array, the algorithm has to check every item, whether it's there or not.

This is done with the .contains() method.

As with the Arrays and Dictionaries, you also have .count and sorted() (which returns a sorted array of the set's items).
