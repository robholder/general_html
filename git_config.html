Git Config

git config is the command to declare your settings for Git. The config command can be run anywhere on the workstation/system (no particular directory).

'git config' + flags + option + value

Example: 

# Set username for system:
git config --global user.name "My Name"
 
# Set email (this one isn't global):
git config user.email "me@myemail.com"

# Set email (this one is global):
git config --global user.email "me@myemail.com"

# Set global .gitignore file:
git config --global core.excludesfile ~/.gitignore_global

# Setup aliases:
git config --global alias.loga "log --online --graph --all"

Set vim as commit editor

git config --global core.editor /usr/bin/vim

--local

The --local flag set the config settings for the local git repo (and stores them in a local file).

The 'local' config file is stored inside the '.git' directory.

This is the default flag! (You have to use --global to NOT be local.)

Also, you have to be in a git repository to run this successfully. (--local will fail if not in git repo.)

--global

The --global flag sets the config setting across all repositories on the system. (Stored in the .gitconfig file.)

Using the --global flag allows you to set the config from any directory.

The '.gitconfig' File

The '.gitconfig' file is where the global settings are stored.

The Global .gitconfig file can be found here:

  • MacOS & Linux - ~/.gitconfig
  • Windows - C:\Users\.gitconfig

Contents of the config file

Example of .gitconfig file: (Notice the format and the config format "section.key value" correponds to "user.name blah". 

[user]
    name = My Name
    email = me@myemail.com
[core]
    editor = code -w
[push]
    default = simple

Unset Settings

git config --unset user.email - unsets the local setting
git config --global --unset user.email - unsets the global setting

Alternatively, edit the .config files directly (local and global).

Echo Config Values

To see the current value as set in the 'config', run the same command as setting but without a value.

NOTE: If git is using the 'install default' for a setting, the echo will be blank

# To see the email setting:
$ git config user.email

> me@myemail.com

To see all the set values: 

$ git config -l

>credential.helper=osxkeychain
>init.defaultbranch=main
>user.name=My Name
>user.email=me@myemail.com

To see where a value setting is defined: 

$ git config --list --show-origin

>file:/Applications/Xcode.app/Contents/Developer/usr/share/git-core/gitconfig    credential.helper=osxkeychain
>file:/Applications/Xcode.app/Contents/Developer/usr/share/git-core/gitconfig    init.defaultbranch=main
>file:/Users/<user>/.gitconfig    user.name=My Name
>file:/Users/<user>/.gitconfig    user.email=me@myemail.com

To see all possible config settings: 

git help -c
# or
git help --config

Git Aliases

You can set aliases within git itself (directly in the config settings rather than a shell or system alias).

The following sets 'loga' as an alias for 'git log --online --graph --all'. 

# To see an alias for a detailed log output.
# (Notice there's no 'git' included in the expanded command):
git config --global alias.loga "log --online --graph --all"

Git aliases allow for the addition of further arguments and flags at use time. So you can add git <commands> as aliases.

Example: 

# To add 'commit' as an alias:
git config --global alias.c "commit"

# Now you can commit by using this shortcut...
git c -m "my commit message"

# ...which expands to:
git commit -m "my commit message"

# To add 'add' as an alias:
git config --global alias.a "add"

# Now you can add using this shortcut...
git a .

# Expands to:
git add .

Show Details of an Alias

# To expand the alias:
git help <alias>

# e.g., 
$ git help c
>'c' is aliased to 'commit'

Use git config --list --show-origin | grep alias to look at them as an echoed list.

Suggestions

Here's a list to copy and paste: 

git config --global alias.loga "log --oneline --graph --all"
git config --global alias.c "commit"
git config --global alias.a "add"
git config --global alias.s "status"
git config --global alias.b "branch"
git config --global alias.sw "switch"
git config --global alias.ch "checkout"
