Open Files From the Commandline

The open command is a general utility on macOS that works with virtually any application. Here’s a brief breakdown:

Default Behavior:

If the file type is associated with a default application, simply running: 

open path/to/file

...will open the file using that default application.

General Use:

You can use open -a ApplicationName <file-or-url> to launch any application with a specified file or even a URL.

For example:

  • Open a text file with TextEdit: 
    open -a TextEdit path/to/file.txt
  • Open a web page in Safari: 
    open -a Safari https://www.example.com
  • Open a PDF file with Preview: 
    open -a Preview path/to/file.pdf

Additional Notes:

  • Make sure you use the exact name of the application as recognized by macOS.
  • If the application is not installed in the standard /Applications directory, you might need to provide the full path.
  • You can also launch apps without specifying a file by just using: 
    open -a ApplicationName

In short, open combined with -a is a versatile command that works with virtually all Mac applications.

Example: PNG in Preview

You can easily open a PNG file in Preview from the command line using macOS’s open command. Here’s how:

Basic Command:

open -a Preview <path-to-file>/image.png

Replace <path-to-file> with the actual path to your PNG file.

Explanation:

  • The open command tells macOS to open a file.
  • The -a flag specifies the application you want to use—in this case, Preview.

Default Application Behavior:

If Preview is the default app for PNG files, you could also simply use: 

open <path-to-file>/image.png

These approaches allow you to call Preview directly from the terminal to open any compatible file.

Opening Current Directory in Finder

Running open . in the terminal will open the current directory in a Finder window. This works because the open command uses the default application associated with the specified item—in this case, the Finder for directories. 

open .
