YouTube - Houdini Digital Assets | 1. Introduction to HDAs

HDAs are Houdini Digital Assets packaged up to be reused and shared, and can be called like any other node.

You can make your own library of tools - HDAs - to support your workflow and pipeline.

Creating an HDA

When you have a working network of nodes performing a function or build:

  • Select all the nodes (except perhaps the feeding node which you'd like to be plugged into a compacted version of your network - your HDA).
  • Menu:
    • Assets > New Digital Asset From Selection
    • In Dialog:
      • Operator Name (for script/code reference)
      • Operator Label (the regular 'name' of the tool)
      • Save to Library (the path and name of the HDA)

NOTE: HDAs used to be called OTLs ('Operator Type Library'). There may be a more complex differentiation, but essentially, HDA and OTL is the same thing (but not really).

Due to this historic naming, the path to the default storage for saved HDAs has OTL in the directory name.
