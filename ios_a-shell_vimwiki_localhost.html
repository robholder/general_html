Getting Vimwiki Html Files To Show Code Colors In Safari On Ipad

If you already have a-Shell installed on the iPad, here is the step-by-step to create a "Vimwiki-Sync-and-Serve" setup.

SSH Keys

Setting up SSH keys in a-Shell is highly recommended because it bypasses the need for tokens and handles the authentication handshake automatically.

Here is how to set up SSH on the new iPad and link it to GitHub:

🔑 1. Generate the SSH Key

In a-Shell, run this command: 

ssh-keygen -t ed25519 -C "your_email@example.com"
  • When it asks to "Enter file in which to save," just press Enter (keep the default).
  • When it asks for a passphrase, you can leave it empty for convenience or enter one for extra security.
  • Run: 
ssh -T git@github.com

Then say 'Yes' to the fingerprint question (since this is the first contact).

📋 2. Copy the Public Key to GitHub

You need to tell GitHub that this specific iPad is allowed to access your account.

  • Display your public key:
    • cat ~/.ssh/id_ed25519.pub
  • Copy the entire output (the long string starting with ssh-ed25519).
  • Go to GitHub.com in Safari:
    • Settings > SSH and GPG keys > New SSH Key.
    • Give it a title like "New iPad" and paste your key into the box.

📂 1. Set Up the Directory & Git

You need to clone your repository into a-Shell’s local storage so it can be served.

  • Open a-Shell.
  • Go to the Documents folder: cd Documents
  • Create a directory called vimwiki_dir, and cd into it.
  • Clone your repo:
  • With SSH:
    • lg2 clone git@github.com:YOUR_USERNAME/YOUR_REPO_NAME.git YOUR_REPO_NAME
      • eg:
      • lg2 clone git@github.com:YOUR_USERNAME/general_html.git general_html
      • Notice that lg2 retains the .git suffix, unless you offer a name as an arg.
    • If asked, select ssh key:
      • If you set up as above, leave password blank and hit enter.
  • Without SSH (not tried0:
    • lg2 clone https://github.com Vimwiki
  • (In both cases, you will likely be prompted for your GitHub username and a Personal Access Token as the password).
  • Navigate into the HTML folder:
  • cd vimwiki/general_html
  • Bookmark it:
    • bookmark vimwiki

Bookmark Management

  • bookmark (auto named for location)
  • bookmark <name>
  • deletemark
  • renamemark
  • showmarks
  • jump
  • pickFolder

📜 2. Create the Sync Script

You’ll want a single file that pulls your changes and starts the server.

Make a directory for binaries:

  • mkdir bin
  • should be:
    • ~/Documents/bin/

Inside Documents/bin/ Create/Edit the script:

  • vim sync_serve.sh
  • Paste this content (Press i to start typing): 
# Keep app active
isForeground

# Kill any zombie servers
killall python

# Jump to the bookmarked folder
jump vimwiki

# Sync with GitHub
lg2 pull

# Start the server in the background
python -m http.server 8000 &
  • Make it executable:
    • chmod +x sync_serve.sh

⚡ 3. Set Up Aliases (server and which)

Now, make your shortcuts permanent in your profile.

  • Open your profile:
    • vim ~/Documents/.profile
  • Add these lines at the bottom: 
alias server='sh ~/Documents/bin/sync_serve.sh'
alias which='alias'

🔖 4. The "Full Safari" Shortcut

To avoid the missing address bar (the "Web App" issue):

  • Open the Shortcuts app.
  • Tap + > Add Action.
  • Search for "Open URLs".
  • Type http://localhost:8000.
  • Tap the "Share" icon (bottom) > Add to Home Screen.

🔗 3. Update your Clone/Remote

lg2 (a-Shell’s git) needs to know to use the SSH URL instead of the HTTPS one. If you haven't cloned yet: lg2 clone git@github.com:YOUR_USERNAME/YOUR_REPO_NAME.git Vimwiki If you already cloned with HTTPS, change the URL: jump vimwiki lg2 remote set-url origin git@github.com:YOUR_USERNAME/YOUR_REPO_NAME.git 🛠️ 4. Final Verification Test the connection by running: lg2 pull Note: The first time you connect, it will ask if you trust the host. Type yes.
