Using dict.fromkeys to Remove Duplicates From List But Retain List Order

This technique is a dictionary trick to remove duplicate filenames while preserving their original order.

Breaking It Down

filenames = list(dict.fromkeys(filenames))
  • dict.fromkeys(filenames)
    • Creates a dictionary where the keys are the elements from filenames.
    • Since dictionaries do not allow duplicate keys, this automatically removes duplicates.
    • The order of elements is preserved because Python dictionaries (since version 3.7) maintain insertion order.
  • list(...)
    • Converts the dictionary keys back into a list, effectively giving you a unique list of filenames while keeping their original order.

Example Usage

filenames = ["file1.txt", "file2.txt", "file1.txt", "file3.txt", "file2.txt"]
filenames = list(dict.fromkeys(filenames))
print(filenames)  # Output: ['file1.txt', 'file2.txt', 'file3.txt']

Why Use This Approach?

  • ✅ Removes duplicates efficiently.
  • ✅ Preserves order (unlike set() which sorts unpredictably).
  • ✅ More concise than manually iterating over the list to remove duplicates.

How does it work?

What does the intermediate dictionary look like, and what does the method 'fromkeys' do?

Let's break down the process step by step.

What Does dict.fromkeys Do?

  • Purpose: dict.fromkeys(iterable, value) creates a new dictionary where the keys come from the provided iterable (e.g., a list), and every key is set to the same default value (which is None if not specified).

Example:

keys = ['file1.txt', 'file2.txt', 'file1.txt', 'file3.txt']
new_dict = dict.fromkeys(keys)
print(new_dict)

The output will be: 

{'file1.txt': None, 'file2.txt': None, 'file3.txt': None}

How it works:

  • The method iterates over the list.
  • For each element, it uses that element as a key in the dictionary.
  • Because dictionaries cannot have duplicate keys, if an element repeats, it simply keeps the first occurrence and ignores the subsequent ones.

What Does the Intermediate Dictionary Look Like?

Assume your original list is: 

filenames = ["file1.txt", "file2.txt", "file1.txt", "file3.txt", "file2.txt"]

When you call: 

intermediate_dict = dict.fromkeys(filenames)

The resulting dictionary looks like: 

{
    "file1.txt": None,
    "file2.txt": None,
    "file3.txt": None
}

Notice that the duplicates ("file1.txt" and "file2.txt") appear only once as dictionary keys.

Converting the Dictionary Back to a List

The line: 

filenames = list(intermediate_dict)

This takes the keys of the dictionary (which are unique) and converts them back into a list, preserving the order in which they were first encountered (thanks to the insertion order preservation in dictionaries since Python 3.7).

So the final result is: 

["file1.txt", "file2.txt", "file3.txt"]

Summary

  • dict.fromkeys(filenames) creates a dictionary with the filenames as keys and None as their associated values.
  • This process inherently removes duplicates because keys in a dictionary must be unique.
  • Converting the dictionary back to a list extracts only the unique keys, resulting in a list with no duplicates.

How do the values ('None' values) disappear?

Do the values evaporate regardless of their values?

When you convert a dictionary to a list using list(dictionary), it only iterates over the keys by default, so the values (whether they're None or any other value) are simply not included in the resulting list.

How It Works

After creating a dictionary with dict.fromkeys(filenames), you get something like: 

{
    "file1.txt": None,
    "file2.txt": None,
    "file3.txt": None
}

When you call: 

keys_list = list(intermediate_dict)

...python iterates over the dictionary keys, producing: 

["file1.txt", "file2.txt", "file3.txt"]

The values (None here) remain in the dictionary, but they’re not returned since list(dictionary) only returns the keys.

Why Does This Happen?

This is a built-in behavior of dictionaries in Python. When you iterate over a dictionary (or convert it to a list), it yields the keys, not the key–value pairs. This behavior is independent of what the values are; they "disappear" from the resulting list simply because the iteration protocol of dictionaries is designed to traverse keys only.

Is there a way of using 'list' to somehow retain the values in some form?

When you call list(a_dict), Python iterates over the dictionary’s keys only. That’s why in your example 

filenames = list(dict.fromkeys(filenames))

...the resulting list contains only the keys (the file names) and the associated values (which are all None, by default) simply aren’t included.

What Happens Internally

  • dict.fromkeys(filenames) This method creates a new dictionary whose keys come from the provided iterable (filenames) and whose values are all set to None (unless you provide a different default). For example: 
keys = ['file1.txt', 'file2.txt', 'file1.txt', 'file3.txt']

intermediate_dict = dict.fromkeys(keys)
print(intermediate_dict)
# Output: {'file1.txt': None, 'file2.txt': None, 'file3.txt': None}

Notice that duplicate keys are eliminated, leaving a dictionary with unique keys.

  • list(intermediate_dict) When you convert a dictionary to a list, Python iterates over the dictionary’s keys. That means: 
unique_files = list(intermediate_dict)
print(unique_files)
# Output: ['file1.txt', 'file2.txt', 'file3.txt']

The values (in this case None) are not part of the iteration result, so they effectively “disappear.”

Retaining the Values

If you want to “retain” or include the values as well, you have a few options:

  • List of Key–Value Tuples Using the .items() method, you can get a list that contains both keys and their associated values: 
unique_files_with_values = list(intermediate_dict.items())
print(unique_files_with_values)
# Output: [('file1.txt', None), ('file2.txt', None), ('file3.txt', None)]

Here, you store each key along with its corresponding value (even if it’s None).

  • Custom Default Value If you want to assign a specific value (other than None) in the process of deduplication, pass it as the second argument to fromkeys(). For example: 
custom_value = "my_default"
intermediate_dict = dict.fromkeys(keys, custom_value)
print(intermediate_dict)
# Output: {'file1.txt': 'my_default', 'file2.txt': 'my_default', 'file3.txt': 'my_default'}
# And to get a list of key–value pairs:
unique_files_with_values = list(intermediate_dict.items())
  • Building a Custom Data Structure If you need to associate something more dynamic with each filename (for example, tracking count or additional metadata), you might need to build the dictionary manually: 
files = ['file1.txt', 'file2.txt', 'file1.txt', 'file3.txt']
file_data = {}
for filename in files:
    if filename not in file_data:
        file_data[filename] = {'count': 1, 'metadata': {}}
    else:
        file_data[filename]['count'] += 1

# Now file_data retains more details for each file.
print(file_data)
# Output: {'file1.txt': {'count': 2, 'metadata': {}}, 'file2.txt': {'count': 1, 'metadata': {}}, 'file3.txt': {'count': 1, 'metadata': {}}}

Summary

  • list(dict.fromkeys(...)) removes duplicates by creating a dictionary where every element in the input list becomes a key (with a default value of None), then converts just the keys back into a list.
  • If you need to keep the values, use dict.items() to get them alongside the keys, or consider constructing your data structure in a way that stores whatever additional information you need.
