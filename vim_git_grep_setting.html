Git Grep

🔍 What “using git grep in Vim” actually means

Vim’s :grep command normally runs your system’s grep program.

If you tell Vim to use git grep instead, then:

  • Vim searches only files tracked by Git
  • It respects .gitignore
  • It’s much faster because Git uses its index
  • Results go into Vim’s quickfix list (just like normal grep)

So you get a project‑aware, lightning‑fast search inside Vim.

⚙️ The correct vimrc setting

Add this to your ~/.vimrc: 

set grepprg=git\ grep\ -n\ $*
set grepformat=%f:%l:%m

What these do

  • grepprg tells Vim which program to run when you type :grep
  • git grep -n prints results as file:line:match, which Vim understands
  • grepformat tells Vim how to parse that output into the quickfix list

This is the canonical, correct setup.

▶️ How to use it inside Vim

Once configured, searching your Git repo is as simple as: 

:grep pattern

Examples: 

:grep myFunction
:grep -i error
:grep -w init

Then open the quickfix window: 

:copen

Navigate results: 

:cnext
:cprev

🧩 Optional: Fugitive’s :Ggrep

If you use vim‑fugitive, you also get: 

:Ggrep pattern

This automatically uses git grep and fills the quickfix list.

📝 Summary

  • git grep in Vim = using Git’s search engine for :grep
  • Faster, respects .gitignore, searches only tracked files
  • Set it with: 
set grepprg=git\ grep\ -n\ $*
set grepformat=%f:%l:%m
  • Use it with :grep pattern and navigate via the quickfix list

Switch Between Vim Grep and Git Grep

Vim can automatically switch between git grep when you’re inside a Git repo and normal grep everywhere else. Once set up, you just type :grep foo and Vim chooses the right backend without you thinking about it.

Here’s a clean, reliable setup that works in both Vim and Neovim.

🔧 The Smart “git‑grep‑when‑possible” Setup

Add this to your ~/.vimrc (or init.vim): 

" Use git grep when inside a Git repo, otherwise fall back to normal grep
function! SmartGrep()
  if system('git rev-parse --is-inside-work-tree') =~# 'true'
    set grepprg=git\ grep\ -n\ $*
    set grepformat=%f:%l:%m
  else
    set grepprg=grep\ -nH\ $*
    set grepformat=%f:%l:%m
  endif
endfunction

" Automatically choose the right grep program when Vim starts
autocmd VimEnter * call SmartGrep()

" Also re-evaluate when changing directories
autocmd DirChanged * call SmartGrep()

🧠 What this does

✔ Detects whether you’re inside a Git repository

It runs: 

git rev-parse --is-inside-work-tree

If the answer is “true”, Vim switches to: 

git grep -n

Otherwise it uses normal grep.

✔ Keeps the quickfix list working

Both branches set: 

grepformat=%f:%l:%m

So results always populate the quickfix window correctly.

✔ Updates automatically

  • When Vim starts
  • When you :cd into another directory
  • When plugins change your working directory

You never have to think about it again.

▶️ How you use it

Exactly the same as before: 

:grep pattern

Then: 

:copen
:cnext
:cprev

Vim silently chooses the right engine.

🎉 Result

You get:

  • blazing‑fast Git‑aware searching when you’re in a repo
  • normal grep everywhere else
  • zero mental overhead

Live Preview

Let’s turn your :grep workflow into something that feels almost like a fuzzy‑finder: instant previews and one‑keystroke searches for the word under your cursor. These two additions pair beautifully with your smart git grep fallback setup.

🔎 1. Live preview of grep results

This trick opens the quickfix window automatically after every :grep so you see matches immediately.

Add this to your vimrc: 

" Automatically open quickfix window after :grep
autocmd QuickFixCmdPost grep cwindow

What this does

  • QuickFixCmdPost grep fires after any :grep command finishes
  • cwindow opens the quickfix window only if there are results
  • If there are no matches, it stays closed (so it doesn’t annoy you)

This makes :grep feel instant and interactive.

🎯 2. Search for the word under the cursor

This is the killer quality‑of‑life improvement. One keypress, and Vim runs :grep on the symbol you’re on.

Add this mapping: 

" Grep for the word under cursor
nnoremap <leader>g :grep <cword><CR>

How it works

  • <cword> expands to the word under your cursor
  • <leader>g becomes your “search this symbol” shortcut
  • Results drop into the quickfix window automatically (thanks to the autocmd above)

If your leader is the default '\', then typing: 

\g

...will search for the identifier under your cursor across your entire Git repo (or the filesystem if you’re outside a repo).

🧩 Optional: case‑insensitive or whole‑word search

If you want the mapping to always do a whole‑word match: 

nnoremap <leader>g :grep -w <cword><CR>

Case‑insensitive: 

nnoremap <leader>g :grep -i <cword><CR>

🚀 How it all works together

With your smart fallback + these two additions, your workflow becomes:

  • :grep foo → instant results
  • <leader>g → search symbol under cursor
  • :copen is no longer needed
  • :cnext / :cprev jump through matches
  • Works with git grep inside repos and normal grep elsewhere

It’s a really smooth, modern search experience inside classic Vim.
