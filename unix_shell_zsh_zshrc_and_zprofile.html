ZSH - The Differnce Between .zshrc and .zprofile

The core difference lies in when they are loaded: .zprofile runs once upon login to set up environment variables (like $PATH), while .zshrc runs every time a new interactive terminal shell is opened to set aliases, prompts, and functions. .zprofile loads first, followed by .zshrc in interactive sessions.

  • .zprofile (Login Shells): Used for one-time, top-level setup. It is ideal for environment variables (\(PATH, \)EDITOR) and settings that don't need to be redefined in every terminal tab.
  • .zshrc (Interactive Shells): Used for daily customizations. It is the best place for aliases, shell functions, prompt configuration (PROMPT), setopt commands, and theme settings.

Key Takeaways:

  • Order: .zprofile loads before .zshrc.
  • Override: Because it loads later, .zshrc can override variables set in .zprofile.
  • Best Practice: Put variable exports (e.g., export PATH=...) in ~/.zprofile and aliases/prompt settings in ~/.zshrc.
  • macOS Behavior: On macOS, even non-login terminals (like opening a new tab) often load .zprofile, but in other Unix environments, it strictly runs upon initial user login.
