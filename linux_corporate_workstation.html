Linux - Pimp My (Corporate/No SUDO) Ride

Here's a question I asked Gemini (Google). I can't try these out yet...

QUESTION

If I have to work on a corporate Linux machine, I half almost no ability or access to install stuff. How can I use things like Nerd Fonts, WezTerm, Starship? Is there a way to have these things work without any sudo privileges?

ANSWER

It is frustrating being locked out of sudo, but on Linux, home is where the binaries are. You can absolutely run this stack by treating your $HOME directory as your personal "root."

1. The "Local Bin" Strategy

Most modern Linux tools are distributed as static binaries or AppImages. You don't need to "install" them; you just need to place them in a folder you own and tell Linux where they are.

  • Create a local bin: mkdir -p ~/.local/bin
  • Add it to your Path: Add export PATH="\(HOME/.local/bin:\)PATH" to your ~/.bashrc or ~/.zshrc.

2. Installing the Tools (No Sudo)

  • Starship: Use the official install script but point it to your local folder.
    • curl -sS https://starship.rs | sh -s -- --bin-dir ~/.local/bin
  • WezTerm: WezTerm is a terminal emulator, so it usually runs on your local machine (Mac/Windows) and connects to the Linux box via SSH. You don't actually need to install WezTerm on the Linux machine—you just need it on your personal computer to "see" the icons.
  • Nerd Fonts: This is the most important part. Fonts are handled by the client side (your Mac).
    • Install the Nerd Font on your macOS system.
    • In your WezTerm config on your Mac, set the font = wezterm.font("YourNerdFontName").
    • When you SSH into the Linux machine, the icons will appear because your local terminal knows how to draw them.

3. Making Icons Work (The Font Cache)

If you must install a font on the Linux machine itself (e.g., for a specific app running there), you can do it without sudo:

  • Move the .ttf or .otf files to ~/.local/share/fonts/.
  • Run fc-cache -fv to refresh your personal font cache Arch Wiki Font Configuration.

4. Use "Homebrew for Linux"

If you have enough disk space, you can install Homebrew for Linux. It installs entirely into /home/linuxbrew/.linuxbrew (or your home folder) and allows you to brew install things like starship, fzf, and neovim without ever touching sudo.

Pro Tip: The "Portable" Workflow

If you move between many corporate machines, look into AppImages. They are single files that contain the entire program. Just chmod +x program.AppImage and run it.

GEMINI QUERY

Do you have SSH access to this machine from your Mac, or are you forced to use a corporate "workspace" in a browser?

MY REPLY

I'm usually directly logged in as a user (not SSH) and usually through a thin client.

ANSWER

Since you are on a thin client (meaning the rendering happens on the server, not your local Mac), the Linux machine must have the font files and binaries locally for you to see the "magic."

Here is your Zero-Privilege Setup Guide:

1. Nerd Fonts (The "Manual" Install)

Since you can't run a font installer, you have to place the files where the system looks for "user-level" fonts.

  • Download the "XZ" or "Zip" version of your favorite font (e.g., JetBrains Mono) from the Nerd Fonts Releases GitHub.
  • Move the files: 
mkdir -p ~/.local/share/fonts
cp ~/Downloads/*.ttf ~/.local/share/fonts/
  • Register them: Run fc-cache -fv.
  • The Catch: You must then go into the Terminal Preferences of the app you are currently using on that Linux box and manually change the font to the "Nerd Font" version you just moved.

2. Starship (The "Pre-compiled" Way)

You can't use the standard installer because it tries to touch /usr/local/bin. Instead, grab the binary directly:

  • Download the starship-x86_64-unknown-linux-gnu.tar.gz from the Starship Releases page.
  • Extract it: tar -xvzf starship-*.tar.gz
  • Move it: mkdir -p ~/.local/bin && mv starship ~/.local/bin/
  • Activate it: Add eval "$(~/.local/bin/starship init bash)" to your ~/.bashrc.

3. WezTerm (The "AppImage" Solution)

WezTerm provides an AppImage, which is a single executable file that requires zero installation.

  • Download the WezTerm-xxx.AppImage.
  • Make it runnable: chmod +x WezTerm-xxx.AppImage
  • Run it: ./WezTerm-xxx.AppImage
  • Note: If the thin client environment is very old, the AppImage might complain about "FUSE" libraries. If that happens, try the "Portable" .tar.xz version from the same WezTerm Downloads page.

4. The "Holy Grail": Nix (Single-User)

If your corporate machine allows outgoing internet access, look into the Nix Package Manager (Single-User Install). It installs entirely within a folder you specify (like ~/.nix-profile) and gives you access to 80,000+ packages (including Starship and Neovim) without ever needing sudo.

Pro Tip: Since you are in a corporate environment, check your disk quota with quota -s. Compiling things or downloading many fonts can fill up a restricted home directory quickly!
