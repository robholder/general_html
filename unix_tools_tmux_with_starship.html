Tmux - Starting Tmux

To pull in a different configuration file for Starship when you are inside a tmux session, you need to set the STARSHIP_CONFIG environment variable.

When Starship initializes, it checks this variable for a file path. If it's not set, it defaults to ~/.config/starship.toml.

Method 1: Automatic Detection in Shell Config (Recommended)

You can add a check to your shell configuration file (.zshrc or .bashrc) that detects if you are in a tmux session and exports the alternative path automatically.

Add this to your ~/.zshrc or ~/.bashrc: 

if [ -n "$TMUX" ]; then
  # Path to your tmux-specific config
  export STARSHIP_CONFIG=~/.config/starship-tmux.toml
fi
  • Why it works: The $TMUX variable is only set when you are inside a tmux session.
  • Note: Make sure this export happens before the eval "$(starship init ...)" line in your config.

Because Starship defaults to ~/.config/starship.toml, you only need to provide the specific path if your file is named or located differently (e.g., inside a subfolder).

To cover both bases correctly, use an if/else block. This ensures that the variable is always defined explicitly, preventing your shell from "remembering" the wrong config if you jump in and out of sessions.

Add this to your .zshrc or .bashrc: 

if [ -n "$TMUX" ]; then
  # Inside Tmux
  export STARSHIP_CONFIG=~/.config/starship/starship-tmux.toml
else
  # NOT in Tmux
  export STARSHIP_CONFIG=~/.config/starship/starship.toml
fi

Why this is safer:

If you only define the Tmux path, when you exit Tmux and go back to a regular WezTerm window, the environment variable might persist in that specific terminal instance, leaving you stuck with the "Tmux look" in a normal window. Explicitly setting it for both scenarios prevents that "ghosting."

  • Pro Tip: Ensure these lines are placed above the eval "$(starship init ...)" line so Starship knows which file to load during startup.
