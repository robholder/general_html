Zsh Script - Copy All Directory Icons to Corresponding Directories

  • Open Terminal.
  • Copy and paste this command, but do not press Enter yet: 

SOURCE="DRAG_SOURCE_HERE"; TARGET="DRAG_TARGET_HERE"; for folder in "$SOURCE"/*(/); do name="${folder:t}"; if [[ -d "$TARGET/$name" && -f "$folder/Icon"$'\r' ]]; then cp "$folder/Icon"$'\r' "$TARGET/$name/"; xattr -wx com.apple.FinderInfo "0000000000000000040000000000000000000000000000000000000000000000" "$TARGET/$name"; fi; done

How to use it:

  • Replace "DRAG_SOURCE_HERE" by dragging your Source enclosing folder into the Terminal.
  • Replace "DRAG_TARGET_HERE" by dragging your Target enclosing folder into the Terminal.
  • Press Enter.

How it works

  • SOURCE="..." and TARGET="..." — We define the exact, absolute paths right at the start. It doesn't matter where they are on your Mac anymore.
  • ${folder:t} — This is a special Zsh modifier. It grabs just the name of the sub-folder (the "tail") and ignores the rest of the messy path.
  • \(TARGET/\)name — It explicitly looks inside your target enclosing folder for a sub-folder with the exact same name, copies the icon file, and flips the Finder visibility flag.

How It Works

To understand exactly how this script works, it helps to think of it as an assembly line. It looks at the big source bucket, grabs a sub-folder, checks if a matching twin exists in the target bucket, and if so, safely transplants the icon DNA and tells macOS to display it.

Here is the complete breakdown of the script, piece by piece.

Part 1: Setting up the Variables

SOURCE="DRAG_SOURCE_HERE"; TARGET="DRAG_TARGET_HERE";
  • What it does: This creates two absolute path variables (＄SOURCE and ＄TARGET) based on the folders you dragged in.
  • Why it matters: By storing the exact blueprint of where these folders live on your hard drive right at the start, the script never gets lost, regardless of what directory Terminal is currently looking at.

Part 2: The Zsh Loop

for folder in "$SOURCE"/*(/); do
  • for folder in...; do: This kicks off a loop. It tells the computer, "Do the following steps for every item we find inside this specific location."
  • "＄SOURCE"/*: The asterisk (*) is a wildcard, meaning "everything inside the source folder."
  • (/): This is a native Zsh glob qualifier. It filters the wildcard so that the script only looks at directories (folders) and completely ignores loose files (like images or text documents) sitting in the root of the source folder.

Part 3: Isolating the Folder Name

name="${folder:t}"
  • What it does: When the script loops through a folder, the ＄folder variable holds the entire, messy absolute path (e.g., /Users/yourname/Desktop/SourceFolder/MyProject).
  • The :t modifier: This is a Zsh-specific trick called the "Tail" modifier. It strips away everything except the very last part of the path. So, /Users/yourname/Desktop/SourceFolder/MyProject becomes just MyProject. We store this clean name in a new variable called ＄name.

Part 4: The Safety Check

if [[ -d "$TARGET/$name" && -f "$folder/Icon"$'\r' ]]; then

Before doing any heavy lifting, the script double-checks two rules to make sure it doesn't cause errors:

  • -d "＄TARGET/＄name": Checks if a directory with that exact name exists inside your Target folder. (If you have a folder named "Photos" in Source, it makes sure "Photos" exists in Target).
  • &&: This means "AND". Both conditions must be true to proceed.
  • -f "＄folder/Icon"＄'\r': Checks if the source folder actually has a custom icon file inside it.

Note: The ＄'\r' is ANSI-C quoting syntax. It translates \r into a literal carriage return character, which is required because Apple hides the icon file under the name Icon followed by an invisible line break.

Part 5: Copying the Icon File

cp "$folder/Icon"$'\r' "$TARGET/$name/"
  • What it does: If the safety check passes, the cp (copy) command fires. It duplicates the hidden icon file from the sub-folder in the Source directory and pastes it directly into the matching sub-folder in the Target directory.

Part 6: Activating the Visuals

xattr -wx com.apple.FinderInfo "0000000000000000040000000000000000000000000000000000000000000000" "$TARGET/$name"
  • xattr -wx: This interacts with "extended attributes"—hidden metadata tags pinned to files and folders in macOS. The -w means write a new attribute, and the -x means the value is written in hexadecimal code.
  • com.apple.FinderInfo: This is the specific metadata category macOS uses to store custom Finder states (like whether a folder has a custom color label, custom icon, or is hidden).
  • The long string of zeros with a 4: In macOS file systems, if the 10th byte of a folder's Finder info is set to 04, it flags the folder as "Has Custom Icon". This command injects that exact flag into your target folder. Without this line, the icon file would copy over, but Finder would completely ignore it and keep showing the default blue folder.

Part 7: Closing the Loop

fi; done
  • fi: Short for "finish" (or "if" backwards). It marks the end of the conditional "if" check.
  • done: Tells the script, "We have finished processing this specific folder. Loop back to the top, grab the next sub-folder, and repeat the whole process."
