Recommit A Previous Commit To Latest

Example

# commits:
four
three
two
one

# to:
three-again
four
three
two
one

To achieve this, you are looking for a Revert or a Cherry-pick. Since you want to keep the history moving forward (adding a new commit that looks like an old one) rather than deleting "four," you want to avoid rebasing or resetting.

The cleanest way to do this is to tell Git: "Make the current files match the state of commit three, and then let me commit that as a new entry."

Option 1: The "Soft Reset" Method (Safest for learning)

This method effectively "copies" the files from version three and puts them in your staging area.

  • Get the hash for commit three:
    • git log --oneline
  • Point your files back to three (without moving your branch):
    • git restore --source <hash-of-three> .
    • (Note the dot at the end; it means "all files".)
  • Commit the "new" version:
  • git add .
  • git commit -m "three-again"

Option 2: The git revert Method (The "Undo" Button)

If you just want to "undo" what happened in commit four, you can revert it. This creates a new commit that is the exact mathematical opposite of whatever you did in "four."

git revert HEAD

  • Result: Git creates a new commit that effectively returns the code to the state it was in at three.
  • Why use this? It's the standard professional way to roll back a mistake while keeping a clear audit trail.

Option 3: The git cherry-pick Method

If you have a lot of commits and specifically want to grab the "essence" of three and slap it on top:

  • Find the hash of three.
  • Run: git cherry-pick <hash-of-three>

Note: This works best if "three" was a single self-contained change. If "three" was just one step in a long journey, Option 1 is usually more reliable for getting the exact state of the project.
