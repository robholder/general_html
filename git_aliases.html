Git Aliases

You can set aliases within git itself (directly in the config settings rather than a shell or system alias).

The following sets 'loga' as an alias for 'git log --online --graph --all'. 

# To see an alias for a detailed log output.
# (Notice there's no 'git' included in the expanded command):
git config --global alias.loga "log --online --graph --all"

Git aliases allow for the addition of further arguments and flags at use time. So you can add git as aliases.

Example: 

# To add 'commit' as an alias:
git config --global alias.c "commit"

# Now you can commit by using this shortcut...
git c -m "my commit message"

# ...which expands to:
git commit -m "my commit message"

# To add 'add' as an alias:
git config --global alias.a "add"

# Now you can add using this shortcut...
git a .

# Expands to:
git add .

Show Details of an Alias

# To expand the alias:
git help <alias>

# e.g., 
$ git help c
>'c' is aliased to 'commit'

Use git config --list --show-origin | grep alias to look at them as an echoed list.

Suggestions

Here's a list to copy and paste: 

git config --global alias.loga "log --oneline --graph --all"
git config --global alias.c "commit"
git config --global alias.a "add"
git config --global alias.s "status"
git config --global alias.b "branch"
git config --global alias.sw "switch"
git config --global alias.ch "checkout"

Listing and Finding Aliases

Because Git aliases are just custom entries sitting inside your Git configuration files, listing them is a matter of querying that configuration through the shell or opening the files directly.

Here is exactly how to find them in both environments.

1. Finding Aliases via the Shell (The Quick Way)

You can use the git config command to scan your configuration files and filter out everything except your aliases.

Option A: The Universal Regex Search (Recommended)

Run this command in your terminal: 

git config --get-regexp '^alias\.'
  • How it works: It looks through all your active configurations and returns any key that starts with alias..
  • The output looks like: 
alias.co checkout
alias.br branch
alias.st status

Option B: The Streamlined List

If you want a slightly cleaner look without the word "alias" repeated on every line, you can list your global configuration and pipe it through grep (if you are on Mac or Linux): 

git config --global --list | grep '^alias\.'

2. Finding Aliases directly in the Config Files

If you want to view, edit, or clean up your aliases manually, you can open Git's underlying configuration files. Git stores aliases in two main places: Global (for your entire computer) and Local (just for a specific repository).

The Global Config File (Where 99% of your aliases live)

Most developers define aliases globally so they work in every project. This file is hidden in your user home directory.

  • File Path: ~/.gitconfig
  • *How to open it in Vim: 
vim ~/.gitconfig
  • What to look for: Scroll down until you see the [alias] header. All your custom shortcuts will be grouped neatly right under it: 
[user]
    name = Jane Doe
    email = janedoe@example.com
[alias]
    co = checkout
    br = branch
    st = status
    lg = log --oneline --graph --decorate

The Local Config File (Project-Specific)

If you created a special alias that only applies to one specific repository, it will be stored inside that project's hidden .git folder.

  • File Path: Look inside your current project folder at .git/config
  • How to open it in Vim: 
vim .git/config
  • What to look for: Just like the global file, look for a [alias] section. (If you haven't made any repository-specific aliases, this section won't exist in this file).

Pro-Tip: Make an alias to list your aliases!

If you find yourself forgetting the shell commands above, you can actually add a Git alias called aliases to quickly audit them in the future.

Run this once in your terminal: 

git config --global alias.aliases "config --get-regexp '^alias\.'"

From now on, typing git aliases in your shell will instantly spit out the whole list!
