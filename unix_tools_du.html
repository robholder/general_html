A complete breakdown of the Linux command du -ha <path>. Here is an overview, its key uses, and a practical "How To Use" guide with examples.

ðŸ’¾ Tool Overview: du -ha

The du (Disk Usage) command is a core Linux utility used to report the estimated disk space used by files and directories. The options -h and -a are almost always combined for easy-to-read, comprehensive reports.

Component Function Why It's Used
du Disk Usage Calculates the total space consumed by the specified directory and all its contents (files and subdirectories).
-h Human-readable Converts large byte counts into understandable units (KB, MB, GB). This is essential for quickly reading the output.
-a All files Lists the disk usage for every individual file inside the directory, in addition to the directories themselves. Without -a, du only reports on the directories.

Key Use Case

The primary use of du -ha is to quickly find where disk space is being consumed within a specific project or filesystem branch.

ðŸš€ How to Use du -ha

The general syntax is straightforward: du -ha [options] <path>

1. Check Current Directory Size

This is the most common use. It shows the size of every file and folder, followed by a total size for the current directory. 

du -ha .

Output Example: 

4.0K    ./config.yaml
12M     ./videos/clip_01.mp4
12M     ./videos
28M     .

The last line (28M .) is the total size of the current directory.

2. Check Specific Directory Size

Use this to inspect a particular folder without having to navigate to it first. 

du -ha /var/log

Output: This will list the size of every file and subdirectory within /var/log and finish with the total size of the /var/log directory.

3. Show Only the Total Size (-sh)

If you are only interested in the grand total size of a directory and don't want the long list of every individual file, you replace the -a option with -s (summarize). 

du -sh /path/to/project

Output Example: 

4.7G    /path/to/project

Note: This is the most efficient command for a quick total size check.

4. Find the Biggest Files/Folders

You can combine du -ha with the sort and tail commands to quickly find the largest items, which is great for cleanup or diagnosis. 

du -ha /path/to/search | sort -hr | head -10
  • Breakdown:
    • du -ha /path/to/search: Generates the file list with human-readable sizes.
    • | sort -hr: Pipes the list to sort.
    • -h: Sorts based on human-readable size (e.g., understands that 1G > 100M).
    • -r: Reverses the order so the largest sizes are at the top.
    • | head -10: Shows only the first 10 lines (the 10 largest items).
  • Result: A list of the top 10 files and folders consuming the most space in that path.
