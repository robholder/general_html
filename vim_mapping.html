Leader Key

You can set a <leader> key to your desire. It's normally \ out of the box.

I have <Space>, but you need to do some housework to make this function correctly:

Here, any mapping already on <Space> is removed, then the mapleader is set to space. There are two different ways, and different systems might prefer one over the other: 

" Remove mapping from <Space> then map as leader:
nnoremap <Space> <nop>
" Adding <Scace> as the leader key (two options):
"let mapleader=" "

See If Mapping Exists

To determine if a specific key combination is already mapped in Vim, you can use the :map family of commands. These commands display mappings for different modes and help you track down where a key combination is being used. Here's how:

Check All Mappings

Use the following command to view all mappings: 

:map

This displays a list of all defined mappings, regardless of mode. You can scroll through the list to see if your desired key combination is already mapped.

Check Specific Mode Mappings

To narrow it down, you can check mappings for specific modes:

Normal mode:

:nmap

Insert mode:

:imap

Visual mode:

:vmap

Command-line mode:

:cmap

Search for a Specific Key Combination

You can use the :verbose command to check if a specific key combination is mapped and where the mapping is defined: 

:verbose map <key>

For example, to check if <C-k> is mapped, you would type: 

:verbose map <C-k>

This will tell you:

  • Whether the key is mapped.
  • The file or script where the mapping is defined (great for debugging!).

Check for Mode-Specific Mappings

To check mappings for a specific mode and a specific key combination, use:

Normal mode:

:verbose nmap <key>

Insert mode:

:verbose imap <key>

Example in Practice

If you suspect <C-j> is mapped and want to see its Insert mode mapping, you would use: 

:verbose imap <C-j>

Bonus: Unmapping Keys

If you find an unwanted mapping, you can remove it with the appropriate :unmap command, like:

Normal mode:

:nunmap <key>

Insert mode:

:iunmap <key>

This should help you easily track and debug key mappings.

Operator-Pending Mappings

These mappings are movement mappings (like motions that follow commands, such as delete or yank).

(The following is from 'Learn Vim The Hard Way'.)

When you’re trying to think about how to define a new operator-pending movement, you can think of it like this:

  1. Start at the cursor position.
  2. Enter visual mode (charwise).
  3. ... mapping keys go here ...
  4. All the text you want to include in the movement should now be selected.

It’s your job to fill in step three with the appropriate keys.

:onoremap p i( <--when used with d, as in dp, will delete inner ().
:onoremap b /return<cr> <--when used with d, as in db, will delete all up to return.

To manipulate where the operator starts:

:onoremap in( :<c-u>normal! f(vi(<cr> <-- This enters command mode, clears, then runs the command.

The above clears command (control-u), then 'finds' (. Next it visually selects the inner ( ).

To test, type "print foo(bar)", then put the cursor in the word 'print', and type cin( in normal mode. The inner brackets are cleared.
