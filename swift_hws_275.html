Hacking With Swift - 275 - Challenge 2

Challenge 2

  • Make views adjust their scale depending on their vertical position, with views near the bottom being large and views near the top being small. I would suggest going no smaller than 50% of the regular size.

Ai Insight

ANSWER

My guess is that I need a modifier after the opacity (or maybe before the rotation3DEffect?).

Can you point me to a solution?

ANSWER

You are exactly on the right track thinking about a new modifier! The modifier you want is called .scaleEffect().

Order-wise, it is generally best practice to put spatial transformations like scaling and rotations before you apply opacity or layout constraints, ensuring that the visual effect scales smoothly from its structural center.

The Scaling Strategy

For this challenge, we want a linear transformation from the top of the screen to the bottom:

  • At the very top of the screen (midY = 0): The scale should be exactly 0.5 (50% size).
  • At the very bottom of the screen (midY = fullView.size.height): The scale should be exactly 1.0 (100% full size).

To translate this into a formula, we can find the card's position as a ratio of the total screen height (proxy.frame(in: .global).midY / fullView.size.height). This gives us a value ranging from 0.0 at the top to 1.0 at the bottom.

Since we want to range from 0.5 to 1.0 instead of 0.0 to 1.0, we can map it like this:

  • Scale = 0.5 + ( 0.5 x midY/ScreenHeight)

The Solution

Here is how you add that modifier to your code. Note that I've also re-aligned your rotation3DEffect back to the horizontal axis (x: 1, y: 0, z: 0) so it doesn't skew weirdly when mixing with the new scale effect: 

struct ContentView: View {
    let colors: [Color] = [.red, .green, .blue, .orange, .pink, .purple, .yellow]
    
    var body: some View {
        GeometryReader { fullView in
            ScrollView(.vertical) {
                VStack(spacing: 0) { // Using a VStack helps keep row frames stable
                    ForEach(0..<50) { index in
                        GeometryReader { proxy in
                            Text("Row #\(index)")
                                .font(.title)
                                .frame(maxWidth: .infinity)
                                .background(colors[index % 7])
                            // 1. SCALE EFFECT FIRST
                                .scaleEffect(
                                    CGFloat(0.5 + (0.5 * (proxy.frame(in: .global)
                                        .midY / fullView.size.height)))
                                )
                            // 2. ROTATION EFFECT
                                .rotation3DEffect(
                                    .degrees(Double(proxy.frame(in: .global)
                                        .midY - fullView.size.height / 2) / 5),
                                    axis: (x: 1, y: 0, z: 0)
                                )
                            // 3. OPACITY EFFECT
                                .opacity(
                                    max(0.0, 1.0 - Double(abs(proxy.frame(in: .global)
                                        .midY - fullView.size.height / 2) / 200.0))
                                )
                        }
                        .frame(height: 40)
                    }
                }
            }
        }
    }
}

Breaking Down the Math

  • At the absolute top edge: midY is 0. The math reads: 0.5 + (0.5 * 0) = 0.5. Your row scales down perfectly to 50% size.
  • At the absolute middle: midY is exactly half the screen height. The fraction becomes 0.5. The math reads: 0.5 + (0.5 * 0.5) = 0.75. Your row sits at 75% size.
  • At the absolute bottom edge: midY matches the full screen height. The fraction becomes 1.0. The math reads: 0.5 + (0.5 * 1.0) = 1.0. Your row blooms to its 100% regular size.
