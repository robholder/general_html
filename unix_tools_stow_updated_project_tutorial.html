Sample Dotfiles Repo Layout

Here’s a realistic, clean layout for a Dotty‑managed dotfiles repo.

This assumes you start with empty package directories and let Dotty populate them. 

dotty/
├── bootstrap.sh
├── README.md
├── bootstrap.1        # man page (optional)
├── .migrateignore     # optional ignore patterns
│
├── zshrc/
│   ├── dot-zshrc
│   ├── dot-zshenv
│   └── dot-zprofile
│
├── vim/
│   ├── dot-vimrc
│   └── dot-vim/
│       └── colors/
│           └── mytheme.vim
│
├── git/
│   ├── dot-gitconfig
│   └── dot-gitignore
│
└── nvim/
    └── dot-config/
        └── nvim/
            ├── init.lua
            └── lua/
                └── plugins.lua

==== A few things to notice: ====

* Package directories are simple: zshrc/, vim/, git/, etc.
* Dotty will create dot-* files automatically when migrating.
* Nested structures (like ~/.config/nvim) become dot-config/nvim/....
* You can add scripts, templates, or README files inside packages — Dotty ignores anything not named dot-*.

== 🚀 Getting Started Tutorial ==

This is the kind of tutorial you’d expect in a polished project.

Feel free to drop it straight into your README or wiki.

=== 🎒 Step 1 — Create your dotfiles directory ===

{{{bash
mkdir ~/dotty
cd ~/dotty

Add the bootstrap.sh script here.

📦 Step 2 — Create package directories

Each package corresponds to a family of dotfiles. 

mkdir zshrc vim git nvim

You can start with empty directories — Dotty will populate them.

🔍 Step 3 — Run Dotty in dry‑run mode

./bootstrap.sh

Dotty will:

  • detect your packages
  • infer matching $HOME files
  • show existing dotfiles (if any)
  • present a numbered selection list
  • show a grouped preview of symlinks

Example: 

INFO: Processing package: zshrc

Existing dotfiles in package (auto-included):
  (none)

Home files matching this package:

  [1] ~/.zshrc
  [2] ~/.zshenv
  [3] ~/.zprofile

Select items to include (e.g. "1 3", "all", or "none"; default: none):
Press Enter to skip, or choose the files you want to migrate.

🛠 Step 4 — Apply changes

Once you’re satisfied with the preview: 

./bootstrap.sh --apply

Dotty will:

  • back up each selected file
  • move it into the package as dot-*
  • create symlinks
  • run Stow to finalize the structure

🔄 Step 5 — Update your dotfiles

Just edit the files inside your package directories: 

dotty/zshrc/dot-zshrc
dotty/vim/dot-vimrc
dotty/git/dot-gitconfig

Stow keeps everything synced.

🧹 Step 6 — Uninstall (optional)

To remove symlinks: 

./bootstrap.sh --uninstall

Dotty will unstow each package cleanly.
