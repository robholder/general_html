Gizmo to Grizmo (Group)

Gizmos are groups or tools in nuke that can be saved as an available node, and when they are updated on disk, comps opened that utilize them are instantly updated to the new version. They are referenced from the Gizmo definition.

However, if the original Gizmo definition is not available, the script will fail, as the script cannot reference the Gizmo.

Solution

The Grizmo

Convert to a Group, but opening the gizmo definition on disk, and edit the file: 

#! /Aplications/Nuke9.0v4/Nuke0.0v4/Contents/MacOs//libnuke-9.0.4.dylib -nx
version 9.0 v4
Gizmo {
    addUserKnob {20 User}
    addUserKnob {41 size T Blur1.size}
}
    Input {
        inputs0
        ...etc.

...edit to: 

#! /Aplications/Nuke9.0v4/Nuke0.0v4/Contents/MacOs//libnuke-9.0.4.dylib -nx
version 9.0 v4
Group {
    name mygrizmo
    addUserKnob {20 User}
    addUserKnob {41 size T Blur1.size}
}
    Input {
        inputs0
        ...etc.

Notice the changes:

  • Gizmo to Group
  • Add the line:
    • name <node-name>

The Result

The Grizmo can still be stored as a generic tool and referenced via a menu addition, but now it's not tied to the definition, and is portable to all systems.

Caveat

Any updates to the Group/Grizmo will not be re-referenced by legacy scripts. The Grizmo will have to be re-imported.
