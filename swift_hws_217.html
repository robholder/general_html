Hacking With Swift - 217 - Challenge 1

Challenge 1

  • The CheckoutView in Cupcake Corner uses an image and loading spinner that don’t add anything to the UI, so find a way to make the screenreader not read them out.

Ai Insight

QUESTION

I tried an accessibilityHidden(true) modifier, but that doesn't work. How can I achieve this?

ANSWER

It is completely understandable why you expected .accessibilityHidden(true) to work here! It feels like the absolute right tool.

The reason it failed is due to how SwiftUI wraps AsyncImage.

AsyncImage isn’t a single visual element; it is a structural container that swaps things out under the hood. When you attach .accessibilityHidden(true) to the outer container, it doesn't always automatically push that setting down into the individual Image or ProgressView elements inside the closure.

To hide it from VoiceOver, you have to apply accessibility modifiers to the internal items inside the closures, or tell SwiftUI to bypass the whole block. Here are the best ways to achieve this.

Solution 1: Define the image as decorative (Most Elegant)

Since the main cupcake image is purely decorative and doesn't convey critical text data, you can declare the image explicitly as a decorative image.

The Image(decorative:) initializer tells iOS to completely ignore the image for accessibility. Then, we apply .accessibilityHidden(true) explicitly to the ProgressView in the placeholder block: 

AsyncImage(url: URL(string: "https://hws.dev/img/cupcakes@3x.jpg"), scale: 3) { phase in
    switch phase {
    case .success(let image):
        // 1. Force the layout to treat the actual downloaded image as decorative
        Image(uiImage: image.asUIImage()) // Or convert via the phase directly
            .resizable()
            .scaledToFit()
    case .failure, .empty:
        // 2. Hide the spinner explicitly
        ProgressView()
            .accessibilityHidden(true)
    @unknown default:
        EmptyView()
    }
}
.frame(height: 233)

Note: Since standard AsyncImage syntax abstracts the raw image away, a slightly cleaner alternative for the standard closure looks like Solution 2 below.

Solution 2: The Element Ignorer (Simplest Fix)

If you don't want to change your standard AsyncImage structure, you can force SwiftUI's accessibility engine to completely disregard the entire layout block by applying .accessibilityElement(children: .ignore) directly to the outer boundaries.

Replace your current AsyncImage block with this: 

AsyncImage(url: URL(string: "https://hws.dev/img/cupcakes@3x.jpg"), scale: 3) { image in
    image
        .resizable()
        .scaledToFit()
} placeholder: {
    ProgressView()
}
.frame(height: 233)
// This line tells VoiceOver: "Ignore absolutely everything inside this layout element"
.accessibilityElement(children: .ignore)

Why this works:

Instead of trying to hide the individual components inside, .accessibilityElement(children: .ignore) creates a shield over the entire AsyncImage box. When VoiceOver crawls down your screen, it encounters this container, sees the instruction to ignore all children, and skips clean over it directly to your "Your total is..." text.

Side Note: Fix a Small Currency Typos

While looking over your code, I noticed a tiny typo in your currency text string: 

Text("Your total is \(order.cost, format: .currency(code: "UDS"))")

You have written "UDS". You'll want to swap that to "USD" for United States Dollars so the local device can render the ＄ symbol and format the cents correctly!
