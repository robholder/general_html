Swift - Haptic Feedback

Here is a comprehensive reference sheet detailing the modern SwiftUI .sensoryFeedback API, the legacy UIKit fallback, and a design guide on when to use each style.

📱 SwiftUI Sensory Feedback Reference Sheet

In iOS 17 and later, SwiftUI introduced the .sensoryFeedback() view modifier. It is the gold standard for haptics because it decouples your heavy application logic from the hardware layer, automatically handles engine latency, and respects system accessibility settings out of the box.

1. The Core Anatomy: How it Works

The modifier requires two things: a Feedback Style (what type of vibration to play) and a Trigger (a value it watches). Whenever the trigger value changes, the haptic plays instantly. 

Text("Value: \(result)")
    .sensoryFeedback(.impact(weight: .medium), trigger: result)

2. The Comprehensive Haptic Catalog

SwiftUI provides three primary categories of sensory feedback: Impacts, Selections, and Notifications.

🛠️ Category A: Impacts (Physical Collisions)

Impacts mimic real-world physical objects colliding. They are perfect for buttons, dropped items, or layout snaps.

SwiftUI Code Hardware Profile Best Used For
.impact(weight: .light) Small, sharp tap Minor UI buttons, typing on a keyboard, toggle switches.
.impact(weight: .medium) Noticeable, solid pop Rolling standard dice (d6), dismiss adjustments.
.impact(weight: .heavy) Deep, thudding tap Rolling massive dice (d20), a dropped heavy card UI element.
.impact(weight: .rigid) Very sharp, metallic click Drag-and-drop boundary collisions, mechanical clicks.
.impact(weight: .soft) Muffled, fleshy tap Pad drumming, organic taps, sliding past subtle milestones.

🎛️ Category B: Selections (Continuous Adjustments)

Selection feedback is a microscopic, light "tick." It is designed to be played rapidly in succession as the user adjusts a value.

  • SwiftUI Code: .sensoryFeedback(.selection, trigger: currentSelection)
  • Best Used For: Wheel pickers scrolling by, sliders moving along intervals, or changing the number of sides on your dice via a dial.

📢 Category C: Notifications (Status Deliveries)

Notification haptics combine multiple rapid pulses to communicate a distinct outcome to the user without them needing to look at the screen.

SwiftUI Code Rhythm Pattern Meaning / Intent
.success Two quick, crisp taps Action completed, database saved, payment processed.
.warning Two slightly heavier, slower taps A non-fatal issue, a boundary limit hit (e.g., trying to scroll history past 50 items).
.error Three rapid, heavy, chaotic vibrations Severe failure, transaction denied, network lost.

3. Dynamic Adjustments (Advanced Customization)

If standard weights aren't exactly hitting the mark, you can customize the intensity and flexibility of an impact: 

// Fully customized impact intensity (0.0 to 1.0)
.sensoryFeedback(.impact(weight: .heavy, intensity: 0.7), trigger: result)

// Flexing feedback based on variables (plays success haptic ONLY if roll is maxed out)
.sensoryFeedback(trigger: result) { oldValue, newValue in
    if newValue == sides {
        return .success
    } else {
        return .impact(weight: .medium)
    }
}

4. Imperative Fallback: The UIKit Method (For Legacy/Closure execution)

If you are writing code inside an action where you do not want to watch a state variable, you must fall back to the UIKit class UIImpactFeedbackGenerator.

⚠️ Crucial Reference Note: You must always call .prepare() ahead of time. Without it, the system leaves the Taptic Engine in a sleep state to save battery, causing an annoying 50–100ms latency gap between the user's touch and the physical rumble. 

Button("Legacy Tap") {
    let generator = UIImpactFeedbackGenerator(style: .medium)
    generator.prepare() // ⚡️ Wakes up hardware engine ahead of time
    
    // Perform complex background math or networking here...
    
    generator.impactOccurred() // Executes cleanly
}

📐 Haptic Design Best Practices

  • Don't Overdo It: Haptics are like salt in cooking; a little elevates the experience, too much ruins it. If an app vibrates on every single button tap, users will find it draining and switch it off.
  • Match Visuals to Tactility: If a view falls quickly from the top of the screen and hits the bottom, use a .heavy or .rigid impact. If a view gently fades in, use no haptic at all or a .soft pulse.
  • Respect User Preferences: The SwiftUI .sensoryFeedback modifier automatically silences itself if the user turns off "System Haptics" in their iOS Settings app. Never rely on a haptic to deliver critical app information - always pair it with a visual queue.
