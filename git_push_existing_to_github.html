The Git Secret:

Here is a reference guide for pushing an existing local Git repository to GitHub, structured for quick scanning whenever you need a refresher.

🧭 Quick Reference: The Standard Steps

If your local repository is ready and you just need the exact commands, follow this sequence: 

# 1. Navigate to your local project directory
cd /path/to/your/local/repo

# 2. Rename your default branch to 'main' (if it isn't already)
git branch -M main

# 3. Link your local repo to your remote GitHub repo
git remote add origin git@github.com:YOUR_USERNAME/YOUR_REPO_NAME.git

# 4. Push your code and set the upstream tracking branch
git push -u origin main

🛠️ Detailed Walkthrough

Step 1: Create the Empty Repository on GitHub

  • Go to github.com and click the + icon in the top-right corner, then select New repository.
  • Repository name: Type your repository name (ideally matching your local folder name).
  • Public/Private: Choose your visibility preference.
  • ⚠️ CRITICAL: Leave "Initialize this repository with:" completely unchecked (do not add a README, .gitignore, or License). Since your local repository already has its own files and history, creating these files on GitHub will cause a "divergent history" error when you try to push.
  • Click Create repository.

Step 2: Grab the Remote URL

Once created, GitHub will show you a Quick Setup page.

  • Look at the top of the Quick setup section and select the SSH toggle button (rather than HTTPS).
  • Click the copy icon next to the repository URL. It will look like this:
    • git@github.com:username/repo-name.git followed by a copy icon (two overlapping squares).

Step 3: Link and Push via Terminal

Open your Terminal, navigate to your folder, and link it: 

# Link the remote URL to the shortcut name "origin"
git remote add origin git@github.com:YOUR_USERNAME/YOUR_REPO_NAME.git

# Verify the link was successful
git remote -v

# Optionally set the branch to 'main'
git branch -M main

Finally, push your code. The -u flag sets the "upstream" tracking, meaning for all future updates, you can simply type git push or git pull. 

git push -u origin main

You should see the progress as a series of reports as the files are pushed to the remote repository.

Use git push to push updates to the remote repo, and git pull to retrieve updates to the local repo.

🔑 Secure Authentication: Setting Up SSH Keys

If GitHub rejects your push with a Permission denied (publickey) error, your Mac needs its SSH keys generated and registered to your GitHub profile.

1. Check for Existing Keys

First, check if your Mac already has an active key bundle: 

ls -al ~/.ssh

If you see files named id_ed25519 and id_ed25519.pub, skip to Step 3. If the folder is empty or doesn't exist, proceed to generate a new pair.

2. Generate a New SSH Key

Run the following command (replace with your GitHub email address). Press Enter to accept all the default file location prompts: 

ssh-keygen -t ed25519 -C "your_email@example.com"

3. Add the Key to your Mac's SSH Agent

To make sure macOS securely holds your key passphrase in your keychain so you don't have to re-type it constantly, activate the agent: 

# Start the agent in the background
eval "$(ssh-agent -s)"

Next, check if you have an SSH configuration file: 

open -e ~/.ssh/config

If it doesn't exist, create it (touch ~/.ssh/config) and paste this standard setup inside it using a text editor: 

Host github.com
  AddKeysToAgent yes
  UseKeychain yes
  IdentityFile ~/.ssh/id_ed25519

Now, save that configuration and officially load the key into your system agent: 

ssh-add --apple-use-keychain ~/.ssh/id_ed25519

4. Link the Key to GitHub

Copy your public key string directly to your clipboard: 

pbcopy < ~/.ssh/id_ed25519.pub
  • Go to GitHub, click your profile picture in the top right, and select Settings.
  • On the left sidebar, click SSH and GPG keys.
  • Click the green New SSH key button.
  • Give it a descriptive title (e.g., "M3 MacBook Pro") and paste your clipboard contents into the Key box.
  • Click Add SSH key.

5. Test the Connection

Verify everything is linked perfectly by running: 

ssh -T git@github.com

If successful, you will see: 

Hi username! You've successfully authenticated, but GitHub does not provide shell access.`

You are now completely cleared to run your git push command flawlessly!
