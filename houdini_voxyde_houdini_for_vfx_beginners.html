Tools & Navigation covers shortcuts

I didn't realize that to get the navigation back (to tumble and move) in the viewport, the space bar only works when you are in handle mode. You have to use <esc> to get control back if you have any other tool/mode.

Voxyde shows the geometry containers in Network View, and explains the Viewport visibility options in the drop-down (when hopping in and out of Geometry nodes to work inside):

  • Show All Objects
  • Ghost Other Objects
  • Hide Other Objects:
    • This not only 'hides' other objects from view, it just doesn't load them either.

Tab

Tab shows you different tools to create depending on context or viewport.

Best newbie tip:

  • You only have to type the first letter or so of each work that makes up a node type name.
  • at ad f will set you up to create Attribute Adjust Float in SOPs for example.
  • atv will setup to create an Attribute VOP node.

Node options

R on a node, to set the Display flag.

Regardless of the node with the Display flag, you can select a node to do edits at that level (<enter> for the handle tool, or to show the tools and options on the top bar of the Viewport).

Operator Types

  • SOPs - Surface operators
  • VOPs - Vector operators
  • DOPs - Dynamic operators

TIP

Instead of saying 'at the geometry level' start learning to think as 'in SOPs'.

Attributes

Quote:

Houdini attributes are the most important part of Houdini. Your ability to create and modify attributes is the single most important thing that will determine your success as a VFX artist. An attribute - in the simplest terms - is just a piece of information that you can attach to geometry. This attribute once you created will transfer down over to all of the other nodes.

The Attribute Adjust Float and Attribute Noise Vector are introduce, showing how you can supply the Attribute Name and perform various changes to the attribute.

The Attribute Create node was shown (to lay down an attribute on a network), but you don't really need to do that for the most part, as the 'adjust' nodes can also create an attribute if the named attribute didn't already exist.

Interesting tip: if the attribute name field has what looks like a Google Maps location marker (an inverted rain drop shape), clicking it on will act as a visualizer for the Viewport and show the attribute in an appropriate manner.

Also, if you have an attribute node selected in the Network Editor, pressing <enter> in the Viewport will give access to the Handle for that adjustment.

The Attribute Adjust Float example shows an object, and this node is set to Operation of Line, and in the Viewport, the Handle is evoked, revealing a start and end point for a linear gradation (shown as a spectrum when visualized). Looking at the Geometry Spreadsheet reveals the values of zero to one being applied - in this case to a new attribute called 'mask'.

An Attribute Noise node is then used to illustrate how the 'mask' attribute can be added into the Blend field, to drive the efficacy of the noise.

Binding Attributes

I haven't got the full picture, but on the Poly Extrude node, the tutorial shows how a created attribute ('zscale') is used to drive the distance of each poly extrude. Instead of a Maya style 'binding' on the distance slider, there is a section lower down the parameter pane (Local Attributes) to activate an attribute assignment to the parameters (and in this case there is a 'Direction' field. The name of the attribute is entered into the field, and since the poly extrude works on primitives, the attribute was created as a primitive attribute (even though the default was 'points').

Attribute VOP

Typing Attribute in the Tab (create node menu) shows a ton of 'Attribute <blah>' nodes.

Instead of learning all these nodes, the tutorial suggest all can be replaced with Attribute VOP, and understanding this node will be all you need to know.

The previously mentioned Attribute nodes (above) can all be replaced with the 'Attribute VOP'. However, sometimes, the pre-built variations can still be used, and often save 'set up' time.

This [Attribute VOP] is the single most important node you have to learn in Houdini.

While VOPs means 'vector attributes', the various 'attribute vop' nodes can operate on floats, ints, vectors or any king of attribute.

Inside

Inside the Attribute VOP node, there are two nodes by default:

  • Input: geometryvopglobal
  • Output: geometryvopoutput

Attributes of parameter input/output widgets are 'colored' according to the data type:

  • green is Vector
  • teal is Float
  • blue is Integer
  • gray (slightly orange) can be custom external Inputs

VOPs Operations

You can re-create the functionality of the Attribute node varieties by connecting nodes inside the Attribute VOP node.

If you use a Noise node, the Turbulent Noise seems to be the most commonly used.

Voxyde shows a default Turbulent Noise node connected directly to the P (position) input node of the Output node, and shows how it doesn't work (the geometry disappers). The reason is that the noise attribute is a 'float' and the 'P' (position) input is expecting a 'vector' data type - and this is were the color codes come into play. The teal (float) doesn't necessarily play well with the green (vector) input.

To fix this, on the Turbulent Noise node, the Signature has to be switched (via drop-down) to 3D Noise. (The color of the 'nose' output widget changes to green.

Creating an Add node enables creating a similar action to the Attribute Noise node.

  • P goes from the input node through the turbulent noise, this then goes into input 2 of the add, and on to the output (P)
  • Add also takes in P from the input node (to its input 1), and the 'sum' output goes onwards to the output node's 'P' input.

The tutorial shows how nodes like the Constant can be substitutes for the sliders on attributes of nodes by plugging into the slots available.

TIP: <shift> R reverses the order of the inputs.
