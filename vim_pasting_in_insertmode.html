Pasting in Insert mode

You can paste the yanked text while you're in insert mode using a combination of Vim commands. Here’s how to do it:

Use <C-r> (Control + r):

  • In insert mode, press <ctrl> r followed by the register name to paste the yanked text.
  • The 'unnamed' register is ":
    • So you can use <ctrl>r " to paste the yanked text (although this get's replaced by deleted text in addition to 'yank').
  • The dedicated 'latest yank' register is 0
    • So you can use <ctrl>0 to guarantee the latest 'yanked' text.

Here’s a step-by-step example:

Yank & Paste

Yank the text in normal mode (e.g., yy to yank a line).

  • Enter insert mode (e.g., press i).
  • Press <ctrl> r followed by " to paste the yanked text.

Regular register

So, the sequence would look something like this:

  • yy --> Yank the current line
  • i --> Enter insert mode
  • <ctrl>r" --> Paste the yanked text while in insert mode

Other register

If you yanked the text into a different register (e.g., register a), you can use Ctrl + r + a to paste it:

  • "ayw --> Yank a word into register 'a'
  • i --> Enter insert mode
  • <ctrl>-ra --> Paste the yanked text from register 'a' while in insert mode

This method allows you to insert the yanked text seamlessly as you type in insert mode.

🔧 Using Vim To Replace An Old Path With A New Path In A File

  • First, copy a path (from terminal or document) onto the system clipboard. (This is the new path.)
  • In the Vim buffer (within which you want to replace paths) place the cursor over the existing path
  • Then construct a replacement string like this: 
:%s@<C-r><C-a>@<C-r>+@g

...where means the Control and R key simultaneously, etc.

Explanation

  • %s → edit whole buffer
  • '@' → safe delimiter (avoids escaping /)
  • <C-r><C-a> → inserts the path under the cursor (Control-R, Control-A)
  • <C-r>+ → inserts the system clipboard (your new path which you copied before commencing the edit.)
  • g → replace all occurrences on each line

This gives you a fully dynamic, cursor‑driven substitution.

⭐ Why this method is excellent

  • You don’t need to type or escape the old path
  • You don’t need to type or escape the new path
  • You avoid delimiter collisions
  • You avoid regex surprises
  • It works even with extremely long or ugly paths
  • It’s repeatable and safe

This is exactly how experienced Vim users handle path‑heavy editing.

🔍 One Small Improvement (Optional But Recommended)

If the path contains regex‑special characters (like ., +, ?, *, [), Vim will treat them as regex unless you escape them.

To avoid that, add \V (very‑nomagic) at the start of the pattern: 

:%s@\V<C-r><C-a>@<C-r>+@g
  • \V tells Vim: “treat everything literally unless escaped.”

This prevents surprises like . matching any character.

🧠 A Smooth Workflow For Replacing Paths

  • Copy the new path from terminal or file (Ctrl+C)
  • In Vim, move the cursor onto the old path
  • Type: 
:%s@\V<C-r><C-a>@<C-r>+@g
  • Hit Enter
  • Done

If it worked, you’ve replaced every instance of the old path with the new one, without typing either.

⭐ Yanking A Whole Path In Vim

When you’re sitting in the middle of a long Linux path, Vim doesn’t treat it as a single “word”, so the usual motions (yiw, yaw, etc.) don’t grab the whole thing. Fortunately, Vim gives you a couple of excellent tools that do understand filename‑like text.

Here are the most reliable approaches.

⭐ Best all‑around method: use in command‑line or insert mode

<C-a> means “the WORD under the cursor, using filename rules”.

So to yank the entire path:

  • Move the cursor anywhere inside the path
  • Enter Normal mode
  • Use: 
"ayy<C-r><C-a>

…but that’s clunky. The cleaner way is:

✔ Yank the whole path into a register:

yi<C-a>

This works because <C-a> is a motion in Normal mode that selects the filename‑like text under the cursor.

Try it:

  • Cursor anywhere inside /very/long/path/to/file.txt
  • Type: 
yi<C-a>

Now the unnamed register " contains the entire path.

⭐ Even simpler: use vi then y

If you prefer visual selection: 

vi<C-a>y

This selects the whole path and yanks it.

⭐ Why works so well

<C-a> in Normal mode is a motion that expands to:

“the WORD under the cursor, using filename characters (letters, digits, /, ., _, -, etc.)”

So it naturally grabs full paths, URLs, filenames, and similar text.

⭐ Alternative: use yiW (less precise)

W treats anything separated by whitespace as a WORD: 

yiW

This works if your path has no spaces and no trailing punctuation, but is much more accurate.

⭐ Summary

For yanking a whole path when your cursor is in the middle of it:

✔ Best:

yi<C-a>

✔ Visual version:

vi<C-a>y

✔ Works but less precise:

yiW
