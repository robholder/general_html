Intro

YouTube - Intro

This YouTube tutorial uses work by Alex Valentina as inspriation, and explores the Karma Renderer (in Houdini 19.5 at the time of making).

Drawing The Outline Curve

YouTube - Drawing The Outline Curve

To create the outline of the geometry, start with a regular Geometry node in the Obj context.

  • Inside the Geometry node:
    • Drop a Curve node:
      • Set the Primitive Type to Bezier to get curved results.
      • Hit <enter> in the Viewport (to activate the handle/tool)
      • H key to select Auto Bezier Draw Mode
      • F key for edit mode
      • <space> 3 to get a front view.
      • Draw the shape and end by clicking back at the first point.
      • <esc> to return to regular Viewport mode.

Preparing The curve

YouTube - Preparing The curve

Next, we need to prepare the curve for use as a polygon generator.

  • Drop a Resample node:
    • Reduce the Length parameter:
    • Switch on the points icon on the Viewport's right-most tool bar.
      • Try 0.01
      • The points should be very densely spaced.
  • To form a uniform mesh:
    • Drop a Remesh node:
      • It might not be very clean at first:
        • Target Size to 0.01 (Zoom in to see the mesh)
      • In the tutorial, there were some polygons that were squished up in areas:
        • Adjust the Iterations (to 6 in the tutorial)

Grouping The Edge

YouTube - Grouping The Edge

Next we'll extrude.

  • Drop a PolyExtrude node:
    • Slight extrude - Distance of 0.01
    • Extrusion tab:
      • Check 'On' the Output Back
  • Check 'On' the Side Group option:
    • This will create a group of all the edge facing polygons of the extrude.
    • You can leave the extrudeSide name.
  • Add a Group Promote node:
    • Convert From to Auto to Points
    • Group Name to extrudeSide
    • New Name to pin

We'll use the group of points to tell Vellum to leave these where they are when inflating the balloon.

Setting Up A Vellum Balloon

YouTube - Setting Up A Vellum Balloon

Next we'll set up a Vellum Balloon simulation.

  • Drop a Vellum Configure Balloon network:
    • Connect the output from the Group Promote node into input 1
    • In the parameters for the vellumcloth node:
      • Scroll down to Pin to Animation:
        • Pin Points to pin (group)
      • Scroll down to the Stretch section:
        • Stiffness - change the scale dropdown (from 100) to 10 000
        • Damping Ratio to 0.001
        • Rest Length Scale to 1.2
      • Scroll down to Bend section:
        • Stiffness - change the scale dropdown (from 0.1) to 1e-9
    • In the parameters for the vellumpressure node:
      • Scroll to the Stretch section:
        • Stiffness - chanage the scale dropdown (from 100) to 1 000 000
        • Rest Length Scale to 12

Vellum Solver Setup

YouTube - Vellum Solver Setup

Continuing with the chain.

  • Drop a Vellum Solver:
    • Solver tab:
      • Set the Subsets to 2
      • Set the Constraint iterations to 32
    • Forces tab:
      • Set the Gravity to 0, 0, 0
      • Disable the Wind
      • Velocity Damping to 0.1
    • Simulation tab:
      • This is system dependant, but the tutorial boosts the memory allocation:
        • Cache Memory (MB) from 5 000 to 25 000

IMPORTANT: Save the file; this is where it's prone to crashes.

Hit play on the timeline to see the simulation run.

  • It takes around 40 frames to settle down in the tutorial.
  • Stop the playhead at this point.

Post Processing The Simulation

YouTube - Post Processing The Simulation

Add a Vellum Post-Process node after the simulation nodes.

  • Spatial Blur to 0.2
  • Subdivision to Loop

Measuring Mesh Curvature

YouTube - Measuring Mesh Curvature

In the tutorial, a Measure node is dropped. But I don't know for what purpose.

    • Measure to Curvature

To finish, drop a Null:

FAQ

What is the primary goal of the tutorial series?

The primary goal of this tutorial series is to explore rendering techniques within Houdini using Karma and Karma XPU, focusing on new features in Houdini 19.5. It aims to recreate abstract artworks, initially created in Photoshop, using Houdini’s procedural tools and simulation capabilities.

How does the geometry creation process begin in Houdini?

The process starts by creating a geometry node and diving into it. A curve node is then used in an orthogonal view to draw abstract, plant-like shapes using Bezier curves. The curves are then resampled to achieve a uniform point distribution, and a remesh node converts the curves to a uniform triangular mesh.

What techniques are employed to give the initial 2D shape a 3D appearance?

The initial 2D mesh is extruded using the "polyextrude" node to give it thickness, creating a 3D plate. The extrusion is kept very small (0.01 units) and then the side polygons of the extrusion are grouped.

How is the Vellum simulation used to inflate and shape the geometry?

A Vellum Balloon constraint is employed, which configures how the surface behaves and applies pressure for inflation. Key adjustments include pinning outer points to maintain their original position using a group created from the extrusion, setting higher stiffness for cloth and pressure, and low bend stiffness to encourage wrinkling. Additionally, a high rest length scale is utilized to allow for expansion.

What modifications are made to the Vellum solver to achieve the desired simulation?

The Vellum solver is modified by decreasing the maximum iterations to 32, increasing substeps for each frame, and disabling gravity and wind forces. Velocity damping is used instead. Memory caching is increased to allow for more data during the simulation.

What post-processing is done after the Vellum simulation and why?

A Vellum post-process node is used to smooth the result with a spatial blur and to subdivide the geometry using loop subdivision. This post-processing is done to improve the visual quality by creating a smoother, higher-quality mesh.

What attributes are calculated and stored on the final geometry before rendering?

A measure node is used to calculate and store the curvature of the geometry on points, and then a normal node calculates point normals for use in shading. These attributes are crucial for controlling shading and rendering in Karma.

What is the significance of the final 'out' null node in the scene?

The final 'out' null node serves as a clean output point for the geometry. This allows for easy selection of the final geometry which can then be shaded, lit, and rendered in Karma for visualization, using the attributes that have been calculated in the previous steps.

Glossary of Key Terms

Key Term Description
Houdini: A 3D animation and VFX software known for its procedural approach to content creation.
KarmaXPU: Houdini's native rendering engine.
Geometry Node: A container node in Houdini that holds operations for creating or modifying geometry.
Curve Node: A node used to create and edit curves, often used as a base for building complex shapes.
Bezier Curve: A type of curve defined by control points, allowing for smooth shapes.
Resample Node: A node that uniformly samples a curve, creating points at even intervals.
Remesh Node: A node that rebuilds the mesh geometry, often used to create a uniform distribution of triangles.
Polyextrude Node: A node that extrudes a polygon surface, adding thickness.
Group Promote: A Houdini node used to convert an existing group into a different grouping type.
Vellum Balloon Constraint: A constraint used to simulate inflated shapes in Houdini.
Vellum Cloth: A component of the Vellum solver in Houdini that defines the properties of the surface.
Vellum Pressure: A component of the Vellum solver that defines the amount of inflation force applied to the cloth.
Vellum Solver: A node that simulates dynamic systems, such as cloth, hair, and soft bodies.
Velocity Damping: A simulation setting that reduces the speed of dynamic motion over time.
Vellum Post Process: A node that performs post-simulation operations, such as smoothing and subdivision.
Spatial Blur: A technique to smooth a mesh by blurring the spatial locations of vertices.
Loop Subdivision: A method of recursively subdividing triangle meshes to increase their density and smoothness.
Measure Node: A node that measures and stores data about the geometry, such as curvature.
Normal Node: A node that calculates and stores surface normals, used for shading and rendering.
Point Cloud: A set of data points in 3D space, used to represent geometry.
Null Node: A non-operational node in Houdini used as a placeholder or for organization.
Orthogonal View: A view in which the viewport is perpendicular to one of the main planes, often the XY, XZ or YZ plane.
Substeps: Simulation iterations run in between frames of a simulation.
Stretch Stiffness: A simulation setting that controls how much an object can be stretched.
Rest Length Scale: A parameter that controls the relative size of an object in a simulation.
Bend Stiffness: A parameter that controls how much an object resists bending in a simulation.
Cache Memory: RAM allocated by the simulation software.
Concave: A surface that curves inwards.
Convex: A surface that curves outwards.
