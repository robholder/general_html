Nuke Refresher

Webinar/Zoom/Seminar 3/25/2022

  • Alex Nijmeh
  • Ken Knobel - San Francisco
  • Terry Riyasat - Toronto

Disney lighting - using Nuke v12.2 - which is the last of the 12 releases

Nuke 13.0 & 13.1 - subject of talk

Talk

  • Monitor out:
    • Has its own Workspace setting
    • Makes a monitor as a floating 'no UI' window
    • Options:
      • Device pop-up options
      • Bit depth
      • Viewer choice as output:
        • or interactive (switches to live viewer)
      • Option to use Viewer adjusts (gamma/exposure)
      • Colorspace transform
      • HDR capable
  • HYDRA (Storm) Viewer:
    • New 3d space viewer
    • Lights, Materials, shadows
    • Much closer now to the Scanline Render (but not 'one-to-one')
    • (Currently being developed/extended to MacOS)
  • 3d Interaction improvements:
    • Snapping
    • Manipulators
  • USD:
    • Imports USD files through 'Read'
    • Options for single or separate node imports
    • Axis nodes can match USD elements:
      • shows location of item from USD hierarchy
      • (Center or vertex options)
  • Timeline Tools:
    • Hiero Review:
      • Sync review
      • Remote sync
      • UI/Tool update in tandem
  • Machine Learning:
    • CopyCat:
      • "Ground Truth":
        • Feed featuring target 'look'
      • "Input":
        • image to apply machine fixes to
      • Set Data directory on disk
    • Example given for actor-dialog changes (on plates)
    • Example given for tracker/marker cleanup
    • Upscale
    • Deblur
    • (Look on Foundry Learn for details/examples)
  • Cryptomatte:
    • Native in v13
    • (Must remove other Cryptomattes to use natively)
    • Encryptomatte:
      • Ability to add items into Cryptomatte stream:
        • example shown: adding cup to scene
  • Performance Improvements:
    • Transforms are quicker
    • Reading/Writing quicker
    • Opening scripts to work status
  • Framehold:
    • Now has 'set to current frame'
  • Color management:
    • Now in can be set in Preferences
    • This means it can be an uber-settings (for whole pipeline compliance)
    • Customization of available OCIO options in UI (prioritisation)
  • Industry Standards:
    • VFX Reference update:
      • VFX Ref Platform 2020
    • Python 3.7x
    • QT 5.12x
    • PyQt 5.12
    • QT for Python (PySide 5.12/5.13)
    • (Nuke might not launch if pipeline is using python 2!)
  • Insertions of multiple nodes will work more intuitively
  • Shake to disconnect nodes (Preferences option)
  • Pipeline and Standards:
    • OCIO v2
    • OpenColorIO v2.0 library
    • GPU implementation
    • Native ACES:
      • ACES 1.2
      • Backward compatible
    • Support for .clf .ctf
  • Blackmagic RAW
  • AIR Tools:
    • Machine Learning updates
    • CatFileCreator:
      • CatFileCreator node <-> pytorch
      • Inference
  • Unreal support:
    • Unreal server
    • Nuke <-> Unreal
    • Pull in sequence or maps
    • Unreal Reader node:
      • Map
      • Sequence
      • Cameras
      • Various Options
      • Render pass selection:
        • feed into Nuke channels (Viewer)
      • Control Variables management
      • File -> export to Unreal
    • Use Position/Camera tools to add/augment/replace
    • Stencils:
      • adjust items in the scene (example; one character) independently
      • To come:
        • Stencils will work like Cryptomatte picking
    • Render Cube-maps (environments):
      • Can record positions to create environment point-clouds
  • Nuke Interop:
    • Katana <-> Nuke
    • Headless Nuke can run comps (from Katana)
    • Example: glows back into Katana renders

To Come

  • Top down rendering (as opposed to Scanline rendering):
    • Potential speed gains
  • Pivot point rotations/match to selections/free rotate
  • OTIO support
  • NDI network protocol:
    • streaming/remote sessions
  • Multi-monitor outputs
  • ARRI and Blackmagic RAW 2.2 updates
