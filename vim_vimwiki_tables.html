Vim - Tables

1. The "Automatic Align" (The Tab Key)

The most important rule in Vimwiki tables is:

  • Never space things manually.
  • Just type | Header | and press Tab.

Vimwiki will automatically create the trailing pipe, align the columns, and move your cursor to the next cell.

If you delete text or add a long word, pressing Tab anywhere in the table will "snap" the entire grid back into perfect alignment.

2. Header Rows & Dividers

To turn a top row into a "Header" (which triggers CSS styling if present), you must provide a separator row. 


| ID | Description | Status |
|----|-------------|--------|
| 01 | Fix CSS     | Done   |

It'll appear like so as HTML:

ID Description Status
01 Fix CSS Done

Tip: You only need one dash |-| for the divider to work; when you press Tab, Vimwiki will expand the dashes to match the column width.

3. Column Alignment

You can control the text alignment of an entire column by adding colons : to the separator row.

  • | :--- | Left-aligned (The default).
  • | --- | Centered (Excellent for status icons or numbers).
  • | ---: | Right-aligned (Best for currency or dates).

4. Spanning Multiple Cells (Cell Merging)

Vimwiki supports a "fake" merge. If you want a cell to "flow" into the next one or look like it’s merged vertically:

Horizontal Merge: Use >. 

| This spans two | > |

Vertical Merge: Use \. 

| Row 1 |
| \     | (This indicates the cell above continues here).

5. Converting Existing Text to a Table

If you have a list of comma-separated values (CSV) and want to turn them into a table instantly:

Select the lines in Visual Mode (V).

Type :VimwikiTable (or use the shortcut \\dt in most setups).

It will convert the commas/tabs into a perfectly aligned table.
