Git - Retrieving All Branches

QUESTION

How do I update my local repo so that it has a current copy of all branches on the remote repo?

ANSWER

To update your local repository with the current state of all branches from the remote, use the following command: 

git fetch --all

What this does

  • Downloads Metadata: It retrieves the latest commits, files, and branch names from all configured remotes (including origin).
  • Updates Tracking Branches: It updates your local "remote-tracking branches" (like origin/main, origin/dev, etc.), which are read-only copies of the remote's state.
  • Safe by Design: This command does not merge any code into your working files or change your currently checked-out branch, so it won't cause merge conflicts.

Helpful Variations

Clean Up Deleted Branches: If branches were deleted on GitHub and you want them gone from your local tracking list too, add the prune flag: 

git fetch --all --prune
  • Check the Results: After fetching, you can see all available branches (both local and remote) with: 
git branch -a

How to actually "get" the code

git fetch only updates the tracking branches. If you want to actually update your local working branch to match the newly fetched remote version, you must then merge or reset:

  • Merge (Safe): git merge origin/your-branch-name.
  • Reset (Aggressive): git reset --hard origin/your-branch-name (this overwrites any local changes to match the remote exactly).

Handle Diverged History (If Necessary)

If your local main branch has diverged significantly from the original origin/main, you may want to reset your local main to match the remote exactly after you have safely pushed your changes to the new branch: 

git checkout main
git reset --hard origin/main
Warning: Only do this after you are sure your changes are safe on your new branch, as this will overwrite your local main with the version from GitHub.
