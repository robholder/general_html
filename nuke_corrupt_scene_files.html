Corrupt Nuke Scene Files

Sometimes, on opening a scene file, half the nodes are missing, and the Nuke file is corrupt.

URLs

  • Troubleshooting Nuke Scripts
  • Foundry Forum posts
  • My v12 scene got corrupted. Wouldn't render on the farm, reopened and half the nodes were lost.

Nafisa Baker - took a look (live on a Zoom call!) and fixed it.

  • Groups:
    • In the text file:
      • Nafisa looked for non-closed groups.
      • In the corrupt scene file, one of the last nodes on a chain was 'painterly_pmnt1':
        • Nafisa added the 'end_group' line.
  • Upon opening the edited file in Nuke:
    • In the script editor's window:
      • "can't read "C1e6a4c00": no such variable"
  • Nafisa then searched for the variable in the file (text edit):
    • For example: 
clone $C9412bd90 {
xpos -2230
ypos 1066
selected false
}
  • Delete this 'clone' node.
  • Nafisa opened again in Nuke, and found another "can't read..." error for another clone.
  • Nafisa deleted this clone block also.
  • Opening scene again, and everything was fine, except for a corrupt Painterly_pmnt group node, and some disconnected pipes in the area.
