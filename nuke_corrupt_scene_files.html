Corrupt Nuke Scene Files

Sometimes, on opening a scene file, half the nodes are missing, and the Nuke file is corrupt.

Walk Through of a Successful Fix

  • My scene file got corrupted. It wouldn't render on the farm, and when I reopened the scene after quitting, half the nodes were lost.

Nafisa Baker - took a look (live on a Zoom call!) and fixed it.

  • Groups:
    • In the text file:
      • Nafisa looked for non-closed groups.
      • In the corrupt scene file, one of the last nodes on a chain (where previously the stream continued) was a 'painterly_pmnt1' Group node:
        • Nafisa added the 'end_group' line where it should be.
  • Upon opening the edited file in Nuke:
    • In the script editor's window message window:
      • "can't read "C1e6a4c00": no such variable"
  • Nafisa then searched for the variable in the file (text edit):
    • For example: 
clone $C9412bd90 {
xpos -2230
ypos 1066
selected false
}
  • This block of the 'clone' node was deleted.
  • Nafisa opened the saved file in Nuke again, and found another "can't read..." error for another clone.
  • Nafisa deleted this clone block also.
  • Opening scene again, and everything was fine, except for a corrupt Painterly_pmnt group node, and some disconnected pipes in the area area of the Group node..
