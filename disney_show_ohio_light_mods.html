ExternalLink: Light Modifiers - Part 1

Light Modifiers

Light mods are

  • TMODS:
    • Transport modifier
    • Controls how light is "transported" from Source to Target.
    • Boost Lower or Tint (influence) on objects
      • direct diffuse
      • direct specular
      • indirect diffuse
      • indirect specular
    • Boost/Lower on:
      • volumetrics
      • caustics
    • Object influences (on other objects)

TMODs

The parm panel for TMODs have Sources & Layer tabs.

Sources:

  • Lights

Layers:

  • Objects:
    • Include
    • Exclude
  • Create in:
    • Shot Modifications Backdrop
    • <Tab> and start to type Transport Modifier
    • Node is called transportmodifier
    • Naming:
      • TMOD_<light_being_modified>_<what_is_affected>
      • TMOD_<sourc>_<target>
      • TMOD_SUN_ground

Add Light/s

  • In Scene Graph Tree panel or viewport:
    • Select light/lights
    • e.g., Key
  • In transportmodifier parm panel:
    • Sources tab:
      • Click Source Includes button
      • In dialog, pick light

Add Objects/items

  • In Scene Graph Tree panel or viewport:
    • Select objects
    • e.g., ivy
  • In transportmodifier parm panel:
    • Layers tab:
      • Click Source Includes button
      • In dialog, pick objects

Rename TMOD to match purpose:

  • e.g., TMOD_KEY_ivy

Adjust light by manipulating the swatches on the TMOD parm panel:

  • Diffuse:
    • color
    • exposure
  • Specular:
    • color
    • exposure
  • Direct Diffuse (DD)
  • Direct Specular (DS)
  • Indirect Diffuse (ID)
  • Indirect Specular (IS)
  • Volume (V)
  • Caustics (C)

Adding Layers

You can add layers to adjust different object by different amounts. Each layer will have controls.

Object to Object TMODs

Bounce Light

You can control bounce light by choosing objects as sources rather than lights.

Visibility/Influence

You can use an object as a source, and then add another as a target, and dial the Visibility value (0 to 1, e.g., 0.6)

This will reduce ALL influence from one object to another - shadows, bounce etc.

Volume mode

Example is shown by adding:

  • Scene objects are Sources
  • Volumes in Targets

Boost to get glow on objects

RegionGrads

  • RegionGrad:
    • Region Gradients
    • Areas of influence:
      • intensities (lights or otherwise)
      • adjust colors (lights or otherwise)
      • cut geometry
      • masking volume density
      • caustics
      • culling xgen
      • restrict TMODs
      • adjust material properties
      • create custom Nuke mattes

Create in the Rgrad Shapes backdrop.

  • Tab and type Region Gradient
    • Node is named RegionGradient by default.
  • Transform like a light or object:
    • Snap it to geo like a light (magnet & MMB)

To assign to parameter (to influence):

  • In the object's parameter panel:
    • Other tab
    • Twist down Region Grad section:
      • Enable
      • Layers + to add
        • In Mask field:
          • add name of RGrad (eg RG_stairs)
        • In Intensity field:
          • Add float number:
            • e.g. 0.5
          • Switch math operator:
            • mult
            • add
            • mix (combines RGrads rather then compounds)
            • etc.

To combine in the network

  • Region gradient - user variable node
  • In parms:
    • RegionGradients option
      • switch to RegionGradients
      • Click + to create slots for upstream RGrads
      • Select RGrads (in Scene Graph Tree for example)
      • plus or drag into fields (one per field)
    • Combiner option:
      • switch to Combiner
      • Enter expression in the Value field:
        • e.g., RG_testShape2 - RG_testShape
WARNING: keep ＄OS in the name fields, rename nodes in the Node Graph.

Now you can add the RGrad to a light or whatever.

  • e.g, Other > Region Grad > Layers on a light

Parms

  • Hard Expand -> size of curvature
  • Soft Falloff -> falloff at edge region
  • Size:
    • Scale:
      • x
      • y
      • z
  • S/SizeMinus (x, y, z negative directions)
  • S/SizePlus (x, y, z positive directions)

Naming Convention

  • RG_<area_being_affected>
    • RG_lowerCharacter
  • RG_<area_being_affected>_<descriptor>
    • RG_room_vignetteCeiling

BONUS: Multiple light influenced by RegionGrad

Drop a Region Gradient - Lights Layer node

  • Tab and type Region Gradient - Lights Layer
  • Position node AFTER the lights in the node graph:
    • Shot modification backdrop
  • In Light Prim:
    • Add regex strings to add lights
    • To pick up all lights ending in lgt:
      • /shot/lrigLOCAL/trs_master/trs_aux/model/matPalette/lr_stage/*lgt
  • Add RGrads to the Layers field(s), and these will influence the included lights
  • Adjust with Intensity field number and operation choice
  • Override Existing Layers:
    • Check on if you want to override existing RGrad layers on a light

GeoMasks

Video Frame #: 61768

  • Create dipeModifier node.
  • Drop below the RGrad
  • in the dipe modifier parms:
    • Template:
      • Choose geoMask
    • Lower on the parm panel:
      • Assignments tab:
        • Targets
        • Add object to cut out from
      • Attributes tab:
        • User Vars dropdown:
          • Choose regionGradients
          • + to add layer fields
          • Add Region Grad Prim Shape here:
            • in Scene Graph Tree make sure to look for:
              • <blah>_rgshp items (not matPalette/shader version)
              • They'll be in lrigLOCAL etc.
          • Rename the RegionGradients field at the top of the RegionGradient area in UserVars:
            • RGGeoMask or something
        • In the Custom area of the same parm panel:
          • geoMaskExpr & <operator> RGGeoMask
          • <operator> is perhaps - to subtract.
          • So:
            • geoMaskExpr & -RGGeoMask
NOTE: 'geoMaskExpr' is pre-populated, it's not related to the name you type in directly under the interface 'RegionGradients' label widget.
