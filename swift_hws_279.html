Hacking With Swift - 279 - Challenge

Challenge

Your challenge this time can be easy or hard depending on how far you want to take it, but at its core the project is simple: you need to build an app that helps users roll dice then store the results they had.

At the very least you should lets users roll dice, and also let them see results from previous rolls. However, if you want to push yourself further you can try one or more of the following:

  • Let the user customize the dice that are rolled: how many of them, and what type: 4-sided, 6-sided, 8-sided, 10-sided, 12-sided, 20-sided, and even 100-sided.
  • Show the total rolled on the dice.
  • Store the results using JSON or SwiftData - anywhere they are persistent.
  • Add haptic feedback when dice are rolled.
  • For a real challenge, make the value rolled by the dice flick through various possible values before settling on the final figure.

When I say "roll dice" you don’t need to create fancy 3D effects - just showing the numbers that were "rolled" is fine.

The only thing that might cause you some work is step 5: making the results flick through various values before settling on the final figure. The easiest way to tackle this is through a Timer that gets cancelled after a certain number of calls.

While you’re working, please take a moment to remember the accessibility of your code - try using it with VoiceOver and make sure it works as well as you can make it.

Ai Insight

QUESTION

What's wrong with this: 

import SwiftUI

struct ContentView: View {
    @State private var sides = 6
    @State private var result = 1
    
    var body: some View {
        NavigationStack {
            Button("Roll") {
                $result = Int.random(in: 1...sides)
            }
            Text("\($result)")
        }
    }
}

ANSWER

The issue here lies in using the ＄ sign when reading and changing your @State variables.

In SwiftUI, the ＄ prefix is used to create a two-way binding (usually passed into UI controls like TextField or Picker so they can read and write a value simultaneously).

When you want to simply assign a new value in an action closure, or display a value inside a Text view, you should use the plain variable name without the ＄.

The Corrected Code

Here is how to fix your view: 

import SwiftUI

struct ContentView: View {
    @State private var sides = 6
    @State private var result = 1
    
    var body: some View {
        NavigationStack {
            // 1. Drop the $ here. You are just assigning a plain Int.
            Button("Roll") {
                result = Int.random(in: 1...sides)
            }
            
            // 2. Drop the $ here. Text only needs to read the plain value.
            Text("\(result)")
        }
    }
}

The Quick Rule of Thumb

  • Use result (No ＄): When you are doing math, reading the value to display it, or setting it to a new value inside a button action.
  • Use ＄result (With ＄): Only when you are handing the variable over to another SwiftUI component that needs permission to change it automatically (e.g., Stepper("Sides", value: $sides, in: 2...20)).

A Polished Basic Single View App

ContentView.swift

import SwiftUI

extension Color {
    static let midnightTop = Color(red: 0.11, green: 0.14, blue: 0.20)
    static let midnightBottom = Color(red: 0.07, green: 0.08, blue: 0.11)
    
    //static let auroraTop = Color(red: 0.94, green: 0.96, blue: 0.95)
    //static let auroraBottom = Color(red: 0.86, green: 0.89, blue: 0.90)
    static let auroraTop = Color(red: 0.50, green: 0.55, blue: 0.75)
    static let auroraBottom = Color(red: 0.70, green: 0.75, blue: 0.90)
}

struct ContentView: View {
    @Environment(\.colorScheme) var currentScheme
    
    @State private var sides = 6
    @State private var result = 1
    
    let diceOptions = [2, 4, 6, 8, 9, 12, 20]
    
    var diceImageName: String {
        // Take integer and format it into a 2-digit string string,
        // adding a leading zero if the number is only a single digit.
        let formattedNumber = String(format: "%02d", sides)
        return "dice_sides_\(formattedNumber)"
    }
    
    var body: some View {
        NavigationStack {
            ZStack {
                LinearGradient(
                    colors: currentScheme == .dark
                    ? [.midnightTop, .midnightBottom]
                    : [.auroraTop, .auroraBottom],
                    startPoint: .top,
                    endPoint: .bottom
                )
                .ignoresSafeArea()
                
                VStack(spacing: 30) {
                    
                    // Spacer added to push the layout contents down gracefully
                    Spacer()
                    
                    // --- THE NEW CUSTOM EMBEDDED TITLE ---
                    Text("Dice")
                        .font(.system(size: 64, weight: .bold, design: .rounded))
                        .foregroundColor(.primary)
                        .padding(.top, 10)
                    
                    
                    // DICE IMAGE AREA (Sized up to 240x240)
                    VStack(spacing: 12) {
                        Image(diceImageName)
                            .resizable()
                            .scaledToFit()
                            .frame(width: 240, height: 240)
                            .foregroundColor(.primary)
                            .padding(24) // Added extra padding for the larger image scale
                            .background(
                                Circle()
                                    .fill(Color(.secondarySystemGroupedBackground))
                                    .shadow(color: .black.opacity(0.06), radius: 12, y: 6)
                            )
                            .contextMenu {
                                Picker("Change Sides", selection: $sides) {
                                    ForEach(diceOptions, id: \.self) { option in
                                        Text("\(option) Sides")
                                    }
                                }
                            }
                        
                        Text("Hold image to quick-change sides")
                            .font(.caption)
                            .foregroundColor(.secondary)
                    }
                    
                    // Result Card Block
                    VStack(spacing: 6) {
                        Text("CURRENT ROLL")
                            .font(.caption2)
                            .fontWeight(.bold)
                            .foregroundColor(.secondary)
                            .tracking(1.5)
                        
                        Text("\(result)")
                            .font(.system(size: 64, weight: .bold, design: .rounded))
                            .foregroundColor(.primary)
                    }
                    .frame(maxWidth: .infinity)
                    .padding(.vertical, 24)
                    .background(
                        RoundedRectangle(cornerRadius: 20, style: .continuous)
                            .fill(Color(.secondarySystemGroupedBackground))
                            .shadow(color: Color.black.opacity(0.05), radius: 15, x: 0, y: 10)
                    )
                    .padding(.horizontal, 24)
                    
                    Spacer()
                    
                    // Action Button
                    Button {
                        let impact = UIImpactFeedbackGenerator(style: .medium)
                        impact.impactOccurred()
                        
                        result = Int.random(in: 1...sides)
                    } label: {
                        Text("Roll \(sides)-Sided Die")
                            .font(.headline)
                            .frame(maxWidth: .infinity)
                            .padding(.vertical, 16)
                    }
                    .buttonStyle(.borderedProminent)
                    .controlSize(.large)
                    .padding(.horizontal, 24)
                    .padding(.bottom, 20)
                }
                // --- THE NAVIGATION WRAPPER CONTROLS ---
                // We keep the toolbar item, but hide the native navigation bar header title completely!
                .toolbar {
                    ToolbarItem(placement: .primaryAction) {
                        Menu {
                            Picker("Sides", selection: $sides) {
                                ForEach(diceOptions, id: \.self) { option in
                                    Text("\(option) Sides")
                                }
                            }
                        } label: {
                            HStack(spacing: 6) {
                                Text("\(sides) Sides")
                                Image(systemName: "chevron.down")
                                    .font(.system(size: 10, weight: .bold))
                            }
                            .font(.subheadline)
                            .foregroundColor(.primary)
                            .padding(.vertical, 6)
                            .padding(.horizontal, 12)
                            .background(
                                Capsule()
                                    .fill(Color(.secondarySystemGroupedBackground))
                                    .shadow(color: .black.opacity(0.04), radius: 4, y: 2)
                            )
                        }
                    }
                }
            }
        }
    }
}

QUESTION

On the 'Dice' title, what styling options are available to us in this circumstance? Can we change font, change color, change background effect (shadow perhaps), or any other options?

ANSWER

Because we replaced the native navigation bar title with a standard SwiftUI Text view, the absolute full library of SwiftUI text styling options is open to us. You are no longer constrained by iOS system defaults.

Here is an overview of the core styling tools you can use to make that title stand out even more, along with code examples for each.

1. Typography & Custom Fonts

You can switch up the weight, design style, or even use a custom system font family installed in your project: 

// Serif Style (Classic, editorial feel)
Text("Dice")
    .font(.system(size: 38, weight: .black, design: .serif))

// Monospaced Style (Tech/Retro look)
Text("Dice")
    .font(.system(size: 34, weight: .medium, design: .monospaced))

// Custom External Font (If you have a custom font file like "Futura")
Text("Dice")
    .font(.custom("Futura-Bold", size: 36))

2. Premium Color & Gradient Styles

Instead of a flat primary color, you can render your text with a multi-color linear gradient or match a premium hue: 

// A clean, metallic gradient mask over the text
Text("Dice")
    .font(.system(size: 36, weight: .bold, design: .rounded))
    .foregroundStyle(
        LinearGradient(
            colors: [.primary, .secondary],
            startPoint: .top,
            endPoint: .bottom
        )
    )

3. Layered Text Shadows

You can add a subtle drop shadow directly onto the text fibers to lift it slightly off your background gradient: 

Text("Dice")
    .font(.system(size: 36, weight: .bold, design: .rounded))
    .shadow(color: .black.opacity(0.15), radius: 4, x: 0, y: 3)

4. Background Plates & Capsuling

If you want the title to look like an interface anchor, you can give it its own background container treatment: 

Text("DICE")
    .font(.system(size: 14, weight: .black, design: .rounded))
    .tracking(4) // Extra spacing between letters
    .foregroundColor(.primary)
    .padding(.vertical, 8)
    .padding(.horizontal, 16)
    .background(
        Capsule()
            .fill(Color(.secondarySystemGroupedBackground))
            .shadow(color: .black.opacity(0.05), radius: 5, y: 2)
    )

🚀 Try This Ultra-Premium Combination Block

If you want a bold, high-contrast title look that still matches your sleek dashboard aesthetic, try swapping your current text block out for this styled variant:

var actionWord: String {

or: 

Text("\(sides == 2 ? "Flip" : "Roll") \(sides == 2 ? "Coin" : "\(sides)-Sided Die")")

Adjusting Scale of Button Font

To scale up the text size inside your button, you just need to swap out .font(.headline) for a larger text style. You have two great paths here depending on how big you want to go.

Option 1: Use a Built-In System Size (Recommended)

Swapping .headline to .title3 or .title2 gives it an instant, clean, system-proportional boost: 

Button {
    let impact = UIImpactFeedbackGenerator(style: .medium)
    impact.impactOccurred()
    
    result = Int.random(in: 1...sides)
} label: {
    // Cleaned up string interpolation so it reads perfectly
    Text("\(actionWord) \(objectWord)")
        .font(.title3) // Large, readable, and perfectly balanced
        .fontWeight(.bold) // Keeps the text thick and prominent
        .frame(maxWidth: .infinity)
        .padding(.vertical, 16)
}
.buttonStyle(.borderedProminent)
.controlSize(.large)
.padding(.horizontal, 24)
.padding(.bottom, 20)

Option 2: Set an Exact Custom Size & Design

If you want total control over the exact pixel footprint to match your massive text elements, you can use .system(size:) and add the matching .rounded design variant: 

Text("\(actionWord) \(objectWord)")
    .font(.system(size: 22, weight: .bold, design: .rounded))
    .frame(maxWidth: .infinity)
    .padding(.vertical, 16)

🎨 Which one to pick?

Go with Option 1 (.title3 with .bold) if you want it to perfectly adapt to user accessibility font scaling choices. Go with Option 2 if you want to hardcode a specific, high-impact punchy size!

Vertical and Horizontal Support

To transition your Dice Roller app beautifully into landscape mode, we will use the Vertical Size Class trick (@Environment(\.verticalSizeClass)).

When the screen turns horizontal, your vertical space is cut in half. To handle this elegantly, our structural code will automatically switch the core container from a vertical VStack into a side-by-side HStack. This places your large 240x240 dice artwork on the left, and groups your result card and action button cleanly on the right.

🛠️ Key Architectural Changes Made

  • Extracted Subviews (diceImageSection, etc.): Because the layout renders elements differently depending on the math inside the if/else wrapper, repeating long code block definitions twice makes files messy. Pulling components down into small variables makes the core layout flow clean and simple.
  • Intelligent Sizing Toggles: Inside the subviews, you will notice ternary inline modifiers like verticalSizeClass == .compact ? 180 : 240. This safely downsizes the image and text sizing slightly when horizontal so that compact screens (like an iPhone SE or Pro Mini) don't get clipping errors or cut-off interfaces!

Here is your fully responsive code, complete with layout optimizations for both orientations:

Storing Results

  • Store the results using JSON or SwiftData - anywhere they are persistent.

In other words, store the history of dice throw results.

To use SwiftData, you do not store the array itself in the database. Instead, you turn the individual item (a single dice roll) into a blueprint class marked with @Model.

SwiftData then automatically generates the database behind the scenes. You use the @Query wrapper in your views to fetch those items into a regular Swift array that updates in real time.

Here is the step-by-step blueprint to set this up for your notes.

Step 1: Define the Blueprint (The Model Class)

Create a new Swift file named DiceRoll.swift. It must be a class, not a struct, and it needs the @Model macro. 

import Foundation
import SwiftData

@Model
final class DiceRoll {
    var timestamp: Date
    var result: Int
    var sides: Int
    
    init(result: Int, sides: Int) {
        self.timestamp = Date() // Automatically timestamps the roll
        self.result = result
        self.sides = sides
    }
}

Step 2: Set Up the App Root

In your main App file (e.g., DiceApp.swift), import SwiftData and attach the container to your window group. You pass the model class name here, not an array. 

import SwiftUI
import SwiftData

@main

    var body: some Scene {
        WindowGroup {
            ContentView()
        }
        // Tell SwiftData to prepare a database for saving DiceRoll objects
        .modelContainer(for: DiceRoll.self) 
    }
}

Step 3: Fetch, Add, Delete, and Cap the List at 50

In your SwiftUI view, you need two tools from SwiftData:

  • @Query: Automatically fetches the data as an array and refreshes the UI when things change.
  • @Environment(\.modelContext): The "scratchpad" used to insert or delete items.

Here is how you handle the UI logic, including your requirement to automatically trim the history to 50 items: 

import SwiftUI
import SwiftData

struct RollHistoryView: View {
    @Environment(\.modelContext) private var modelContext
    
    // Fetch rolls, sorting them from newest to oldest
    @Query(sort: \DiceRoll.timestamp, order: .reverse) private var rolls: [DiceRoll]
    
    var body: some View {
        NavigationStack {
            List {
                ForEach(rolls) { roll in
                    HStack {
                        Text("Rolled a d\(roll.sides)")
                        Spacer()
                        Text("\(roll.result)")
                            .font(.headline)
                    }
                }
                .onDelete(perform: deleteRolls) // Standard swipe-to-delete
            }
            .navigationTitle("History")
        }
    }
    
    // MARK: - Core Database Functions
    
    /// Call this function from your Dice app when a user rolls a die
    func saveRoll(result: Int, sides: Int) {
        // 1. Create a new roll object
        let newRoll = DiceRoll(result: result, sides: sides)
        
        // 2. Insert it into the database
        modelContext.insert(newRoll)
        
        // 3. Enforce the 50-item limit
        enforceHistoryLimit()
    }
    
    /// Automatically drops the oldest items if history exceeds 50
    private func enforceHistoryLimit() {
        // Because @Query sorts them newest first, the oldest items are at the end.
        // If we have more than 50, we remove everything past index 49.
        if rolls.count > 50 {
            // Grab items starting from index 50 to the end of the array
            let excessRolls = rolls.suffix(from: 50)
            
            for oldRoll in excessRolls {
                modelContext.delete(oldRoll)
            }
        }
    }
    
    /// Handles manual List swipe-to-delete action
    private func deleteRolls(offsets: IndexSet) {
        for index in offsets {
            let rollToDelete = rolls[index]
            modelContext.delete(rollToDelete)
        }
    }
}

📝 Key Takeaways for your Notes

  • .modelContainer(for: DiceRoll.self): Tells the app, "Get a database ready to store individual instances of DiceRoll."
  • @Query: Acts like a magnet. It reaches into that container, extracts the items, puts them in a standard Swift array ([DiceRoll]), and watches for changes.
  • modelContext: The brain that handles the actual data manipulation (.insert() and .delete()). SwiftData saves automatically behind the scenes when views shift or apps close.

final class Keyword

QUESTION

What is the significance of final class <ClassName>? What is the final keyword?

ANSWER

In Swift, the final keyword is an access modifier applied to classes. When you mark a class as final, you are telling the compiler: "This class is complete. It is the end of the line, and no other class is allowed to inherit from it or subclass it."

By default, regular Swift classes can be extended through inheritance (e.g., a Dog class might inherit from an Animal class). Adding final shuts that door permanently.

Here is exactly why this keyword is significant, especially when working with modern app architectures like SwiftData or MVVM.

1. Preventing Accidental Inheritance (Safety)

The primary architectural reason to use final is to protect your data models.

In your Dice app, your DiceRoll class is a rigid blueprint meant to map directly to a database table. It has specific properties like timestamp and result. Allowing another developer (or your future self) to subclass it could accidentally break your database layout or introduce unexpected bugs. 

final class DiceRoll { ... }

// ❌ COMPILER ERROR:
// "Inheritance from a final class 'DiceRoll'"
class SuperDiceRoll: DiceRoll { ... }

By making it final, you guarantee that a DiceRoll object will always just be a plain DiceRoll object anywhere in your app.

2. Speed Boosts Under the Hood (Static vs. Dynamic Dispatch)

This is where final becomes a secret weapon for app performance.

When a class is not marked final, the CPU has to use a process called Dynamic Dispatch at runtime. Every time your app calls a method on that class, the computer has to pause and double-check: "Wait, is this the base class method, or did a subclass override this function with a different version?" This lookup happens via an internal table (called a vtable) and takes a tiny fraction of a second.

When you add final, you promise the compiler that these methods will never be overridden. The compiler can throw away the lookup table and use Static Dispatch (or Direct Dispatch). The compiler points the CPU directly to the exact memory address of the code it needs to run.

Analogy for Your Notes:

  • Dynamic Dispatch (Standard Class): Looking up a word in a physical dictionary every time you want to see what it means.
  • Static Dispatch (final Class): Having the definition memorized so you can recite it instantly without checking a book.

🛠️ Summary for Your Reference Sheet

  • What it does: Prevents a class from being subclassed (no inheritance allowed).
  • Why we use it:
    • Intent: Signals that this class is a specialized, standalone tool or data model (like a SwiftData entity).
    • Performance: Optimizes your code, resulting in faster execution speeds by bypassing runtime method lookups.
  • Best Practice: In modern Swift, default to making every class final. If you later discover that you absolutely need to inherit from it, you can easily remove the keyword.

Challenge - Haptic Feedback

  • Add haptic feedback when dice are rolled.

QUESTION

I would like to add haptic feedback on the actionButtonSection which holds the dice roll function. How do I do this?

ANSWER

You already have the foundation for it right inside your button block! 

let impact = UIImpactFeedbackGenerator(style: .medium)
impact.impactOccurred()

This code works perfectly, but there is one modern catch: UIImpactFeedbackGenerator belongs to UIKit (Apple's older framework). While it works fine in SwiftUI, modern SwiftUI apps have a native, cleaner way to handle haptics using the .sensoryFeedback modifier.

Using the SwiftUI native way is cleaner because it completely separates your logic from your layout, automatically prepares the Taptic Engine for zero latency, and matches modern iOS design standards.

Here are the two best ways to do this.

Option 1: The Modern SwiftUI Way (Recommended)

Instead of instantiating a generator inside the button action, you can trigger a haptic pulse by telling SwiftUI to watch a variable (like your result) for changes. Whenever result changes, iOS plays the haptic feedback. 

private var actionButtonSection: some View {
    Button {
        let newResult = Int.random(in: 1...sides)
        result = newResult
        
        let newRoll = DiceRoll(result: result, sides: sides)
        modelContext.insert(newRoll)
    } label: {
        Text("\(actionWord) \(objectWord)")
            // ... your styles ...
    }
    // 🎲 Add this modifier to the button or the outer view container:
    // It triggers a tactile "impact" whenever the 'result' variable changes value.
    .sensoryFeedback(.impact(weight: .medium, intensity: 1.0), trigger: result)
}

Option 2: The CoreHaptics / UIKit Way (Fixing why yours might feel latent)

If you prefer keeping the haptic inside the button closure closure (Option A style), the code you have is correct. However, if you notice a tiny, frustrating delay between pressing the button and feeling the click, it's because the phone didn't have time to "spin up" the vibration motor.

To fix that, you must call .prepare() a split second before the impact: 

Button {
    let impact = UIImpactFeedbackGenerator(style: .medium)
    impact.prepare() // ⚡️ Wakes up the Taptic engine so there is 0ms delay
    impact.impactOccurred()
    
    let newResult = Int.random(in: 1...sides)
    result = newResult
    // ... rest of saving code ...
}

📝 Haptic Style Guide for Your Notes

When picking a sensory feedback style, match the physical action of your UI:

  • .sensoryFeedback(.impact(weight: .heavy), trigger: result): Perfect for rolling a heavy d20, creating a satisfying "thud."

Rolling Number Illusion

To implement this, you will use a Timer that fires very quickly (e.g., every 0.1 seconds), randomizes the number on screen, keeps track of how many times it has fired, and then destroys itself after a set duration to reveal the true final score.

Here is the step-by-step logic blueprint to add to your notes, along with the refactored code to make it work.

🧠 The Logic Blueprint

Instead of updating the number and saving it to the database instantly when the button is pressed, the process now splits into a timeline:

  • The Click: The user taps the button. We disable the button temporarily so they can't spam it.
  • The Rattle (The Loop): A timer starts ticking every 0.08 seconds. On every tick, we overwrite the result property with a random number.
  • The Stop: Once the timer ticks say, 15 times (~1.2 seconds), we cancel the timer, calculate the actual final result, save it to SwiftData, and unlock the button.

🛠️ Step-by-Step Code Update for ContentView.swift

We need to add a few pieces of state to coordinate the timeline, and then update the button's action block.

1. Add New State Variables

Add these near your other @State properties at the top of your ContentView: 

@State private var isRolling = false              // Disables the button during the animation
@State private var timerTicksRemaining = 0       // Counts down how many times the number changes
@State private var rollTimer: Timer? = nil        // Holds our active timer object

2. Update the actionButtonSection

Replace your current button action with this animated timer sequence: 

private var actionButtonSection: some View {
    Button {
        // 1. Prevent overlapping rolls
        guard !isRolling else { return }
        isRolling = true
        
        // 2. Set the length of the animation (e.g., 15 rapid updates)
        timerTicksRemaining = 15
        
        // 3. Play an initial "startup" haptic
        let impact = UIImpactFeedbackGenerator(style: .medium)
        impact.prepare()
        impact.impactOccurred()
        
        // 4. Start the rapid rattling timer
        rollTimer = Timer.scheduledTimer(withTimeInterval: 0.08, repeats: true) { timer in
            if timerTicksRemaining > 0 {
                // --- THE RATTLE ---
                // Keep flickering the onscreen display to a random number
                result = Int.random(in: 1...sides)
                timerTicksRemaining -= 1
                
                // Optional: Provide a light tick sound/vibration per flip
                let tick = UIImpactFeedbackGenerator(style: .light)
                tick.impactOccurred(intensity: 0.5)
                
            } else {
                // --- THE SETTLE ---
                // 5. Time's up! Stop the timer loop
                timer.invalidate()
                rollTimer = nil
                
                // 6. Establish the permanent, final score
                let finalResult = Int.random(in: 1...sides)
                result = finalResult
                
                // 7. Push to your SwiftData History database
                let newRoll = DiceRoll(result: finalResult, sides: sides)
                modelContext.insert(newRoll)
                
                // 8. Re-enable the button for the next turn
                isRolling = false
            }
        }
        
    } label: {
        Text(isRolling ? "Rolling..." : "\(actionWord) \(objectWord)")
            .font(.title3)
            .fontWeight(.bold)
            .frame(maxWidth: .infinity)
            .padding(.vertical, verticalSizeClass == .compact ? 12 : 16)
    }
    .buttonStyle(.borderedProminent)
    .controlSize(.large)
    // 9. Gray out or disable the button while the timer is actively ticking
    .disabled(isRolling) 
    .padding(.horizontal, verticalSizeClass == .compact ? 0 : 24)
    .padding(.bottom, verticalSizeClass == .compact ? 5 : 20)
}

📝 Critical Timer Notes for Your Reference Sheets

  • Timer.scheduledTimer(...): This creates an asynchronous loop running on the device's main processing loop (RunLoop). It executes whatever code is inside its trailing closure block at your specified withTimeInterval rate.
  • timer.invalidate(): This is the absolute most important command when dealing with timers. If you do not call .invalidate(), the timer will loop endlessly in the background forever, eating up battery life and crashing your logic.
  • guard !isRolling else { return }: This is a defensive programming wall. If the user mashes the button three times rapidly, this line completely stops execution so that the app doesn't spin up three conflicting timers at the same time.
  • The Micro-Haptic Integration: By pairing a .light haptic with every decrement of timerTicksRemaining, the physical phone will literally shake rhythmically in the user's hand matching the numbers changing on screen. It creates a highly responsive, physical sensation.

QUESTION

Can we use the more modern haptic approach here? (Instead of the UIKit approach?)

ANSWER

Yes, absolutely! We can migrate this entirely to the modern SwiftUI approach.

To do this natively, we shouldn't use result as the trigger directly, because if the randomizer rolls the same number twice in a row during the rattle, the value won't change, and a haptic tick will be skipped.

Instead, we can create a custom state enum called RollHapticState. As the timer ticks, we update this state, and a single .sensoryFeedback modifier reads it to play light impacts during the roll, and a heavy impact when the die settles!

🛠️ The Modern SwiftUI Native Implementation

1. Add the Haptic State tracking

Add this small enum and state variable to the top of your ContentView: 

// A clean tracking enum for our custom haptic timeline
enum RollHapticState: Equatable {
    case idle
    case rattling(Int) // Stores the ticks remaining so every tick is uniquely recognized
    case settled
}

Select All Button

QUESTION

For the RollHistoryView, is it possible to have a 'Select All' button appear when the 'Edit' button is pressed? (In that way, in addition to selecting items to delete, all items can be selected to delete.)

ANSWER

Yes, that is absolutely possible! It is a great quality-of-life feature for users who want to purge their entire roll history with a single tap instead of clicking 50 circles individually.

To do this, we can take advantage of SwiftUI's editMode environment variable. This allows RollHistoryView to actively spy on the native EditButton(). When it sees the environment shift into .active, we can conditionally show a "Select All" button in our toolbar.

Here is how to update your RollHistoryView.swift to add the "Select All" functionality.

🛠️ Updated RollHistoryView

We will grab the built-in \.editMode from the environment and add a button that dumps every single roll ID into your selectedRollIDs set at once. 

struct RollHistoryView: View {
    @Environment(\.modelContext) private var modelContext
    @Query(sort: \DiceRoll.timestamp, order: .reverse) private var rolls: [DiceRoll]
    
    // 1. Read the system edit mode state to know when the user presses "Edit"
    @Environment(\.editMode) private var editMode
    
    @State private var selectedRollIDs = Set<PersistentIdentifier>()
    
    var body: some View {
        NavigationStack {
            List(selection: $selectedRollIDs) {
                ForEach(rolls) { roll in
                    HStack {
                        Text("Rolled a d\(roll.sides)")
                        Spacer()
                        Text("\(roll.result)")
                            .font(.headline)
                    }
                }
                .onDelete(perform: deleteRolls)
            }
            .navigationTitle("History")
            .toolbar {
                // --- LEFT SIDE TOOLBAR ---
                ToolbarItem(placement: .topBarLeading) {
                    HStack(spacing: 16) {
                        // 2. Trash icon appears when items are selected
                        if !selectedRollIDs.isEmpty {
                            Button(role: .destructive, action: deleteSelectedRolls) {
                                Label("Delete Selected", systemImage: "trash")
                            }
                        }
                        
                        // 3. "Select All" appears ONLY when Edit Mode is active and list isn't empty
                        if editMode?.wrappedValue.isEditing == true && !rolls.isEmpty {
                            Button(action: selectAllRolls) {
                                Text(selectedRollIDs.count == rolls.count ? "Deselect All" : "Select All")
                            }
                        }
                    }
                }
                
                // --- RIGHT SIDE TOOLBAR ---
                ToolbarItem(placement: .topBarTrailing) {
                    EditButton()
                }
            }
            .onChange(of: rolls.count) {
                enforceHistoryLimit()
            }
            // 4. Clean up selections automatically if the user presses "Done" to exit edit mode
            .onChange(of: editMode?.wrappedValue) { _, newValue in
                if newValue?.isEditing == false {
                    selectedRollIDs.removeAll()
                }
            }
        }
    }
    
    // MARK: - Core Database Functions
    
    /// Toggles between selecting every item in the database or clearing the selection
    private func selectAllRolls() {
        if selectedRollIDs.count == rolls.count {
            // If everything is already selected, clear it out (Deselect All)
            selectedRollIDs.removeAll()
        } else {
            // Otherwise, map all current roll database IDs into our selection basket
            let allIDs = rolls.map { $0.id }
            selectedRollIDs = Set(allIDs)
        }
    }
    
    private func deleteSelectedRolls() {
        for roll in rolls {
            if selectedRollIDs.contains(roll.id) {
                modelContext.delete(roll)
            }
        }
        selectedRollIDs.removeAll()
    }
    
    private func enforceHistoryLimit() {
        if rolls.count > 50 {
            let excessRolls = rolls.suffix(from: 50)
            for oldRoll in excessRolls {
                modelContext.delete(oldRoll)
            }
        }
    }
    
    private func deleteRolls(offsets: IndexSet) {
        for index in offsets {
            modelContext.delete(rolls[index])
        }
    }
}

📝 Key Details for Your Reference Sheets

  • @Environment(\.editMode): This reads a binding managed by SwiftUI. It contains an enum that is either .inactive or .active.
  • editMode?.wrappedValue.isEditing: This is a convenient Boolean helper property provided by Apple that returns true specifically when the interface shifts into its editing state.
  • Dynamic Button Text: Notice how the button reads selectedRollIDs.count == rolls.count. This makes the button context-aware. If the user hits "Select All", the button text instantly transforms into "Deselect All" to allow them to reverse their decision cleanly!
