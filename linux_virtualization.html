Linux - Rocky Linux Virtualization (M1/Macmini)

Download

Download UTM from:

  • https://mac.getutm.app

Download Linux 'image' from Rocky Linux's page:

  • https://rockylinux.org/download

Choose the DVD ISO option. (Currently over 8Gb.)

Breakdown of Image Versions

For the goal of setting up a GUI-based environment on an M1 Mac to explore configurations like Vim and shell scripting, the DVD ISO (specifically the aarch64 version) is the best choice.

Choosing the Right Image

Since you want a Full UI, here is the breakdown of your options:

  • DVD ISO (Recommended): This is a "standalone" installer that includes all necessary packages to install a full Desktop/Workstation environment without needing an active internet connection during the process. It is the largest file (often 10GB+) because it contains everything.
  • Minimal ISO: Installs a bare-bones system with only a command-line interface (no GUI). You can add a GUI later, but it requires manually installing desktop packages via the terminal after the initial setup.
  • Boot ISO: This is a tiny "net install" image. It contains only the installer itself; every single package (including the UI) must be downloaded from the internet during the installation process.

Other Specialized Images

You will likely not need these for a standard VM setup:

  • Desktop/Workstation Live Images: These are "boot and try" images. They allow you to run the OS directly from the "CD" without installing it. While you can install from them, they are primarily for testing hardware compatibility or quick tasks.
  • Cloud Images: Pre-configured virtual disk files (like .qcow2) designed for cloud providers (AWS, Azure) or OpenStack. They typically do not have a GUI and are built for automated deployment.
  • Docker (OCI) Images: These are not operating systems you can boot in UTM; they are "templates" used to create isolated Docker containers for running specific apps or services.

Quick Summary for M1 Mac mini

  • Go to the Rocky Linux Download Page.
  • Click aarch64 (this is the ARM architecture for M1).
  • Download the DVD ISO.
  • In UTM, select "Virtualize" and point it to this ISO.

During the installation, make sure to select "Workstation" under the "Software Selection" menu to ensure you get the full graphical interface you're looking for.

Installation Guide for Apple Silicon (M1)

  • Download the ISO: Visit the Rocky Linux download page and select the aarch64 (ARM64) architecture. The "Minimal" ISO is usually sufficient for a base install, while the "DVD" ISO includes more pre-packaged software.
  • Create VM in UTM:
    • Select "Virtualize" (do NOT choose "Emulate" unless you specifically need to run an Intel/x86 version, which will be much slower).
    • Select "Linux" and browse for your downloaded Rocky Linux ISO.
  • Hardware Allocation:
    • Memory: Allocate at least 4GB to 8GB of RAM for a smooth desktop experience.
    • CPU: Allocate at least 4 cores (half of the M1's 8 cores) for better performance.
  • Display Settings: For better graphics performance, ensure Virtio-GPU is selected in the VM's Display settings within UTM.

Rosetta (For non Intel emulation)

This setup runs directly on top of the ARM64 version of Rocky Linux

Running x86_64 VFX software on an ARM64 Rocky Linux VM requires using Apple's Virtualization framework within UTM to enable Rosetta 2 translation. This setup allows your ARM-based Linux system to "understand" and execute Intel-based binaries.

1. Enable Rosetta in UTM Settings

You must configure the VM to use Apple's native virtualization instead of standard QEMU to access Rosetta.

  • Install Rosetta on macOS: If you haven't already, run this in your Mac's terminal: softwareupdate --install-rosetta.
  • UTM Configuration:
    • Open your VM settings in UTM.
    • Go to the Virtualization tab.
    • Check "Use Apple Virtualization".
    • Check "Enable Rosetta on Linux (x86_64 Emulation)".

2. Mount the Rosetta Share in Linux

Once enabled in UTM, macOS provides a special "virtfs" share containing the Rosetta runtime. You must mount this inside your Rocky Linux VM to use it.

  • Create a mount point: sudo mkdir -p /mnt/rosetta.
  • Mount the share: 
sudo mount -t virtiofs rosetta /mnt/rosetta
  • Make it permanent: Add the following line to your /etc/fstab file so it mounts automatically on boot: 
rosetta /mnt/rosetta virtiofs rosetta 0 0

3. Register Rosetta as a Binary Handler

To make Linux automatically use Rosetta whenever you try to run an x86_64 file, you need to register it with the kernel.

  • Install the handler tool: 
sudo dnf install binfmt-support
  • Register the format: 
sudo /mnt/rosetta/rosetta --register-binfmt

(Note: The exact path inside the mount may vary; check for a file named rosetta in /mnt/rosetta.)

Important Limitations for VFX

  • No GPU Acceleration: Rosetta translates CPU instructions but does not provide 3D hardware acceleration (OpenGL/Vulkan) for Linux guests in UTM. This means software like Maya or Nuke will likely be very slow or crash when trying to render the 3D viewport.
  • Library Dependencies: You may still need to install x86_64 versions of system libraries (like glibc or libstdc++) inside Rocky Linux to satisfy the software's requirements.

Disk Space/Allocation

For a basic install of Rocky Linux with a full UI on your M1 Mac mini using UTM, you should allocate between 20 GB and 40 GB of virtual disk space.

Disk Space Breakdown

  • Minimum Requirement: A base installation with a graphical user interface (GUI) typically requires at least 10–15 GB of space just to function.
  • Recommended for Light Work: Allocating 40 GB is the standard recommendation for general use. This provides a "comfortable" buffer for:
    • Operating system updates.
    • Development tools like Vim, compilers, and shell scripting utilities.
    • Swap space (virtual memory), which the installer often sets to 2–4 GB by default.
  • VFX Software (Future): If you eventually plan to install large VFX tools like Maya or Houdini, you will need significantly more, typically 64 GB to 100 GB or more.

Space-Saving Tip in UTM

When you create the VM in UTM, ensure you do not select "Preallocate Disk".

  • Dynamic Allocation: By default, UTM uses "sparse" disk images. If you set a 40 GB limit, the actual file on your Mac will only take up about 5–8 GB initially. It will only grow as you actually add files or install new software within Linux.

Other Basic Requirements

To ensure the UI runs smoothly on your M1:

  • RAM: Allocate at least 4 GB (though 2 GB is the absolute minimum for a GUI).
  • CPU: Assign 4 cores to keep the interface snappy.

Sharing (with macOS)

To share folders between your M1 Mac and Rocky Linux using UTM, you use the VirtioFS (or VirtFS) protocol. This allows you to mount a specific folder from your Mac directly into the Linux file system, making it accessible from your terminal and GUI.

Step 1: Configure UTM Settings

You must set up the share on the Mac side before starting the VM:

  • Shut down your Rocky Linux VM.
  • In the UTM main window, select your VM and click the Settings icon (top right).
  • Go to the Sharing tab on the left.
  • Click New... or Browse to select the folder on your Mac you want to share (e.g., your Projects folder).
  • Set the Share Name (or "Tag") to something simple like share.
  • Click Save.

Step 2: Mount the Folder in Rocky Linux

Now, start your VM and use the terminal to connect to that folder:

  • Create a mount point: This is where the Mac folder will appear inside Linux. 
sudo mkdir -p /mnt/mac_share
  • Mount the folder: Run the following command (replace share with the tag name you chose in Step 1): 
# If using Apple Virtualization Framework:
sudo mount -t virtiofs share /mnt/mac_share

# If using the standard QEMU backend:
sudo mount -t 9p -o trans=virtio share /mnt/mac_share
  • Verify: Run ls /mnt/mac_share to see your Mac files.

Step 3: Make it Permanent (Optional)

To ensure the folder mounts automatically every time you start the VM, add it to your file system table:

Open the configuration file: sudo vi /etc/fstab (or use nano).

Add this line at the very bottom: 

share /mnt/mac_share virtiofs rw,nofail 0 0

_(Note: If you used the 9p command in Step 2, use share /mnt/mac_share 9p trans=virtio,rw,nofail 0 0 instead.)_

Troubleshooting Access Denied

If you can see the files but cannot edit them, you may need to take ownership of the mount point: 

sudo chown -R $USER:$USER /mnt/mac_share
