PySide6 - VirtualEnv

Take a look at Pyenv and Pyenv Virtualenv:

Pyenv - Virtualenv

Setting up a new Pyside6 VirtualEnv

# Navigate to location to create a virt env:
cd <dir>
 
# Create working folder:
mkdir <my_env>
cd !$

## If you don't have pyenv set up, make sure to install (see link above)

# List installed python packages on the system (including existing virtual envs):
pyenv versions

# If there's one already created, move on to the next part, otherwise:

pyenv virtualenv <existing_python_install> <env_to_create_based_on_existing_python_install>

# Example:
pyenv virtualenv 3.11.9 pyside6_3.11.9

# Running pyenv versions will now include the new package/env

# set the new virtual environment as the directories python package:
pyenv local <env_package_of_choice>

# Example:
pyenv local pyside6_3.11.9

# Pyenv has now activated the virtualenv (pyenv doesn't need to be activated like regular virtual envs).

Installing PySide6

# To install the latest package:
pip3 install PySide6

# To install a specific version of PySide6:
pip3 install PySide6==5.15.0

Reminder: Remove VirtualEnv and PyEnv

# remove Pyenv virtualenv:
pyenv virtualenv-delete <name_of_virtual_env>

# Example:
pyenv virtualenv-delete pyside6_3.11.9

# then remove directory

rm -rf <name_of_directory_containing_virtualenv>

# Example:

rm -rf pyside6_gui
