Swift - Enums

Here is a thorough, developer-to-developer guide on Swift Enums (Enumerations), focusing heavily on how they work generally and how we use them in SwiftUI to build rock-solid, non-instantiable layout engines.

What is an Enum? (The Core Concept)

While a struct is a blueprint for building a multi-property object (like a User with a name, age, and email), an enum is a finite list of distinct choices. It defines a type by giving it a strict set of mutually exclusive values called cases. 

// A struct holds data combinations
struct User {
    var name: String
    var isPremium: Bool
}

// An enum represents a single choice from a list
enum AppTheme {
    case light
    case dark
    case highContrast
}

Why use Enums?

  • Type Safety: You cannot accidentally type "Darke" instead of "dark". The compiler forces you to pick a valid case.
  • Exhaustiveness: When using a switch statement with an enum, Swift forces you to handle every single case. If you add a case later, your code won't compile until you update your layout to handle it, preventing silent bugs.

1. Standard Enums: The Fundamentals

A standard enum defines a brand-new data type with a strict, closed list of possible choices called cases. Think of it as a custom toggle or a multi-option dial you design for your app.

Declaring and Using a Basic Enum

Here is how you write a basic enum to manage layout modes or states: 

// 1. Define the type and its available choices
enum CardState {
    case expanded
    case collapsed
    case hidden
}

// 2. Create a variable using your new type
var currentLayout = CardState.collapsed

Once Swift knows that currentLayout is a CardState, you can use short-dot notation to change its value cleanly: 

currentLayout = .expanded  // Swift automatically infers it's a CardState choice

2. The Two Superpowers of Standard Enums

To make enums useful for everyday programming, Swift gives them two incredible features:

  • Raw Values (backing data)
  • Associated Values (attached data).

Superpower A: Raw Values (Backing Data)

Sometimes you need each enum case to map to a matching primitive value, like a string, character, or integer. This is called a Raw Value. 

// This enum is backed by explicit layout sizes (CGFloats)
enum ComponentHeight: CGFloat {
    case small = 44
    case medium = 88
    case large = 150
}

// Extracting the raw value using `.rawValue`
let cardHeight = ComponentHeight.medium.rawValue // Returns 88.0

You can do the same thing with strings, which is incredibly useful for mapping cases to UI text labels or asset names: 

enum IconAsset: String {
    case profile = "user-avatar-image"
    case settings = "gear-icon-image"
}

// Swift uses the exact case name as the string automatically if you don't assign one
enum ViewTitle: String {
    case home      // rawValue is "home"
    case favorites // rawValue is "favorites"
}

Superpower B: Associated Values (Attached Data)

This is where enums become exponentially more powerful than simple lists. Associated values allow each case to carry unique extra information alongside it.

Imagine you are modeling a layout notification banner: 

enum BannerType {
    case success                     // No extra data needed
    case warning(code: Int)          // Carries an error code integer
    case critical(message: String)   // Carries a custom warning message string
}

// Using the type safely with its attached payloads
let currentAlert = BannerType.critical(message: "Disk space is completely full!")

3. Driving Logic with Enums: The switch Statement

The absolute best way to read and react to an enum is using a switch statement. Swift enforces a rule called Exhaustiveness: your switch must handle every single case listed in the enum, or your app will refuse to compile.

Here is how you open up an enum case, extract its associated values, and use them to safely run your layout code: 

func configureLayout(for alert: BannerType) {
    switch alert {
    case .success:
        print("Show a nice green checkmark.")
        
    case .warning(let code):
        print("Show a yellow triangle. Error code was: \(code)")
        
    case .critical(let message):
        print("Flash a full-screen red warning. Message: \(message)")
    }
}

4. In Context: Driving SwiftUI Layouts with Enums

In modern SwiftUI development, you will constantly use standard enums to manage UI states. Because views automatically re-render when state changes, toggling an enum case is the cleanest way to swap out entirely different layout hierarchies or styles. 

import SwiftUI

// Defining our layout states
enum CardStyle {
    case minimalist
    case informationDense
}

struct AdaptiveCardView: View {
    // 1. Establish the enum as a state property
    @State private var currentStyle: CardStyle = .minimalist
    
    var body: some View {
        VStack {
            // 2. Use a switch statement to completely transform the view hierarchy
            switch currentStyle {
            case .minimalist:
                Text("Minimal View")
                    .font(.title)
                    .frame(width: 200, height: 100)
                    .background(.blue)
                    
            case .informationDense:
                VStack(alignment: .leading) {
                    Text("Detailed Analysis").font(.headline)
                    Text("Row 1: Analytics Data").font(.caption)
                    Text("Row 2: Geometric Bounds").font(.caption)
                }
                .frame(width: 200, height: 200) // Stretches taller
                .background(.purple)
            }
            
            // 3. Button to toggle the enum state seamlessly
            Button("Toggle Layout") {
                withAnimation {
                    if currentStyle == .minimalist {
                        currentStyle = .informationDense
                    } else {
                        currentStyle = .minimalist
                    }
                }
            }
            .padding()
        }
        .foregroundColor(.white)
        .clipShape(RoundedRectangle(cornerRadius: 12))
    }
}

Summary Guide for Your Reference

  • Plain Enum: Use when you just need a strict list of clear-cut options (.light, .dark).
  • Raw Values (enum Name: String): Use when those options need a permanent structural counterpart like a database string or a size number (.medium = 88).
  • Associated Values (case name(payload)): Use when a specific state needs to dynamically wrap up and pass around varying extra context (.critical(message: "...")).

5. The Power Move: Caseless Enums for Namespacing

Now let's tackle a specific use: Using an enum in place of a struct to create a non-instantiable namespace.

When you are designing a complex SwiftUI layout, you often want to create a central file that holds layout constants, custom math helpers, or geometric calculations.

The Problem with using a Struct

If you use a struct to organize your layout utilities, the compiler leaves a tiny backdoor open: 

struct LayoutMetrics {
    static let cardCornerRadius: CGFloat = 12
    static let gridSpacing: CGFloat = 16
    
    static func calculateWidth(for screenWidth: CGFloat) -> CGFloat {
        return screenWidth * 0.85
    }
}

// ⚠️ THE BUG: This is valid Swift, but makes no sense!
// You just allocated memory for an empty box that serves no purpose.
let accidentalInstance = LayoutMetrics()

Because LayoutMetrics is a struct, a junior developer (or you, late at night) can accidentally write LayoutMetrics(), creating an instance of a view/utility container that was only ever meant to hold static math formulas.

The Solution: The Caseless Enum

If you define that exact same container as an enum but give it zero cases, it becomes a ghost type. 

enum LayoutConstants {
    static let cardWidth: CGFloat = 200
    static let cardHeight: CGFloat = 200
    static let rotationDampener: CGFloat = 5
}

// 🛑 COMPILER ERROR: 'LayoutConstants' cannot be constructed because it has no accessible initializers
let blockedInstance = LayoutConstants()

Because an enum's entire job is to represent its cases, an enum with zero cases can never be instantiated. It cannot exist as an object in your device's memory. It serves purely as an unbreakable, static blueprint - a perfect "namespace" for organizing your layout code.

6. Practical Example: A Non-Instantiable Layout Namespace

Here is how you wrap this concept up into a clean, professional SwiftUI extension system. We will create a caseless enum called CarouselDesign to store the math formulas for 3D rotation effects, keeping our view code incredibly clean. 

import SwiftUI

// 1. The pure structural namespace (Un-instantiable!)
enum CarouselDesign {
    static let cardSize = CGSize(width: 200, height: 200)
    static private let dampener: CGFloat = 5
    
    // Custom layout code calculated dynamically
    static func rotationDegrees(for cardMidX: CGFloat, screenWidth: CGFloat) -> Double {
        let centerOffset = cardMidX - (screenWidth / 2)
        return -Double(centerOffset / dampener)
    }
}

// 2. Clean use inside your view
struct CenteredCarouselView: View {
    var body: some View {
        GeometryReader { fullView in
            ScrollView(.horizontal) {
                HStack(spacing: 0) {
                    ForEach(1..<20) { num in
                        GeometryReader { proxy in
                            Text("Card \(num)")
                                .frame(
                                    width: CarouselDesign.cardSize.width, 
                                    height: CarouselDesign.cardSize.height
                                )
                                .background(.red)
                                .rotation3DEffect(
                                    .degrees(CarouselDesign.rotationDegrees(
                                        for: proxy.frame(in: .global).midX,
                                        screenWidth: fullView.size.width
                                    )),
                                    axis: (x: 0, y: 1, z: 0)
                                )
                        }
                        // Using the namespace properties safely
                        .frame(
                            width: CarouselDesign.cardSize.width, 
                            height: CarouselDesign.cardSize.height
                        )
                    }
                }
            }
        }
    }
}

Quick Reference Cheat Sheet

  • Use a Struct when you need to make instances of things that hold unique, stateful data (e.g., let user1, let user2).
  • Use a Standard Enum when your layout can exist in a few distinct states (e.g., enum CardState { case expanded, collapsed, hidden }).
  • Use a Caseless Enum when you want to build a shared toolbox of styles, math extensions, or configuration constants that should never be turned into physical instances.
