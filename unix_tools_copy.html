Unix Tools

cp

Below is a concise, note-form breakdown of the cp utility:

Purpose:

  • Copies files and directories from one location to another.

Basic Syntax:

cp [options] source... destination

Standard File Copying:

  • Single File: 
cp file1.txt file2.txt

Copies file1.txt to file2.txt (overwriting if file2.txt exists).

  • Multiple Files: 
cp file1.txt file2.txt directory/

Copies file1.txt and file2.txt into the specified directory.

Copying Directories:

Recursive Copy:

  • Use -r or -R to copy entire directories: 
cp -r sourcedir/ destinationdir/

Without recursion, you'll get an error when trying to copy directories.

Commonly Used Options:

-a (archive):

  • Combines multiple options (-r, -p, -d) to copy directories, preserve attributes, symlinks, and timestamps. 
cp -a sourcedir/ destinationdir/
  • -i (interactive):
    • Prompts for confirmation before overwriting files. 
cp -i file1.txt destination_directory/
  • -v (verbose):
    • Displays details about the files being copied. 
cp -v file1.txt destination_directory/
  • -p (preserve):
    • Preserves file attributes (mode, ownership, timestamps). 
cp -p file1.txt file2.txt
  • -u (update):
    • Copies files only if the source file is newer than the destination file or if the destination file is missing. 
cp -u file1.txt destination_directory/

Relative and Absolute Paths:

  • cp accepts both relative (e.g., ../folder) and absolute paths (e.g., /home/user/folder).
  • Ensure correct path formatting to avoid errors like "not copied" for directories.

Important Considerations:

  • Overwriting Behavior:
    • By default, cp overwrites files in the destination without prompt. Use -i for safety.
  • Empty Directories:
    • Git tracks files, not directories. When using cp in project reorganizations, remember that empty directories may need placeholder files if you want version control to track them.

Examples in Normal Use:

  • Copy a file to a new file: 
cp myfile.txt newfile.txt
  • Copy multiple files into a directory: 
cp file1.txt file2.txt dir/

Copy a directory with all its files and subdirectories (recursively): 

cp -r mydir/ ../another_dir

Copy with verbose output: 

cp -rv sourcedir/ destinationdir/

This breakdown covers the basic to moderately advanced usage of cp in everyday file management tasks.

Copy Directories

When copying a directory using the cp command, you must include the recursive flag so that the directory’s contents (including subdirectories) are copied. For example: 

cp -r mydir/ ../another_dir

Here’s a brief note-form breakdown:

  • Default Behavior:
    • The cp command, by default, only copies regular files and not directories.
  • Recursive Copy:
    • Adding the -r or -R flag tells cp to copy directories recursively.
  • Example Command:
    • cp -r mydir/ ../another_dir Copies the entire mydir directory (and its contents) into ../another_dir.
  • Alternative Flags:
    • Use -a to archive, which preserves file attributes (e.g., cp -a mydir/ ../another_dir).

Using these flags will allow you to copy directories to relative (or absolute) paths successfully.
