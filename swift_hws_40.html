Hacking With Swift - How to work with variables inside classes

Swift classes function like signposts, with each instance pointing to the same data, impacting how variable properties are treated.

This one small code sample demonstrates how things work: 

class User {
    var name = "Paul"
}

let user = User()
user.name = "Taylor"
print(user.name)

A constant User instance can have its internal data changed, but the instance itself remains unchanged. Making the name property a constant prevents its change, while making both the instance and name variable allows for changes to both.

Example code

Look at this code with a variable instance: 

class User {
    var name = "Paul"
}

var user = User()
user.name = "Taylor"
user = User()
print(user.name)


// >> Paul

Now with a constant instance (to show how it fails): 

class User {
    var name = "Paul"
}

let user = User()
user.name = "Taylor"
user = User()
print(user.name)

// >> MyPlayground.playground: 7:1: note: change 'let' to 'var' to make it mutable
let user = User()
^~~
var

Four Configurations

We end up with four options:

  • Constant instance, constant property:
    • the same user
    • who always has the same name.
  • Constant instance, variable property:
    • the same user
    • but their name can change.
  • Variable instance, constant property:
    • different users
    • but their names never change.
  • Variable instance, variable property:
    • different users
    • and those users can also change their names.

Variable properties within a class instance, even if the instance itself is constant, indicate that the data can be changed elsewhere in the program.

Structs vs Classes

Constant structs cannot have their properties changed because they hold data directly, unlike classes which use signposts.

Classes in Swift no longer require the mutating keyword for methods that modify data, as Swift can determine mutability based on property declaration.

Struct ~ mutating func doWork { ... } Class ~ func doWork { ... }

Examples from Tests

Mutating func is NOT required for Classes: 

class Pizza {
	private var toppings = [String]()
	func add(topping: String) {
		toppings.append(topping)
	}
}
var pizza = Pizza()
pizza.add(topping: "Mushrooms")

Mutating func IS required for Structs: 

struct Park {
	var numberOfFlowers = 1000
	mutating func plantFlowers() {
		numberOfFlowers += 50
	}
}
let park = Park()
park.plantFlowers()

Classes created as constants ('let') CAN be modified if the properties are variables ('var'): 

class Light {
	var onState = false
	func toggle() {
		if onState {
			onState = false
		} else {
			onState = true
		}
		print("Click")
	}
}
let light = Light()
light.toggle()

struct Code {
	var numberOfBugs = 100
	mutating func fixBug() {
		numberOfBugs += 3
	}
}
var code = Code()
code.fixBug()

class Phasers {
	var energyLevel = 100
	func firePhasers() {
		if energyLevel > 10 {
			print("Firing!")
			energyLevel -= 10
		}
	}
}
var phasers = Phasers()
phasers.firePhasers()

class Sun {
	var isNova = false
	func goNova() {
		isNova = true
	}
}
let sun = Sun()
sun.goNova()

struct Barbecue {
	var charcoalBricks = 20
	mutating func addBricks(_ number: Int) {
		charcoalBricks += number
	}
}
var barbecue = Barbecue()
barbecue.addBricks(4)
