Git Log & History

You have several powerful options to explore a file’s history, including viewing commits, comments, and changes over time.

Basic File History

To see all commits related to a specific file: 

git log -- <filename>

This will show the commit history for the file, including commit messages and authors.

View Changes in Each Commit

If you want to see the actual changes made to the file in each commit: 

git log -p -- <filename>

The -p flag shows the patch (diff) for each commit.

Follow File History Across Renames

If the file has been renamed and you want to track its history: 

git log --follow -- <filename>

The --follow flag ensures Git continues tracking the file even if it was renamed.

Compact History View

For a one-line summary of commits affecting the file: 

git log --oneline -- <filename>

This gives a concise view of commit hashes and messages.

Graphical History Exploration

If you prefer a visual representation: 

gitk -- <filename>

This opens a GUI tool showing the file’s history.

Blame: See Who Changed What

To see who last modified each line in the file: 

git blame <filename>

This is useful for understanding when and why specific changes were made.

Deciding How to Proceed

  • If the file has frequent changes, you might want to review diffs (git log -p).
  • If you suspect renames, use --follow to track its full history.
  • If you need to revert a change, find the commit hash and use git revert <commit-hash>.
