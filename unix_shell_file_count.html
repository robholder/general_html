LS

1. The Quick Standard Command (Files only, no subfolders)

To count only the files in your immediate directory (excluding folders, hidden files, and subdirectories): 

ls -p | grep -v / | wc -l

How it works:

  • ls -p lists files and appends a / to the end of any folder names.
  • grep -v / filters out (excludes) any lines containing / (removing the folders from the list).
  • wc -l counts the remaining lines.

2. Count Everything in the current directory (Files & Folders)

If you want a quick count of absolutely everything (files, folders, and symlinks) sitting directly in the current folder: 

ls -1 | wc -l

(Note: That is a number 1 after the ls, which forces the output into a single column so wc -l can count it accurately.)

3. Count absolutely everything recursively (Including inside subfolders)

If you want to count every single file hidden away inside subfolders and sub-subfolders as well: 

find . -type f | wc -l

How it works:

  • find . searches the current directory and all subdirectories.
  • -type f restricts the search specifically to physical files (ignoring directory folders).
  • wc -l counts the total matches.

EZA

You can also use eza! In fact, eza has built-in features that make counting files even easier and cleaner than traditional ls because it has native flags to separate files from folders.

If you have eza aliased to ls, running your alias as-is might behave slightly differently because eza defaults to its own colorized, column-based grids.

1. The Clean Way: Use eza Native Flags

eza features a -f (or --only-files) flag that completely eliminates the need to use grep to filter out directories.

To count only the files in your current directory using eza: 

eza -1f | wc -l

How this works:

  • eza is invoked.
  • -1 forces the output into a single column (one file per line).
  • -f tells eza to only list files (automatically ignoring all folders).
  • wc -l counts the lines.

2. The "Escaped" Way: Force the Original ls

If you are writing a script or want to guarantee you are using the classic, system-native ls (bypassing your alias entirely), you can escape the command by prepending a backslash \. 

\ls -p | grep -v / | wc -l

Why the backslash works:

In Unix shells, putting a backslash \ before a command tells the shell: *"Ignore any aliases I have defined and run the literal binary found in $PATH."* This is extremely useful for keeping one-liners portable if you plan to share them with others who might not have eza installed.
