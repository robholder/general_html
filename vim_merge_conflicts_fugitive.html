Merge Conflict Resolution with Fugitive

Gdiff (Gvdiffsplit!) on a Conflicted Merge

Definitions:

  • TARGET:
    • Current branch
  • WORKING COPY:
    • Current CONFLICTED copy (with the 'git' conflicts)
  • MERGE:
    • From the 'merge' branch

Opening a 3-way Gdiff layout

  • Open conflicted file in Vim.
  • Run 'Gvdiffsplit!'
  • 3 panes open:
    • left: target (current branch)
    • mid: conflict file
    • right: merge file (from merge branch)
      bufspec: //2 conflicted file bufspec: //3
      TARGET (HEAD) WORKING COPY MERGE

Two Ways of Working

  • Working in the middle (Working conflict buffer).
  • Working in either of the other windows (Target or Merge, Left or Right)

Working in the Working/Conflict buffer (middle)

This is based on :diffget

While in the middle buffer, specify which buffer to 'get' the changes from:
:diffget //2 <-- Obtain (get) from the Target/HEAD window (left). NOTE: the left buffer is always Taret and //2
:diffget //3 <-- Obtain (get) from the Merge window (right). NOTE: the right buffer is always Merge and //3

Working in the Target or Merge buffer (left/right)

This is based on :diffput

While in either left or right buffers, 'put' changes, and they will automatically go to the Working (HEAD) copy.
:diffput <-- Puts into the Working Copy by default. (You can specify the file name if necessary.)
