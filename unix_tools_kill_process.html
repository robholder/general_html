Setup Dummy situation for exercising

Exercise in the terminal/shell window:

  • Tab One:
    • dd if=/dev/zero of=/dev/null
  • Tab Two:
    • top

This will start a dummy process, and you check on it in the 'top' feedback screen. 

# tab one:
dd if=/dev/zero of=/dev/null
# tab two:
top

Kill

You can look at the processes with 'top', then quit with either:

  • q

or

  • <cntl> c

To kill a PID/process with 'kill': 

kill <PID>

# Example:
kill 33278

# If the job is not being killed with the regular 'kill' (This is more severe and can leave disrubpted files/data):
kill -9 <PID>

# Example:
kill -9 33278

On Linux (not the same on MacOS) to see all SIG options (like '-9') list them with:

  • kill [[-l]]

kill <user's processes>

As an admin, you can try to kill a users processes: 

sudo killall -u <username>

# Example:
sudo killall -u myusername
password: .....

pkill

To kill with name rather than the PID: 

pkill <process_name>

# Example:
pkill firefox

# To force kill a process:
pkill -f <process_name>

# Example:
pkill -f firefox

PS

To show all shell processes: 

ps
>>>
  PID TTY           TIME CMD
  664 ttys000    0:01.20 -zsh
  668 ttys001    0:01.11 -zsh
39155 ttys002    0:00.63 -zsh
33198 ttys003    0:00.59 -zsh
87742 ttys004    0:00.07 -zsh
88002 ttys005    0:00.07 -zsh
88262 ttys005    8:03.00 dd if=/dev/zero of=/dev/null

To show all machine processes: 

ps aux
<<will result in a huge list of processes>>

List of options:

  • ps
  • ps aux

psgrep

psgrep will return the PID of the process(es) matching a search term: 

psgrep <search_term>
