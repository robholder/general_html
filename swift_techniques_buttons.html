Swift - Buttons

Here is a scannable reference guide for SwiftUI’s Button. It covers the core syntax, how to attach functionality, and the primary styling options available to change its aesthetic.

1. Core Syntax (The Mechanics)

A SwiftUI Button separates what it does (the action closure) from what it looks like (the label view).

Simple Text Buttons

For basic text-only buttons, you can pass a string directly as the first argument. 

Button("Tap Me") {
    print("Button was tapped!")
}

Complex Custom Content

For buttons that require shapes, icons, text configurations, or stacks, use the trailing closure syntax for the label. 

Button {
    // 1. The Action (What it does)
    print("Faved!")
} label: {
    // 2. The Appearance (What it looks like)
    Label("Add to Favorites", systemImage: "heart.fill")
        .font(.headline)
}

2. Aesthetic Styles & Configurations

SwiftUI provides built-in system modifiers that transform a plain button into a styled interface element instantly.

Button Roles (Semantic Intent)

Roles tell SwiftUI the purpose of the button so it can style it contextually (e.g., turning a destructive button red). 

Button("Delete File", role: .destructive) { /* Delete logic */ }
Button("Cancel", role: .cancel) { /* Cancel logic */ }

Primitive Button Styles

Apply the .buttonStyle(_:) modifier to change the overall shape, background behavior, and interaction feedback.

Modifier Visual Result Best Used For
.buttonStyle(.automatic) Default platform look (plain text on iOS). Standard secondary actions.
.buttonStyle(.bordered) Standard button shape with a subtle background tint. Secondary or standard forms.
.buttonStyle(.borderedProminent) Filled, bright background using the app's accent color. Primary call-to-action (CTA).
.buttonStyle(.borderless) Pure text/icon button without structural borders. Toolbars and clean list actions.
.buttonStyle(.plain) Text/icon button that completely drops system highlight tints. Custom complex components. 
// Example of a prominent primary CTA button
Button("Submit Order") {
    processOrder()
}
.buttonStyle(.borderedProminent)

Size Modifiers

Adjust the padding and height constraints using .controlSize(_:). 

Button("Small Action") { }
    .buttonStyle(.bordered)
    .controlSize(.small) // Options: .mini, .small, .regular, .large, .extraLarge

Corner Shapes

When using bordered styles, reshape the button background with .buttonBorderShape(_:). 

Button("Capsule") { }
    .buttonStyle(.bordered)
    .buttonBorderShape(.capsule) // Options: .automatic, .capsule, .roundedRectangle(radius: 12)

3. Advanced Custom Aesthetics

For a completely unique layout, apply standard view modifiers directly to the label contents. Important Layout Rule: Apply custom padding, frames, and background colors inside the label closure to ensure the entire visual area remains interactive. 

Button {
    print("Custom styled button pressed")
} label: {
    Text("Get Started")
        .font(.title2)
        .bold()
        .foregroundColor(.white)
        .padding(.vertical, 12)
        .frame(maxWidth: .infinity) // Stretches button width to fit container
        .background(
            LinearGradient(colors: [.blue, .purple], startPoint: .topLeading, endPoint: .bottomTrailing)
        )
        .clipShape(RoundedRectangle(cornerRadius: 15))
        .shadow(radius: 5)
}
.padding(.horizontal)

4. State & Functionality Options

Disabling a Button

Prevent user interaction conditionally by pairing the .disabled() modifier with a boolean @State variable. 

@State private var isFormValid = false

var body: some View {
    Button("Submit") {
        saveData()
    }
    .buttonStyle(.borderedProminent)
    .disabled(!isFormValid) // Button automatically greys out and ignores taps if false
}

Simultaneous & Multi-actions

A button's action block can execute any procedural code, state mutations, or trigger external network calls. 

Button("Update Account") {
    isLoading = true          // 1. Mutate UI State
    SoundManager.playClick()  // 2. Trigger side effect
    networkService.sync()     // 3. Fire business logic
}

Images & Symbols

You can use an image as a button. In fact, you can turn any SF Symbol or custom image asset into a tappable element using two different approaches, depending on the look you want.

Here is the quick syntax breakdown to add to your reference guide:

1. The Standard Icon Button (SF Symbols)

The cleanest way to build an icon button is to pass an SF Symbol image name right inside the systemImage parameter of the initializer. 

Button(systemImage: "gearshape.fill") {
    print("Settings opened!")
}
.font(.title) // Controls the physical size of the icon symbol

2. Custom Images & Assets (The Label Closure Approach)

If you want to use a custom image from your Asset Catalog or apply specific frame resizing to an image, place the image view inside the trailing label closure. 

Button {
    print("Profile avatar tapped!")
} label: {
    Image("avatar-placeholder") // Your custom asset name
        .resizable()
        .scaledToFill()
        .frame(width: 60, height: 60)
        .clipShape(Circle())
        .overlay(Circle().stroke(Color.blue, lineWidth: 2))
}

Design Tip: When using a plain image asset as a button label, SwiftUI automatically applies a subtle blue tint or dims the graphic momentarily when tapped to give the user visual feedback. If you don't want SwiftUI to alter the colors of your image asset, add .buttonStyle(.plain) to the end of the button code.
