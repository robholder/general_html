02-06-The Value Ladder

The Value Ladder

The Value Ladder is a way of interactively adjusting values in increments, usually by powers of ten (0.01, 0.1, 1, 10, 100 etc.). It appears as a vertical stack of powers.
The value appears in the active power box of the ladder graphic.

  • Parameter Pane:
    • MMB <-- over parameters in the parameters pane and adjust by sliding left/right and jumping powers vertically.
    • <shift> MMB <-- allows you to move the cursor off the 'value ladder' to continue adjustments.
    • MMB <-- over parameters label to adjust all axis together.
  • Viewport (on handles of transform tools):
    • MMB <-- over handle axis
    • Note:
      • When adjusting Rotation in the viewport via the Value Ladder, the increments are logical for rotations (1, 15, 30, 45).
      • On parameter pane, it reverts to powers of 10.
  • Strings & Expressions:
    • Select the numerals in a string or expression
    • MMB over the selection to activate adjustments via the Value Ladder graphic.

02-09-Transforming An Object's Pivot

Editing and Transforming the Pivot

(while in Handle, Move, Rotate or Scale tool)

The pivot is the source point of transformations.

  • To set the pivot to adjust mode:
  • While in HANDLE, MOVE, ROTATE or SCALE tools:
    • Insert key
    • <shift> " key (double-quote character)
    • RMB on an object and choose pivot mode

When in 'pivot' mode, the tips of the transform tool's handle axis are wireframe (solid for regular transforms).
Switching to one of the dedicated move tools (t, r or e), exits the pivot edit mode.

The pivot is not restricted to the object's bounds. It can be anywhere in space.

The parameters pane has transforms cells for the pivot as well as the object proper.

Parameter & Viewport adjustments

When interacting with the pivot transformations in the viewport, Houdini treats each edit as a resolved step. This is the most intuitive way to make several adjustments one after another.

In the parameters pane, you have to use the Modify Pre-Transform pop-up to 'Clean transforms' between separate edits. (Otherwise, the numerical adjustments are accumulated into one transform.)

03-01-Handle Detachment

Handle Detachment

This is similar to the pivot edit, but this is a temporary detachment of the center (rather than the pivot itself, which is done using the " key).

  • To detach the handles of a transform tool:
  • While in a transform tool mode:
    • ' key (single-quote)
    • RMB on an object and choose Detach handles

TIP: Grid snapping is handy for this (Magnet on grid icon - left of Viewport panel).
TIP: X key will show a radial menu from which 'snap to grid' can be invoked.

  • Deselecting and re-selecting the object will restore the default position of the transform tool handles.
  • RMB and choose persistent to keep the handle visible and in the edited/detached position. (You can actually use the Handle tool AND the transform tools simultaneously in this way).

Forcing restore of position of Handle tool location

  • RMB > Handle parameters - choose snap to centroid or pivot (as well as Object and World space).

03-11-Snapping Basics

Snapping tools

  • The main snapping options are represented and set by the 'magnet' icons on the left panel of the viewport.
  • x key brings up the snapping radial tool to choose snapping mode too

You can snap to reference planes, c-planes and geometry features.

  • Point snapping:
    • 'Magnet' and point icon;
    • Allows snapping to points of geometry.
    • You can still use the ' detach features and use the point snapping
  • Primitive snapping:
    • 'Magnet' and curve icon
    • Allows snapping to polygons (Houdini primitives)
    • Snapping does not orient by default (only position)
  • Multi-snapping:
    • Solo 'Magnet' icon
    • <cntl> <shift> J to open parameters
    • or
    • RMB on icon to open parameters

Multi-snapping dialog

Use sliders to set 'off' of priority of snapping. This allows multiple snapping features at once.

  • Click 'Clear' to set all to 'off'

03-12-Snapping - Gravity & Occlusion

  • RMB on the Magnet icon(s) to open Snapping parameters.
  • x and choose snapping options from the radial menu.

Gravity

The 'gravity' slider sets the strength of the snap effect. (It's in screen space pixels.)

Occlusion

The 'occlusion' checkbox sets the behavior whereby only 'visible' features are snapped to. (So not to backside features or features behind object features.)

03-13-Snapping - Depth

  • RMB on the Magnet icon(s) to open Snapping parameters.
  • x and choose snapping options from the radial menu.

Depth Snapping

  • Depth on - snapping to actual position of 'snap to' features.
  • Depth off - snapping either to the view plane or if engaged, a c-plane

Depth off is useful in orthographic style working.

Note: Transforms along a single axis (clicking and dragging a move axis for example, still allows you to keep along that axis even with Depth switched 'on'.

03-14-Snapping - Interior Snapping

  • RMB on the Magnet icon(s) to open Snapping parameters.
  • x and choose snapping options from the radial menu.

Interior Snapping

Interior snapping is only available when 'Depth Snapping' is enabled.

Interior snapping works on the basis of snapping to a mid point along an imaginary line between the selected component and the opposite side of the geometry. So a front and back point on a sphere for example where they overlap in an orthographic view for example. (The resulting position is usually embedded within the geometry.)

There are two sub-options:

  • View based - the line is between the selected components and opposite 'side' according to the view direction of the 'snap to' component
  • Normal based - the line is between the selected component and opposite 'side' according to the normal directions of the snap to component

03-15-Snapping - Orient on Snap

Orient on Snap

The orient on snap checkbox rotates the object to the orientation of the snap to component.

  • Three modes:
    • Standard Axis to Frame:
      • 'y' axis to component normal
        • Snapping to primitive (polygon face) can sometimes flip the 'y' direction
      • 'x' axis to the tangent of the components
        • This is sometimes unpredictable, and each snap can orient wildly on the x axis
    • Nearest Axis to Frame:
      • 'y' axis to nearest direction of components
      • 'x' axis to nearest tangent
    • Nearest Axis to Normal:
      • 'y' axis to normal
      • 'x' is ignored

04-05-Connecting Nodes Across Networks - OBJ-MAT

Network Paths

Items in the Houdini scene are stored and organized by 'paths'. These are like an internal path structure.

Geo/Mat Connection example

If you look at the parameters for a geometry item, and switch to the 'Render' tab, you'll see a path to the Material container/manager if one is assigned (or you can assign one from here).

  • Jump to the material item/container by clicking on the curved arrow icon to the right of the path field.
  • You can also choose an item to assign using the floating tree dialog with the right most icon.

As an aside, you can cut and paste nodes from location to location in the path structure using copy and paste techniques.

04-06-Root Level Network Directories vs Custom Network Managers

Self Contained Networks

You can copy and paste nodes between Houdini scene files, but if assets are located throughout the network of Houdini's directories (within the Houdini file), these paths and assets might be disconnected and lost.

One solution is to nest things like materials within an asset's Geometry node.

  • Create a Material Context Manager (container) inside the geometry node
  • Set up material and apply to the geometry (set the path field to the node in the parameters panel)

Now this self contained set up can be copied & pasted between scene files and maintain the directory path links.

HDA assets (Houdini Digital Assets) are a more robust technique for transferring assets however.

04-07-Contexts Overview Part 1 - Geo & Obj

Wiring Nodes Together

Nodes are wired together, and the flow of data is affected by each node as the data travels through. (The use of 'null' nodes is utilized to signify bookmarks in the flow.)

Parent/Child Relationships

Dragging a 'wire' from one node to another sets up a Parent -> Child relationship. The 'from' node becomes the parent of the 'to' node. This can be illustrated by wiring two objects together, The transforms of the 'parent' will be inherited by the 'child' object.

04-08-Contexts Overview Part 2 - COPs

COPs Compositing Context

The compositing context can be used to prepare material maps. These are piped into material assests via the path field. However, the path field for images (like material maps) usually expect to path to files on an external drive or file system. If this is the case:

  • Add op: to the start of the path location:
    • e.g. op:/img/Box_texture/texture_out

You can filter for 'types' of assets in the file structure by clicking on the filter buttons above the 'Tree View' <shift> w in the network panel.

04-09-Contexts Overview Part 3 - ROPs

ROP or Render/Output Context

Look at the Render Tab, and see a rendering window. There are dropdowns to choose the render nodes from a list of all the render drivers in the file.

  • /out/blah/render_blah

The render parameters allow you to choose Candidate Objects and Phantom Objects.

  • Candidate Objects:
    • Regular visibility to render.
  • Phantom Objects:
    • Invisible to camera but still influence the scene with shadows and reflections etc.

Chaining Render Nodes

Wiring render nodes together make them dependencies, where the first will complete before the next (child) and so on.

Usually, separate nodes will render by frame, and one by one:

  • Frame 1:
    • render node A
    • render node B
    • render node C
  • Frame 2:
    • render node A
    • render node B
    • render node C

04-10-Contexts Overview Part 4 - MAT & VOPs

MAT/VOP Shader Building

The MAT context is basically VOPs. (The network editor panel displays the VEX Builder label).

Shaders in MAT/VOPs can be dragged and dropped onto objects in the Render View tab to assign

The MAT context allows for low-level shader building using nodes like 'constant', 'multiply' etc.

  • Networks are built Left to Right by wiring nodes together

High-level shaders are available (like the PrincipledShader) and can be used together with low-level nodes.

Shader nodes can be used (pathed to) from other contexts (like the COPs composite context etc.).

04-13-Contexts Overview Part 7 - TOPs

Task Operations (TOP)

The tutorial gives an example of a TOPs network whereby, a Foreground (FG) and Background (BG) layers are to be rendered, then composited (FG over BG). There is the notion of a 'wait for all' node which starts a video render as soon as all frames are complete.

The example features the interface while the TOP network is run, showing the Dots on the node previews (invoked via RMB menu to show tasks) where gray dots indicate idle frames/tasks, bright-green dots where rendering is in progress, and dark-green where renders have completed.

TOPs can be used to organize processes using object versions, or dynamics versions, in addition to render management.

04-14-Object Level vs Geometry Level Transforms and Coordinates

Object Level & Geometry Level

This lesson introduced the drop-down button next to the shade mode on the top right of the Viewport pane.

  • Show All Objects
  • Ghost Other Objects
  • Hide Other Objects

Using an example of a platonic solid (geometry, teapot) inside an Object container, the lesson shows how moving the geo within the object can create confusion with local space within the Object container and the global space of the Object container object itself.

Hide Other Objects while looking at the teapot geo (inside the Object container) shows a custom grid oriented to the teapot (as seen if the teapot is rotated in some way).

The takeway is to organize transforms logically within the Object container, to perhaps keep the geometry at the center of the Object container's space, and to transform the object to move and place the geo ultimately.

Matching transforms (using the icons at the end of the parameter rows) gets transform data from the Object container and not the internal geo for example.

Another quick tip was the D key to open the display options when hovering over the viewport display. (To change the background color for example.)

05-01-Working in the Network Editor

Object Level Node Flags and Node Rings

Node Flags

The Object node flags have colored tags:

  • Left - green <-- Selectability E key (while in the Network editor)
  • Right - blue <-- Visibility R key (while in the Network editor)

Multi-selecting nodes, then toggling these flags will affect all the selected nodes.

Node Ring

In the Network editor pane, hovering over a node will reveal a Node Ring.

  • Eye icon (blue) <-- visibility
  • Arrow (Green) <-- selectibility
  • i <-- info panel (persistent and pinable)
  • MMB <-- temp info panel

Display options

You can set the way that the node ring is evoked by setting the screen size of nodes before the node ring appears in the Display options. (Useful for busy Network editors!)

  • Display icon (top right of the Network Editor pane - eye icon)
  • D key while in the Network Editor pane.

05-02-Working With Geometry Level Nodes

Geometry Nodes

Geometry nodes have 4 flags:

  • Right - Blue - Visibility:
    • Only one node can have the visibility flag 'on'
    • This is the node that is visible on the outside at the Object level.

Inputs go into the dots on the tops of nodes.
Outputs come from the dots on the underside or bottom of the nodes.

05-03-Geometry Level Display, Render and Bypass Flags

Render, Display and Bypass Flags

Flags:

  • Render:
    • This item will be the render point
    • <cntl> LMB
    • T key
    • right-most
    • Either ONLY 'blue' in geometry container
    • Purple if 'blue is present in other nodes.
  • Display:
    • Viewport only
    • <alt> LMB
    • R key
    • right-most
    • Blue (if no other item has 'purple')
  • Render & Display:
    • LMB
    • Blue flag and Purple & Blue rings
  • Bypass:
    • Ignored by render/viewport
    • yellow
    • left-most
    • Q or B key
    • MMB Ring - down-arrow yellow icon

Render Flag will dictate which node marks the render process termination.
Display Flag will dictate what appears in the viewport (can be different to the actual render node).
Bypass Flag will be completely ignored and not processed.

Leaving loose nodes (not wired to any nodes being rendered/displayed and with no display/render flags) will effectively make them unprocessed by Houdini. There is no performance hit in having unflagged, unwired nodes.

05-04-Wiring Disconnecting Copying and Merging Nodes

If a node has a Warning icon, you can MMB on the node to read the error. Sometimes the error is in regard to 'wiring'.

Wiring Nodes

  • Drag out-dot to input-dot
  • Drag and drop a node between connected nodes (over the wires)
  • Dragging from an output node when nodes are multi-selected will result in a merge node
  • RMB on out-dot or input-dot to get a TAB menu of nodes; LMB to drop the pre-connected node

Disconnecting Nodes

  • Select a wire and hit Delete key
  • Y key and drag across wires (scissors tool)
  • Shake a node loose

Copying Nodes

  • Select node(s)
    • <alt> and drag to copy selected node(s)
    • <alt> <cntl> and drag to copy node and ALL subsequent nodes in the network
    • <alt> <shift> and drag to copy node and ALL previous nodes in the network
    • <alt> <cntl> <shift> and drag to copy node(s) as reference (instance) nodes:
      • RMB on node and select Actions > Create Reference Copy

To see reference connections in node schematic:

  • View > (Dependency Links) Show for All Nodes
  • Or
  • View > (Dependency Links) Show for Selected Nodes
  • These will show instances with orange links/wires

Merging Nodes

  • Select multiple nodes:
    • LMB drag from one of the out-dots
    • TAB to choose Merge node.

05-05-Template Flag

Template Flag

The template flag is the 2nd right-most flag on geometry notes inside the geometry container.

  • Purple when active.
  • Geometry is visible as a 'template' in the viewport
  • None, one or more can be flagged with the Template Flag:
    • LMB on flag to activate
    • Pink section on the ring menu
    • <shift> LMB to add items to the template flag group.
    • <cntl> LMB to make templated object 'selectable' in the viewport.
