Launch

Regular launch: 

vim

Launch WITH Vimrc WITHOUT plugins: 

vim --noplugin

Launch WITH plugins but WITHOUT Vimrc: Unlike vim -u NONE, which disables all configuration loading, NORC allows Vim to use default behaviors while avoiding user customizations. 

vim -u NORC

Launch WITHOUT vimrc and WITHOUT plugins: 

vim -u NONE

Launch WITHOUT vimrc and WITHOUT plugins and ignores 'compatible' setting: 

vim -u NONE -N

Launch WITHOUT vimrc and WITHOUT plugins and in modifiable mode but with a warning before quit: 

vim -u NONE -m

More on Options

Vim has a wide range of command-line options that allow you to customize how it starts. Here’s a breakdown of some of the most useful ones:

Basic Startup Options

Option Description Purpose
vim filename Opens filename in Vim Standard way to edit a file
vim -R filename Opens file in readonly mode Prevents accidental edits
vim -m filename Opens file in modifiable but warns on save Useful for cautious editing
vim -y Starts Vim in Easy Mode Makes Vim behave like a simple text editor

Session & Configuration Control

Option Description Purpose
vim -u NONE Starts Vim without any configuration files Debugging issues with .vimrc
vim -U NONE Starts GVim without configuration Similar to -u NONE but for GUI Vim
vim -u ~/.vimrc Loads a specific Vim configuration file Useful for testing different setups
vim -N Starts Vim in "normal" mode (ignores compatible setting) Ensures modern Vim behavior

File Handling & Navigation

Option Description Purpose
vim + Opens file and places cursor at last line Quick navigation
vim +25 filename Opens file and places cursor at line 25 Jump to a specific line
vim +/pattern filename Opens file and searches for pattern Useful for jumping to specific text
vim -o file1 file2 Opens files in horizontal splits Multi-file editing
vim -O file1 file2 Opens files in vertical splits Side-by-side editing
vim -p file1 file2 Opens files in tabs Organizing multiple files

Executing Commands on Startup

Option Description Purpose
vim -c 'command' Runs a Vim command after startup Automate tasks
vim -c 'set number' filename Opens file and enables line numbers Quick settings adjustment
vim -c 'vimgrep /T0DO/ */.py' Searches for T0DO in Python files Useful for code review

Restricted & Safe Modes

Option Description Purpose
vim -Z Restricted mode (disables shell commands) Security-conscious editing
vim -M Stricter readonly mode (prevents modifications) Ensures no accidental edits

Debugging & Performance

Option Description Purpose
vim --startuptime log.txt Logs Vim startup time to log.txt Debugging slow startup
vim --noplugin Starts Vim without plugins Troubleshooting plugin issues

When Would You Use These?

  • Debugging: vim -u NONE to test without configs.
  • Multi-file editing: vim -p file1 file2 for tabs.
  • Automating tasks: vim -c 'command' to run commands on startup.
  • Safe viewing: vim -M for strict readonly mode.

Launch Vim with a Command

The $(command) syntax in a shell substitutes the output of a command inline before executing the rest of the command. When used with vim, it allows you to dynamically specify files or content to edit.

How $(command) Works

  • $(command) runs the command inside parentheses and replaces it with its output.
  • Useful when you want Vim to open a file dynamically generated by another command.

Examples in Action

Open the Most Recently Modified File 

vim $(ls -t | head -n 1)
  • ls -t lists files by modification time, newest first.
  • head -n 1 picks the first file (the most recently modified).
  • Vim opens that file automatically!

Open Multiple Files That Match a Pattern

vim $(find . -name "*.txt")
  • find . -name "*.txt" searches for all .txt files.
  • Vim launches with all those files opened.

Open Files Modified in the Last Hour

vim $(find . -type f -mmin -60)
  • -mmin -60 finds files modified within the last 60 minutes.
  • Vim opens those files for quick review.

Open a File Based on a Keyword Search

vim $(grep -l "T0DO" *.txt)
  • grep -l "T0DO" *.txt lists files containing "T0DO".
  • Vim launches those files for editing.

Key Considerations

  • Use $(command) only when you know the command will return valid filenames.
  • If no matching files are found, Vim may return an error.
  • You can combine it with other shell commands for advanced automation.

Quit

Quitting Vim has multiple variations depending on whether you have unsaved changes, need to force quit, or want to close multiple buffers. Here’s a detailed breakdown:

Basic Quit Commands

Full Command Abbreviation Description
:quit :q Quits Vim if no unsaved changes exist
:q! - Quits without saving, even if changes exist
:wq - Saves and then quits Vim
:x - Saves only if changes were made, then quits

Example Usage:

  • If you haven’t made changes, simply type :q.
  • If Vim warns of unsaved changes and you don’t want to save, type :q!.

Saving Before Quitting

Full Command Abbreviation Description
:write :w Saves the current file without quitting
:wq - Saves and quits Vim
:x - Saves only if changes were made, then quits
ZZ - Equivalent to :wq (save and quit)

Example Usage:

  • You’ve made changes and want to save before quitting → use :wq or :x.
  • Instead of typing :wq, you can also press ZZ (Shift+Z Shift+Z).

Quitting Multiple Files or Buffers

Full Command Description
:qa Quit all open windows/tabs
:qa! Force quit all open windows/tabs without saving
:wqa Save all files, then quit

Example Usage:

  • If you’re editing multiple files and want to quit all at once → use :qa.
  • If some buffers have unsaved changes and you want to exit anyway → use :qa!.

Handling Specific Buffers or Windows

Full Command Description
:close Closes the current split window
:q Closes the current file
:bd (buffer delete) Closes the current buffer without quitting Vim

Example Usage:

  • If you’re working with split windows and want to close only one, use :close.
  • If you’re using multiple buffers and want to remove just one, use :bd.

Emergency Quit Options

Full Command Description
:q! Force quit without saving
:qa! Force quit all windows without saving
:wq! Save and force quit

Example Usage:

  • If Vim refuses to quit due to error messages → use :q!.
  • If you have many open files and need to quit instantly → use :qa!.

Final Thoughts

  • If you haven't edited anything, a simple :q works.
  • If you want to save changes, use :wq, :x, or ZZ.
  • If you need an emergency exit, :q! or :qa! will force quit.

Update

:update (or its abbreviation :up) fits into the quitting & saving category because it ensures that any unsaved changes are written to the file before proceeding.

What :update Does

  • Saves only if the file has been modified (does nothing if there are no changes).
  • Useful when you want to make sure changes are saved before quitting or performing another action.

How It Relates to Quitting

You can use :up alongside quit commands:

  • :update | quit → Saves changes only if needed, then quits.
  • :up | q → Same as above, but shorter.
  • :up | qa → Ensures all modified files are saved before quitting all buffers.

Why Use :update Instead of :w?

  • :w always saves, even if nothing changed.
  • :up only saves if modifications exist, preventing unnecessary disk writes.

Other Uses of :update

Can be combined with commands like :next, ensuring a file is saved before navigating: 

:update | next

Works well in auto-saving workflows, reducing unnecessary writes.
