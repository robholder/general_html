Hacking With Swift - 09 - How to store and find data in dictionaries

Swift's dictionaries behave in a similar why to Python, but the syntax is a little different for declaration and retrieval.

Dictionary

You can build a dictionary and the types will be defined at declaration: 

let employee2 = ["name": "Roger Mellie", "job": "anchor", "location": "London"]

Or: 

let employee2 = [
    "name": "Roger Mellie",
    "job": "anchor",
    "location": "London"
]

Unexpected output

Swift will give a weird result if you try to print a dictionary value like this: 

print(employee2["name"])

# >> Optional("Roger Mellie")

Swift needs a 'default' fallback to avoid this output: 

print(employee2["name", default: "Unknown"])
print(employee2["job", default: "Unknown"])
print(employee2["location", default: "Unknown"])

Mixing Value Types

String and Boolean: 

let hasGraduated = [
    "Eric": false,
    "Maeve": true,
    "Otis": false,
]

Integer and String: 

let olympics = [
    2012: "London",
    2016: "Rio de Janeiro",
    2021: "Tokyo"
]

print(olympics[2012, default: "Unknown"])

Declaring Data Types

Use [String: Int] syntax to declare 'string' keys and 'integer' values in a dictionary, as in this case): 

var heigths[String: Int]()

heights["Yao Ming"] = 229
heights["Shaquille O'Neal"] = 216
heights["LeBron James"] = 206

No Duplicate Keys

You can't have duplicate keys in Dictionaries. Assigning a value to an existing key will replace the value. 

var archEnemies = [String: String]()

archEnemies["Batman"] = "The Joker"
archEnemies["Superman"] = "Lex Luthor"

# If another value is assigned to "Batman", it gets replaced in the dictionary:
archEnemies["Batman"] = "Penguin"

Methods

You have many methods available to dictionaries.

For example:

  • .count():
    • This will give the number of key, value pairs.
  • .removeAll()
    • This will remove all entries in the dictionary (keys and values).
