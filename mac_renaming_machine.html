MacOS - Renaming Machines

(I did this to help with common settings across Macs on my network - so wanted a consistent naming strategy.)

CLI

While Apple's System Settings (General > About) allows for easy "ComputerName" changes, it often fails to update the underlying HostName or LocalHostName, leading to the exact mismatch you experienced with your Starship palettes.

The "Standard" Three-Step Rename

Run these in order to ensure everything is in sync:

ComputerName: The "friendly" name seen in Finder and AirDrop.

sudo scutil --set ComputerName "MyAwesomeMac"

LocalHostName: The name used by local network services (Bonjour). Usually has no spaces and ends in .local.

sudo scutil --set LocalHostName "MyAwesomeMac"

HostName: The name reported by the hostname command in your shell.

sudo scutil --set HostName "MyAwesomeMac"

Post-Rename Cleanup

After running these, it is highly recommended to flush your DNS cache so other devices (and Starship) see the change immediately: 

sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder

Check Results

scutil --get ComputerName
scutil --get LocalHostName
scutil --get HostName

Only Use Approved Letters/Characters

The Rule for LocalHostName

Unlike your "ComputerName" (which can have spaces, emojis, and apostrophes), the LocalHostName must:

  • NOT have spaces. (Use a hyphen - instead).
  • NOT have special characters (like !, @, #, or ').
  • NOT have 'underscores' ('_')
  • Only use letters, numbers, and hyphens.
  • Be under 63 characters.
