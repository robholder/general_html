Starship - Setting Alternative and Simple Prompts

QUESTION

Is it possible to 'load' a starship config on the shell I'm in?

ANSWER

Yes, you can "load" it immediately by re-exporting the STARSHIP_CONFIG variable and then re-initializing the shell's prompt. You don't need to restart your terminal.

Run these two commands in your current shell:

Set the path: 

export STARSHIP_CONFIG=~/.config/starship/starship-tmux.toml

Force a reload (for Zsh/Bash): 

eval "$(starship init $(basename $SHELL))"

Why this works

Starship doesn't "watch" the config file for changes in real-time. It reads the file once when the shell starts (via that eval command). By running the eval again manually, you're telling Starship to look at the environment variable, find the file, and re-draw the prompt using the new settings.

A Pro Tip: Create a "Reload" Alias

If you're tinkering with your .toml files, add this to your .zshrc or .bashrc: 

alias sreload='eval "$(starship init $(basename $SHELL))"'

Now, whenever you save your .toml file, just type sreload to see the changes instantly.

  • One warning: If you are using Tmux, remember that environment variables set in one pane don't automatically update in other panes. You'll need to run the export and eval in each pane where you want the new look.

QUESTION

How do I force the default prompt from Starship? I notice that if my export - config line is not pointing to an existing file, the prompt falls back to a Starship default. How do I invoke this directly?

ANSWER

To force Starship to use its internal defaults, you simply need to point STARSHIP_CONFIG to a non-existent path or an empty file.

Since Starship can’t find any configuration instructions, it falls back to its "factory settings." You can do this on the fly in your current shell with: 

export STARSHIP_CONFIG=/dev/null

Why this works:

  • /dev/null is a special "empty" file on Unix systems.
  • When Starship tries to read it, it finds nothing.
  • Because it finds no format string or module overrides, it triggers its built-in default layout (which usually includes the user, directory, and git branch).

Other ways to "Reset" to Default:

1. Unset the variable entirely

If you want to return to your standard ~/.config/starship.toml (after you've been messing with a Tmux-specific one), run: 

unset STARSHIP_CONFIG

2. Temporary Default (Single Command)

If you just want to see the default prompt for one second without changing your session variables: 

STARSHIP_CONFIG=/dev/null starship explain

3. The "Pure" Default (No Shell Integration)

If you want to see what the prompt looks like without any of your shell's colors or extra logic interfering, run: 

starship prompt --config /dev/null

QUESTION

Is there a way to add this as a zshrc alias so that I have an instant 'clean' prompt?

ANSWER

To get an instant "clean" prompt via an alias, you need the alias to do two things: point the config to /dev/null and then re-initialize the shell to apply it.

Add this to your ~/.zshrc: 

alias star-clean='export STARSHIP_CONFIG=/dev/null && eval "$(starship init zsh)"'

Why use eval in the alias?

If you only use export, the prompt won't actually change until you execute a command or open a new shell. Adding the eval line forces Starship to re-read the configuration (or lack thereof) and redraw your prompt immediately.

The "Reset" Alias

You’ll likely want a way to get back to your custom setup without restarting your terminal. Add this companion alias: 

alias star-reset='unset STARSHIP_CONFIG && eval "$(starship init zsh)"'

Note: This assumes your main config is in the default location (~/.config/starship.toml).

How to use them:

  • Type star-clean: Your prompt instantly reverts to the Starship factory defaults.
  • Type star-reset: Your prompt instantly switches back to your custom themed version.
  • One tip for macOS Mojave: If you notice a slight delay when running these, it's because Starship is re-running its initialization. It’s fast, but for an even "cleaner" look, you could also add clear to the start of the alias to wipe the screen at the same time.

Toggle Prompt Alias

Would you like to include a toggle function instead, so one command switches back and forth?

To make a single command that flips between your Custom Theme and the Clean Default, add this function to your ~/.zshrc.

I've named it toggle-prompt, but you can rename the alias to whatever is easiest to type (like tp). 

# Toggle between custom Starship config and factory defaults
toggle-prompt() {
  # Check if the config is currently set to /dev/null (the 'clean' state)
  if [[ "$STARSHIP_CONFIG" == "/dev/null" ]]; then
    # Return to normal (unsetting defaults it back to ~/.config/starship.toml)
    unset STARSHIP_CONFIG
    echo "✨ Custom prompt restored."
  else
    # Switch to factory defaults
    export STARSHIP_CONFIG="/dev/null"
    echo "🧹 Prompt cleaned."
  fi
  
  # Re-initialize to apply the change immediately
  eval "$(starship init zsh)"
}

# Alias for quick access
alias tp='toggle-prompt'

Why this version is great:

  • Smart Detection: It looks at the current value of $STARSHIP_CONFIG so it always knows which "mode" you are in.
  • Instant Feedback: The echo lines let you know exactly what happened (you can remove them if you want it silent).
  • No New Shell Required: The eval line forces Zsh to redraw the prompt the moment you hit Enter.

One Small Tweak for your Tmux Setup:

If your "normal" state isn't the default ~/.config/starship.toml (e.g., you want it to toggle back to your specific starship-tmux.toml), change the unset line to:

  • export STARSHIP_CONFIG=~/.config/starship/starship-tmux.toml

To apply this now:

  • Save your ~/.zshrc.
  • Run source ~/.zshrc.
  • Type tp and watch the magic happen.
