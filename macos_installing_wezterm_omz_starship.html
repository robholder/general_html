Installing Wezterm, OMZ & Starship

I'm thinking NOT to use OMZ, as Starship (for my purposes) achieves the same thing. I install it in this page, but then uninstall it at the end.

QUESTION

I want to install wezterm, omz and starship on macOS. What's a good strategy?

ANSWER

To install WezTerm, Oh My Zsh (OMZ), and Starship on macOS, the most efficient strategy is to use Homebrew for the binaries and the official install script for OMZ.

1. Install Homebrew (If not already installed)

Open the default macOS Terminal and run the official command from brew.sh: 

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

2. Install WezTerm

Use Homebrew's cask system to install the WezTerm GUI application: 

brew install --cask wezterm
  • Font Requirement: Starship and OMZ themes often require icons. It is recommended to install a Nerd Font (like JetBrains Mono Nerd Font) and set it in your WezTerm configuration.

3. Install Nerd Font(s)

brew install --cask font-jetbrains-mono-nerd-font

4. Install Oh My Zsh

Run the official Oh My Zsh installation script: 

sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"
  • This script will create a default ~/.zshrc file and set Zsh as your default shell if it isn't already.

5. Install Starship

Install the Starship cross-shell prompt via Homebrew: 

brew install starship

6. Configure Everything to Work Together

To make Starship your primary prompt while using Oh My Zsh features:

  • Initialize Starship: Open your Zsh configuration file (nano ~/.zshrc) and add the following line to the very end: 
eval "$(starship init zsh)"
  • Configure WezTerm: Create a configuration file at ~/.wezterm.lua (or ~/.config/wezterm.lua) to define your font and appearance: 
local wezterm = require 'wezterm'
local config = wezterm.config_builder()

-- 1. Font (Use the exact name WezTerm confirmed earlier)
-- config.font = wezterm.font("JetBrainsMono Nerd Font", { weight = Medium } )
config.font = wezterm.font("JetBrainsMono Nerd Font", { weight = 700 } )
config.font_size = 12
config.color_scheme = 'Nord (Gogh)' -- Example scheme

-- 2. The "Thickening" Combo (No errors this time!)
-- 'Normal' provides the most 'body' to the letters.
config.freetype_load_target = 'Normal'
config.freetype_render_target = 'HorizontalLcd'

-- 3. Horizontal Presence
-- This is your best tool for making 'Regular' weight look 'Medium'.
config.cell_width = 1
config.line_height = 1.1

-- 4. Subpixel Antialiasing (The 'Smoothness' factor)
-- This mimics the 'Font Smoothing' in macOS Terminal.
config.freetype_load_flags = 'DEFAULT'

return config

Pro Tip: Because Starship handles your prompt entirely, you can leave the ZSH_THEME="" variable empty in your ~/.zshrc to avoid conflicts.

Also, look at this link for color schemes for WezTerm:

  • https://wezterm.org/colorschemes/

QUESTION

At which point to I install the required nerd fonts, and how is this done?

ANSWER

Installing Nerd Fonts is best done after installing Homebrew but before final configuration of WezTerm or Starship. These fonts are essential for displaying the icons and glyphs used by Starship and modern Zsh themes.

How to Install via Homebrew

Historically, fonts required a separate "tap," but they are now integrated directly into the main Homebrew repository.

Search for a font: Most users prefer JetBrains Mono or Fira Code. 

brew search nerd-font

Install your choice: For example, to install the popular JetBrains Mono Nerd Font: 

brew install --cask font-jetbrains-mono-nerd-font

How to Enable in WezTerm

Once installed, you must tell WezTerm to use it by editing your ~/.wezterm.lua (or ~/.config/wezterm) file: 

local wezterm = require 'wezterm'
local config = wezterm.config_builder()

-- Set the font to the one you just installed
config.font = wezterm.font 'JetBrains Mono Nerd Font'

return config

Note: WezTerm actually bundles its own "Nerd Font Symbols" as a fallback, meaning icons often work even without a patched font, but installing a dedicated Nerd Font ensures better alignment and styling.

Alternative: Manual Install

If you prefer not to use Homebrew for fonts, you can:

  • Download a font zip from the Nerd Fonts Downloads page.
  • Open the Font Book app on your Mac.
  • Drag and drop the unzipped .ttf or .otf files into Font Book and click Install.

Note On Wezterm Config Location

You do not have to leave it in the home directory. In fact, many users prefer moving it to a dedicated folder to keep their home directory clean.

WezTerm automatically looks for a configuration file at ~/.config/wezterm/wezterm.lua. This is the standard "XDG" location for configuration files on macOS and Linux, keeping your home directory tidy.

Create the directory: 

mkdir -p ~/.config/wezterm

Move your file there: 

mv ~/.wezterm.lua ~/.config/wezterm/wezterm.lua

Note: When moving to this folder, the leading dot in the filename is typically removed (use wezterm.lua instead of .wezterm.lua).

Install Order

QUESTION

Does it matter what order I install WezTerm, OMZ and Starship?

ANSWER

While the absolute installation order doesn't technically "break" the apps, following this specific sequence ensures the smoothest setup on macOS:

1. Homebrew & WezTerm (First)

Install the terminal emulator first so you have a modern environment to perform the rest of the installation steps.

  • Action: Install Homebrew and then WezTerm.
  • Why: Default macOS Terminal doesn't always render modern Nerd Font icons correctly during the setup process.

2. Nerd Fonts (Second)

Install your chosen font immediately after getting WezTerm.

  • Action: brew install --cask font-jetbrains-mono-nerd-font (or your preferred font).
  • Why: Without this, the next two steps (OMZ and Starship) will look "broken" with squares or question marks instead of icons.
See below for how to verify the nerd fonts before moving on.

3. Oh My Zsh (Third)

Install the shell framework.

  • Action: Use the Official OMZ curl script.
  • Why: The OMZ installer replaces or creates a new ~/.zshrc file. If you install Starship before OMZ, the OMZ installer might overwrite the Starship initialization lines you just added.

4. Starship (Last)

Add the prompt to your final shell configuration.

  • Action: brew install starship and add eval "$(starship init zsh)" to the very bottom of your ~/.zshrc.
  • Why: Starship needs to be initialized after Oh My Zsh to ensure it correctly overrides any default themes provided by OMZ.

Would you like the specific command to verify that your Nerd Fonts are active in WezTerm before you start the OMZ install?

Verify the Nerd Fonts

To verify that WezTerm is correctly seeing and using your Nerd Font before proceeding with your Oh My Zsh installation, you can use these built-in CLI commands:

1. Verify Configuration

Run this to see which font WezTerm is currently using based on your wezterm.lua file: 

wezterm ls-fonts

Look for the "Primary font" section in the output. It should display the name of the Nerd Font you installed (e.g., JetBrains Mono Nerd Font).

2. Verify System Availability

If the first command doesn't show your font, check if WezTerm can see it on your macOS system at all: 

wezterm ls-fonts --list-system | grep -i "Nerd Font"

This lists all available system fonts that WezTerm can access.

3. The "Visual Test" (Crucial)

Even if the name is correct, the icons must actually render. Run this command to print common Nerd Font symbols to your terminal: 

echo -e "\uf001 \uf002 \uf00c \uf00d \uf179 \uf121"
  • Success: You see a music note, magnifying glass, checkmark, "X", Apple logo, and code brackets.
  • Failure: You see empty boxes or question marks. This means your wezterm.lua isn't correctly pointing to the installed font.

4. Troubleshooting Specific Icons

If a specific icon in Starship looks broken later, you can use this to see exactly which font WezTerm is using to render that specific character: 

wezterm ls-fonts --text "  "

(Replace the icon inside the quotes with whatever icon is appearing as a box).

Once you've confirmed icons are visible, you are safe to proceed with the Oh My Zsh installation script.

More On Fonts

QUESTION

How do I know what a font should be called in the lua config?

When I install a font, the name is concatenated with hyphens, but in the lua config, they need to be called by a different layout.

i.e. JetBrainsMono Nerd Font when at install time it is:

font-jetbrains-mono-nerd-font

ANSWER

It's easy to get mixed up because you’re dealing with three different "names" for the same thing: the Homebrew Formula, the File Name, and the PostScript Name.

1. The Homebrew Formula (The "Package")

  • Example: font-jetbrains-mono-nerd-font
  • Purpose: This is just the "box" the font comes in. It tells brew what to download from the internet Homebrew Formulae. It has no relationship to what you type in your Lua config.

2. The File Name (The "Storage")

  • Example: JetBrainsMonoNerdFont-Regular.ttf
  • Purpose: This is how macOS sees the file on your hard drive. WezTerm can sometimes find fonts by filename, but it isn't the standard way.

3. The Full Family Name (The "Lua Name")

  • Example: "JetBrainsMono Nerd Font"
  • Purpose: This is the internal metadata (the "Face Name") that the font designer embedded inside the file. This is what WezTerm wants.

How to find the "Lua Name" for any font

Instead of guessing, use these two foolproof methods:

Method A: The WezTerm "Source of Truth" (Best)

Run this command to see exactly what WezTerm sees in its brain: 

wezterm ls-fonts --list-system

Scroll through (or grep) the list. Look for the line that says family: "...". Whatever is inside those quotes is the exact string you must put in your wezterm.lua.

Method B: The macOS "Font Book"

  • Open the Font Book app on your Mac.
  • Click on your installed font.
  • Look at the top of the window or the info panel (⌘+i).
  • The "Family Name" listed there is almost always what you need for WezTerm.

Pro Tip: Concatenation

Most "Nerd Font" versions of popular fonts (like JetBrains or Fira) remove the spaces from the original name (e.g., "JetBrains Mono" becomes "JetBrainsMono") but keep the "Nerd Font" suffix separate.

  • Quick Test: If you just installed a new font and want to see if WezTerm "found" it, try: 
wezterm ls-fonts --list-system | grep "Family Name You Think It Is"

Configuring Starship

To make Starship icons match the "beefy" 700-weight look of your font, we need to initialize Starship within your Oh My Zsh configuration and then apply a Preset that uses heavy-duty Nerd Font symbols.

1. Initialize Starship in OMZ

Open your Zsh configuration file: 

nano ~/.zshrc

Go to the very bottom of the file (after the Oh My Zsh lines) and add: 

# Initialize Starship
eval "$(starship init zsh)"

2. Apply a "Beefy" Icon Preset

Starship icons can sometimes look "thin" even if the text is thick. To fix this, use the Nerd Font Symbols preset which uses larger, more substantial glyphs.

Run this command in your terminal to create the Starship config file with the preset: 

mkdir -p ~/.config && starship preset nerd-font-symbols -o ~/.config/starship.toml

3. Sync Starship Weights with WezTerm

To ensure the icons don't look "spindly" compared to your 700-weight text, you can tell Starship to use bold styling for its modules. Open the newly created Starship Configuration: 

nano ~/.config/starship.toml

Look for (or add) these global settings to ensure the prompt matches your WezTerm "beefiness": 

# This makes the prompt symbols as bold as your 700-weight font
[character]
success_symbol = "[➜](bold green)"
error_symbol = "[➜](bold red)"

[directory]
style = "bold cyan"

4. Remove OMZ Theme Conflict

Since Starship is now your "Painter," tell Oh My Zsh to stop trying to paint the prompt by clearing the theme in ~/.zshrc: 

# Find this line and set it to empty
ZSH_THEME=""

5. Final Visual Check

Restart WezTerm. You should now see a prompt that:

  • Uses your 700-weight JetBrainsMono Nerd Font.
  • Has thick, bold icons for Git, Python, Node, etc.
  • Retains the subtle colors we configured in your wezterm.lua.

Pro Tip: If the icons still look a tiny bit smaller than the text, you can "fake" a larger icon size in wezterm.lua by adding a fallback rule specifically for the symbols range, but usually, the nerd-font-symbols preset is enough.

Uninstall OMZ (Oh My Zsh)

Uninstall OMZ with this command: 

uninstall_oh_my_zsh

This will look for an existing .zshrc.<backup> file if present, and replace the current OMZ .zshrc file with this one. (It'll rename the .zshrc.<backup_suffix> to .zshrc and backup the existing .zshr from the OMZ install to another backup, with the format: .zshrc.omz-uninstalled<date>.)

You'll have to add anything that ISN'T OMZ back into this replacement if it's out of date (with other things added since using OMZ). For example, you'll have to add Startship lines, or your own aliases and entries.

If there is an OMZ directory present, you can delete it: {{sh rm -rf ~/.oh-my-zsh }}}

In my case, I didn't have this directory.

When the .zshrc is restored (with any extra items you've re-added from the backed up .zshrc.omz-uninstalled<date> file), you can just restart the terminal.
