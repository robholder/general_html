Vimwiki and Github

A Journey to Deploy Github Backups For Vimwiki

Goal

  • Get Vimwiki to backup to a Github repo, and to be able to 'pull' the wiki to local machines (work and home) to be able to consolitdate notes in one place.
  • This involves a number of strategic arrangements:
    • Github:
      • Store both regular wiki pages and converted html files.
      • Pull the latest vimwiki from Github when editing the Vimwiki
      • Upload edited files to Github from whichever location I'm working from.
    • Local Machine:
      • Setup Vim to use Vimwiki:
        • Configure vimrc
      • Setup the actual Vimwiki pages:
        • Clone from Github, then maintain the Vimwiki ready for pull/push.
        • Store on a common file path
        • Add bin shell script to 'git push' the edited files to the repository

Items

  • SSH:
    • setup ssh-key on machines to allow communication with Github
      • Configure vimrc
  • Create initial Vimwiki:
    • Choose directory/file location
    • Setup vimrc
    • Clone from Github (1st time)
    • Pull from Github (subsequent edits)
    • or:
      • Make initial Repo:
      • Git init
  • Create Github Repo:
    • Git push
  • Pull Repo:
    • Clone the vimwiki on additional machines
