Comment Out Code

You can comment out a block of code in Xcode using several methods, ranging from a quick keyboard shortcut to a more "permanent" block syntax.

1. Standard Comment Shortcut

The fastest way to comment out a block is to highlight the lines and use the standard toggle:

  • Keyboard Shortcut: Command (⌘) + /.
  • Result: This inserts // at the beginning of every selected line.
  • Pressing it again on the same selection will uncomment the lines.

2. C-Style Block Comments

If you want to wrap code in a single /* ... */ block rather than prefixing every line with //:

  • Manual Entry: Type /* at the start and */ at the end.
  • Xcode 15+ Feature: In newer versions of Xcode, if your selection starts or ends in the middle of a line, using ⌘ + / may automatically use the /* ... */ block style instead of the line-by-line // style.

3. Documentation Block Comments

For creating structured documentation comments for functions or variables:

  • Keyboard Shortcut: Option (⌥) + Command (⌘) + /.
  • Result: This generates a doc-comment block (e.g., /// or /** ... */) and often automatically includes fields for parameters and return values if placed above a function.

4. Alternatives for Large Sections

If you need to "comment out" very large blocks that might already contain comments (which can break standard block comments), you can use conditional compilation:

  • Syntax: Wrap the code in #if 0 and #endif.
  • Benefit: This ensures the compiler completely ignores the section, regardless of any internal /* */ or // markers.

    Toggle Line Comment ⌘ + /
    Add Doc Comment ⌥ + ⌘ + /
