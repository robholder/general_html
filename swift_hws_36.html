Hacking With Swift - How to make one class inherit from another

Swift allows classes to inherit functionality from existing classes (inheritance), enabling customization through access to parent class (parent or 'super' class) properties and methods.

Inheritance

Inheritance in Swift: Colon after child class name, followed by parent class name.

Example

Parent or Super Class:

class Employee {
    let hours: Int

    init(hours: Int) {
        self.hours = hours
    }
    func printSummary() {
        print("I work \(hours) hours a day.")
    } 
}

Sub-classes

class Developer: Employee {
    func work() {
        print("I'm writing code for \(hours) hours.")
    }
}

class Manager: Employee {
    func work() {
        print("I'm going to meetings for \(hours) hours.")
    }
}

let robert = Developer(hours: 8)
let joseph = Manager(hours: 10)
robert.work()
joseph.work()

// >> I'm writing code for 8 hours.
// >> I'm going to meetings for 10 hours.

robert.printSummary()
// >> I work \(hours) hours a day.

Overriding Methods

Swift will refuse to build code if overrides are in error (either not overridden, or if original doesn't exist).

Overrides are preceded by the key word "override": 

class Developer: Employee {
    func work() {
        print("I'm writing code for \(hours) hours.")
    }
    override func printSummary() }
        print("I'm a developer who will sometimes work \(hours) hours a day, but other times will just argue.")
    }
}

Exceptions

If the parent class has a method that doesn't return a value, but the child (sub-class) method DOES, then you don't need the 'override' keyword.

Swift is smart about how method overrides work:

  • if your parent class has a work() method that doesn't return something, but the child class does, your don't need the override keyword, and...
  • if your parent class has a work() method that takes no parameters, but the child class has a work() method that accepts a string to designate where the work is being done, that does not require override because you aren’t replacing the parent method.

(This isn't entirely correct.)

🎯 The real rule

You must use override whenever your subclass defines a method (or property) with the same signature as one in the superclass. You can omit override only when the method you write does not match any method in the superclass.

That’s it.

🧠 What counts as “the same signature”?

A method signature includes:

  • method name
  • parameter labels
  • parameter types
  • return type

Example when override isn't needed: 

class Animal {
    func speak() { }
}

class Dog: Animal {
    func speak(volume: Int) { }   // different signature → NOT an override
}

Final

If you want to ensure that your current class CANNOT be inherited by a sub-class, use the 'final' keyword. 

final class Developer: Employee {
    func work() {
        print("I'm writing code for \(hours) hours.")
    }
}

The tutor tends to make all classes 'final' (default) and flip them by removing the keyword when the class needs to be an inheritable class.

Examples from Tests

class Vehicle {
    var wheels: Int
    init(wheels: Int) {
        self.wheels = wheels
    }
}

class Truck: Vehicle {
    var goodsCapacity: Int
    
    init(wheels: Int, goodsCapacity: Int) {
        self.goodsCapacity = goodsCapacity
        super.init(wheels: wheels)
    }
}

let truck = Truck(wheels: 6, goodsCapacity: 5000)
print(truck.wheels)         // 6
print(truck.goodsCapacity)  // 5000

class Student {
	var name: String
    init(name: String) {
        self.name = name
    }
}
class UniversityStudent: Student {
	var annualFees: Int
	init(name: String, annualFees: Int) {
		self.annualFees = annualFees
		super.init(name: name)
	}
}

class Handbag {

	init(price: Int) {
		self.price = price
	}
}
class DesignerHandbag: Handbag {
	var brand: String
	init(brand: String, price: Int) {
		self.brand = brand
		super.init(price: price)
	}
}

class Product {
	var name: String
	init(name: String) {
		self.name = name
	}
}
class Book: Product {
	var isbn: String
	init(name: String, isbn: String) {
		self.isbn = isbn
		super.init(name: name)
	}
}

class Bicycle {
	var color: String
	init(color: String) {
		self.color = color
	}
}
class MountainBike: Bicycle {
	var tireThickness: Double
	init(color: String, tireThickness: Double) {
		self.tireThickness = tireThickness
		super.init(color: color)
	}

class Instrument {
	var name: String
	init(name: String) {
		self.name = name

class Piano: Instrument {

	}

	var cost: Int
	init(cost: Int) {
		self.cost = cost
	}
}
class LaserPrinter: Printer {
	var model: String
	init(model: String, cost: Int) {
		self.model = model

	var sides: Int
	init(sides: Int) {
		self.sides = sides
	}
}
class Rectangle: Shape {
	var color: String
	init(color: String) {
		self.color = color
		super.init(sides: 4)
	}
