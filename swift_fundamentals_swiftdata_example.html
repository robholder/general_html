Minimal Example

Here is a complete, minimal, and fully functioning SwiftData project contained in a single file. You can drop this directly into a new Xcode project to use as a cheat sheet.

It covers the three pillars:

  • the @Model blueprint,
  • the .modelContainer setup,
  • and the @Query property wrapper to pull data in and out.

The Complete Minimal Example

import SwiftUI
import SwiftData

// 1. THE MODEL: Define the data structure
@Model
class Item {
    var timestamp: Date
    
    init(timestamp: Date = .now) {
        self.timestamp = timestamp
    }
}

// 2. THE APP ROOT: Inject the database container
@main
struct SwiftDataMiniApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
        // This sets up the database storage in the background for our Item class
        .modelContainer(for: Item.self)
    }
}

// 3. THE VIEW: Read from and write to the database
struct ContentView: View {
    // Access the 'manager' context used to insert or delete data
    @Environment(\.modelContext) private var modelContext
    
    // Fetch and automatically update the list of items from the database
    @Query private var items: [Item]

    var body: some View {
        NavigationStack {
            List {
                ForEach(items) { item in
                    Text("Saved at: \(item.timestamp.formatted(
                        date: .omitted,
                        time: .standard)
                    )")
                }
                .onDelete(perform: deleteItems) // Swipe to delete from DB
            }
            .navigationTitle("Mini SwiftData")
            .toolbar {
                Button("Add Item", systemImage: "plus", action: addItem)
            }
        }
    }

    // CREATE Logic
    func addItem() {
        let newItem = Item()
        modelContext.insert(newItem) // Saves it instantly to the database
    }

    // DELETE Logic
    func deleteItems(offsets: IndexSet) {
        for index in offsets {
            modelContext.delete(items[index]) // Vaporizes it from the database
        }
    }
}

How the Data Flows (In & Out)

  • Going In (Write): You grab the environment's modelContext. Think of this context as your active scratchpad. When you call modelContext.insert(newItem), SwiftData takes that scratchpad object and flushes it down into the permanent database file managed by your app container.
  • Coming Out (Read): You use @Query private var items: [Item]. This acts as a live, open pipe directly into the database file. You don't have to call a "refresh" function when you add or delete something; the moment the data file changes, @Query notices it, pulls the fresh data out, and tells SwiftUI to redraw the List.
