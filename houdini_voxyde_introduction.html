Incidential nodes covered

attribute create - adding arbitrary attribute (not really needed as subsequent nodes can create).
attribute adjust float - editing the arbitrary attribute (in example, randomized). Select component (point, prim etc.)
add - create points, polys or particles
attribute adjust vector - editing the 'P' attribute (noise, and noise type animationg)
attribute noise vector - like above, but with blend option (to blend with input state)
attribute vop - This is the super-duper attribute node. Can reproduce all the others.
Vector to float - Takes vector (3d) and offers three outputs
Fit range - remap values (like -3 to 3 --> 0 to 1: source min is -3, source max is 3, dest min is 0, dest max is 1)
attribute wrangle - This is a VEX version of the attribute vop node (which uses nodes to apply edits)


Inside attribute vop

Color outputs on vop nodes:

  • green - vector
  • teal (brighter than green) - float
  • blue - integer
  • sap green/yellow - 4d

Promote parameters to Node

To show an inner parameter to the enclosing node parameters panel:

  • MMB on parameter of inner node:
    • Promote Parameter

Solvers

  • popsolver - particles (points):
    • input: pop object
  • pyrosolver - fire:
    • input: smoke object
  • flipsolver - fluid:
    • input: flip object
  • rigidbodysolver - rigid body:
    • input: rbd packed object
  • vellumsolver - cloth:
    • input: vellum object
  • (simple) solver - base:
    • input: anything?

Animating particles (or whatever) with noise is different in a solver. The solver is more organic as it calculates motion from the previous frame, whereas, animating parameters always derives from the original position of the particle(s).

YouTube VOPs Intro

VOPs are a gui for VEX based operations.

At the start of the video, the following nodes are previewed (inside a geometry node):

  • attribute vop
  • Point vop
  • Vertex vop
  • Primitive vop

These are the same thing with different settings. The attribute vop and the point vop are identical.

A Volume vop is shown breifly, but is not the same as the nodes previewed above.

Inside a VOP node

Inside is the input (left) and the output (right). These can be joined with other nodes between to perform procedures and adjustments.

Constant Node

Can be plugged into nodes such as 'add' to add value to stream of data - position perhaps (P).

  • Constant:
    • select from:
      • Float
      • 3d Vector
      • etc.

Float To Vector

Like the Constant node when set to 3d Vector. Offers sliders; one for each 'axis'.

  • Float To Vector:
    • You can plug values/variables into it too:
      • eg. Time into one of the vectors

Vector To Float

Opposite of Float to Vector.

  • Vector To Float:
    • Split vector axis:
      • perform actions on each
      • re-assemble back to vector with Float to Vector

Promote Parameter

MMB on a parameter (input on node or parameter on parameter pane) and chose Promote Parameter to have it show up on the outer (attribute vop) node's parameter panel.

Bind node

Bind and Bind Export are the same but with different settings.

You can add attributes from outside the network (like 'P'). The export option node can replace the regular output node of the attribute vop node's contents too.

Any of the inputs (on the default input node) can be replicated with a 'bind' node (e.g., 'Time'). This can replace the default input node, just like the export replacing the output.

You can export parameters to new attributes like this too.

Geometry vop input & output nodes

At any time, you can renew the default input (geometry vop global) and default output (geometry vop output).

Import Point

The import point node can be used to import from other geo as plugged into the 'attribute vop' node (other inputs).

Connect OpInput# out (geometry vop global - input) to the new 'import point' node. These OpInputs map to the inputs on the exterior of the attribute vop node.

Or you can just set the import point node to invisibly import from these inputs (using the 'input' drop-down on the node's parameter panel). Set to 'file' to use by connection to the input node.

Mix node

The 'mix node' blends between two inputs. Maybe positions of two version of a geometry.

Displace Along Normals

In the lesson, the displace along normals node was plugged into the output node's 'P' slot. When used on a grid, the grid just rises along the normals (upward), but adding a turbulent nose node, and plugging the inputs 'P' into the 'pos' of the noise node, and output (noise) into the 'scale' input of the 'displace along normals' node, results in a noise pattern of vertical displacements. The 'Displacement Amount' slider adjusts the effect.

Negate

The Negate node reverses the polarity of a value. Minus becomes positive etc. It's basically it's a multiply by -1.

Compliment

compliment subtracts the input value from 1: 1-x.

Absolute

The absolute node will take all negative values and make them positive.

  • example:
    • -1 becomes 1
    • 1 remains 1

Clamp

Clamp just clamps values to that set in Min and/or Max

Power

The power node takes the input and raises it by an exponent (power of).

Fit To Range

The fit to range node, remaps values (source) to destination values.

Ramp Parameter

A visual node, much like the 'fit to range' idea, the ramp parameter node allows for more complex remaps (even colors).

You can choose from RGB Ramp or Spline Ramp (float).

Spline Ramp

You can expand the visual for the spline ramp, to see a graph on which you can 'draw' profiles.

Confusingly, the node's parameters don't actually affect the network until you go up to the enclosing (attribute vop) node's parameters and MMB click on the 'ramp' which appears on there. This will then transfer the edit from the internal ramp node to the external parameter. Weird.

LMB click to add control points.

<shift> click on the little tab handles (underneath the ramp graphic) to select them, and then change the interpolation on the drop-down if desired.

NOTE: Gotchas!!

  • The ramp clamps from zero to one! Perhaps use multiply to expand later.
  • Change the 'name' parameter for each ramp to be able to edit correctly:
    • symptom is 'grayed out' parameters on the new ramp.

Maximum

The maximum node takes two inputs and the higher value per sample 'wins'.

Minimum

The minimum node takes two inputs and the lower value per sample 'wins'.

Ceiling

The ceiling node pushes values to the next integer value above the 'point'. (However, the value is still a float.)

  • Example:
    • 1.3 to 2
    • 8.6 to 9

Round To Integer

The round to integer will 'round' to the nearest integer value for a given float. (However, the value is still a float.)

  • Example:
    • 1.3 to 1
    • 8.6 to 9

Floor

The floor node pushes values to the next integer value below the 'point'. (However, the value is still a float.)

  • Example:
    • 1.3 to 1
    • 8.6 to 8

Float to Integer

The float to integer node takes a float as an input, and outputs a genuine integer.

Integer to Float

The integer to float node takes an integer as an input, and outputs a float. (It is important to convert integers to floats if something like a divide node is used. Passing an integer to a divide will not acta as expected!)

Fraction

The fraction node leaves only the decimal fraction of an input. (Removes the 'whole' number from the float.)

  • Example:
    • 1.3 to 0.3
    • 8.6 to 0.6

Modulo

The modulo node, divides input 1 by input 2, and returns the remainder.

  • Example:
    • 1 divided by 5 --> remainder of 1
    • 5 divided by 5 --> remainder of 0
    • 8 divided by 5 --> remainder of 3
    • 12 divided by 5 --> remainder of 2

NOTE: with 'frame' as input 1 and 5 as input 2, the result will loop from 0 to 4.

Compare

The compare node compares input 1 to input 2, and if it's the same, it outputs a 1 (else 0) if set to equal.

Comparisons:

  • Equal
  • Less Than
  • Greater Than
  • Less Than Or Equal
  • Greater Than or Equal
  • Not Equal

This can be used with the Modulo node above - maybe to kick out a 1 every 5 frames, but comparing the output of modulo with zero. In the case of 5, every fifth frame will be a modulo of zero, resulting in 'compare' emitting a 1.

Sine

The sine node generates a sine wave based on an input. You may have to use a vector to float and a float to vector node to use it in practice (for point positions or colors for example).

Use a multiply or multiply constant node BEFORE the input to the sine node to increase frequency.

Use a add or add constant node BEFORE the input to the sine node to offset the wave.

Trigonometric Functions

The trigonometric functions node is a swiss army knife version of the above.

This node generates the sine/cosine wave, but with frequency and offset built-in.

Using waves with absolute and other nodes can help with different varieties of osculations.

Random

The random node takes in a value (constant, (P)ositions, ptnum etc.) and outputs random values (between zero and one).

It can take in various data types (float, vector, int) and can output values of different types.

  • Example:
    • input: Int (ptnum)
    • output: 3d Vector (normals):
      • add 'fit range':
        • Source Min: 0 0 0
        • Destination Min: -1 -1 -1

Gaussian Random

The Gaussian Random node is like a regular random node, but instead of values between zero and one, it kicks out values around zero (negative & positive). Values over 1 are possible as are values below -1 (using Std Deviation slider).

You need a seed input.

The 'gaussian random' output doesn't need the 'fit range' trick as shown on the random node example.

Turbulent Noise

The Turbulent Noise node produces a real workhorse in terms of noise.

Types (drop-down menu):

  • Perlin Noise
  • Original Perlin Noise
  • Sparse Convolution Noise
  • Alligator Noise
  • Simplex Noise:
    • -0.6 to 0.6
    • Add a 'fit range' node if you need to pull to a 0 to 1
  • Zero Centered Perlin Noise

Output:

  • 1d: float
  • 3d: vector (color, position etc.)

Anti-Aliased Flow Noise

The Anit-Aliased Flow Noise node allows for 4d vectors (3d + time), which can be useful for animated noise.

You can use a Vector to Vector 3 node to plug in 4d variables/values.

This helps produce animated noise that doesn't repeat or osculate in an obvious way.

Vector to Vector4

The Vector to Vector4 node turns a 3d vector (3 components) into a 4d vector (with a time component).

Curl Noise

The Curl Noise node has similar options to the turbulent noise node (drop-down menu).

It is similar to the AA Flow Noise node in the parameters.

NOTE: the fourth cell in the 4d fields is equivalent to 'time', so think about that when adjusting numbers. (The last cell will be controlling things like speed or frequency.)

The Step size parameter can be increased (slightly!) to reduce stray points.

This node is especially good in simulations.

Worley Noise

The Worley Nose node maps points and interpolates between them, resulting in ramps resembling cells.

There are three versions of the interpolation, using a drop-down on the parameters:

  • Euclidean (most commonly used)
  • Manhattan
  • Chebyshev

There are output options:

  • dist1
  • dist2
  • dist3
  • dist4
  • seed:
    • this is like a cluster of huge random numbers (useful as an input for random).
    • useful for making cluster patches
    • output to P perhaps.

Adding or subtracting these various outputs can be fun and useful.

TIP: you can 'add' another noise (perhaps Turbulent Noise) to the input (perhaps P) and pipe this into the pos of the Worley Noise node for more organic looking effects.

Relative to Bounding Box

Great for grads across objects!

First, the regular Bound node, when dropped after an object, will map the bounding cube around the extents - min and max boundaries (containing volume).

Example:

  • Rubber toy > then bound node:
    • check the templating of the bound node ('e') to see the bounding cube.
    • The x, y and z 'dimensions' are zero to one, regardless of shape and size of the box.

The Relative to Bounding Box node maps these zero to one values across the input based on the bounds.

Vector to float after the 'Relative to Bounding Box' (from a single output) can be used to do a y-grad when mapped to color. Use 'fit to range' or a ramp to adjust perhaps.

Add noise to the position parameter and the terminator of the ramp can be more organic.

VOPS_03 - Houdini - Voxyde VOPs - Vector Operations

YouTube VOPs Vector Operations

Knowing how to manipulate vectors...

Subtracting a control vector (location) from vectors (like normals) to get them to point to the control vector.

Create a geo object (rubber toy?), and pipe into an attribute vop, make an add node, and make a single point (and plug into slot 2 on the 'attribute vop'.

In the video tutorial, inside an attribute vop:

  • the P is piped into a subtract (input 2)
  • then on into the output 'N' (normals).
  • From the input (input2) we attach an import point attribute node (into the 'file' input)
  • then on into the subtract (input 1)
  • a constant node (MMB click on the ptnum input of the 'import point attribute' node, to 'promote to constant') is set to vector, and plugged into the 'ptnum' of the 'import point' node.

Now, the normals of the initial geo (rubber toy) will point to the second object - the single point. You can move this point with the handle tool.

TIP: Houdini shows an mismatched node connection with a dotted line - indicating it is not ideal. An example might be a float value being plugged into a parameter expecting a vector.

Normalize

The Normalize node 'normalizes' vector values (returns the vector with a 'unit length' or length of one).

When performing Dot or Cross Product calculations, its good to have normalized vector values as inputs.

Length

The Length node shows the length (float) value of a vector. Normalized vectors will be a unit length (value one).

Length can give you the velocity if a vector is going to be used for velocity driven effects.

Cross Product

The Cross Product node calculates the cross product of two vectors.

The cross product will give you the perpendicular direction/vector from two vectors. Imagine an imaginary plane defined by two edges - the cross product is in the direction of the natural normal of that imaginary plane.

The lesson shows how to use the subtract a location from vectors to get the vectors to 'point' to the location (or away from but swaping inputs on a subtract), and a cardinal direction to give two vectors from which to calculate a third vector (perhaps for velocity/direction).

I saved a file out with examples at this point. It was too visual to list here.

After several explorations using the 'rubber toy' model:

The example uses points on a torus:

  • Scatter points on the torus
  • Subtract a constant (vector) from the points normals to get a direction to the center of the torus
  • Use normalize to get a normalized vector output
  • Get the cross product to get a vector in the direction of the torus ring at the point position
  • Add another vector (vertical perhaps) to get points to move around an up like a tornado.

TIP: RMB on 'texture' icon on the scene view panel (right side) to set particles to:

  • Points
  • Pixels
  • Lines
  • Discs
  • Lit Spheres

Dot Product

The Dot Product of two vectors gives a value of -1 to 1, based on the comparison of directions.

Two vectors...

  • Aligned:
    • 1
  • Perpendicular 90º:
    • 0
  • Opposite:
    • -1

Plugging in the position of a point (a 3d vector value essentially) into the Dot Product node (with Normal vectors for example) can control the direction of values.

Also, adding noise to the point target location vector can help to get variation.

The resulting vectors could be used for...

  • Define direction on a surface (like a 'top' surface for rain impact)
  • Define fields based on velocity
  • Proximity to something's position

Reflect

The Reflect node does a vector reflect.

To imagine the process result, imagine one vector is acting like a plane (direction) and the second input vector gets reflected 'back' as if its base is striking the plane defined by the first vector. The second vector appears to be a reflection off the first vector's plane.

I guess to imagine the direction of the reflection that results, you have to imagine the second vector has reverse direction, to join at the base of the two, rather than diverse away from the base or join of the two.
