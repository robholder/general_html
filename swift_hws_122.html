Hacking With Swift - 122 - Programmatic navigation with NavigationStack

Programmatic navigation allows us to move from one view to another just using code (rather than waiting for the user to take a specific action).

For example, maybe your app is busy processing some user input and you want to navigate to a result screen when that work is finished - you want the navigation to happen automatically when you say so, rather than as a direct response to user input.

In SwiftUI this is done by binding the path of a NavigationStack to an array of whatever data you're navigating with. So, we might start with this: 

struct ContentView: View {
    @State private var path = [Int]()

    var body: some View {
        NavigationStack(path: $path) {
            VStack {
                // more code to come
            }
            .navigationDestination(for: Int.self) { selection in
                Text("You selected \(selection)")
            }
        }
    }
}

That does two important things:

  • It creates an @State property to store an array of integers.
  • It binds that property to the path of our NavigationStack, meaning that changing the array will automatically navigate to whatever is in the array, but also changes the array as the user presses Back in the navigation bar.

So, we could replace the // more code to come with buttons such as these: 

Button("Show 32") {
    path = [32]
}

Button("Show 64") {
    path.append(64)
}

In the first button we're setting the whole array so that it just contains the number 32. If anything else happened to be in the array it will be removed, meaning that the NavigationStack will return to its original state before navigating to the number 32.

In the second button we're appending 64, meaning that it will be added to whatever we were navigating to. So, if our array already contained 32, we'd now have three views in the stack: the original view (called the "root" view), then something to show the number 32, and finally something to show the number 64.

You can also push multiple values at the same time, like this: 

Button("Show 32 then 64") {
    path = [32, 64]
}

That will present a view for 32 then a view for 64, so the user needs to tap Back twice to get back to the root view.

You can mix user navigation and programmatic navigation as much as you want - SwiftUI will take care of making sure your path array stays in sync with whatever data you show, regardless of how it's shown.

In Context

import SwiftUI


struct ContentView: View {
    @State private var path = [Int]()
    
    var body: some View {
        NavigationStack(path: $path) {
            VStack {
                Button("Show 32") {
                    path = [32]
                }
                
                Button("Show 64") {
                    path.append(64)
                }
                Button("Show 32 then 64") {
                    path = [32, 64]
                }
            }
            .navigationDestination(for: Int.self) { selection in
                Text("You selected \(selection)")
            }
        }
    }
}

#Preview {
    ContentView()
}

AI Summary

This is the perfect "lightbulb moment" for understanding how Programmatic Navigation works. The path array isn't just a "teleport" button; it is a literal map of the stack currently sitting on top of your home screen.

1. The "Stack" Analogy

Think of your NavigationStack like a physical stack of playing cards.

  • The Home View (your VStack with buttons) is the table. It never moves.
  • Every number in your path array is a card placed on top of the table.

2. What's happening in your buttons?

Button 1: "Show 32" (path = [32])

  • You tell SwiftUI: "The stack should have exactly one card: 32."
  • Result: You see the view for 32.

Button 2: "Show 64" (path.append(64))

  • You tell SwiftUI: "Take whatever is there and put 64 on top."
  • Result: You see 64. (If you were at Home, it's just 64. If you were already at 32, you'd now be at [32, 64]).

Button 3: "Show 32 then 64" (path = [32, 64])

  • You tell SwiftUI: "Instantly make the stack look like this: 32 on the bottom, 64 on top."
  • The Result: SwiftUI looks at the last item in the array to decide what you are currently looking at. Since 64 is at the end, that's what you see.
  • The "Back" Button: Because 32 is still in the array (underneath 64), the Back button doesn't go home - it goes to the previous item in the array.

3. Why is this useful?

This behavior is actually a superpower. It allows you to Deep Link.

Imagine a user receives a notification about a specific mission in your Moonshot app. You don't want them to just land on the home screen; you want them to land on the Mission Detail view, but you also want them to be able to hit "Back" and see the Astronaut list.

By setting the path to [Mission, Astronaut], you've pre-constructed the user's history so they don't feel lost.

4. The Vim "Visual" Test

If you want to see this in action while you play with the app, add this line inside your VStack:

Text("Current path: \(path.description)")

In Vim, you can quickly duplicate one of those buttons to experiment:

  • Put your cursor on the "Show 32" button line.
  • Type 5yy (Yank 5 lines).
  • Move to the end of the VStack and press p.
  • Change the code to path = [10, 20, 30, 40].

When you click that new button, you'll see "40", but you'll have to hit "Back" four times to get home!

Summary

  • The last number in the array is the screen you see.
  • The other numbers are the "breadcrumbs" or "history" waiting behind the Back button.
  • An empty array [] means you are back at the very beginning.
