Python - lseq

This is a Python3 tool to list sequences of files in a directory, as a range syntax based on file stems and numbered files. Inspired by a tool written by Andrew Whitehurst.

Example output

bg_plate.[(-003)-002].exr
hero_char.[0001-0003],[0005-0006],[0010].exr
    * missing 4 files: [0004], [0007-0009]
smoke_puff.[01-05].exr
vfx_spark.0150.exr

The Script

#!/usr/bin/env python3

import sys
import os
import argparse
from collections import defaultdict

"""
lseq: Scans a directory and returns a compact summary of frame sequences
      and dynamically highlights missing frames in red.
"""

__author__ = "Robert Holder"
__copyright__ = "Copyright 2026, Robert Holder"
__license__ = "MIT"
__version__ = "2.0.2"
__maintainer__ = "Robert Holder"
__email__ = ""
__status__ = "Production"

# ANSI escape codes for coloring terminal text
COLOR_RED = "\033[91m"
COLOR_RESET = "\033[0m"

# ---------------------------------------------------------------------
# SAFETY VALVE: Prevents hanging on non-sequential numbers
# ---------------------------------------------------------------------
# No standard VFX sequence will have a gap larger than 100,000 frames.
# This prevents O(N) range allocations from locking up the system.
MAX_ALLOWED_GAP = 100000


def format_single_frame(val, pad):
    """
    Formats a single frame number, preserving its absolute padding and wrapping
    negative numbers in parentheses.
    """
    if val < 0:
        # Wrap negative numbers in parentheses, e.g., (-003)
        padded_digits = str(abs(val)).zfill(pad)
        return f"(-{padded_digits})"
    else:
        # Standard positive padding, e.g., 002
        return str(val).zfill(pad)


def makeFrameList(inputList, pad):
    """
    Takes a sorted list of integer frame numbers and aggregates them into a
    compact string, grouping consecutive runs inside square brackets.
    """
    if not inputList:
        return ""
    if len(inputList) == 1:
        return format_single_frame(inputList[0], pad)

    segments = []
    start = inputList[0]
    prev = inputList[0]

    for val in inputList[1:]:
        if val == prev + 1:
            prev = val
        else:
            if start == prev:
                segments.append(format_single_frame(start, pad))
            else:
                segments.append(f"[{format_single_frame(start, pad)}-{format_single_frame(prev, pad)}]")
            start = val
            prev = val

    # Handle final segment
    if start == prev:
        segments.append(format_single_frame(start, pad))
    else:
        segments.append(f"[{format_single_frame(start, pad)}-{format_single_frame(prev, pad)}]")

    return ",".join(segments)


def makeMissingFrameList(missingList, pad):
    """
    Groups missing frames into contiguous runs and formats them,
    enclosing ALL segments (including single frames) in square brackets,
    separated by ', '.
    """
    if not missingList:
        return ""
    if len(missingList) == 1:
        return f"[{format_single_frame(missingList[0], pad)}]"

    segments = []
    start = missingList[0]
    prev = missingList[0]

    for val in missingList[1:]:
        if val == prev + 1:
            prev = val
        else:
            if start == prev:
                segments.append(f"[{format_single_frame(start, pad)}]")
            else:
                segments.append(f"[{format_single_frame(start, pad)}-{format_single_frame(prev, pad)}]")
            start = val
            prev = val

    # Handle final segment
    if start == prev:
        segments.append(f"[{format_single_frame(start, pad)}]")
    else:
        segments.append(f"[{format_single_frame(start, pad)}-{format_single_frame(prev, pad)}]")

    return ", ".join(segments)


def stringToInt(inputString):
    """
    Safely parses a string value into a signed integer, accounting for leading negative signs.
    """
    if not inputString:
        return 0
    
    tweakVal = inputString
    if inputString.startswith("-"):
        multVal = -1
        tweakVal = inputString[1:]
    else:
        multVal = 1
        
    if not tweakVal.isdigit():
        print(f"Error: {tweakVal} is a non-numeric frame value.", file=sys.stderr)
        return 0
    else:
        return int(tweakVal) * multVal


def getFileName(inputString):
    """
    Extracts the root name stem of the sequence, cleanly stripping any trailing
    minus signs or separator dots.
    """
    base = os.path.basename(inputString)
    noExt = base.rsplit(".", 1)
    name_part = noExt[0]
    
    idx = len(name_part)
    while idx > 0 and name_part[idx-1].isdigit():
        idx -= 1
        
    outString = name_part[:idx]
    
    # Strip the trailing minus sign if it was a negative frame
    if outString.endswith("-"):
        outString = outString[:-1]  
        
    # Strip the separator dot so we don't return "stem." as the stem name
    if outString.endswith("."):
        outString = outString[:-1]
        
    return outString


def getFrameNumber(inputString):
    """
    Extracts the frame number string from a path.
    """
    base = os.path.basename(inputString)
    noExt = base.rsplit(".", 1)
    name_part = noExt[0]
    
    idx = len(name_part)
    while idx > 0 and (name_part[idx-1].isdigit() or name_part[idx-1] == "-"):
        if name_part[idx-1] == "-" and (idx - 1 > 0 and name_part[idx-2].isdigit()):
            break
        idx -= 1
        
    return name_part[idx:]


def getExtension(inputString):
    """
    Extracts the file extension of a path.
    """
    ext = inputString.rsplit(".", 1)
    return "." + ext[1] if len(ext) > 1 else ""


def getFramePadding(inputFrame):
    """
    Evaluates only the absolute digit padding width.
    Negative signs are ignored so that '-003' and '002' are grouped into 
    the same padding bin (width of 3).
    """
    if not inputFrame:
        return 0
    
    # Strip leading minus sign to count absolute digits
    clean_frame = inputFrame[1:] if inputFrame.startswith("-") else inputFrame
    return len(clean_frame)

# MAIN EXECUTION
# =====================================================================

def main():
    # Initialize the standard argument parser
    parser = argparse.ArgumentParser(
        description="List numbered sequences in a directory using a compact, readable syntax.",
        formatter_class=argparse.RawTextHelpFormatter
    )

    # Expose the metadata version directly to the CLI
    parser.add_argument(
        "-v", "--version",
        action="version",
        version=f"%(prog)s {__version__}"
    )

    # Define the directory argument as an optional positional argument
    parser.add_argument(
        "directory", 
        nargs="?", 
        default=".", 
        help="The directory path to scan. Defaults to the current working directory."
    )

    args = parser.parse_args()

    # Validate directory path
    testPath = args.directory
    if not os.path.exists(testPath):
        print(f"Error: Specified directory path does not exist: '{testPath}'", file=sys.stderr)
        sys.exit(1)
    if not os.path.isdir(testPath):
        print(f"Error: Specified path is a file, not a directory: '{testPath}'", file=sys.stderr)
        sys.exit(1)

    # Ensure trailing folder slash
    if not testPath.endswith(os.sep):
        testPath += os.sep

    # Locate files reliably using os.listdir
    try:
        raw_files = os.listdir(testPath)
    except OSError as e:
        print(f"Error: Unable to read directory '{testPath}': {e}", file=sys.stderr)
        sys.exit(1)

    # Filter to files with an extension (equivalent to glob's *.*)
    fileList = [os.path.join(testPath, f) for f in raw_files if "." in f and os.path.isfile(os.path.join(testPath, f))]

    if len(fileList) < 1:
        print(f"Warning: No files found in '{testPath}'", file=sys.stderr)
        sys.exit(0)

    # Group files by signature: (name_stem, absolute_padding_width, extension)
    sequences = defaultdict(list)

    for path in fileList:
        frame_str = getFrameNumber(path)
        if frame_str and frame_str != "NG":
            stem = getFileName(path)
            pad = getFramePadding(frame_str)
            ext = getExtension(path)
            frame_val = stringToInt(frame_str)
            
            key = (stem, pad, ext)
            sequences[key].append(frame_val)

    # ---> NEW CHECK <---
    # 2.0.2: If files were found but none parsed as valid sequences (e.g., only .txt or .py files),
    # warn the user rather than exiting silently.
    if not sequences:
        print(f"Warning: No valid sequential files found in '{testPath}'", file=sys.stderr)
        sys.exit(0)

    # Accumulate output
    outputString = ""

    for (stem, pad, ext), frames in sorted(sequences.items()):
        frames.sort()
        
        # Process ranges
        formatted_frames = makeFrameList(frames, pad)
        
        # Clean, single-dot formatted line
        line = f"{stem}.{formatted_frames}{ext}\n"

        # Check for missing files if the sequence has more than one frame
        has_gaps = False
        missing_report = ""
        
        if len(frames) > 1:
            min_frame = frames[0]
            max_frame = frames[-1]
            total_gap = max_frame - min_frame
            
            # -------------------------------------------------------------
            # SAFETY CHECK: Skip analysis if sequence gap is unrealistic
            # -------------------------------------------------------------
            if total_gap <= MAX_ALLOWED_GAP:
                # Use sets to find mathematical differences
                expected_set = set(range(min_frame, max_frame + 1))
                present_set = set(frames)
                missing_set = expected_set - present_set
                
                if missing_set:
                    has_gaps = True
                    missing_list = sorted(list(missing_set))
                    missing_count = len(missing_list)
                    formatted_missing = makeMissingFrameList(missing_list, pad)
                    
                    # Format pluralization and construct the secondary info line
                    plural_suffix = "s" if missing_count > 1 else ""
                    missing_report = f"    * missing {missing_count} file{plural_suffix}: {formatted_missing}\n"

        # Output logic
        if has_gaps:
            # Color both the sequence name and its missing file report red
            outputString += f"{COLOR_RED}{line}{missing_report}{COLOR_RESET}"
        else:
            outputString += line

    # Output final report
    print(outputString, end="")


if __name__ == "__main__":
    main()

This function has a very specific and critical job in VFX pipelines: it extracts the frame number from a file path while keeping negative signs intact (like -003), but making sure it doesn't accidentally grab a hyphen used as a word separator (like shot-01).

Let’s trace how this function works line-by-line using a complex filename: "/path/to/my-shot_v01.-0045.exr" (Frame -45, with a hyphen in the shot name).

Step 1: Strip down to the name part

    base = os.path.basename(inputString)

What it does: Extracts just the file name from the full directory path.

In our example: base becomes "my-shot_v01.-0045.exr". 

    noExt = base.rsplit(".", 1)
    name_part = noExt[0]

What it does: Splits the string starting from the right at the last dot (.) to separate the extension, and grabs the left side.

In our example:

  • noExt is ["my-shot_v01.-0045", "exr"].
  • name_part becomes "my-shot_v01.-0045".

Step 2: Set up a backward-scanning index

    idx = len(name_part)

What it does: Sets an index pointer (idx) to the very end of the string. We are going to scan backwards (from right to left) to find where the frame number begins.

In our example: len("my-shot_v01.-0045") is 17. So idx = 17.

Step 3: Scan backward for digits and minus signs

    while idx > 0 and (name_part[idx-1].isdigit() or name_part[idx-1] == "-"):

What it does: This loop keeps moving backward (left) as long as the character it looks at is either a digit (0-9) or a minus sign (-).

In our example:

  • Character at index 16 is "5" (digit) → Keep going.
  • Character at index 15 is "4" (digit) → Keep going.
  • Character at index 14 is "0" (digit) → Keep going.
  • Character at index 13 is "0" (digit) → Keep going.
  • Character at index 12 is "-" (hyphen) → Keep going (but look at the special rule inside the loop next!).

Step 4: The Negative Number Safety Check

This is the trickiest part of the loop. Inside the while loop, we find this safety check: 

        if name_part[idx-1] == "-" and (idx - 1 > 0 and name_part[idx-2].isdigit()):
            break

What it does in English: "If the character we just found is a hyphen ("-"), look at the character directly to its left. If that character is a number, STOP looping immediately."

Why does this exist?

  • If you have a file named render-1-0045.exr, and we scan backward:
  • We scan through 5, 4, 0, 0.
  • We reach the - before 0045. This is a negative sign. The character to its left is 1.
  • Since 1 is a digit, this rule triggers a break and stops scanning. This prevents the script from accidentally turning -1-0045 into a single, giant, invalid number!

In our example ("my-shot_v01.-0045"):

  • We reach the - at index 12.
  • The character to its left (index 11) is . (a dot, not a digit).
  • The check evaluates to False, so we do not break. 
        idx -= 1

What it does: Decrements our index to keep scanning left.

In our example: We successfully pass the - character, and the index decrements to pointing at the dot ..

On the next loop iteration, the character at index 11 is .. Since a dot is neither a digit nor a hyphen, the while loop cleanly terminates.

1. Why we can't just break at any hyphen (-)

In standard VFX pipeline naming conventions, a hyphen can mean two completely different things depending on where it sits:

  • It can be a dash separator between the name and a version/token (e.g., shot-name-v01-0001.exr).
  • It can be a negative sign indicating a negative frame number (e.g., shot-name-v01.-0001.exr).

If the code simply did a break the moment it hit any hyphen while scanning backwards, it would break completely on negative frames.

Look at my-shot_v01.-0045.exr

If we scan backward:

  • We collect 5, 4, 0, 0.
  • We encounter the -.
  • If we just blindly break here, the function returns "0045".
  • The negative sign is completely dropped! The script will misread frame -45 as frame 45.

By not breaking immediately, the loop decreases idx one more time, includes the - in the slice, and correctly extracts "-0045".

2. The Real Purpose of the isdigit() Look-Behind

Now, look at the exact check that is confusing: 

if name_part[idx-1] == "-" and (idx - 1 > 0 and name_part[idx-2].isdigit()):
    break

This specifically handles files that use hyphens as separators right next to another number, like a version number or a shot number.

The Example: render-1-0045.exr

  • What it means to a human: This is frame 0045 of a sequence named render-1.
  • What the backward loop does step-by-step:
    • It reads 5, 4, 0, 0. (idx moves left).
    • It hits the hyphen right before 0045.
    • It looks to the left of that hyphen and sees 1 (which .isdigit() evaluates to True).
    • The check triggers a break.
    • The loop stops, and name_part[idx:] returns 0045. (Correct!)

What would happen without that check?

If that check wasn't there, the loop would say: "Oh, a hyphen! Let me check if it's part of a negative number." It wouldn't break. It would keep scanning left.

  • It swallows the -.
  • It sees 1, which is a digit, so it swallows that too.
  • It hits the next - (before the 1). To its left is r (not a digit), so it swallows that hyphen too.
  • The loop finally stops at the letters.

The function would return "-1-0045". The script would then try to turn "-1-0045" into an integer, completely breaking your frame logic.

Step 5: Slice and return the frame number

    return name_part[idx:]

What it does: Slices the string from our final stopped index (idx) to the end of the string.

In our example: The loop stopped when idx reached 12 (the index of the hyphen -).

name_part[12:] extracts everything from index 12 to the end: "-0045".

Summary of what it returns:

  • shot_v01.0024.exr → returns "0024"
  • shot_v01.-0024.exr → returns "-0024" (correctly identifies negative frames)
  • my-shot-01-0024.exr → returns "0024" (correctly ignores the hyphen in my-shot-01)
  • notes.txt → returns "" (no digits at the end)

def getFileName(inputString):
    """
    Extracts the root name stem of the sequence, cleanly stripping any trailing
    minus signs or separator dots.
    """
    base = os.path.basename(inputString)
    noExt = base.rsplit(".", 1)
    name_part = noExt[0]
    
    idx = len(name_part)
    while idx > 0 and name_part[idx-1].isdigit():
        idx -= 1
        
    outString = name_part[:idx]
    
    # Strip the trailing minus sign if it was a negative frame
    if outString.endswith("-"):
        outString = outString[:-1]  
        
    # Strip the separator dot so we don't return "stem." as the stem name
    if outString.endswith("."):
        outString = outString[:-1]
        
    return outString

getFramePadding(inputFrame)

def getFramePadding(inputFrame):
    """
    Evaluates only the absolute digit padding width.
    Negative signs are ignored so that '-003' and '002' are grouped into 
    the same padding bin (width of 3).
    """
    if not inputFrame:
        return 0
    
    # Strip leading minus sign to count absolute digits
    clean_frame = inputFrame[1:] if inputFrame.startswith("-") else inputFrame
    return len(clean_frame)

The ultimate goal of this function is to look at a frame string—like "0001", "12", or "-003" - and figure out how many digits wide it is, entirely ignoring whether it’s a positive or negative frame.

This is crucial for lseq, because if you have a sequence that transitions from negative frames to positive frames (e.g., -002, -001, 0000, 0001), they all need to be grouped into the exact same "padding bin" (a padding width of 4). If they aren't, the script will split them into separate, broken sequences.

Here is how the code evaluates a string line-by-line:

1. The Safety Guard

if not inputFrame:
    return 0

What it does: Checks if inputFrame is empty (like "") or None.

Why it's there: If a file somehow bypasses the parser without a valid frame number, this prevents the script from crashing when it tries to read the string's length, cleanly returning a padding of 0.

2. Stripping the Negative Sign

clean_frame = inputFrame[1:] if inputFrame.startswith("-") else inputFrame

This is a Python conditional expression (a one-liner if-else statement). Let's see how it behaves under two different scenarios:

Scenario A: The frame is negative (inputFrame = "-003")

  • inputFrame.startswith("-") evaluates to True.
  • The code runs the first part: inputFrame[1:].
  • In Python, string slicing [1:] means "give me everything from index 1 to the very end of the string, skipping index 0."
    • Index 0 is "-"
    • Index 1 is "0"
    • Index 2 is "0"
    • Index 3 is "3"
  • clean_frame becomes the string "003". The minus sign is completely stripped.

Scenario B: The frame is positive (inputFrame = "0002")

  • inputFrame.startswith("-") evaluates to False.
  • The code jumps to the else block: else inputFrame.
  • clean_frame simply becomes a direct copy of the original string: "0002".

3. Calculating the Width

return len(clean_frame)

What it does: Measures the total character length of the now-sanitized string and returns it as an integer. The Result:

  • len("003") returns 3
  • len("0002") returns 4

Because of this function, both "-003" and "003" yield a padding value of 3, ensuring they are matched together perfectly under the same sequence key!

getExtension(inputString)

Let’s break down getExtension(inputString) line-by-line, focusing heavily on what that rsplit(".", 1) is doing.

The goal of this function is to reliably extract a file's extension (like .exr or .mp4) from a full file path, even if the filename contains multiple dots (e.g., shot_010.0001.exr).

The Secret of rsplit(".", 1)

Python’s standard .split() splits a string from the left side. However, .rsplit() splits a string from the right side.

The two arguments mean:

  • ".": The delimiter to split on (a dot).
  • 1: The maximum number of splits allowed to occur.

By telling Python to look from the right and only split once, you guarantee that you isolate the very last element after the final dot—which is always the file extension.

Let's trace an example:

If inputString is /path/to/render.0001.exr:

  • Without the 1 limit (rsplit(".")): Python would split on every dot, resulting in: ['/path/to/render', '0001', 'exr'].
  • With the 1 limit (rsplit(".", 1)): Python scans from the right, finds the first dot (the one right before exr), performs exactly one split, and stops. It returns a list with exactly two elements: 
['/path/to/render.0001', 'exr']
#       ext[0]          ext[1]

Line-by-Line Breakdown of the Function

1. The Split

ext = inputString.rsplit(".", 1)
  • What it does: As shown above, it splits the string once from the right at the final dot. The variable ext now holds a list.
  • If a dot was found, ext will have 2 items: [everything_before, extension].
  • If no dot was found (e.g., a folder named Downloads), no split happens, and ext will have 1 item: ['Downloads'].

2. The Conditional Return

return "." + ext[1] if len(ext) > 1 else ""

This is a clean Python conditional one-liner. It checks the size of our ext list:

  • Scenario A: A dot was found (len(ext) > 1)
    • If len(ext) is 2, it means an extension exists at index 1 (ext[1] is "exr"). The function prepends a dot back onto it ("." + "exr") and returns ".exr".
  • Scenario B: No dot was found (else)
  • If len(ext) is 1, it means there is no extension in the path. The function falls back to the else statement and returns an empty string "".

stringToInt(inputString)

Let's take a deep dive into stringToInt(inputString).

The core job of this function is to turn a frame string (like "0042" or "-0012") into an actual mathematical integer (42 or -12).

The tricky part of coding this natively in Python is that Python's built-in string checker, .isdigit(), only returns True if a string contains pure numbers. If a string has a minus sign in it (like "-0012"), .isdigit() returns False. This function gets around that limitation by isolating the minus sign, validating the remaining digits, and manually re-applying the negative sign using multiplication.

Here is the step-by-step breakdown:

1. The Empty Check

if not inputString:
    return 0
  • What it does: If the incoming string is empty ("") or None, it immediately short-circuits the function and returns 0. This keeps the script from throwing errors later on.

2. Setting Up the "Tweak" and Sign Detection

tweakVal = inputString
if inputString.startswith("-"):
    multVal = -1
    tweakVal = inputString[1:]
else:
    multVal = 1

This is where the magic happens for negative numbers. It sets up two variables: tweakVal (a temporary string we can safely check for digits) and multVal (a mathematical multiplier).

  • If the string is negative (e.g., "-0012"):
    • It detects the starting "-".
    • It sets multVal = -1.
    • It sets tweakVal = inputString[1:], which strips off the minus sign. If the input was "-0012", tweakVal becomes "0012".
  • If the string is positive (e.g., "0042"):
    • It skips to the else block.
    • It sets multVal = 1.
    • tweakVal remains unchanged as "0042".

3. Validation and Conversion

if not tweakVal.isdigit():
    print(f"Error: {tweakVal} is a non-numeric frame value.", file=sys.stderr)
    return 0
else:
    return int(tweakVal) * multVal

Now that the minus sign has been temporarily stripped away, tweakVal is a clean string of numbers.

  • The if block (Error Handling):
    • If tweakVal still contains non-numeric characters (like "00a2" or "NG"), .isdigit() fails. The function prints an error message to the system's error stream (sys.stderr) and returns 0.
  • The else block (The Conversion):
    • If tweakVal is purely numbers, it runs:
      • int(tweakVal): Converts the cleaned string to a positive integer. Python automatically discards leading zeros here (e.g., "0012" becomes 12).
      • multVal: Multiplies that integer by our saved multiplier (1 or -1).

Let's Trace Two Quick Examples

Example A: inputString = "-0012"

  • startswith("-") is True → multVal = -1 and tweakVal = "0012".
  • "0012".isdigit() is True → jumps to the else block.
  • Math: int("0012") * -1 → 12 * -1 → Returns -12.

Example B: inputString = "0042"

  • startswith("-") is False → multVal = 1 and tweakVal = "0042".
  • "0042".isdigit() is True → jumps to the else block.
  • Math: int("0042") * 1 → 42

Defaultdict

In Python, defaultdict is a highly convenient subclass of the standard dictionary (dict) found in the built-in collections module: 

from collections import defaultdict

Its sole purpose is to prevent KeyError exceptions by automatically creating a default value whenever you try to access or modify a key that doesn't exist yet.

The Problem with Standard Dictionaries

With a standard Python dictionary, if you try to append something to a list inside a key that hasn't been created yet, Python will crash: 

# This will CRASH with a KeyError:
sequences = {}
key = ("render_layer", 4, ".exr")

sequences[key].append(101)  # ❌ KeyError! The key doesn't exist yet.

To make this work with a standard dictionary, you have to manually check for the key every single time: 

# The manual way:
if key not in sequences:
    sequences[key] = []  # Initialize it first
sequences[key].append(101)

How defaultdict Solves This

When you initialize a defaultdict, you pass it a "default factory" - a callable object (like list, int, or set) that tells the dictionary what to put there if a key is missing.

In a script: 

sequences = defaultdict(list)

By passing list as the factory, you are telling Python: "If I ask for a key that isn't there, instantly create it and set its value to an empty list []."

This allows you to safely write this single line: 

sequences[key].append(frame_val)

What happens behind the scenes:

  • First time encountering a sequence (e.g., ("shot_01", 4, ".exr")):
    • defaultdict notices this key is missing.
    • It quietly runs list() to generate an empty list [].
    • It inserts the key: {("shot_01", 4, ".exr"): []}.
    • It immediately appends your frame number (e.g., 101) to that new list.
  • Second time encountering the same sequence:
  • defaultdict sees the key already exists.
  • It skips initialization and simply appends the next frame number (e.g., 102) to the existing list.

Other Common Uses for defaultdict

You can use different factories depending on what you are trying to group or count:

  • defaultdict(int): Great for counting things. Missing keys start at 0. 
counts = defaultdict(int)
counts["apple"] += 1  # Automatically starts at 0, becomes 1
  • defaultdict(set): Great for gathering unique, non-duplicate items. Missing keys start as set(). 
unique_exts = defaultdict(set)
unique_exts["render_passes"].add(".exr")

NG Explanation

"NG" is not a Python feature or keyword. It is just a plain text string that the programmer who wrote this script chose to look out for.

In Python, anything inside quotation marks like "NG" is just text. The operator != means "is not equal to".

So, this line of code: 

if frame_str and frame_str != "NG":

...translates to English as:

"If we successfully found a frame number, and that frame number is not the exact text 'NG', then proceed."

Why is "NG" there? (The VFX Context)

In the Visual Effects (VFX) and animation industries, "NG" sometimes stands for "No Good".

When artists render out thousands of frames of an animation, sometimes a specific frame glitches or fails. Instead of deleting it and leaving a gap, or if they want to flag a bad render to their team, they might rename that file or frame to include "NG" (for example, character_render.NG.exr).

This script add frame_str != "NG" as a safety check. It ensures that if the script encounters a file marked as "No Good", it will ignore it and won't try to turn the letters "NG" into a mathematical frame number (which would cause the script to crash).
