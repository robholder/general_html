Vim-Plug (Plugin Manager)

Here's a concise breakdown of vim-plug, its purpose, and how to use it:

What is vim-plug?

  • vim-plug is a minimalist and fast plugin manager for Vim and Neovim.
  • It simplifies the process of installing, updating, and managing plugins in Vim.
  • Designed to be efficient: it uses parallel downloads and supports lazy loading (on-demand loading of plugins).

How to Install vim-plug

Download the vim-plug script:

  • For Vim: 
curl -fLo ~/.vim/autoload/plug.vim --create-dirs \
https://raw.githubusercontent.com/junegunn/vim-plug/master/plug.vim
  • For Neovim: 
curl -fLo ~/.local/share/nvim/site/autoload/plug.vim --create-dirs \
https://raw.githubusercontent.com/junegunn/vim-plug/master/plug.vim

Verify Installation:

Open Vim or Neovim and run:

  • :echo filereadable(expand('~/.vim/autoload/plug.vim'))
    • If it returns 1, vim-plug is installed.

How to Use vim-plug

  • Add Plugins in Your Configuration File:
    • For Vim: Update ~/.vimrc.
    • For Neovim: Update ~/.config/nvim/init.vim.

Example:

call plug#begin('~/.vim/plugged')
Plug 'preservim/nerdtree'          " File Explorer
Plug 'tpope/vim-fugitive'          " Git Integration
Plug 'junegunn/fzf.vim', { 'do': './install --all' } " Fuzzy Finder
call plug#end()
  • Plug is used to define plugins.
  • The plug#begin and plug#end block contains all plugin declarations.

Install Plugins:

  • Run :PlugInstall in Vim to download and install all plugins specified in your configuration.

Update Plugins:

  • Run :PlugUpdate to fetch and apply updates for installed plugins.

Remove Unused Plugins:

Comment out or remove the Plug line from your configuration.

  • Run :PlugClean to remove the unused plugin files.

Other Commands:

  • :PlugStatus: Lists the status of installed plugins.
  • :PlugUpgrade: Updates vim-plug itself.
  • :PlugDiff: Shows differences between the current plugin state and the config.

Features of vim-plug

  • Parallel Installation: Installs multiple plugins simultaneously, speeding up the process.
  • Post-update Hooks: Use {'do': 'command'} to run additional commands after installing/updating a plugin (e.g., compiling native extensions).
  • Lazy Loading: Load plugins only when needed to reduce startup time: 
Plug 'scrooloose/syntastic', { 'on': 'SyntasticCheck' }
  • Lazy Loading with Hooks: You can combine lazy loading and post-update hooks for efficient plugin management: 
Plug 'scrooloose/syntastic', { 'on': 'SyntasticCheck', 'do': 'helptags ALL' }

Using the Repo 'Branch' Feature

Many repositories have transitioned to using main or other branch names as their default. Here's how you can troubleshoot and resolve this issue:

  • Check the Repository's Default Branch: Visit the GitHub pages for the plugins:
  • Look for the default branch name (e.g., main, release, or something else).
  • Specify the Correct Branch in Vim-Plug: Update your .vimrc or init.vim to explicitly set the branch for each plugin. For example: 
Plug 'username/vim-poolside', { 'branch': 'release' }
Plug 'username/vim-srt-sync', { 'branch': 'main' }
  • Replace 'main' with the actual branch name if it's different.
  • Clean and Reinstall Plugins: If the error persists, try cleaning up the plugin installation and reinstalling: 
:PlugClean
:PlugInstall
  • Update Vim-Plug: Ensure your Vim-Plug installation is up-to-date. Run: 
:PlugUpgrade
  • Manually Clone the Repository: If all else fails, manually clone the repository and check out the correct branch: 
git clone https://github.com/username/vim-poolside.git
cd vim-poolside
git checkout main

Common Post-Update Hooks

Compile a Plugin:

  • For plugins that require compilation (e.g., plugins with C extensions), you can use: 
Plug 'junegunn/fzf', { 'do': './install --all' }
  • This runs the ./install script for the fzf plugin after installation or update.

Run Shell Commands:

  • You can execute any shell command to set up the plugin: 
Plug 'Valloric/YouCompleteMe', { 'do': './install.py --all' }
  • This is often used for plugins like YouCompleteMe, which require Python-based installation scripts.

Run Vim Commands:

  • Post-update hooks can also execute Vim commands: 
Plug 'tpope/vim-sensible', { 'do': 'helptags ALL' }
  • helptags ALL generates help documentation for plugins.

Run Multiple Commands:

  • Use a shell to chain multiple commands if needed: 
Plug 'my/plugin', { 'do': 'cmd1 && cmd2' }

Custom Scripts:

  • Run custom setup scripts: 
Plug 'example/plugin', { 'do': 'bash setup.sh' }

Important Notes

  • Shell Commands vs. Vim Commands:
    • Post-update hooks default to running shell commands. If you want to run a Vim command, you can use the :execute command in Vimscript.
  • Error Handling: If the post-update hook fails, the plugin installation/update process will notify you of the error. Fix the issue, and re-run :PlugInstall or :PlugUpdate.

With vim-plug, managing Vim plugins becomes efficient and user-friendly.
