Stow - Updated Project - Tests

Copilot Guide

You don’t have to do everything at once—we can make this very incremental.

Let’s walk it step by step, starting from where you are now: you have bootstrap.sh in your dotfiles repo.

Step 1: Install Bats

On macOS (with Homebrew): 

brew install bats-core

On Ubuntu/Debian: 

sudo apt-get update
sudo apt-get install -y bats

To confirm: 

bats --version

Step 2: Create the tests/ directory

From your repo root (where bootstrap.sh lives): 

mkdir tests
cd tests

Step 3: Add test_helper.bash

Create tests/test_helper.bash: 

cat > test_helper.bash << 'EOF'
setup_fake_home() {
  export TEST_HOME
  TEST_HOME="$(mktemp -d)"
  export HOME="$TEST_HOME"
}

setup_fake_dotty() {
  export DOTTY_DIR
  DOTTY_DIR="$(mktemp -d)"
  cp "$BATS_TEST_DIRNAME/../bootstrap.sh" "$DOTTY_DIR/bootstrap.sh"
  chmod +x "$DOTTY_DIR/bootstrap.sh"
}

create_home_file() {
  local name="$1"
  echo "# test file" > "$HOME/$name"
}

run_dotty() {
  cd "$DOTTY_DIR"
  ./bootstrap.sh "$@" 2>&1
}
EOF

This gives all tests a clean $HOME and a fresh copy of bootstrap.sh.

Step 4: Add your first test file

Let’s start with a single, simple test: “does the script run at all?”

Create tests/00_smoke.bats: 

cat > 00_smoke.bats << 'EOF'
#!/usr/bin/env bats

load test_helper

@test "bootstrap.sh runs in dry mode" {
  setup_fake_home
  setup_fake_dotty

  mkdir "$DOTTY_DIR/zshrc"

  run run_dotty
  [ "$status" -eq 0 ]
  [[ "$output" =~ "Processing package: zshrc" ]]
}
EOF

Make it executable (not strictly required, but nice): 

chmod +x 00_smoke.bats

Step 5: Run the tests

From your repo root: 

bats tests

You should see something like: 

 ✓ bootstrap.sh runs in dry mode

1 test, 0 failures

If you see that, you’ve got a working test harness.

Step 6: Add a second, slightly richer test

Now let’s test the prefix inference.

Create tests/01_inference.bats: 

cat > 01_inference.bats << 'EOF'
#!/usr/bin/env bats

load test_helper

@test "prefix inference detects matching home files" {
  setup_fake_home
  setup_fake_dotty

  mkdir "$DOTTY_DIR/zshrc"
  create_home_file ".zshrc"
  create_home_file ".zshenv"

  run run_dotty

  [[ "$output" =~ "$HOME/.zshrc" ]]
  [[ "$output" =~ "$HOME/.zshenv" ]]
}
EOF

Run again: 

bats tests

Now you should see 2 tests passing.

Step 7: Grow from there (one test at a time)

Once that’s working, you can add more tests gradually:

  • 02_selection_ui.bats — check that default is “none” when pressing Enter
  • 03_apply_migration.bats — pipe 1\n into --apply and assert file moved + symlink created
  • 04_existing_dotfiles.bats — create dot-zshrc in package and assert it’s listed as auto-included

You don’t have to build the whole suite in one sitting—just add tests as you touch behavior.

If you’d like, next we can:

  • write 03_apply_migration.bats together and debug it live
  • or wire up GitHub Actions once you’re happy running bats tests locally.
