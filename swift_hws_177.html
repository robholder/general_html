Hacking With Swift - 177 - Showing empty states with ContentUnavailableView

SwiftUI's ContentUnavailableView shows a standard user interface for when your app has nothing to display. I know, that sounds redundant, right? After all, if you have nothing to display, you have nothing to display!

But ContentUnavailableView is perfect for times your app relies on user information that hasn't been provided yet, such as when your user hasn't created any data, or if they are searching for something and there are no results.

As an example, if you were making an app that let users write down Swift code snippets they wanted to remember, it might start with no snippets by default. So, you could use ContentUnavailableView like this: 

ContentUnavailableView("No snippets", systemImage: "swift")

That will show a big Swift icon from SF Symbols, plus title text below saying "No snippets".

You can also add an extra line of description text below, specified as a Text view so you can add extra styling such as a custom font, or a custom color: 

ContentUnavailableView("No snippets", systemImage: "swift", description: Text("You don't have any saved snippets yet."))

...or same code split into a vertical block: 

ContentUnavailableView(
    "No snippets",
    systemImage: "swift",
    description: Text(
        "You don't have any saved snippets yet."
    )
)

And if you want full control, you can provide individual views for the title and description, along with some buttons to display to help the user to get started: 

ContentUnavailableView {
    Label("No snippets", systemImage: "swift")
} description: {
    Text("You don't have any saved snippets yet.")
} actions: {
    Button("Create Snippet") {
        // create a snippet
    }
    .buttonStyle(.borderedProminent)
}

It's a really simple view to use, but it's much better than just showing a blank screen when the user first comes to your app!

Ai Insight

Convention for splitting long lines of SwiftUI code.

When a single line of code starts stretching horizontally, it becomes incredibly difficult to read, especially on smaller laptop screens or when you have Xcode’s code-navigator sidebar open.

The golden rule for splitting initializers (like ContentUnavailableView) is to give each argument its own dedicated line, aligned vertically.

The Standard Swift Convention

Here is the cleanest, most universally accepted way to format that exact view: 

ContentUnavailableView(
    "No snippets", 
    systemImage: "swift", 
    description: Text("You don't have any saved snippets yet.")
)

Why this format is the industry standard:

  • The "Scan" Test: Your eyes can scan straight down the left margin. You instantly see the Title ("No snippets"), the Icon ("swift"), and the Description (Text(...)) as distinct, individual puzzle pieces.
  • Easy Diffing in Git: If you ever change the icon from "swift" to "doc.text" in the future, your version control (Git) will highlight only that single line as changed. If it's all on one line, Git highlights the entire paragraph, making code reviews annoying.
  • Trailing Punctuation: Notice how the closing parenthesis ) sits on its own line at the very bottom, perfectly aligned with the start of ContentUnavailableView. This cleanly "closes the box" visually.

The Pro-Tip: Let Xcode Do It For You Automatically

You don't have to manually hit Return and space out these lines yourself. Xcode has a built-in feature designed exactly for this.

  • Put your cursor right inside the line (or select lines)
  • Go to Editor ──► Structure:
    • ──► Reformat to Width (<Ctrl><Option> i)
    • ──► Format to multiple lines (<Ctrl> m)
    • ──► Format Selection with 'swift-format' (<Ctrl><Option><Shift> i)
    • ──► Format File with 'swift-format' (<Ctrl><Shift> i)
  • Xcode will instantly snap the arguments down into this clean, vertical stack.

What if it's still too long?

If an argument inside the list is still stretching too far - like your description parameter - you can even break down the internal elements, like this: 

ContentUnavailableView(
    "No snippets", 
    systemImage: "swift", 
    description: Text(
        "You don't have any saved snippets yet. Click the plus button above to add your first one!"
    )
)

By keeping the open/close brackets ((, )) vertically aligned with the code that triggered them, your SwiftUI files will stay neat, legible, and easy to maintain.
