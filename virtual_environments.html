Virtual Environments - Python

Depending on the installed python, you might have to run python3 in place of python in the steps ahead. (Python 3.3 introduced Virtual Environments using venv.)

  • Make sure to check out the PyEnv pages; PyEnv offers an alternative

Creating a Virtual Environment

python -m venv /path/to/new/virtual/env

Activate the Virtual Environment

# either navigate to the env and:
source bin/activate
# or enter whole path from anywhere:
source /path/to/virtual/env/bin/activate

Deactivate the Virtual Environment

From within the venv: 

deactivate

Save 'requirement' file for recreating the environment packages

python -m pip freeze > requirements.txt
# or
pip freeze # to the commandline stdout
# or
pip freeze > requirements.txt
# or
pip3 freeze > requirements.txt

...and to install from the requirements file:

python -m pip install -r requirements.txt
# or
pip install -r requirements.txt
# or
pip3 install -r requirements.txt
