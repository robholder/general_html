Short Message (shortmess) Flags

You can set various options for shorter or suppression of messages (usually in the .vimrc settings file. (Setting values for the flags interactively is NOT sticky.)

To check or set the flags, use the following: 

" to show current flags
:set shortmess?

" to add flags
:set shortmess+=<flags>

" to remove flags
:set shortmess-=<flags>

You can add (+) or remove (-) flags en mass with one set command.

Removing the Opening Flash Page

To remove the opening message page from Vim (when launching), I have the following in my .vimrc: 

set shortmess+=I
NOTE: This setting has to be AFTER the 'set nocompatible' setting if present. The nocompatible setting reverts shortmess to its defaults.

Flags

The shortmess option in Vim uses flags (letters) to control message display and verbosity. Here's a more comprehensive list of the flags you can use:

  • A: Avoid showing the "ATTENTION" message when a swap file exists.
  • C: Suppress the "recording" message while recording a macro.
  • F: Skip file info messages like "Reading file..." or "New file...".
  • I: Omit the intro message when Vim starts.
  • O: Suppress "hit-enter" prompts for :open commands.
  • S: Hide "search hit BOTTOM, continuing at TOP" messages.
    • Suppresses the "search hit BOTTOM, continuing at TOP" or "search hit TOP, continuing at BOTTOM" messages when a search wraps around the file. This is particularly useful if you find these messages disruptive when navigating with / or ?.
    • S also hides the match count.
    • Removing this with :set shortmess-=S will enable the match count.
  • T: Avoid error messages about reading or writing files when open() or write() fails.
  • W: Skip warnings when changing files with unsaved changes.
  • c: Shorten messages in completion menus.
  • f: Suppress "[readonly]", "[modified]", or "[file format]" flags.
  • i: Shorten messages for insertion or change operations.
  • l: Suppress "loading ..." messages when opening files/plugins.
  • n: Avoid "search hit N match" messages during searches.
  • o: Skip prompts when overwriting files.
  • q: Avoid showing "read/write" messages for quickfix commands.
  • s: Hide "search hit BOTTOM/TOP" messages.
    • Suppresses the "search hit BOTTOM" and "search hit TOP" messages entirely. Unlike the S flag, this means you won't see any notifications when reaching the start or end of a file during a search.
  • t: Suppress truncation messages for long file names.
  • x: Avoid redundant "extra" error messages.

Default Values for Shortmess

The default value for the shortmess option in Vim is typically: 

filnxtTo

Here's a breakdown of the default flags:

  • f: Suppress [readonly], [modified], or [file format] flags.
  • i: Shorten messages for insertion or change operations (e.g., fewer words).
  • l: Suppress "loading ..." messages when opening files/plugins.
  • n: Skip "search hit N match" messages during searches.
  • x: Avoid redundant "extra" error messages.
  • t: Suppress truncation messages for long file names.
  • T: Avoid error messages about reading or writing files when open() or write() fails.
  • o: Skip prompts when overwriting files.

These defaults balance verbosity and conciseness for most users. If you modify shortmess during a session and want to reset it to these defaults, you can use: 

:set shortmess=filnxtTo
REMEMBER: Interactive shortmess changes are not sticky, they revert if not set in the .vimrc.
