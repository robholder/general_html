Python - CSV Splitter

Splitting CSVs by Month

Here is a complete Python script to handle this for you. It uses Python's built-in csv and datetime modules, so you don't need to install any external libraries (like pandas).

The script reads your master CSV file, looks at the Posting Date for each row, figures out the month and year, and automatically groups and saves them into individual monthly files with the original headers intact.

The Python Script

import csv
from datetime import datetime
import os

# --- CONFIGURATION ---
# Replace 'master_bank_file.csv' with the actual path to your downloaded file
INPUT_FILE = 'master_bank_file.csv'
# The directory where you want the monthly files to be saved (current directory by default)
OUTPUT_DIR = 'monthly_statements'
# ---------------------

def split_csv_by_month(input_file, output_dir):
    # Create the output directory if it doesn't exist
    if not os.path.exists(output_dir):
        os.makedirs(output_dir)
        print(f"Created directory: {output_dir}")

    # Dictionary to hold file handles and csv.writer objects for each month
    # Structure: {(year, month): (file_object, csv_writer_object)}
    monthly_writers = {}

    try:
        with open(input_file, mode='r', newline='', encoding='utf-8') as infile:
            # Use DictReader so we can access columns by their heading names
            reader = csv.DictReader(infile)
            headers = reader.fieldnames
            
            if not headers:
                print("Error: The input CSV file is empty or missing headers.")
                return

            print("Processing rows...")
            row_count = 0

            for row in reader:
                # 1. Extract the Posting Date
                date_str = row.get('Posting Date')
                if not date_str:
                    continue  # Skip rows with missing dates
                
                # 2. Parse the date to extract year and month
                # NOTE: Adjust the format '%m/%d/%Y' if your bank uses a different format 
                # (e.g., '%Y-%m-%d' or '%d/%m/%Y')
                try:
                    # Trying standard MM/DD/YYYY first
                    date_obj = datetime.strptime(date_str.strip(), '%m/%d/%Y')
                except ValueError:
                    try:
                        # Fallback for YYYY-MM-DD just in case
                        date_obj = datetime.strptime(date_str.strip(), '%Y-%m-%d')
                    except ValueError:
                        print(f"Skipping row due to unexpected date format: {date_str}")
                        continue

                year_month_key = (date_obj.year, date_obj.month)

                # 3. If this is a new month, create a new file and write the headers
                if year_month_key not in monthly_writers:
                    file_name = f"Statement_{date_obj.year}_{date_obj.month:02d}.csv"
                    file_path = os.path.join(output_dir, file_name)
                    
                    outfile = open(file_path, mode='w', newline='', encoding='utf-8')
                    
                    # ADDED extrasaction='ignore' HERE to bypass Chase's trailing columns safely
                    writer = csv.DictWriter(outfile, fieldnames=headers, extrasaction='ignore')
                    writer.writeheader()  # Re-instate headings
                    
                    monthly_writers[year_month_key] = (outfile, writer)
                
                # 4. Write the row to the appropriate monthly file
                _, writer = monthly_writers[year_month_key]
                writer.writerow(row)
                row_count += 1

            print(f"Success! Processed {row_count} rows into {len(monthly_writers)} monthly files.")

    except FileNotFoundError:
        print(f"Error: The file '{input_file}' was not found. Please check the path.")
    finally:
        # Clean up: Close all open output files safely
        for outfile, _ in monthly_writers.values():
            outfile.close()

if __name__ == "__main__":
    split_csv_by_month(INPUT_FILE, OUTPUT_DIR)

How to use it:

  • Save the Script: Copy the code above and save it as split_bank_csv.py.
  • Place your CSV: Put your large, multi-month bank CSV file in the same folder as the script, and make sure its filename matches the INPUT_FILE variable in the script (currently set to 'master_bank_file.csv').
  • Run the Script: Open your terminal or command prompt, navigate to the folder, and run: 
python split_bank_csv.py
  • Check the Output: A new folder called monthly_statements will appear, containing files named like Statement_2026_01.csv, Statement_2026_02.csv, etc., each complete with your original headings.

⚠️ One Quick Thing to Double-Check

Banks format dates differently depending on where you live and who you bank with. In the script, I used the standard US format: MM/DD/YYYY (e.g., 05/23/2026).

If your bank formats dates differently—for instance, DD/MM/YYYY - you'll just need to tweak line 39 in the script: For DD/MM/YYYY (e.g., 23/05/2026), change '%m/%d/%Y' to '%d/%m/%Y'.

QUESTION

Can we incorporate the whole script into the Automator script (so there are no dependencies)?

ANSWER

Yes, absolutely. Embedding the Python code directly into Automator is much cleaner because it makes the Quick Action entirely self-contained. You can move or share the Quick Action without worrying about breaking a file path.

Here is how to embed the entire script directly into Automator:

Step 1: Configure Automator

  • Create a new Quick Action in Automator.
  • Ensure the top settings are still set to:
    • Workflow receives current: files or folders
    • in: Finder.app
  • In the left-hand library search bar, type "Run Shell Script" and drag that action into the main workflow area on the right. ("Run Shell Script" is under 'Utilities'.)
  • In your Run Shell Script action, change the Shell dropdown from /bin/bash (or /bin/zsh) to /usr/bin/python3.
  • Ensure Pass input is set to as arguments.

Step 2: Paste the Self-Contained Code

Clear out any text currently in the box and paste the following code block. This version is specifically adapted to read the file paths directly from macOS's system arguments array (sys.argv): 

import csv
from datetime import datetime
import os
import sys

def split_csv_by_month(input_file):
    # Dynamically find the directory where the source file lives
    source_dir = os.path.dirname(os.path.abspath(input_file))

if __name__ == "__main__":
    # sys.argv[1:] captures all files passed by Finder when you right-click

Step 3: Save and Run

  • Press Cmd + S to save the workflow.
  • Name it something like Split Bank CSV by Month.

Now, the entire Python engine lives right inside the workflow file (~/Library/Services/Split Bank CSV by Month.workflow). You can delete the old standalone .py file from your computer entirely - this Quick Action is now fully independent and portable!

QUESTION

Where can I find the Automator scripts? (I'm curious if these can be transferred or shared with other account/machines.)

ANSWER

You can absolutely transfer and share these scripts with other accounts or machines! Because we eliminated external script dependencies and packed the whole Python code inside the Quick Action, sharing it is incredibly easy.

Where Quick Actions Are Located

Every custom Quick Action you create is saved as a packaged bundle with a .workflow extension. They live in a hidden folder inside your user directory: 

~/Library/Services/

How to find it quickly in Finder:

  • Open a Finder window.
  • In the top macOS menu bar, click Go > Go to Folder... (or press Cmd + Shift + G).
  • Type or paste exactly this: ~/Library/Services and press Enter.

Inside that folder, you will see a file named exactly what you named your action (e.g., Split Bank CSV by Month.workflow).

How to Share or Move It

To transfer the Quick Action to another user account or an entirely separate Mac, follow these steps:

1. Copy the File

Grab the .workflow file from your ~/Library/Services folder and share it however you like (via AirDrop, email, USB drive, Slack, etc.).

2. Install on the Destination Mac

On the target Mac or account, simply double-click the .workflow file. macOS will recognize it as an Automator package and show a prompt asking: "Do you want to install this Quick Action?"

Click Install, and macOS will automatically move it into that specific user's ~/Library/Services folder.

3. Ready to Use

The new machine will instantly have the right-click menu functionality. Because it uses /usr/bin/python3 (built straight into macOS) and handles any standard CSV formatting automatically, it will work right out of the box without requiring any environment setup!
