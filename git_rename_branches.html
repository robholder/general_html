Git - Renaming Branches

I needed to rename branches on a 'dotfiles' repo (with branches for each machine).

Renaming a branch is a common part of keeping a "fleet" organized, especially if you decide to change a machine's hostname later.

The "Local" Rename

If you are currently on the branch you want to rename, use the -m (move) flag: 

git branch -m new-branch-name

If you are on a different branch (like main) and want to rename a different one: 

bash
git branch -m old-branch-name new-branch-name

The "Remote" Sync (If you've pushed to GitHub/Lab)

Renaming locally doesn't automatically fix the server. You have to "swap" them out:

  • Push the new name:
    • git push origin -u new-branch-name
  • Delete the old name:
    • git push origin --delete old-branch-name

Does Renaming a Local Branch Leave The Old Branch on GitHub?

Yes, for a remote repository (like GitHub or Bitbucket), there is no "rename" command. When you follow the rename process, you are essentially creating a new pointer and deleting the old one.

If you only push the new name without deleting the old one, you will indeed end up with two identical branches on the server.

The "Rename-Push-Delete" Process

To correctly "rename" a remote branch, you must perform these three steps:

  • Rename Locally: This changes the "sticky note" on your own computer. 
git branch -m old-name new-name
  • Push the New Name: This creates a brand new branch on the server and links your local branch to it. 
git push origin -u new-name
  • Delete the Old Remote: This removes the outdated branch from the server. 
git push origin --delete old-name

GitHub - Rename Branch

If you are using GitHub, you can rename a branch directly in the browser by clicking the "pencil" icon next to the branch name. This handles the "push and delete" for you on the server side, but you will still need to run a few commands locally to update your computer: 

git fetch origin
git branch -u origin/new-name new-name
git remote prune origin

Would you like to see how to "prune" your local list so those old, deleted remote names stop showing up when you run git branch -a?

Pruning

To "prune" your Git branches and keep your local environment clean after renaming or deleting branches on a server like GitHub, you need to address two different types of "stale" data: remote-tracking references and orphaned local branches.

1. Pruning Remote References

When a branch is deleted from the remote server, your local machine still keeps a "bookmark" of it (e.g., origin/old-name). Running a standard git fetch does not remove these.

The Command:

git fetch --prune
  • What it does: It connects to the server, sees which branches are gone, and deletes the local origin/ references for them.
  • Pro-Tip: You can make this the default behavior for every fetch by running:

git config --global fetch.prune true.

2. Cleaning Up "Orphaned" Local Branches

Even after pruning the remote references, the actual local branch (e.g., old-name) still exists on your hard drive. Git marks these as [gone].

The Manual Check:

Run git branch -vv to see which branches are no longer on the server. You will see a label like [origin/old-name: gone] next to the branch name.

The One-Line Cleanup (Bash/Zsh):

If you have several orphaned branches, you can delete all of them at once with this command: 

git branch -vv | grep ': gone]' | awk '{print $1}' | xargs git branch -d
  • grep ': gone]': Filters for only the orphaned branches.
  • awk '{print $1}': Extracts just the branch names.
  • git branch -d: Safely deletes the branch only if it has been merged. Use -D if you want to force-delete unmerged work.

3. Summary of Cleanup Commands

Command What it cleans
git fetch -p Removes stale origin/branch-name references.
git remote prune origin Same as above, but doesn't download new data.
git branch -d Deletes the actual local branch.
git remote prune origin --dry-run Shows what would be pruned without doing it.
