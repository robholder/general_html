Push to, or get from, GitHub

Create A New Repo - And Push To GitHub

# Navigate to or create a working directory/project
# Then setup git:
# the -b option is only available in versions after 2.28.
git init -b main
# prior to version 2.28, use:
git init
# Add your existing files:
git add .
# commit the files:
git commit -m "first commit"
# set up repo at Github - and use the Quick setup (copy) URL. (e.g. myrepo)
# Do not create a read me!! (or there'll be a mismatch)
git remote add origin https://github.com/rholder/myrepo.git
# Verify new URL
git remote -v
git push -u origin main
# you'll be asked to input your (github) username
# you'll be asked for a password - this is your personal token (which is pasted somewhere safe!)
# (see below for hints on personal access tokens)

Push An Existing Repo

git remote add origin https://github.com/rholder/myrepo.git
git push -u origin main

To Push All Branches To GitHub

git push --all

To Pull From A GitHub Repo

# To get the 'changes' from a remote repo:
git fetch
# ...followed by a merge to 'merge':
get merge

Or... To get and merge the 'changes' from a remote repo: 

git pull

To Clone A Github Repo To Local Machine

  • Go to (or setup) a repo on Github.
  • Click on 'Code' and use the HTTPS option
  • Click on the copy button at the end of the HTTPS line
  • In the commandline: 
    # git clone, and paste the URL to your repo:
git clone http://githb.com/YOUR_USERNAME/YOUR_REPO
  • it turns out you need to set up a token instead of the password.
  • GitHub -> Settings -> Developer Settings - and set up a token
  • copy and paste this token when the terminal on MacOS asks for a password (and it doesn't show anything, but it works)

Note on SSH

ssh-agent is not persistent across reboots. You need to start agent... 

eval $(ssh-agent)

...and then you need to add them manually after reboot or setup some script to load them automatically.

If you have have your keys in "default location", like ~/.ssh/id_rsa, it should be enough to run... 

ssh-add

...otherwise you need to run... 

ssh-add path/to/your/private/key.
