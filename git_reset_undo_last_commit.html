Reset - Undo Last Commit

Is it possible to undo a commit in Git?

Yes, it is absolutely possible to undo a commit in Git. In fact, Git has a few different ways to do it depending on whether you want to keep your work or completely wipe it out, and whether you've already pushed the commit to GitHub (or another remote).

Here is a breakdown of the three most common ways to undo a commit.

1. The Safe Option: git reset --soft

Use this if you made a commit but realized you forgot to add a file, made a typo, or want to split the commit into two. It undoes the commit but keeps all your changes in your staging area. 

git reset --soft HEAD~1
  • What it does: It moves your branch pointer back by one commit (HEAD~1).
  • Result: Your files remain exactly as they were, sitting in your staging area ready to be re-committed.

2. The Nuclear Option: git reset --hard

Use with caution! Use this only if you want to completely destroy the last commit and throw away all the work associated with it. 

git reset --hard HEAD~1
  • What it does: It rolls back your branch to the previous commit and obliterates any uncommitted changes in your working directory.
  • Result: The commit is gone, and your code goes back to exactly how it looked before that commit was made.

3. The Public Option: git revert

If you have already pushed your commit to a shared remote repository, using git reset can mess up your teammates' history. Instead, you should use git revert. 

git revert HEAD
  • What it does: Instead of deleting the bad commit, it creates a brand new commit that does the exact opposite of the bad one (e.g., if you added a line in the bad commit, the revert commit deletes it).
  • Result: Your history remains safe and linear, making it the polite way to fix mistakes in shared projects.

Summary Cheat Sheet

Situation Command to Use What happens to your code? Safe for Pushed Commits?
Want to fix the commit message or add more files git reset --soft HEAD~1 Saved as staged changes ❌ No (creates history conflicts)
The commit was a mistake and I want to destroy the code git reset --hard HEAD~1 Permanently deleted ❌ No
I already pushed the bad commit to GitHub/GitLab git revert HEAD Preserved, but undone in a new commit Yes

💡 Quick Tip: If you just made a commit and realized you made a typo in the commit message, you don't need to undo the whole thing. Just type git commit --amend to update the message of your most recent commit!
