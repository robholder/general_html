Hacking With Swift - How to create a deinitializer for a class

Swift classes can have deinitializers, called when objects are destroyed.

Deinitializers

Deinitializers, unlike initializers, don’t use 'func', take parameters, or return data. They are automatically called when the final copy of a class instance is destroyed.

This comes with a few small provisos:

  • You don’t use func with deinitializers.
  • Deinitializers can never take parameters or return data
    • (As a result aren’t even written with parentheses).
  • Deinitializers are run when the final copy of a class-instance is destroyed.
    • (That might mean it was created inside a function that is now finishing, for example.)
  • We never call deinitializers directly.
    • (They are handled automatically by the system.)
  • Structs don’t have deinitializers.
    • (Because you can’t copy them).

Scope

'Deinitializer call timing' depends on scope, the context where information is available, similar to variable accessibility within functions, conditions, and loops.

Exactly when your deinitializers are called depends on what you’re doing, but really it comes down to a concept called scope. Scope more or less means “the context where information is available”, and you’ve seen lots of examples already:

  • If you create a variable inside a function, you can’t access it from outside the function.
  • If you create a variable inside an 'if' condition, that variable is not available outside the condition.
  • If you create a variable inside a for loop, including the loop variable itself, you can’t use it outside the loop.

Local scopes are created by conditions, loops, and functions using braces. When a value exits scope, its context is destroyed, and for classes, the underlying data is destroyed when the last reference is removed.

deinit

Class creation and destruction can be demonstrated with print statements. 

class User {
    let id: Int

    init(id: Int) {
        self.id = id
        print("User \(id): I'm alive!")
    }

    deinit {
        print("User \(id): I'm dead!")
    }
}

Now we can create and destroy instances of that quickly using a loop – if we create a User instance inside the loop, it will be destroyed when the loop iteration finishes (since it's in the 'scope' of the loop): 

for i in 1...3 {
    let user = User(id: i)
    print("User \(user.id): I'm in control!")
}

// >> User 1: I'm alive!
// >> User 1: I'm in control!
// >> User 1: I'm dead!
// >> User 2: I'm alive!
// >> User 2: I'm in control!
// >> User 2: I'm dead!
// >> User 3: I'm alive!
// >> User 3: I'm in control!
// >> User 3: I'm dead!

When that code runs you’ll see it creates and destroys each user individually, with one being destroyed fully before another is even created.

Remember, the deinitializer is only called when the last remaining reference to a class instance is destroyed. This might be a variable or constant you have stashed away, or perhaps you stored something in an array.

For example, if we were adding our User instances as they were created, they would only be destroyed when the array is cleared: 

var users = [User]()

for i in 1...3 {
    let user = User(id: i)
    print("User \(user.id): I'm in control!")
    users.append(user)
}

print("Loop is finished!")
users.removeAll()
print("Array is clear!")

// >> User 1: I'm alive!
// >> User 1: I'm in control!
// >> User 2: I'm alive!
// >> User 2: I'm in control!
// >> User 3: I'm alive!
// >> User 3: I'm in control!
// >> Loop is finished!
// >> User 1: I'm dead!
// >> User 2: I'm dead!
// >> User 3: I'm dead!
// >> Array is clear!
