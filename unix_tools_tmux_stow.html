Stow Setup for Tmux

To layout tmux configs with plugins using Stow, create a ~/.config/tmux structure in your dotfiles directory, placing tmux.conf and plugin management scripts inside. Stow this to ~/.config/ so tmux automatically detects the configuration and TPM installs plugins there, keeping your home directory clean.

Recommended Directory Structure:

~/dotfiles/
└──  tmux
    └──  dot-config
        └──  tmux
            ├──  plugins
            │   ├── 󰊢 .gitignore
            │   └──  tpm
            └──  tmux.conf

Steps to Implement:

  • Create the directory structure in your dotfiles repository:
    • mkdir -p ~/dotfiles/tmux/dot-config/tmux
  • Move your existing configuration to this new location:
    • mv ~/.tmux.conf ~/dotfiles/tmux/dot.config/tmux/tmux.conf
  • Edit tmux.conf and include Plugin Manager (TPM) settings:
NOTE: This is old. Look below for tpm, resurrection and continuum. 
# 1. Path (keeps Stow/Symlinks happy)
set-environment -g TMUX_PLUGIN_MANAGER_PATH '~/.config/tmux/plugins'

# 2. List of plugins
set -g @plugin 'tmux-plugins/tpm'
set -g @plugin 'tmux-plugins/tmux-sensible' # Recommended starting plugin

# 3. Initialize (MUST be the last line)
run '~/.config/tmux/plugins/tpm/tpm'

A standard git pull will get you 90% of the way there to including all this Tmux plugin stuff, but because we have introduced a submodule (TPM) and changed the directory structure, we need to run two extra commands on those machines to "activate" the changes.

Here is the exact sequence to run on other machines

1. The Pull & Submodule Sync

A regular pull won't automatically download the contents of the new tpm folder. Run this: 

bash
git pull
git submodule update --init --recursive

This tells Git: "I see a new submodule pointer in the history; go fetch the actual code from GitHub." [1]

2. The Stow Update

Since we moved dot-tmux.conf into dot-config/tmux/tmux.conf, our old symlinks are now broken (pointing to a file that doesn't exist) or outdated. 

stow --restow tmux

The --restow (or -R) flag is a lifactor: it prunes the old dead links and creates the new ones in ~/.config/tmux/. [2]

3. The Plugin Install

Finally, since the plugins/ folder (except for TPM) was gitignored, those machines don't have tmux-sensible or resurrect yet.

  • Open tmux.
  • Press Prefix + I (Capital I).

TPM will now read your updated tmux.conf and download the plugins locally on that machine. [3]

Summary Checklist for Remote Machines:

  • git pull
  • git submodule update --init
  • stow -R tmux
  • tmux -> Prefix + I

Pro-Tip: If you see an error about "conflicting files" during the stow step, it’s because you might have an old ~/.tmux.conf file still sitting there. Delete it manually (rm ~/.tmux.conf) and run the stow command again.

Modern TPM, Resurrection & Continuum Install/Setup

NOTE: This is the modern approach to the setup.

This is the complete, self-contained reference guide to setting up TPM, Tmux-Resurrect, and Tmux-Continuum inside your shared GNU Stow layout.

This design keeps all third-party plugin code contained directly inside your ~/dotfiles repository, while routing the session save files to a local, untracked system directory. This ensures each machine maintains its own unique session history without polluting your shared Git repository.

The Target Architecture

Because you have a single top-level directory symlink (~/.config/tmux pointing directly to ~/dotfiles/tmux/dot-config/tmux), your filesystem structure will look like this: 

~/dotfiles/tmux/dot-config/tmux/
├── tmux.conf            <-- Tracked in Git
└── plugins/             <-- Tracked in Git (Stores third-party code)
    └── tpm/             <-- The package manager bootstrap

Step 1: Bootstrap the Package Manager (TPM)

Run this one-time command on your Mac to create your target directory and clone TPM directly into your local dotfiles layout: 

mkdir -p ~/dotfiles/tmux/dot-config/tmux/plugins/
git clone https://github.com ~/dotfiles/tmux/dot-config/tmux/plugins/tpm

Step 2: Write the Portable tmux.conf

Open /Users/<myusername>/dotfiles/tmux/dot-config/tmux/tmux.conf and replace its entire contents with this configuration block.

Note: This configuration uses universal ~ path expansions so it functions automatically across multiple machines with different usernames.tmux 

# =====================================================================
# 1. PATH CONFIGURATIONS (GNU STOW & MACHINE-LOCAL COEXISTENCE)
# =====================================================================
# Forces TPM to install third-party plugins inside your stowed dotfiles path
set -g @gpm_plugins_path '~/.config/tmux/plugins/'

# Forces session save data to generate locally on each individual machine
# (Bypasses dotfiles repo completely so setups remain unique)
set -g @resurrect-dir '~/.local/share/tmux/resurrect'

# =====================================================================
# 2. PLUGIN DEFINITIONS
# =====================================================================
set -g @plugin 'tmux-plugins/tpm'
set -g @plugin 'tmux-plugins/tmux-resurrect'
set -g @plugin 'tmux-plugins/tmux-continuum'

# =====================================================================
# 3. BACKGROUND CONTINUUM SYSTEM AUTOMATION
# =====================================================================
# Automatically load the last saved session state when tmux server boots up
set -g @continuum-restore 'on'

# =====================================================================
# 4. INITIALIZE TMUX PLUGIN MANAGER
# =====================================================================
# This line MUST be kept at the absolute bottom of this file.
run -b '~/.config/tmux/plugins/tpm/tpm'

The warning that the 'run' command has to be at the end isn't strictly true (especially if you have color settings being overridden by plugin configs).

To maintain color settings, move your styles below the TPM run line.

You can easily secure your styles by taking advantage of a loophole in tmux's configuration parser: The plugin manager line does not actually have to be the final line of your file.

Any command placed after the TPM boot line runs last, effectively overriding any plugin or theme settings that loaded right before it.

Open the ~/dotfiles/tmux/dot-config/tmux/tmux.conf file and move the color commands to the absolute bottom of the file, below your plugin runner: 

# =====================================================================
# 4. INITIALIZE TMUX PLUGIN MANAGER 
# =====================================================================
# TPM boots up here, initializing all your chosen themes and plugins
run '~/.config/tmux/plugins/tpm/tpm'

# =====================================================================
# 5. ABSOLUTE USER OVERRIDES (THE NEW ULTIMATE BOTTOM)
# =====================================================================
# Placed here, these commands intercept and overwrite ANY theme's defaults 
# the millisecond after the plugin manager finishes loading them.
set -g mode-style "bg=yellow,fg=black"
set -g copy-mode-match-style "bg=yellow,fg=black"
set -g copy-mode-current-match-style "bg=yellow,fg=black"

(Note: Change run -b to a standard blocking run command as shown above. This forces tmux to wait until TPM completely finishes loading its assets before executing your yellow style overrides).

Step 3: Initialize the Environment

Since your top-level directory link is already established by Stow, you are ready to boot up the environment:

Kill any hung or legacy background tmux operations to start fresh: 

tmux kill-server

Launch a clean tmux instance: 

tmux

Download and install the remaining plugins: Press your tmux prefix key combo prefix, release it, and then press capital I (Shift + i).

A small terminal overlay will appear confirming that tmux-resurrect and tmux-continuum have successfully cloned into your plugins/ folder.

Step 4: Verify the Saving Pipelines

To guarantee that everything is working across your machine-local boundaries:

  • Force a manual environment save: Press prefix, then Ctrl + s.
  • Look for the brief "Tmux environment saved!" notification on the status bar.
  • Confirm that the local machine generated its unique text storage matrix: 
ls -la ~/.local/share/tmux/resurrect/

You will see a brand new timestamp file mapping today's date. Your dotfiles directory is now fully self-contained, and tmux-continuum will auto-save your active window splits in the background every 15 minutes.

Whenever you clone this configuration onto a brand new machine, all you need to do is run Step 1 to bootstrap TPM, launch tmux, and press prefix + I!

Your Daily Tmux Routine

To keep your setup perpetual and seamless across restarts, follow this reliable workflow:

  • Daily Work: Work inside your tmux sessions normally. Every 15 minutes, tmux-continuum will quietly save your exact environment (windows, splits, paths) in the background.
  • Before a Reboot/Shutdown: You do not strictly need to do anything because of the 15-minute auto-save, but it is highly recommended to manually force a final save right before closing down. Press prefix then Ctrl + s.
  • After a Reboot/Launch: Open WezTerm and type tmux. Because @continuum-restore 'on' is active, tmux will automatically reconstruct your entire environment exactly as it was during the last save.

If Automatic Restore Ever Fails: The Manual Intervention

If your machine crashes or you launch tmux and find a blank slate, you can manually force a complete restoration using this explicit hotkey:

  • Press prefix followed by Ctrl + r (Restore).

Tmux will read your most recent local text snapshot from ~/.local/share/tmux/resurrect/ and immediately spin your workspace layout back into existence.

How to Keep the Server Running "Forever" (No Terminal Dependency)

By default, closing your terminal application (WezTerm) will not kill your tmux sessions, but turning off or restarting your Mac will.

If you want your sessions to stay alive in the background even if you quit WezTerm or close your laptop lid, you are leveraging tmux's native architecture:

  • Detach Safely: When you want to close your terminal window but leave all your code running in the background, press prefix then d (Detach). This exits the interface but leaves the background server completely untouched.
  • Re-Attach Later: Next time you open WezTerm, run tmux attach to instantly jump back into your live session.

Pro-Tip: Deep Application Restoration

Right now, your layout restores your terminal directories, window splits, and layout shapes. However, if you want your text editors (like Vim/Neovim) or active CLI tools to automatically reopen inside those windows when it restores, add these explicit instruction flags to your tmux.conf: 

# Restore Neovim and Vim sessions
set -g @resurrect-strategy-vim 'session'
set -g @resurrect-strategy-nvim 'session'

# Restore additional programs (like pane-running commands)
set -g @resurrect-processes 'ssh psql mysql sqlite3'

(Note: For Vim/Neovim restoration to work, you must also be using a session plugin inside your editor like tpope/vim-obsession or an automated session manager).


Setting Globals for Submodule Recursive Modules

You can change Git's default behavior for the pull command. This ensures that submodules are updated across all your machines and repositories without needing extra commands.

Run this once on each machine:

git config --global submodule.recurse true

What this does: It tells Git to always include the --recurse-submodules flag for every git pull. This automatically runs git submodule update --init --recursive immediately after the pull is complete.

Stow the config:

Run stow tmux from inside your ~/dotfiles directory to symlink ~/.config/tmux/.

If you don't have a .stowrc setup and you've used the 'dot' substitution: 

# If you used the 'dot' substitution in your file structure names
# use the --dotfiles flat to force '.' in the symlink name.
stow --dotfiles tmux

Final Synchronization Tip

Since you are using Stow, remember that while your Git files are now auto-updating, you still need to refresh the symlinks if you change your folder structure. I recommend occasionally running: 

stow -R tmux

Key Rules for --dotfiles

  • Replacement logic: Stow only replaces the prefix dot- at the beginning of a file or folder name.
  • Nested files: You do not need to prefix every file. For example, in dot-config/tmux/tmux.conf, only the top-level dot-config needs the prefix to become ~/.config. The files inside it will retain their original names.
  • Multiple dots: A file named dot-tmux.conf will be stowed as .tmux.conf.
  • Version requirement: This feature was introduced in Stow 2.3.0. Ensure your version is current by running stow --version.

Install Plugins:

  • Clone TPM into your configured directory:
    • git clone https://github.com/tmux-plugins/tpm ~/.config/tmux/plugins/tpm
  • Reload:
    • Restart or reload tmux (prefix + I to install plugins).

Using ~/.config/tmux is superior because it is the standard location for modern tmux versions (3.1+), and it keeps plugins, session data, and configurations in one organized directory.
