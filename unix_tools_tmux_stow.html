Stow Setup for Tmux

To layout tmux configs with plugins using Stow, create a ~/.config/tmux structure in your dotfiles directory, placing tmux.conf and plugin management scripts inside. Stow this to ~/.config/ so tmux automatically detects the configuration and TPM installs plugins there, keeping your home directory clean.

Recommended Directory Structure:

~/dotfiles/
└──  tmux
    └──  dot-config
        └──  tmux
            ├──  plugins
            │   ├── 󰊢 .gitignore
            │   └──  tpm
            └──  tmux.conf

Steps to Implement:

  • Create the directory structure in your dotfiles repository:
    • mkdir -p ~/dotfiles/tmux/dot-config/tmux
  • Move your existing configuration to this new location:
    • mv ~/.tmux.conf ~/dotfiles/tmux/dot.config/tmux/tmux.conf
  • Edit tmux.conf and include Plugin Manager (TPM) settings: 
# 1. Path (keeps Stow/Symlinks happy)
set-environment -g TMUX_PLUGIN_MANAGER_PATH '~/.config/tmux/plugins'

# 2. List of plugins
set -g @plugin 'tmux-plugins/tpm'
set -g @plugin 'tmux-plugins/tmux-sensible' # Recommended starting plugin

# 3. Initialize (MUST be the last line)
run '~/.config/tmux/plugins/tpm/tpm'

A standard git pull will get you 90% of the way there to including all this Tmux plugin stuff, but because we have introduced a submodule (TPM) and changed the directory structure, we need to run two extra commands on those machines to "activate" the changes.

Here is the exact sequence to run on other machines

1. The Pull & Submodule Sync

A regular pull won't automatically download the contents of the new tpm folder. Run this: 

bash
git pull
git submodule update --init --recursive

This tells Git: "I see a new submodule pointer in the history; go fetch the actual code from GitHub." [1]

2. The Stow Update

Since we moved dot-tmux.conf into dot-config/tmux/tmux.conf, our old symlinks are now broken (pointing to a file that doesn't exist) or outdated. 

stow --restow tmux

The --restow (or -R) flag is a lifactor: it prunes the old dead links and creates the new ones in ~/.config/tmux/. [2]

3. The Plugin Install

Finally, since the plugins/ folder (except for TPM) was gitignored, those machines don't have tmux-sensible or resurrect yet.

  • Open tmux.
  • Press Prefix + I (Capital I).

TPM will now read your updated tmux.conf and download the plugins locally on that machine. [3]

Summary Checklist for Remote Machines:

  • git pull
  • git submodule update --init
  • stow -R tmux
  • tmux -> Prefix + I

Pro-Tip: If you see an error about "conflicting files" during the stow step, it’s because you might have an old ~/.tmux.conf file still sitting there. Delete it manually (rm ~/.tmux.conf) and run the stow command again.

Setting Globals for Submodule Recursive Modules

You can change Git's default behavior for the pull command. This ensures that submodules are updated across all your machines and repositories without needing extra commands.

Run this once on each machine:

git config --global submodule.recurse true

What this does: It tells Git to always include the --recurse-submodules flag for every git pull. This automatically runs git submodule update --init --recursive immediately after the pull is complete.

Stow the config:

Run stow tmux from inside your ~/dotfiles directory to symlink ~/.config/tmux/.

If you don't have a .stowrc setup and you've used the 'dot' substitution: 

# If you used the 'dot' substitution in your file structure names
# use the --dotfiles flat to force '.' in the symlink name.
stow --dotfiles tmux

Final Synchronization Tip

Since you are using Stow, remember that while your Git files are now auto-updating, you still need to refresh the symlinks if you change your folder structure. I recommend occasionally running: 

stow -R tmux

Key Rules for --dotfiles

  • Replacement logic: Stow only replaces the prefix dot- at the beginning of a file or folder name.
  • Nested files: You do not need to prefix every file. For example, in dot-config/tmux/tmux.conf, only the top-level dot-config needs the prefix to become ~/.config. The files inside it will retain their original names.
  • Multiple dots: A file named dot-tmux.conf will be stowed as .tmux.conf.
  • Version requirement: This feature was introduced in Stow 2.3.0. Ensure your version is current by running stow --version.

Install Plugins:

  • Clone TPM into your configured directory:
    • git clone https://github.com/tmux-plugins/tpm ~/.config/tmux/plugins/tpm
  • Reload:
    • Restart or reload tmux (prefix + I to install plugins).

Using ~/.config/tmux is superior because it is the standard location for modern tmux versions (3.1+), and it keeps plugins, session data, and configurations in one organized directory.
