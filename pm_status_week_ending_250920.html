CellSelect Memory Issues

Updated page and posted a heads-up on the comp-rads Slack channel 

CellSelect Tool
Even though this tool can be ideal in many situations, I've added a Warning section to the top of the tool page about memory issues as follows.
It may be a great solution in many cases, but be aware that it can sometimes cause your memory usage to bloat (and potentially cause failure locally and on the farm). It's been used many times without issue, but be aware of possible memory drawbacks.
To help mitigate issues, these guides might help:
Consider precomping the mattes rather than generating them live with each render.
ALWAYS consider using garbage roto mattes in place of this tool - it may be quicker and less impactful in many cases
Try to place the CellSelect nodes as near to the source of the cell color image as possible (and try to steer away from placing CellSelect nodes after Colorpick nodes or color process in the character templates. This is an anecdotal observation of memory issues, but be aware of the possibility.
The tool is very unforgiving if the sample strays off specific color values. It’ll cause major issues if the tool finds itself over non-color pixels (as in outside of crop areas or off-screen areas). Even a single frame will cause it to fail.
[edit - tip added from Nafisa B and Mathew T] A last ditch memory management tweak is to try setting the chunking to a smaller size on the render farm. (There is a ‘Chunk’ size parameter on the Deadline Render Submit interface.)
https://paramount.atlassian.net/wiki/spaces/PFA/pages/580027139/CellSelect (edited) P
