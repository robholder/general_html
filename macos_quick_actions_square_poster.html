Python - Quick Action - Square Poster (Image Edit)

macOS Tahoe

  • Run Shell Script
  • Workflow receives current: image files
  • in: Finder.app
  • Shell <path to python3>
  • Pass input: as arguments 
    import os
import shutil
import subprocess
import sys

bg_col_suffix = "black"
bg_col_hex = '000000'

def make_image_square(file_path):
    # 1. SPLIT FILE PATH AND GENERATE UNIQUE SUFFIX
    base_path, extension = os.path.splitext(file_path)
    
    # Try the default destination first
    new_square_path = f"{base_path}_square{extension}"
    
    # If the file already exists, loop until a unique number suffix is found
    counter = 1
    while os.path.exists(new_square_path):
        new_square_path = f"{base_path}_{bg_col_suffix}_square_{counter}{extension}"
        counter += 1
    
    # 2. GENERATE THE SAFE DUPLICATE
    try:
        shutil.copy2(file_path, new_square_path)
    except Exception:
        return # Skip if file copying fails
        
    # 3. READ ORIGINAL DIMENSIONS VIA SIPS
    try:
        cmd = ['sips', '-g', 'pixelWidth', '-g', 'pixelHeight', new_square_path]
        result = subprocess.run(cmd, capture_output=True, text=True, check=True)
        
        width = 0
        height = 0
        
        for line in result.stdout.split('\n'):
            if 'pixelWidth:' in line:
                width = int(line.split(':')[1].strip())
            elif 'pixelHeight:' in line:
                height = int(line.split(':')[1].strip())
        
        # 4. CALCULATE CANVAS & APPLY BLACK PADDING
        max_dimension = max(width, height)
        
        if max_dimension > 0:
            subprocess.run([
                'sips',
                '--padToHeightWidth', str(max_dimension), str(max_dimension),
                '--padColor', bg_col_hex,
                new_square_path
            ], check=True)
            
    except Exception:
        # Cleanup broken copy on failure
        if os.path.exists(new_square_path):
            os.remove(new_square_path)

if __name__ == "__main__":
    for path in sys.argv[1:]:
        if os.path.exists(path):
            make_image_square(path)

cat > "$PYTMP" << 'EOF'
import os
import shutil
import subprocess
import sys

bg_col_suffix = "black"

def make_image_square(file_path):
    base_path, extension = os.path.splitext(file_path)
    new_square_path = f"{base_path}_square{extension}"
    counter = 1
    while os.path.exists(new_square_path):
        new_square_path = f"{base_path}_{bg_col_suffix}_square_{counter}{extension}"
        counter += 1

    try:
        shutil.copy2(file_path, new_square_path)
    except Exception:
        return

    try:
        # Convert to RGB JPEG so Mojave can pad it
        subprocess.run([
            'sips',
            '-s', 'format', 'jpeg',
            new_square_path
        ], check=True)

        # Read dimensions
        cmd = ['sips', '-g', 'pixelWidth', '-g', 'pixelHeight', new_square_path]
        result = subprocess.run(cmd, capture_output=True, text=True, check=True)

        width = height = 0
        for line in result.stdout.split('\n'):
            if 'pixelWidth:' in line:
                width = int(line.split(':')[1].strip())
            elif 'pixelHeight:' in line:
                height = int(line.split(':')[1].strip())

        max_dimension = max(width, height)

        if max_dimension > 0:
            # Mojave requires padColor as three separate values
            subprocess.run([
                'sips',
                '--padColor', '0', '0', '0',
                new_square_path
            ], check=True)

            # Mojave requires padToHeightWidth in a separate command
            subprocess.run([
                'sips',
                '--padToHeightWidth', str(max_dimension), str(max_dimension),
                new_square_path
            ], check=True)

    except Exception:
        if os.path.exists(new_square_path):
            os.remove(new_square_path)

if __name__ == "__main__":
    print("Python received:", sys.argv[1:])
    for path in sys.argv[1:]:
        if os.path.exists(path):
            make_image_square(path)
EOF

Getting this workflow running on macOS Mojave (as show above), involves the following changes:

This will help you understand why Mojave behaved the way it did, and why the fixes were necessary.

Below is a clean, structured breakdown of every meaningful difference.

⭐ 1. Python execution method

Tahoe (modern macOS)

  • You ran Python 3 directly inside Automator using a here‑document.
  • Modern macOS has a stable shell environment and supports pyenv shims inside Automator.

Mojave

  • Automator’s shell environment is restricted.
  • pyenv shims do not work inside Automator.
  • Here‑documents often mis-parse, causing Automator to try executing file paths as commands.

Fix

  • Use a temporary Python file written via cat > "$PYTMP" and executed directly.
  • Call the real Python interpreter, not the shim: 
    /Users/myusername/.pyenv/versions/3.10.15/bin/python3

This guarantees Python runs correctly on Mojave.

⭐ 2. Argument passing

Tahoe

  • Automator reliably passed file paths to Python.
  • "$@" after the here‑document worked normally.

Mojave

  • Automator often fails to pass arguments unless:
    • "Pass input" is set to 'as arguments'
    • The workflow receives files or folders (or 'images')
    • The workflow is set to run in Finder

Fix

  • Verified argument passing with debug output.
  • Ensured correct Automator Quick Action settings.

⭐ 3. sips behavior differences

This is the biggest functional difference.

Tahoe

  • sips supports:
    • --padColor 000000
    • --padToHeightWidth in the same command
    • padding PNGs, JPEGs, HEIC, etc.
    • padding images with alpha channels

Mojave

Mojave’s sips is much more limited:

  • Cannot pad PNGs with alpha
  • Cannot pad indexed PNGs
  • Cannot pad CMYK JPEGs
  • Cannot pad HEIC
  • Cannot pad GIF
  • Silently fails when padding unsupported formats
  • Requires:
    • --padColor as three separate integers (0 0 0)
    • --padColor and --padToHeightWidth in separate commands

Fix

We added:

Step 1 — Convert the duplicate image to RGB JPEG

This makes the image compatible with Mojave’s padding: 

sips -s format jpeg new_square_path

Step 2 — Use Mojave‑compatible padding syntax 

sips --padColor 0 0 0 new_square_path
sips --padToHeightWidth max max new_square_path

This is the exact syntax Mojave accepts.

⭐ 4. Silent failure handling

Tahoe

  • sips errors are usually visible.
  • Padding failures are rare.

Mojave

  • sips fails silently.
  • No stderr output.
  • No error codes.
  • The duplicate file is created but unchanged.

Fix

  • Added debug prints.
  • Added logging (optional).
  • Added explicit conversion step to avoid silent failures.

⭐ 5. Image format compatibility

Tahoe

  • Modern sips supports padding PNGs with transparency.

Mojave

  • Mojave’s sips cannot pad images with alpha.
  • It must be converted to JPEG first.

Fix

  • Convert to JPEG before padding.

⭐ Final takeaway

Your original script was perfectly valid for modern macOS, but Mojave’s older sips tool and Automator environment required:

  • a different Python execution strategy
  • different argument handling
  • different padding syntax
  • image format conversion
  • avoiding pyenv shims

Once those Mojave-specific quirks were addressed, your workflow works exactly as intended.
