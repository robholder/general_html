Git Ignore (.gitignore)

Create this file to add files and folder to ignore (and NOT track) as direct entries or regex references.

The 'git ignore' file should be named .gitignore, and should be tracked and commited with the repository.

Regex are very useful in the .gitignore, so extensions can be listed with *.ext

Folders must end with a /, otherwise they would be considered references to files.

Comments are preceded by #. These lines will be ignored by the interpreter.

Standard 'glob' patterns work with .gitignore entries.

  • '* ' - Zero or more characters
  • '?' - matches any single character
  • '!' - negates the patterns that follow.
  • [abc] - match any character in the brackets
  • [1-9] - match any character in a range
  • 'a/**/z' - match nested directories (this would match a/one/z, a/two/z etc.)
  • '/' - at the start of a line indicates 'at current level only' and not recursive

You can add a .gitignore to any directory in the repository if needed (multiple .gitignore files).

.gitignore commit suggestion: precede your note with "chore" (e.g. "chore: add .gitignore file").

Random Tip: If you have already added a file to the repo, use git rm to remove it, add a reference to the file to the .gitignore, and re-commit.

Global .gitignore

You can set up a workstation 'global' .gitignore that will be rejected by the repositories on the workstation.

This sets the path, and you can create the file separately for the config to pick up. (For example at ~/.gitignore_global.) 

# This sets up a file and config setting to use it.
git config --global core.excludesfile ~/.gitignore_global

# To see where the current gitignore config has been set:
git config --global core.excludesfile

Example for MacOS: 

# General
.DS_Store
.AppleDouble
.LSOverride

# Icon must end with two \r
Icon

# Thumbnails
._*

# Files that might appear in the root of a volume
.DocumentRevisions-V100
.fseventsd
.Spotlight-V100
.TemporaryItems
.Trashes
.VolumeIcon.icns
.com.apple.timemachine.donotpresent

# Directories potentially created on remote AFP share
.AppleDB
.AppleDesktop
Network Trash Folder
Temporary Items
.apdisk

Example for Windows: 

# Windows thumbnail cache files
Thumbs.db
Thumbs.db:encryptable
ehthumbs.db
ehthumbs_vista.db

# Dump file
*.stackdump

# Folder config file
[Dd]esktop.ini

# Recycle Bin used on file shares
$RECYCLE.BIN/

# Windows Installer files
*.cab
*.msi
*.msix
*.msm
*.msp

# Windows shortcuts
*.lnk

Example for Linux: 

*~

# temporary files which can be created if a process still has a handle open of a deleted file
.fuse_hidden*

# KDE directory preferences
.directory

# Linux trash folder which might appear on any partition or disk
.Trash-*

# .nfs files are created when an open file is removed but is still being accessed
.nfs*

Checking Gitignore Status

When you have Git initialized in a directory, you can check which .gitignore settings are in effect by inspecting both local and global configurations. Here’s how:

Checking Local .gitignore File

Each repository can have its own .gitignore file, typically located in the root of the project. You can view its contents with: 

cat .gitignore

Or, if using Windows PowerShell: 

Get-Content .gitignore

Checking Global .gitignore Settings

Git also allows for a global .gitignore file, which applies across all repositories. To find the location of your global ignore file, run: 

git config --get core.excludesFile

If a file is specified, you can open it to inspect its contents: 

cat <global_gitignore_path>

Checking Ignored Files (Effective Rules)

To see which files are ignored in your repository, you can use: 

git status --ignored

Or, list all ignored files explicitly: 

git ls-files --ignored --exclude-standard

This helps confirm which files are being ignored based on .gitignore settings.

Checking Environment Variables

Git’s ignore rules can also be influenced by environment variables. To check if any are set, run: 

env | grep GIT

On Windows (PowerShell): 

Get-ChildItem Env:GIT*

Summary:

  • Local .gitignore affects only the specific repository.
  • Global .gitignore applies system-wide.
  • git status --ignored shows ignored files.
  • Environment variables might define additional exclusions.
