SSH

  • To be able to clone and push files to Github, you need an SSH key.
  • On a local machine, you need to create an SSH public and private key.
  • The public key needs to be entered into the Github account's settings:
    • Copy the contents of the .pub file and paste it onto Settings>SSH Security > add key area of Github.
  • (You might need to set the permissions of the ssh private key as readable by owner only.)
  • The ssh-agent might be needed to register the ssh key with the local machine.

Local Machine Terminal

Create the keys: 

(In a terminal)
ssh-keygen -b 4096

>>Generating public/private rsa key pair.
>>Enter file in which to save the key (/Users/noddy/.ssh/id_rsa): 

(just hit 'return' for default location and file name)

>>Created directory '/Users/noddy/.ssh'.
>>Enter passphrase (empty for no passphrase): 

(enter a password - this is for the ssh key)

>>Enter same passphrase again: 
>>Your identification has been saved in /Users/noddy/.ssh/id_rsa
>>Your public key has been saved in /Users/noddy/.ssh/id_rsa.pub
The key fingerprint is:
SHA256:IT<blah, blah, blah>kzY myaccount@MyMachine.local
The key's randomart image is:
(this is faked)
+---[RSA 4096]--+
|    ...   .    |
|     o.o.o     |
|    . ...o     |
|    o ... o    |
|  . S ooo +.   |
|  . +o.+ .. =  |
|      .o+.o =+ |
|     .o.o oo.= |
|     o.   .+*..|
+----[SHA256]---+
  • If you need to, you can register the key with ssh-agent 
    ssh-agent -s
ssh-add ~/.ssh/<your key>
(e.g. ssh-add ~/.ssh/id_rsa)
