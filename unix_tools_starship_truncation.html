Startship - Truncation of Directory Path

The prompt can show, hide or truncate the present working directory. There are various settings that control the result.

In [directory] section

To show the full path in Starship, you need to modify the [directory] module inside the specific .toml file associated with that theme.

Starship’s default behavior is to truncate the path to 3 directories (e.g., ~/a/b/c) to save space. To see the whole path, you need to set the truncation_length to 0.

The Fix

  • Open the config file for the theme you want to change (e.g., nano ~/.config/starship/tokyo_night.starship.toml).
  • Search for the [directory] section.
  • Add or change the truncation_length and truncate_to_repo variables as shown below:

Example: 

[directory]
truncation_length = 0      # 0 means "don't truncate at all"
truncate_to_repo = false   # Prevents it from cutting off at the Git root

Understanding the Key Settings

Here is what those specific lines actually do:

Setting Effect
truncation_length = 0 By default, this is 3. Setting it to 0 tells Starship to display every single parent directory from the root/home.
truncate_to_repo = false By default, Starship often "starts" the path at the folder containing your .git folder. Setting this to false ensures you see the full path even when inside a project.
fish_style_pwd_dir_length If this is set (e.g., to 1), it turns directories into single letters (like ~/u/l/bin). Remove this line or set it to 0 for full names.

More Ai Insight

QUESTION

I have Starship prompts, and I was curious if it's possible to make them longer? (I need them wider on the screen to accommodate longer directory paths in the pwd.)

ANSWER

Yes, absolutely! Starship truncates long directory paths by default (often shortening them to 3 parent directories), but you can easily configure it to expand and display longer or complete paths in your ~/.config/starship.toml file.

Here is how you can customize the [directory] section to make your working directory paths wider and much more detailed:

1. Show More Parent Directories (or the Entire Path)

By default, Starship uses truncation_length = 3. You can increase this limit or disable truncation entirely so it shows every directory in your path: 

[directory]
# Increase the number of directories shown before truncation happens:
truncation_length = 8

# OR, disable truncation completely to show the FULL path:
# truncation_length = 0

2. Don't Truncate to Single Characters

By default, when Starship does truncate a path, it shrinks parent folders to a single character (e.g., /u/l/b/bin). You can turn off that behavior so parent folders retain their full names: 

[directory]
# Keeps parent folder names intact even when truncating:
truncate_to_repo = false
fish_style_pwd_dir_length = 0

3. Add a Multi-Line Prompt for Extra Breathing Room

If your paths get so long that they push your command input area off the side of the screen, you can configure Starship to put your text cursor on a second line below the path: 

# Adds a line break so long paths have the entire width of the terminal:
add_newline = true

[line_break]
disabled = false

Example Configuration Block

Here is a solid starship.toml snippet to make directory paths wider and less restricted: 

# ~/.config/starship.toml

[directory]
truncation_length = 10         # Show up to 10 folder levels
truncate_to_repo = false       # Don't truncate when inside a Git repo
fish_style_pwd_dir_length = 0  # Don't shorten parent folders to single letters
style = "bold cyan"

[line_break]
disabled = false               # Keep command typing area on a new line

Once you save the file, hit Enter in your terminal and Starship will immediately render the updated path settings!

QUESTION

Can you explain more about this? 


# Adds a line break so long paths have the entire width of the terminal:
add_newline = true

ANSWER

There is a subtle distinction in Starship configuration between spacing out commands and splitting your prompt into two lines:

1. What add_newline = true actually does

At the global level, add_newline = true inserts an empty blank line above your prompt before every new command.

Without add_newline (false): 

user@host ~/long/path/here $ ls
file1.txt file2.txt
user@host ~/long/path/here $

With add_newline = true: 

user@host ~/long/path/here $ ls
file1.txt file2.txt

user@host ~/long/path/here $

It helps visual clarity by giving previous command outputs some breathing room, but it doesn't give your current path any extra horizontal space.

2. What actually gives your path the "entire width"

If you want a multi-line prompt - where your current directory path sits on top and your typing cursor sits on the line underneath—that is controlled by the [line_break] module.

When [line_break] is active, Starship layout looks like this: 

~/a/very/long/and/deeply/nested/directory/path/that/spans/far (git:main)
❯ _ <--- Your typing cursor gets the entire terminal width!

How to configure both in starship.toml

To get maximum room for long working directory paths without cramping where you type commands: 

# Adds an empty space above the prompt to separate commands visually
add_newline = true

# Splits the prompt into two lines (Path on top, cursor below)
[line_break]
disabled = false

# Ensure the prompt symbol (❯) appears on the bottom line
[character]
success_symbol = "[❯](bold green)"
error_symbol = "[❯](bold red)"
