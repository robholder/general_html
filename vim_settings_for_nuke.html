Vim Settings for Nuke

To make Vim automatically treat .nk files as a special filetype with brace‑based folding enabled. That way, whenever you open a Nuke script, folding is ready to go.

🔎 Step 1: Create a filetype detection rule

Add this to your ~/.vim/filetype.vim (or ~/.vim/ftdetect/nuke.vim): 

au BufRead,BufNewFile *.nk set filetype=nuke

This tells Vim: “Whenever I open a .nk file, set the filetype to nuke.”

🔎 Step 2: Define settings for the nuke filetype

Create a file ~/.vim/ftplugin/nuke.vim with: 

" Enable folding by braces for Nuke scripts
setlocal foldmethod=marker
setlocal foldmarker={,}
setlocal foldenable

This ensures folding is automatically on, using { and } as fold markers.

🔎 Step 3: Optional syntax highlighting

If you want syntax highlighting for .nk files, you can also create a simple syntax file at ~/.vim/syntax/nuke.vim. For example: 

syntax keyword nukeNode Read Write Shuffle Remove
syntax match nukeBrace "{"
syntax match nukeBrace "}"

highlight link nukeNode Keyword
highlight link nukeBrace Delimiter

This isn’t required for folding, but it makes .nk files easier to read.

✅ Result

Now, whenever you open a Nuke scene file (.nk):

  • Vim sets the filetype to nuke.
  • Folding is enabled automatically, with { and } as fold markers.
  • You can fold/unfold blocks with zc, zo, za, etc.

⚡ Summary:

Define a custom filetype for .nk files (nuke), then add an ftplugin that sets foldmethod=marker and foldmarker={,}. This way, brace folding is always enabled when editing Nuke scripts.

Adding Fold Labels

Let’s make folded blocks in your .nk files display something more meaningful than just {…}. In Vim you can customize the text shown for a fold with the foldtext option. By writing a small function, you can show the node name (the first word of the block) instead of the generic fold marker.

🔎 Step 1: Define a foldtext function

Add this to your ~/.vim/ftplugin/nuke.vim: 

function! NukeFoldText()
  " Get the first line of the fold
  let lnum = v:foldstart
  let line = getline(lnum)

  " Trim leading/trailing whitespace
  let line = substitute(line, '^\s*', '', '')
  let line = substitute(line, '\s*$', '', '')

  " Show node name plus fold size
  return line . ' … ' . (v:foldend - v:foldstart + 1) . ' lines'
endfunction

setlocal foldtext=NukeFoldText()

🔎 Step 2: How it works

  • v:foldstart and v:foldend are special Vim variables pointing to the first and last line of the fold.
  • getline(v:foldstart) grabs the first line (usually something like Read {).
  • The function returns a string that becomes the fold’s label in the editor.

Example: 

Read { … 25 lines

🛠️ Optional Enhancements

Highlight the node type only: 

let node = matchstr(line, '^\w\+')
return node . ' block … ' . (v:foldend - v:foldstart + 1) . ' lines'

Add icons or special characters if your font supports them.

✅ Result

Now when you fold a block in a Nuke .nk file:

  • Instead of +-- { … }, you’ll see something like: 
    Transform { … 18 lines

Much easier to navigate large scripts.

⚡ Summary:

Define a custom foldtext function in your ftplugin/nuke.vim that grabs the first line of the fold and displays the node name plus the number of lines. This makes folded blocks in Nuke scripts far more informative.

Color Markers

Let’s make your folded blocks in Vim not only show the node name but also stand out visually with custom highlighting. By default, Vim uses the Folded highlight group for fold text. You can override this or even define your own group and apply it inside your foldtext function.

🔎 Step 1: Define a custom highlight group

In your ~/.vim/ftplugin/nuke.vim (or colorscheme file), add: 

" Define a custom highlight group for node names
highlight NukeFoldNode ctermfg=Yellow guifg=#FFD700 cterm=bold gui=bold

This makes node names appear bold yellow in both terminal and GUI Vim.

🔎 Step 2: Update your foldtext function

Modify your earlier NukeFoldText() to embed the highlight group: 

function! NukeFoldText()
  let lnum = v:foldstart
  let line = getline(lnum)

  " Extract node name (first word)
  let node = matchstr(line, '^\w\+')

  " Build fold text with highlighting
  return printf('%s%s%s … %d lines',
        \ '%#NukeFoldNode#', node, '%#Folded#',
        \ (v:foldend - v:foldstart + 1))
endfunction

setlocal foldtext=NukeFoldText()

🔎 How It Works

  • %#GroupName# switches highlighting to a specific group.
  • %#Folded# switches back to the default fold text style.
  • So the node name is highlighted with NukeFoldNode, while the rest uses the normal fold style.

✅ Example Result

When you fold a block like: 

Read {
   file /path/to/image.exr
   ...
}

The fold line will display something like: 

Read … 12 lines

with “Read” in bold yellow, and the rest in the standard folded style.

⚡ Summary:

Define a custom highlight group (NukeFoldNode), then use %#GroupName# inside your foldtext function to colorize the node name differently from the rest of the fold text. This makes folded blocks in Nuke .nk files visually distinctive and easier to scan.

Number of Inputs

Let’s extend your fold text so it not only shows the icon and node name, but also the number of inputs/outputs that node has. This makes folded blocks even more informative when scanning a big Nuke script.

🔎 How to Detect Inputs/Outputs

In a .nk file, each node block often contains lines like: 

inputs 2

or 

input0 Read1
input1 Transform1

So we can scan the folded block between v:foldstart and v:foldend for those keywords.

🛠️ Updated Foldtext Function

Add this to your ~/.vim/ftplugin/nuke.vim: 

function! NukeFoldText()
  let lnum = v:foldstart
  let line = getline(lnum)

  " Extract node name (first word of the block)
  let node = matchstr(line, '^\w\+')

  " Default icon
  let icon = '⚙️'
  if node =~? 'Read'
    let icon = '📥'
  elseif node =~? 'Write'
    let icon = '📤'
  elseif node =~? 'Shuffle'
    let icon = '🔀'
  elseif node =~? 'Transform'
    let icon = '🔧'
  elseif node =~? 'Remove'
    let icon = '❌'
  elseif node =~? 'Merge'
    let icon = '🧩'
  endif

  " Detect inputs/outputs by scanning folded lines
  let inputs = 0
  let outputs = 0
  for i in range(v:foldstart, v:foldend)
    let l = getline(i)
    if l =~? '^inputs'
      let inputs = str2nr(matchstr(l, '\d\+'))
    elseif l =~? '^outputs'
      let outputs = str2nr(matchstr(l, '\d\+'))
    endif
  endfor

  " Build fold text with icon, node name, I/O counts, and line count
  return printf('%s %s%s%s (in:%d out:%d) … %d lines',
        \ icon,
        \ '%#NukeFoldNode#', node, '%#Folded#',
        \ inputs, outputs,
        \ (v:foldend - v:foldstart + 1))
endfunction

setlocal foldtext=NukeFoldText()

✅ Example Result

When folding a block like: 

Merge {
  inputs 2
  outputs 1
  ...
}

The fold line will display: 

🧩 Merge (in:2 out:1) … 25 lines

⚡ Summary

By scanning the folded block for inputs and outputs lines, you can display the node’s connectivity right in the fold text. Combined with icons and highlighting, this gives you a compact “dashboard” view of your Nuke script inside Vim.

Show disconnected nodes

Let’s extend the fold text so the input/output counts are color‑coded. Vim lets you switch highlight groups inline with %#GroupName# inside your foldtext function. We’ll define two groups: one for “good” (green) and one for “bad” (red), then apply them depending on the values.

  elseif node =~? 'Remove'
    let icon = '❌'
  elseif node =~? 'Merge'

Vertical Indication

Let’s add indentation guides so nested folds visually show their depth. In Vim, you can’t literally draw vertical bars in the fold gutter, but you can prepend characters to the fold text based on the fold’s nesting level. That way, deeper folds show more bars or arrows, giving you a clear hierarchy.

🔎 Step 1: Compute fold level

Vim exposes foldlevel(v:foldstart) which tells you how deep the fold is. We can use that to generate indentation markers.

🛠️ Updated Foldtext Function

function! NukeFoldText()
  let lnum = v:foldstart
  let line = getline(lnum)

  " Extract node name
  let node = matchstr(line, '^\w\+')

  " Default icon
  let icon = '⚙️'
  if node =~? 'Read'
    let icon = '📥'
  elseif node =~? 'Write'
    let icon = '📤'
  elseif node =~? 'Shuffle'
    let icon = '🔀'
  elseif node =~? 'Transform'
    let icon = '🔧'
  elseif node =~? 'Remove'
    let icon = '❌'
  elseif node =~? 'Merge'
    let icon = '🧩'
  endif

  " Detect inputs/outputs
  let inputs = 0
  let outputs = 0
  for i in range(v:foldstart, v:foldend)
    let l = getline(i)
    if l =~? '^inputs'
      let inputs = str2nr(matchstr(l, '\d\+'))
    elseif l =~? '^outputs'
      let outputs = str2nr(matchstr(l, '\d\+'))
    endif
  endfor

  " Color coding for I/O
  let inGroup  = (inputs  > 0 ? '%#NukeFoldGood#' : '%#NukeFoldBad#')
  let outGroup = (outputs > 0 ? '%#NukeFoldGood#' : '%#NukeFoldBad#')

  " Indentation guides based on fold level
  let level = foldlevel(v:foldstart)
  let indent = repeat('│ ', level - 1)

  " Build fold text
  return printf('%s%s %s%s%s (in:%s%d%s out:%s%d%s) … %d lines',
        \ indent,
        \ icon,
        \ '%#NukeFoldNode#', node, '%#Folded#',
        \ inGroup, inputs, '%#Folded#',
        \ outGroup, outputs, '%#Folded#',
        \ (v:foldend - v:foldstart + 1))
endfunction

setlocal foldtext=NukeFoldText()

✅ Example Result

For nested blocks, you’ll see: 

📥 Read (in:0 out:1) … 12 lines
│ 📤 Write (in:1 out:0) … 8 lines
│ │ 🔀 Shuffle (in:2 out:1) … 15 lines
  • Each deeper fold adds a vertical bar │ as an indentation guide.
  • Icons and color coding remain intact.
  • The hierarchy of nodes is visually clear.

⚡ Summary

By using foldlevel(v:foldstart) and repeat('│ ', level-1), you can prepend indentation guides to fold text. Combined with icons and color‑coded I/O counts, this gives you a tree‑like view of your Nuke .nk scripts inside Vim.

Automatically Collapse

Let’s make Vim automatically collapse all folds into a tree view when you open a .nk file. That way you start with a clean overview of your Nuke script, and can expand only the blocks you want to inspect.

🔎 Step 1: Auto‑collapse folds on file open

In your ~/.vim/ftplugin/nuke.vim, add: 

" Enable folding by braces
setlocal foldmethod=marker
setlocal foldmarker={,}
setlocal foldenable

" Custom foldtext (from earlier steps)
setlocal foldtext=NukeFoldText()

" Collapse all folds when opening a .nk file
autocmd BufReadPost *.nk normal! zM
  • zM closes all folds to the maximum level (everything collapsed).
  • This runs automatically after you open a .nk file.

🔎 Step 2: Optional — open only top‑level folds

If you prefer to collapse everything but leave top‑level blocks open, use: 

autocmd BufReadPost *.nk normal! zR | normal! zM | normal! zO

Explanation:

  • zR opens all folds.
  • zM closes them all.
  • zO opens just the fold under the cursor (usually the first top‑level block). You can adjust this sequence to fit your preferred “tree view” style.

✅ Result

When you open a Nuke .nk file:

  • All node blocks are folded into a compact tree.
  • Fold text shows icons, node names, I/O counts, and indentation guides.
  • You can expand with zo or collapse again with zc.

⚡ Summary:

Add an autocmd BufReadPost *.nk normal! zM to your ftplugin/nuke.vim. This collapses all folds automatically when opening a Nuke script, giving you a clean tree view right away.

Toggle Switch

Let’s add a toggle keybinding so you can quickly switch between a collapsed “tree view” and a fully expanded view of your Nuke .nk folds. Vim makes this easy with a custom mapping.

🔎 Step 1: Define a toggle function

Add this to your ~/.vim/ftplugin/nuke.vim: 

function! ToggleTreeView()
  " Check if folds are currently mostly closed
  if foldclosed(1) != -1
    " If closed, open all folds
    normal! zR
  else
    " If open, close all folds
    normal! zM
  endif
endfunction
  • foldclosed(1) checks if the first line is folded.
  • If it is, we assume you’re in “tree view” and expand everything (zR).
  • If not, we collapse everything (zM).

🔎 Step 2: Bind it to a key

Pick a convenient key — for example t: 

nnoremap <leader>t :call ToggleTreeView()<CR>

Now pressing t toggles between collapsed and expanded views.

✅ Example Workflow

  • Open a .nk file → folds auto‑collapse into tree view.
  • Press t → all folds expand.
  • Press t again → everything collapses back into tree view.

⚡ Summary

By defining a ToggleTreeView() function and mapping it to t, you can instantly switch between collapsed and expanded fold views while editing Nuke scripts in Vim. This gives you a clean overview and quick access to details without manual zM/zR keystrokes.

highlight default NukeFoldNode ctermfg=Yellow guifg=#FFD700 cterm=bold gui=bold
highlight default NukeFoldGood ctermfg=Green  guifg=#00FF00 cterm=bold gui=bold
highlight default NukeFoldBad  ctermfg=Red    guifg=#FF0000 cterm=bold gui=bold

" ------------------------------
" Custom foldtext
" ------------------------------
function! NukeFoldText()
  let lnum = v:foldstart
  let line = getline(lnum)

  " Node name
  let node = matchstr(line, '^\w\+')

  " Icons
  let icon = '⚙️'
  if node =~? 'Read'
    let icon = '📥'
  elseif node =~? 'Write'
    let icon = '📤'
  elseif node =~? 'Shuffle'
    let icon = '🔀'
  elseif node =~? 'Transform'
    let icon = '🔧'
  elseif node =~? 'Remove'
    let icon = '❌'
  elseif node =~? 'Merge'
    let icon = '🧩'
  endif

  " Detect inputs/outputs
  let inputs = 0
  let outputs = 0
  for i in range(v:foldstart, v:foldend)
    let l = getline(i)
    if l =~? '^inputs'
      let inputs = str2nr(matchstr(l, '\d\+'))
    elseif l =~? '^outputs'
      let outputs = str2nr(matchstr(l, '\d\+'))
    endif
  endfor

  " Color coding
  let inGroup  = (inputs  > 0 ? '%#NukeFoldGood#' : '%#NukeFoldBad#')
  let outGroup = (outputs > 0 ? '%#NukeFoldGood#' : '%#NukeFoldBad#')

  " Indentation guides
  let level = foldlevel(v:foldstart)
  let indent = repeat('│ ', level - 1)

  " Build fold text
  return printf('%s%s %s%s%s (in:%s%d%s out:%s%d%s) … %d lines',
        \ indent,
        \ icon,
        \ '%#NukeFoldNode#', node, '%#Folded#',
        \ inGroup, inputs, '%#Folded#',
        \ outGroup, outputs, '%#Folded#',
        \ (v:foldend - v:foldstart + 1))
endfunction

setlocal foldtext=NukeFoldText()

" ------------------------------
" Auto-collapse into tree view
" ------------------------------
autocmd BufReadPost *.nk normal! zM

" ------------------------------
" Toggle tree view <leader>t
" ------------------------------
function! ToggleTreeView()
  if foldclosed(1) != -1
    normal! zR
  else
    normal! zM
  endif
endfunction

nnoremap <buffer> <leader>t :call ToggleTreeView()<CR>

# vim-nuke-tools

A Vim/Neovim plugin that adds smart folding, icons, syntax highlighting, and a tree‑view UI for editing Nuke `.nk` script files.

This plugin makes large Nuke scripts dramatically easier to navigate by turning each node block into a clean, informative fold.

---

## ✨ Features

- Automatic detection of `.nk` files
- Folding based on `{ ... }` node blocks
- Beautiful fold text including:
  - Node icons (📥 📤 🔀 🔧 ❌ 🧩 ⚙️)
  - Color‑coded input/output counts
  - Indentation guides (`│ │ │`)
- Auto‑collapse into a tree view when opening a file
- Toggle tree view with `<leader>t`
- Lightweight syntax highlighting for node names and braces

---
