Swift - Button Shapes for Text Objects

To turn a Text view into a pill or lozenge shape in SwiftUI, you need to combine padding, a background color, and a clip shape.

Here is the cleanest and most idiomatic way to do it: 

Text("SwiftUI")
    .font(.subheadline)
    .fontWeight(.semibold)
    .foregroundStyle(.white)
    .padding(.horizontal, 16) // Generates breathing room on the sides
    .padding(.vertical, 8)     // Generates breathing room on top and bottom
    .background(.blue)         // Applies the lozenge background fill
    .clipShape(Capsule())      // Automatically rounds the corners into a perfect lozenge

Key Architectural Concepts to Keep in Mind:

1. Order of Modifiers Matters Systematically

In SwiftUI, modifiers wrap views sequentially like layers of an onion. If you apply .clipShape() or .background() before you add .padding(), you will end up with a tiny, cramped shape because the background is colored before the view frame expands. Always pad your text first, then style the background and shape.

2. Why Use Capsule Instead of RoundedRectangle?

While RoundedRectangle(cornerRadius: 25) can create a similar look, it requires you to guess or hardcode a corner radius value. If your text changes length or the user increases their system font size, a fixed corner radius can distort.

Using Capsule() ensures that no matter how long the text is, or how large the font grows, the shortest edges will always be perfectly and seamlessly rounded into a flawless lozenge shape.

Fixed Width

To make your lozenges a completely uniform width regardless of how short or long the text label inside is, you need to explicitly define the text view's frame before you apply the background color and shape.

Here are the two best approaches, depending on whether you want a fixed size or a dynamically scaling grid.

Approach 1: Fixed Uniform Width

If you want every lozenge to have an exact pixel width (for example, a vertical stack of uniform buttons), use a specific .frame(width:) value. The following illustrates two buttons with different length text, but with fixed width regardless): 

Text("Short")
    .foregroundStyle(.white)
    .padding(.vertical, 8)
    .frame(width: 120)       // 1. Forces the frame width to an exact value
    .background(.blue)
    .clipShape(Capsule())

Text("Extremely Long")
    .foregroundStyle(.white)
    .padding(.vertical, 8)
    .frame(width: 120)       // 2. Matches the exact same width seamlessly
    .background(.blue)
    .clipShape(Capsule())

Important Order Warning: Notice that .frame() replaces .padding(.horizontal). You must put .frame() after .padding(.vertical) but before .background(). If you put it after .background(), the text will clip or the background color won't expand to fill the width.

Approach 2: Flexible Uniform Width (Proportional)

If you don't want to hardcode a pixel value because you want to respect the user's Dynamic Type font sizes, you can specify a maximum infinity frame. This forces the lozenges to automatically stretch and fill the entire width of whatever container layout they sit inside. 

VStack(spacing: 12) {
    Text("Yes")
        .foregroundStyle(.white)
        .padding(.vertical, 12)
        .frame(maxWidth: .infinity) // Spreads to fill the parent container width
        .background(.green)
        .clipShape(Capsule())
        
    Text("No, Cancel")
        .foregroundStyle(.white)
        .padding(.vertical, 12)
        .frame(maxWidth: .infinity) // Matches the exact same width as the "Yes" button
        .background(.red)
        .clipShape(Capsule())
}
.padding(.horizontal, 40) // Controls the overall width of the lozenge group from the outside

Summary Checklist for Consistent Sizing:

  • Vertical padding goes first (.padding(.vertical, 8)).
  • Width management goes second (.frame(width: 120) or .frame(maxWidth: .infinity)).
  • Aesthetics go last (.background() and .clipShape(Capsule())).

Other Enhancements

This guide outlines essential SwiftUI layout techniques for designing polished, uniform, and structurally organized text buttons or labels. When designing a user interface, using structural adjustments like indents, hierarchical font weights, and alignment grids helps guide a user's eye through data much more effectively than just changing colors.

Here are four core techniques for building platform-agnostic, professional text components.

1. Architectural Hierarchy with Indentation

Indentation is a powerful way to visually nest secondary elements under a primary element, or to indicate a state change (such as an active vs. inactive item). You can control this dynamically by applying conditional padding to the leading edge of your button content. 

HStack {
    VStack(alignment: .leading) {
        Text("Primary Label")
            .font(.headline)
        Text("Secondary descriptive text")
            .font(.subheadline)
            .foregroundStyle(.secondary)
    }
    Spacer()
}
// Apply an indent based on a conditional state (e.g., isSubItem)
.padding(.leading, isSubItem ? 24 : 0)

2. Multi-Component Alignment Grids (AlignmentGuides)

When trying to make a list of buttons look orderly, our instinct is often to hardcode a fixed .frame(width:) onto individual elements. However, if your labels vary wildly in length or icon size, hardcoded widths can fall out of alignment or clip your text.

Using custom alignment guides allows you to anchor disparate components to a shared vertical axis, ensuring that the text labels line up perfectly regardless of the icon or checkbox size sitting to their left. 

HStack(alignment: .center) {
    Image(systemName: "square.and.arrow.up")
        // Forces a custom baseline shift to align different icon widths
        .alignmentGuide(.leading) { d in d[.leading] - 12 } 
    
    VStack(alignment: .leading) {
        Text("Share File")
            .font(.body)
        Text("Send a copy to your team")
            .font(.caption)
    }
}

3. Dynamic Type Scaled Padding (@ScaledMetric)

When designing an interface, hardcoding a layout padding value like .padding(.horizontal, 16) means that if a user scales up their system text via device accessibility options, the text grows but the button's padding stays tiny. This results in a cramped, broken appearance.

The @ScaledMetric property wrapper automatically scales your spatial layouts, indents, and button margins relative to the user's system font size selection. 

struct AdaptiveButton: View {
    // Automatically scales the base value of 16 points up or down with system font choices
    @ScaledMetric(relativeTo: .body) var horizontalIndent: CGFloat = 16
    @ScaledMetric(relativeTo: .body) var verticalIndent: CGFloat = 12
    
    var body: some View {
        Text("Interactive Button")
            .font(.body)
            .padding(.horizontal, horizontalIndent) // Adaptive side padding
            .padding(.vertical, verticalIndent)     // Adaptive top/bottom padding
            .background(.blue)
            .clipShape(Capsule())
    }
}

4. Grouped Containment (Custom Card Backgrounds)

Instead of relying on basic text lines or native full-width list items, clearing the default row styling and wrapping elements in custom-indented cards gives your interface a modern, modular aesthetic. 

VStack(spacing: 12) {
    ForEach(items) { item in
        HStack {
            Text(item.title)
                .font(.body)
            Spacer()
            Image(systemName: "chevron.right")
        }
        .padding() // Inner cushioning for the card contents
        // Outer styling of the card container
        .background(
            RoundedRectangle(cornerRadius: 12)
                .fill(Color(.secondarySystemBackground))
        )
        // Controls the margin/indent of the entire card away from the screen edges
        .padding(.horizontal, 16) 
    }
}

Summary of Best Practices for Text Buttons:

The Layer Principle: Always apply padding before the background color to expand the clickable region. Apply clipping shapes after the background.

Flexibility Over Rigidity: Use @ScaledMetric instead of hardcoded point values to ensure your layout respects accessibility settings.

Containment: Use visual containers (like cards or capsules) to bundle related text and icons into clear, distinct touch targets.
