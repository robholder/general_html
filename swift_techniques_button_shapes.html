Swift - Button Shapes for Text Objects

To turn a Text view into a pill or lozenge shape in SwiftUI, you need to combine padding, a background color, and a clip shape.

Here is the cleanest and most idiomatic way to do it: 

Text("SwiftUI")
    .font(.subheadline)
    .fontWeight(.semibold)
    .foregroundStyle(.white)
    .padding(.horizontal, 16) // Generates breathing room on the sides
    .padding(.vertical, 8)     // Generates breathing room on top and bottom
    .background(.blue)         // Applies the lozenge background fill
    .clipShape(Capsule())      // Automatically rounds the corners into a perfect lozenge

Key Architectural Concepts to Keep in Mind:

1. Order of Modifiers Matters Systematically

In SwiftUI, modifiers wrap views sequentially like layers of an onion. If you apply .clipShape() or .background() before you add .padding(), you will end up with a tiny, cramped shape because the background is colored before the view frame expands. Always pad your text first, then style the background and shape.

2. Why Use Capsule Instead of RoundedRectangle?

While RoundedRectangle(cornerRadius: 25) can create a similar look, it requires you to guess or hardcode a corner radius value. If your text changes length or the user increases their system font size, a fixed corner radius can distort.

Using Capsule() ensures that no matter how long the text is, or how large the font grows, the shortest edges will always be perfectly and seamlessly rounded into a flawless lozenge shape.

Fixed Width

To make your lozenges a completely uniform width regardless of how short or long the text label inside is, you need to explicitly define the text view's frame before you apply the background color and shape.

Here are the two best approaches, depending on whether you want a fixed size or a dynamically scaling grid.

Approach 1: Fixed Uniform Width

If you want every lozenge to have an exact pixel width (for example, a vertical stack of uniform buttons), use a specific .frame(width:) value. 

Text("Short")
    .foregroundStyle(.white)
    .padding(.vertical, 8)
    .frame(width: 120)       // 1. Forces the frame width to an exact value
    .background(.blue)
    .clipShape(Capsule())

Text("Extremely Long")
    .foregroundStyle(.white)
    .padding(.vertical, 8)
    .frame(width: 120)       // 2. Matches the exact same width seamlessly
    .background(.blue)
    .clipShape(Capsule())

Important Order Warning: Notice that .frame() replaces .padding(.horizontal). You must put .frame() after .padding(.vertical) but before .background(). If you put it after .background(), the text will clip or the background color won't expand to fill the width.

Approach 2: Flexible Uniform Width (Proportional)

If you don't want to hardcode a pixel value because you want to respect the user's Dynamic Type font sizes, you can specify a maximum infinity frame. This forces the lozenges to automatically stretch and fill the entire width of whatever container layout they sit inside. 

VStack(spacing: 12) {
    Text("Yes")
        .foregroundStyle(.white)
        .padding(.vertical, 12)
        .frame(maxWidth: .infinity) // Spreads to fill the parent container width
        .background(.green)
        .clipShape(Capsule())
        
    Text("No, Cancel")
        .foregroundStyle(.white)
        .padding(.vertical, 12)
        .frame(maxWidth: .infinity) // Matches the exact same width as the "Yes" button
        .background(.red)
        .clipShape(Capsule())
}
.padding(.horizontal, 40) // Controls the overall width of the lozenge group from the outside

Summary Checklist for Consistent Sizing:

  • Vertical padding goes first (.padding(.vertical, 8)).
  • Width management goes second (.frame(width: 120) or .frame(maxWidth: .infinity)).
  • Aesthetics go last (.background() and .clipShape(Capsule())).

Other Enhancements

If you are looking to make your list or your lozenges look more polished and structurally organized, there are several highly effective SwiftUI layout techniques you can use.

Using structural adjustments like indents, hierarchical font sizing, and alignment grids can guide a user's eye through data much better than just changing colors.

Here are four great techniques to enhance your views, along with how to build them.

1. Contextual Row Indentation (.listRowSeparatorTint & Padding)

If you want to visually group certain types of elements or create a visual hierarchy in your list (for example, indenting uncontacted prospects to make them stand out), you can apply conditional horizontal padding directly to the row content. 

HStack {
    VStack(alignment: .leading) {
        Text(prospect.name)
            .font(.headline)
        Text(prospect.emailAddress)
            .font(.subheadline)
    }
    Spacer()
}
// If uncontacted, indent the row content to create a nested visual hierarchy
.padding(.leading, prospect.isContacted ? 0 : 24)

2. Multi-Column Alignment grids (AlignmentGuides)

When trying to make data look orderly, our instinct is often to hardcode a .frame(width:) onto components. However, if your labels change, hardcoded widths can fall out of alignment.

You can use a hidden grid structure by embedding a custom alignment guide. This ensures that the text fields of your rows line up perfectly on a vertical axis, even if the icons or checkboxes on the left change sizes. 

HStack(alignment: .center) {
    Image(systemName: "person.crop.circle")
        .alignmentGuide(.leading) { d in d[.leading] - 10 } // Forces custom baseline
    
    VStack(alignment: .leading) {
        Text(prospect.name)
        Text(prospect.emailAddress)
    }
}

3. Dynamic Type Scaled Padding (@ScaledMetric)

When designing a uniform lozenge, hardcoding a padding value like .padding(.horizontal, 16) means that if an elderly user scales up their system text via Apple's accessibility options, the text grows but the capsule's padding stays tiny, making it look broken.

You can use the modern @ScaledMetric property wrapper. This automatically scales your spatial layouts and indents relative to the user's system font size. 

struct LozengeView: View {
    // Automatically scales the base value of 16 points up or down with system font choices
    @ScaledMetric(relativeTo: .body) var horizontalPadding: CGFloat = 16
    
    var body: some View {
        Text("Prospect")
            .padding(.horizontal, horizontalPadding) // Perfectly adaptive indent
            .background(.blue)
            .clipShape(Capsule())
    }
}

4. Grouping Data with Custom Row Backgrounds

Instead of using standard white list rows, you can clear the default list background and group related elements into custom-indented cards. This gives your interface a modern, dashboard-like aesthetic. 

List(prospects) { prospect in
    HStack {
        // Your text layout here
    }
    .padding()
    // 1. Clears default row styling and injects a clean custom card background
    .listRowBackground(
        RoundedRectangle(cornerRadius: 12)
            .fill(Color(.systemGroupedBackground))
            .padding(.vertical, 4)  // Spacers between the cards
            .padding(.horizontal, 8) // Indents the cards away from screen edges
    )
    .listRowSeparator(.hidden) // Removes the harsh system line divider
}
